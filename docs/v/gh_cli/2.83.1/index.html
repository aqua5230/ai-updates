zh-TW

PR 移除審查者不再失敗，編輯更新傳送清楚

你用 gh pr edit 編輯 Pull Request（程式審核單）並移除所有審查者（reviewer，幫你審程式的人）時，現在會在 PR API 呼叫中傳送審查者的空陣列，確保名單能正確清空。空陣列（一個沒有項目的清單）明確表達「不需要任何審查者」。就像在清單上直接填寫「不需要任何審查者」，負責人就知道名單該是空白。

Original changelog
`gh pr edit`: Ensure empty arrays for reviewers in PR API calls by @BagToad in https://github.com/cli/cli/pull/12122

內部開發流程升級，授權檢查回歸、工具版本更新

這版本主要是開發工具與流程的內部改進。首先，原本獨立的授權檢查（license check，確保程式碼裡用的開源套件不違法）被重新整合到主要的程式碼檢查流程（lint workflow）裡面。其次，第三方授權、第三方套件與相依的軟體工具也跟著更新，包括 golangci-lint 的 GitHub Action（在 GitHub 上自動跑程式碼檢查的動作元件）從 8.0.0 升到 9.0.0。這些改動都在開發流程裡，就像汽車廠替檢修廠更新工具，改的是車間裡的設備。

PR reviewer removal succeeds reliably with clearer API signals

When you use gh pr edit to remove all reviewers from a Pull Request, it now sends an empty reviewer array in PR API calls so the list is handled correctly. An empty array clearly records that no reviewers are wanted. It is like writing "no reviewers needed" directly on a list.

Original changelog
`gh pr edit`: Ensure empty arrays for reviewers in PR API calls by @BagToad in https://github.com/cli/cli/pull/12122

License checks restored to workflow, development toolchain upgraded

This release focuses on under-the-hood improvements to the development workflow and toolchain. License checks (verifying that all open-source components used in the code comply with legal requirements) have been integrated back into the main linting workflow. Third-party licenses and dependencies were also updated, and the golangci-lint GitHub Action (a component that runs the code checks automatically on GitHub) was upgraded from 8.0.0 to 9.0.0. Like a repair shop updating its equipment, the changes are in the workshop.
