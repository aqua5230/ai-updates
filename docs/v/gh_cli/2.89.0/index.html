建立提取要求與議題時，可搜尋負責人並依登入身分送出變更

你用 gh pr create、gh issue create 或 gh issue edit 建立或修改提取要求、議題時，現在能用搜尋方式挑選負責人。在 github.com 上，指令也會依你目前登入的身分處理這些變更；不必先記住完整帳號再手動填入。這讓「要交給誰跟進」這一步更容易從候選人中找到正確的人，就像在通訊錄搜尋名字後再把信交給收件人。

可用實驗開關試用 huh 提示介面

你想試用新的互動式提問畫面時，可以設定 GH_EXPERIMENTAL_PROMPTER 這個環境變數（啟動程式前交給它的設定值），開啟只使用 huh 的實驗性提示介面。它被明確放在實驗開關後面，代表這是可選的測試功能，而不是所有人都會自動換上的正式預設，也可隨時不使用。就像先拿到一把試用鑰匙，再決定要不要進新裝潢的房間。

企業雲端的 Copilot 連線更能找到正確位置，並標記代理呼叫

你在 ghe.com 租戶使用 agent-task 時，GitHub CLI 現在會動態找出正確的 Copilot API（程式彼此溝通的入口）網址，不再固定使用一個地址。同時，由代理執行的呼叫會記錄在 User-Agent（請求附帶的用戶端身分標籤）標頭中，並把 actor-mode 的判斷集中到 ApiActorsSupported。這些是連線與識別層的調整，像客服總機會依公司分機轉接，並在來電單上標出代辦來源。

When you create or edit pull requests and issues with gh pr create, gh issue create, or gh issue edit, you can now search for the person to assign. On github.com, these changes are also applied using the identity you are signed in with, so you do not have to recall and type a full account name first. It is like finding a name in your contacts before handing over a letter.

Try the experimental huh prompt interface behind a feature flag

If you want to try a new interactive prompt screen, set GH_EXPERIMENTAL_PROMPTER, an environment variable supplied before the program starts. This enables an experimental prompter that uses huh only. Because it sits behind an experimental switch, it is an optional test feature rather than a new default for everyone. It is like receiving a trial key before deciding whether to enter a newly renovated room.

Find the right Copilot endpoint for enterprise cloud accounts and mark agent calls

When you use agent-task in a ghe.com tenancy, GitHub CLI now resolves the appropriate Copilot API URL dynamically instead of relying on one fixed address. Calls made by an agent are recorded in the User-Agent header, the label sent with a request to identify its client, and actor-mode signals are consolidated in ApiActorsSupported. These are connection and identification changes, like a switchboard routing calls by company extension and marking who initiated each request.

Fetch less data for issues and stabilize Codespaces port forwarding

When you create or transfer an issue, issue create and issue transfer no longer fetch unnecessary fields, avoiding an unneeded data request. Codespaces port forwarding, which exposes a service from a cloud development environment, also fixes a data race: two tasks colliding while changing the same data. The documentation now clarifies that gh pr edit --add-reviewer can request a review again. It is like leaving irrelevant attachments out of a parcel and lining people up before they pass through a narrow door.
