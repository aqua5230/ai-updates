Windows 簽署改用 OIDC，並撤回兩項專案權限改動

你下載 Windows 版 gh 時，發行端的程式碼簽署已從 client secret（應用程式保存的登入密鑰）改為 OIDC（讓工作流程向身分服務即時換取短期身分證明）；這是發行流程的安全調整，不會改變你的使用指令。同時，上一版整理 project 查詢權限錯誤、以及讓建立 project issue 的範圍錯誤更清楚的兩項修改都被撤回。像門禁改成每次進門領一張短效通行證，但管理員也把兩條剛換、尚不合用的指示牌先換回原樣。

en

Move Windows signing to OIDC and revert two project-scope changes

When you download gh for Windows, the release process now signs code with OIDC instead of a client secret. This changes how the publisher proves its identity during release, not the commands you use. Two changes from the previous release were also reverted: shared handling for project-query scope errors and clearer scope errors while creating project issues. It is like replacing a permanent door key with a short-lived entry pass for each visit, while putting two newly changed signs back until they are ready.

ja

Windows 署名を OIDC へ移し、プロジェクト権限の2変更を戻す

Windows 版 gh を受け取る時、公開側のコード署名は client secret（アプリが保管するログイン用の秘密情報）から OIDC（作業手順が身元確認サービスから短期の証明を受け取る仕組み）へ移った。これは公開手順の安全性を変えるもので、利用するコマンドは変わらない。前版の project 問い合わせにおける権限範囲エラーの共通処理と、project issue 作成時の範囲エラーを分かりやすくする変更は、いずれも取り消された。門ごとに短期の通行証を受け取る方式へ替えつつ、まだ合わない2枚の新しい案内板は元に戻すようなものである。

Windows 서명을 OIDC로 옮기고 프로젝트 권한 관련 두 변경을 되돌린다

Windows용 gh를 내려받을 때 배포 측 코드 서명은 client secret(앱이 보관하는 로그인 비밀 키) 대신 OIDC(작업 흐름이 신원 서비스에서 짧게 쓰는 증명을 즉시 받는 방식)를 사용한다. 이는 배포자가 배포 중 신원을 증명하는 방식을 바꾸는 것이며, 사용자가 쓰는 명령은 바뀌지 않는다. 이전 버전의 project 조회 권한 범위 오류 공통 처리와 project issue 생성 시 범위 오류를 더 분명히 한 두 변경도 되돌렸다. 매번 짧은 출입증을 받는 문으로 바꾸되, 아직 맞지 않는 새 안내판 두 장은 원래대로 돌려놓는 것과 같다.
