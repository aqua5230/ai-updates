You can now preview and install custom extensions with more flexibility, and easily comply with corporate data residency policies. Clean logs to prevent terminal injection and add security warnings to extensions

You can now review logs and install third-party extensions with much better protection against malicious exploits. The CLI now filters unsafe escape codes when outputting logs to prevent terminal injection from manipulating your screen. Additionally, help pages for extensions now display a clear trust disclaimer, reminding you to only run extensions you trust. A syntax fix was also applied to variable interpolation in deployment workflows (deployment.yml) to keep automated processes stable. It's like looking at a public bulletin board: previously someone could hide sticky buttons under posters to trick you, but now the posters are behind safety glass and there's a warning sign reminding you to watch out for scams.

커스텀 확장 기능을 더 유연하게 미리 보고 설치할 수 있으며, 기업의 데이터 스토리지 지역 규정도 쉽게 준수할 수 있다. 스킬을 미리 볼 때 숨겨진 디렉터리 접근을 허용하는 플래그를 지정할 수 있고, 데이터 레지던시를 지원하는 GitHub Enterprise Cloud(GHEC)도 정상 작동한다. 또한 스킬을 설치할 때 복잡한 네임스페이스 대신 스킬 고유의 이름으로 저장되어 경로가 한결 깔끔해졌다. 스마트폰에 위젯을 설치할 때, 예전엔 개발자 이름별로 여러 폴더에 복잡하게 얽혀 있던 것이 이제는 홈 화면에 곧바로 설치되고, 데이터 레지던시 호스트를 쓰는 GHEC를 지원하는 것과 같다.

프로젝트 보드 권한 오류를 무시하고, Linux 자동 완성 및 원격 측정 메트릭을 수정한다.

다양한 Linux 환경에서 더 매끄러운 자동 완성과 안정적인 명령어 실행을 누릴 수 있다. Debian 환경의 zsh에서 탭 키로 명령어를 완성할 때 후보 목록이 올바르게 나타난다. 프로젝트 보드에 접근하다가 '리소스 접근 불가' 오류가 발생하더라도 이제 프로그램을 중단시키지 않고 가볍게 무시하고 넘어간다. 아울러 원격 측정의 샘플링 비율이 갱신되지 않던 문제도 고쳐졌다. 도서관에서 책을 찾을 때, 예전엔 권한이 없는 서가와 마주치면 안내 로봇이 완전히 멈춰버리고 검색 키오스크는 오타 투성이에 자동 추천도 되지 않았던 반면, 이제는 제한 구역을 알아서 우회해 가며 책 검색이 매끄럽게 이루어지는 것과 같다.
