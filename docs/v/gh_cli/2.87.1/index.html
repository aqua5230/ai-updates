移除授權檔殘留，固定 agent-task 的 API 版本

你更新 gh 後，安裝內容會少一點不該跟著帶上的授權檔殘留；它們不是你要使用的功能，移除後讓套件更乾淨。另一項調整把 agent-task/CAPI 使用的 API 版本固定下來。API（讓不同程式彼此溝通的介面）若每次自動挑版本，外部服務改動時可能出現難追的差異；固定版本就是先鎖定同一套說法。這版沒有替 agent-task 增加操作功能，而是清掉行李、把溝通規格釘住，像出門前丟掉包包裡多餘的包裝紙，並把轉接頭固定在正確規格。

en

Remove bundled license debris and pin the agent-task API version

After updating gh, its package will carry less leftover license material that is not part of the feature you use. The release also pins the API version used by agent-task/CAPI. An API is the interface programs use to talk to one another; choosing a version automatically can produce hard-to-trace differences when an outside service changes, while pinning it keeps both sides speaking the same version. This does not add an agent-task feature. It removes packing debris and fixes the connection standard, like throwing spare wrapping paper out of a bag and locking in the right adapter.

ko

묶인 라이선스 잔여물을 없애고 agent-task API 버전을 고정한다

gh를 업데이트하면 실제 기능과 관계없는 묶음 라이선스 잔여물이 줄어 패키지가 더 깔끔해진다. 또 agent-task/CAPI가 쓰는 API 버전을 고정했다. API는 서로 다른 프로그램이 대화하는 창구인데, 매번 자동으로 버전을 고르면 외부 서비스가 바뀔 때 추적하기 어려운 차이가 생길 수 있다. 버전을 고정하면 같은 규격으로 계속 말하게 된다. agent-task에 새 조작 기능을 더한 판이 아니라 짐을 정리하고 연결 규격을 붙잡는 변화이다. 가방 속 남은 포장지를 버리고 맞는 변환 플러그를 고정하는 것과 같다.
