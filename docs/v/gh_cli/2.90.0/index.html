New gh skill commands let you find, install, preview, update, and publish skills

You can now use `gh skill` to search for, preview, install, update, and publish skills: instruction packages that teach an AI or tool a particular job. If you enter an unknown command, GitHub CLI can suggest an official extension to install. Installing extensions and local-only skill options no longer require an authentication check. The `api` and `auth` commands also record agentic invocations. Publishing now pushes unpushed commits first and uses the target directory's remote. Search also keeps same-named skills from different namespaces separate, handles encoded parent paths, and `gh release list --limit 0` no longer loops forever. It is like a tool shop where you can inspect an item, take it home, and have the clerk send your pending paperwork with it.

Skill publishing documentation and checks are clearer and steadier

You now get clearer security and operating notes when verifying downloads or publishing skills. Documentation adds PGP key fingerprints, SHA/MD5 checksums for keyring files, the corrected SHA512 value, and a dependency CVE policy; it also documents skill commands and agent skills. Internally, the CLI updates Go, replaces a compression package, removes an old Debian development script, and adds recommendations, code reviewers, shared discovery, and tests around official extensions and skill packages. Publish tests no longer invoke real Git, `--dry-run` messages match, and redundant fallback logic is gone. It is like registered mail: the seal, verification code, and delivery rules are explicit, and the route has been rehearsed at a practice counter.

gh skill을 추가해 스킬을 찾고 설치하고 미리 보고 갱신하고 공개한다

이제 `gh skill`로 스킬(skill, AI나 도구에 특정 작업을 가르치는 설명 묶음)을 찾고, 미리 보고, 설치하고, 갱신하고, 공개할 수 있다. 알 수 없는 명령어를 입력하면 설치할 수 있는 공식 확장 기능도 제안한다. 확장 기능 설치와 로컬 전용 스킬 옵션은 더 이상 먼저 인증을 확인하지 않는다. `api`와 `auth` 명령어는 AI 에이전트 호출도 기록한다. 공개할 때는 아직 보내지 않은 커밋을 먼저 올리고 대상 디렉터리의 원격 설정을 사용한다. 서로 다른 네임스페이스의 같은 이름 스킬을 검색에서 합쳐 버리던 문제, 특수 문자가 든 상위 경로, `gh release list --limit 0`의 무한 반복도 고쳤다. 공구점에서 물건을 살펴보고 가져가며, 점원이 밀린 서류까지 함께 보내 주는 것과 같다.
