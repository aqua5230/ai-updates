gh CLI 終端機直接開討論串

gh CLI 現在能直接在命令列操作 GitHub Discussions（討論區功能，社群提問與閒聊的看板）了，這是預覽版（preview，功能還在測試、之後可能調整）。新增「gh discussion」指令組：list 列出討論串、view 看內容、create 開新討論、edit 編輯，另外「gh discussion comment」可以直接留言或回覆別人的討論。內建的 gh 技能（skill，讓 AI 代理知道怎麼用 gh 指令的說明書）也同步補上這批 discussion 指令和 Issues 2.0 的用法說明。像原本只能用網頁逛的社群留言板，現在終端機也能直接開窗留言。

Original changelog

議題可以分類、拆子任務、互相牽線了

GitHub 的 Issues 2.0（新一代議題系統，議題可以分類、拆子任務、彼此關聯）現在 gh CLI 也支援了：issue 可以標上類型（issue types）、拆成子議題（sub-issues，大任務底下的小任務）、跟其他議題建立關聯（relationships）。像把一張寫滿待辦的便條紙，換成可以分類貼、拆小張、互相牽線的看板，管理大專案的議題終於不用全部擠在一張清單裡。

Original changelog
Add Issues 2.0 support: issue types, sub-issues, and relationships by @BagToad in [#13057](https://github.com/cli/cli/pull/13057)

gh skill 系列指令一次補齊

gh skill（技能，讓 AI 代理照著說明書做事的功能包）系列指令更好用了。新增「gh skill list」可以一次看清楚裝了哪些技能。「gh skill install」加上 --all 旗標，能把一個 repository（儲存庫）裡的技能全部裝起來，不用一個一個裝。「gh skill update --all」批次更新時，遇到缺 metadata（描述資訊）的技能會自動跳過、不會卡住整批更新。技能檔案放在比較深的巢狀資料夾裡，以前掃描不到，現在也找得到了。像倉庫裡的工具箱終於有清單可以點名，一鍵進貨、缺標籤的工具不會卡住盤點，藏在櫃子深處的工具也翻得出來。

gh CLI can now drive GitHub Discussions from the terminal

gh CLI can now drive GitHub Discussions (a community Q&A and chat board) straight from the command line — this is a preview (the feature is still being tested and may change). The new gh discussion command set covers list, view, create, and edit, and gh discussion comment lets you post or reply to a discussion directly. The built-in gh skill (a skill is a manual that tells an AI agent how to use gh commands) has also been updated with these discussion commands and Issues 2.0 usage notes. It's like a community message board you used to have to browse on the web now getting its own window right in your terminal.

Issues can now be typed, split, and linked to each other

GitHub's Issues 2.0 (the next-generation issue system, where issues can be typed, split into sub-issues, and linked to each other) is now supported in gh CLI: issues can carry a type (issue types), be broken into sub-issues (smaller tasks under a bigger one), and form relationships with other issues. It's like swapping a sticky note crammed with to-dos for a board where you can categorize, split into smaller notes, and draw connecting lines — managing a big project's issues no longer means cramming everything onto one list.

Original changelog
Add Issues 2.0 support: issue types, sub-issues, and relationships by @BagToad in [#13057](https://github.com/cli/cli/pull/13057)

The `gh skill` command family gets rounded out

The gh skill (a skill is a bundle of instructions that tells an AI agent how to do a task) command family got more usable. The new gh skill list shows everything installed at a glance. gh skill install gained an --all flag to install every skill in a repository at once instead of one by one. gh skill update --all now skips skills missing metadata (description info) instead of getting stuck on the whole batch. And skill files buried in nested subfolders, which used to go undetected, are now found too. It's like a warehouse toolbox finally getting an inventory list, one-click restocking, tools without labels no longer jamming the count, and gear buried deep in the cabinet finally turning up.

이슈를 종류별로 나누고 쪼개고 서로 연결할 수 있게 됐다

GitHub의 Issues 2.0(차세대 이슈 시스템, 이슈에 종류를 붙이고 하위 이슈로 쪼개고 서로 연결할 수 있는 방식)이 이제 gh CLI에서도 지원된다: issue에 종류(issue types)를 붙이고, 하위 이슈(sub-issues, 큰 작업 아래 딸린 작은 작업)로 쪼개고, 다른 이슈와 관계(relationships)를 맺을 수 있다. 할 일로 빼곡한 메모지 한 장을, 종류별로 붙이고 작게 쪼개고 서로 선으로 연결할 수 있는 보드로 바꾼 셈이라, 큰 프로젝트의 이슈 관리가 더는 리스트 한 장에 다 욱여넣지 않아도 된다.

gh skill 계열 명령이 한 번에 갖춰졌다

gh skill(스킬, AI 에이전트에게 작업 방법을 알려주는 설명서 묶음) 계열 명령이 더 쓰기 좋아졌다. 새로 생긴 'gh skill list'로 설치된 스킬을 한눈에 볼 수 있다. 'gh skill install'에 --all 플래그가 붙어서, 저장소 안의 스킬을 한 번에 전부 설치할 수 있다. 'gh skill update --all'로 일괄 업데이트할 때 metadata(설명 정보)가 없는 스킬은 자동으로 건너뛰어 전체가 멈추지 않는다. 깊이 중첩된 폴더에 있던 스킬 파일도 예전엔 못 찾았는데 이제 찾아낸다. 창고 공구함에 드디어 재고 목록이 생기고, 클릭 한 번으로 한꺼번에 들여놓고, 라벨 없는 공구가 재고 조사를 멈추게 하지 않고, 서랍 깊숙이 묻힌 도구도 찾아지는 셈이다.

확장 기능 별칭과 CI 자동 설치 소소한 개선

이번엔 명령어 별칭과 자동화 쪽의 자잘한 개선이다. 'gh extension uninstall'을 이제 'gh extension remove'로도 칠 수 있어서, 이름 두 개를 다 외울 필요 없이 같은 곳으로 통한다. CI(지속적 통합, 자동으로 빌드와 테스트를 도는 흐름) 환경에서는 공식 extension(확장 기능)이 자동으로 설치돼 따로 설정할 필요가 없다. 마케팅용 터미널 화면 스크린샷을 만드는 terminal-mockup canvas extension도 새로 추가됐다. 나머지에는 내부 정리와 의존 패키지 버전 업데이트가 있다. 평소 사용에는 영향 없다. 가게가 '반품'과 '환불' 버튼을 하나로 합친 것과 같아서, 어느 쪽을 눌러도 같은 곳으로 간다.
