zh-TW

用 gh 直接安裝與啟動 Copilot CLI

現在你想在終端機使用 Copilot CLI（讓你透過文字指令使用 Copilot 的工具）時，可以直接輸入 `gh copilot`；電腦尚未安裝它就會先詢問你是否安裝，裝好後接著啟動。若你已經知道要交代什麼，也能在後面接上參數與旗標，`gh` 會原封不動轉交給 Copilot CLI 執行。這不是讓 Copilot 多了新本事，而是把原本要另外找安裝程式、再切換指令的兩段路接起來，像在車站同一個窗口買票後直接上車。

Original changelog
`gh copilot` will prompt to install, then run Copilot CLI
`gh copilot <args>` will execute the Copilot CLI, forwarding any arguments and flags
`gh copilot`: add native `copilot` command to execute/install copilot cli by @devm33 in https://github.com/cli/cli/pull/12444
Fix Windows asset URL in `copilot` command tests by @babakks in https://github.com/cli/cli/pull/12500

可清除特定分支的全部快取，建立重複 PR 會即時擋下

現在你要清理某個 ref（Git 用來指向分支或版本的名稱）留下的快取（把先前結果暫存、下次直接使用的資料）時，`gh cache delete` 可以一次刪掉該 ref 的所有快取，不必逐筆處理。建立 Pull Request（PR，把程式修改送人審核的請求）時，如果 head 與 base ref 相同，`gh pr create` 也會直接報錯；同一個起點與目標沒有可合併的差異，早點停下比建立後才發現更省事。貢獻指南同時改得更清楚，並記錄了兩位首次貢獻者。像整理倉庫時按區清空舊紙箱，門口也先攔住要把同一份文件送給自己的表單。

en

Install and launch Copilot CLI directly through gh

You can now get to Copilot CLI from the same terminal workflow you use for GitHub. Run `gh copilot` and gh will offer to install Copilot CLI if it is missing, then launch it. Add arguments or flags after the command and gh passes them through to Copilot CLI unchanged. Copilot has not gained a new capability here; the update simply joins installation and launch into one route instead of making you find a separate installer and switch commands, like buying a ticket and boarding at the same station counter.

Fix Windows asset URL in `copilot` command tests by @babakks in https://github.com/cli/cli/pull/12500

Clear every cache for a ref and catch duplicate PR targets early

You can now clear all cached results attached to one ref—a Git name that points to a branch or version—with a single `gh cache delete` action instead of removing them one by one. `gh pr create` also now errors when its head and base refs are identical, because there is nothing to merge between the same starting point and destination. The contributor guide was clarified, and the release records two first-time contributors. It is like emptying every old box in one warehouse aisle while a guard stops a form addressed from you to you before it reaches the desk.

ja

gh から Copilot CLI を直接インストールして起動できる

端末で Copilot CLI（文字のコマンドで Copilot を使う道具）を使いたいとき、`gh copilot` と入力すればよくなった。未インストールなら gh が先にインストールするか尋ね、完了後に起動する。指示用の引数やフラグを後ろに付ければ、それらは変更されず Copilot CLI に渡される。Copilot 自体に新しい能力を足したのではなく、別のインストーラーを探してコマンドを切り替える二段階を一本につないだ変更であり、駅の同じ窓口で切符を買ってそのまま乗車するようなものである。

特定の ref の全キャッシュを削除でき、同じ PR の作成を早期に止める

Git でブランチや版を指す名前である ref に残ったキャッシュ（以前の結果を一時保存して次回そのまま使うデータ）は、`gh cache delete` でその ref 分をまとめて削除できるようになった。Pull Request（PR、変更をレビューへ回す依頼）を作る際も、head と base の ref が同じなら `gh pr create` がエラーで止める。同じ出発点と宛先には統合する差分がないためである。貢献ガイドも分かりやすくなり、初参加の貢献者二人も記録された。倉庫の一区画の古い箱を一度に空にし、自分から自分宛ての書類を入口で止めるようなものである。

ko

gh에서 Copilot CLI를 바로 설치하고 실행한다

터미널에서 Copilot CLI(글자 명령으로 Copilot을 쓰는 도구)를 쓰고 싶다면 이제 `gh copilot`만 입력하면 된다. 설치되어 있지 않으면 gh가 먼저 설치할지 묻고, 설치가 끝나면 실행한다. 뒤에 인수나 플래그를 붙이면 gh가 바꾸지 않고 Copilot CLI에 넘긴다. Copilot에 새 능력이 생긴 것이 아니라, 별도 설치 프로그램을 찾고 명령을 바꾸던 두 단계를 하나로 이은 변화이다. 역의 같은 창구에서 표를 사고 바로 타는 것과 같다.

특정 ref의 모든 캐시를 지우고 중복 PR 생성을 일찍 막는다

이제 Git에서 브랜치나 버전을 가리키는 이름인 ref에 남은 캐시(이전 결과를 잠시 보관해 다음에 바로 쓰는 데이터)를 `gh cache delete`로 한꺼번에 지울 수 있다. Pull Request(PR, 코드 변경을 검토에 보내는 요청)를 만들 때 head와 base ref가 같으면 `gh pr create`가 오류로 멈춘다. 같은 출발점과 목적지 사이에는 합칠 차이가 없기 때문이다. 기여 안내도 더 분명해졌고, 처음 기여한 두 사람도 기록됐다. 창고 한 구역의 낡은 상자를 한 번에 비우고, 자신이 자신에게 보내는 서류를 입구에서 막는 것과 같다.
