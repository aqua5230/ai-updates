免登入下載公開專案的發布檔

現在下載公開專案的發布檔案時，不需要登入 GitHub 帳號也能直接下載了。以前如果你用指令 `gh release download` 去抓取軟體的安裝檔或壓縮包，即使那個專案是完全對外公開（public repositories）的，系統還是會死板地要求你先登入（authentication，證明你是誰的登入驗證程序）才能放行。現在這個不合理的門檻移除了，只要是公開專案，任何人都能免登入直接把檔案載走。就像走進免費開放的市立圖書館拿一份導覽手冊，以前警衛還會擋在門口硬要你掏出借書證來登記，現在只要是公開擺在架上的東西，你推開門就能直接拿走。

安裝 AI 代理輔助工具的技能時，指令變得更靈活，也不會再因為你講得太清楚反而被刁難了。這次 `gh skill` 指令擴大了支援範圍，懂得辨識出 `antigravity-cli` 與 `antigravity2.0`，並且能偵測到更多第三方的寫程式碼代理（coding agents，能幫你寫程式碼的 AI 助手）。在底層設定上，安裝通用代理技能的預設位置被修正為 `~/.agents/skills`。更聰明的是，以前如果你已經在指令裡用 `--dir` 參數明確告訴它「我要把技能安裝到這個資料夾」，它居然還會跳出互動提示（prompt，在這裡指跳出選單問你問題）強迫你再選一次代理工具；現在它會直接尊從你的指定路徑，不再多嘴。就像你買了一支萬用遙控器，它現在學會了最新款電視的紅外線頻率，而且當你已經精準指著那台電視按下開關時，它不會再跳出畫面囉唆地問你「請問你要遙控哪一台設備」。

修正 Codespace 轉發當機與 CI 狀態空白

你的開發環境不會再因為同時搶著寫資料而當機，檢查清單取消時也不會再留下一片混亂的空白了。以前在用 Codespace 的對外連線（codespace port forwarding，把雲端開發機器的網路連接埠打通到你本機的功能）時，如果背後的程式碼遇到併發寫入衝突（concurrent map writes，指兩隻程式同時搶著改同一份資料表造成的衝突當機），整個轉發功能就會崩潰。現在修好了這個衝突讓它不再當機。此外，如果自動檢查流程（CI checks，自動跑的一系列測試與檢查項目）被全面取消了，以前的總結畫面會直接印出一片死板的空白，讓你一頭霧水；現在它會老老實實顯示「全部已取消」的摘要。就像一家餐廳，以前如果兩個服務生同時把點菜單寫在同一格版子上，版子就會撕破導致廚房大當機；現在這個容易弄破版子的問題修好了。而且如果那桌客人決定全部退單，廚房的看板上會清楚標明「此桌已取消」，而不是留下一塊讓人瞎猜的空白黑板。

Download public releases without authentication

You can now download release files from public projects without needing to sign into a GitHub account at all. Previously, if you used the `gh release download` command to grab an installer or zip file, the system rigidly demanded authentication (the login process proving your identity) before letting you proceed—even if the project was completely public and open to the world. That unnecessary hurdle has been removed, allowing anyone to fetch files from public repositories without logging in. It's like walking into a free public library to grab a visitor brochure: the security guard used to stubbornly block the door demanding your library card just to take a free leaflet, but now if it's sitting on the public rack, you can just walk in and take it.

More AI agents supported and smarter skill installation

Installing skills for AI coding agents is now much more flexible and won't bombard you with redundant questions. The `gh skill` command has expanded its horizons: it now fully supports `antigravity-cli` and `antigravity2.0`, and successfully detects a broader range of third-party coding agents (AI assistants that help you write code). Under the hood, the universal agent's skill installation now correctly targets `~/.agents/skills`. Most importantly, if you previously told the command exactly where to install a skill by passing the `--dir` flag, it used to stubbornly interrupt you with an interactive prompt (a menu halting execution to ask you a question) demanding you choose an agent anyway; it now honors your exact folder path and keeps quiet. It's like buying a universal remote that just learned the infrared codes for all the newest TVs; and if you explicitly point it at the TV and press power, it no longer pops up a tedious menu asking "Which device would you like to control?"

Fixes for Codespace port forwarding crashes and blank CI summaries

Your cloud environment will no longer crash from background data collisions, and cancelled checks won't leave you staring at a confusing blank screen. Previously, when using Codespace's port forwarding (a feature that connects your local machine to the network ports of your cloud-based development environment), if the underlying code experienced concurrent map writes (a fatal crash caused when two threads frantically try to edit the same data table at the exact same time), the entire forwarding feature would collapse. The concurrent write conflict has been fixed so it no longer crashes. Furthermore, when all the automated CI checks (a series of tests and verifications that run automatically) happen to get cancelled, the summary screen used to just display a dead, empty void that left you guessing. It now clearly and explicitly states that the checks were cancelled. It's like a busy restaurant kitchen: previously, if two waiters tried to scribble on the same order ticket at the same time, the paper would rip and shut down the kitchen; now the bug causing the paper to rip has been fixed. And if an entire table cancels their order, the kitchen board correctly reads "All Cancelled" instead of just sitting there as a bafflingly blank chalkboard.

Codespace 転送時のクラッシュと CI 状態が空白になるバグを修正

開発環境がデータの同時書き込みでクラッシュすることはなくなり、検査リストが取り消された際も訳の分からない空白が残ることはなくなった。以前、Codespace のポートフォワーディング（codespace port forwarding、クラウド上の開発マシンのネットワーク接続口を手元のパソコンと繋ぐ機能）を利用している最中、背後のコードで並行書き込みの衝突（concurrent map writes、2つのプログラムが同じデータ表を同時に書き換えようとして起こる致命的なクラッシュ）が発生すると、転送機能全体が崩壊していた。今回この衝突バグが修正され、クラッシュしなくなった。また、自動チェック（CI checks、自動で走る一連のテストや検証）がすべてキャンセルされた場合、以前の要約画面は無機質な空白を表示するだけでユーザーを困惑させていたが、今は正直に「すべてキャンセル済み」という要約を明示してくれる。繁盛しているレストランの厨房で、以前は2人のウェイターが同じ伝票の枠に同時に注文を書き込もうとして紙が破れ、厨房が完全にストップしていたのが、今は紙が破れる問題が直ったようなものである。さらに、あるテーブルの客が注文をすべて取り消した場合、厨房の黒板には何も書かれずに謎の余白が残るのではなく、「このテーブルはキャンセル」と明確に記されるようになった。

Codespace 포워딩 충돌 및 CI 상태가 빈 화면으로 뜨는 문제 해결

동시에 데이터를 밀어 넣느라 개발 환경이 뻗어버리는 일이 사라졌고, 검사 목록이 취소되었을 때 혼란스러운 빈 화면만 남는 일도 없어졌다. 예전엔 Codespace의 포트 포워딩(codespace 포트 포워딩, 클라우드 개발 머신의 네트워크 연결 통로를 내 컴퓨터와 뚫어주는 기능)을 쓸 때, 뒤에 있는 코드가 동시 쓰기 충돌(concurrent map writes, 두 프로그램이 똑같은 데이터 표를 동시에 고치려다 부딪혀 죽는 현상)을 일으키면 포워딩 기능 전체가 와르르 무너졌다. 이제 이 동시 쓰기 충돌 버그가 고쳐져서 더 이상 죽지 않는다. 또한 자동 검사(CI checks, 자동으로 실행되는 일련의 테스트와 점검)가 몽땅 취소되어 버리면, 예전의 요약 화면은 딱딱하게 텅 빈 백지상태만 띄워서 사람을 어리둥절하게 만들었다. 이제는 '모두 취소됨'이라는 요약을 정직하게 보여준다. 바쁜 식당 주방에서, 예전엔 두 종업원이 같은 주문표 칸에 동시에 글씨를 쓰려다 종이가 찢어져 주방이 마비되었다면 이제는 종이가 찢어지던 문제가 해결된 것과 같다. 그리고 그 테이블 손님이 주문을 싹 다 취소하면, 주방 게시판이 사람 헷갈리게 텅 비어 있는 대신 '이 테이블 취소됨'이라고 분명히 적어주는 식이다.
