技能可裝進更多 AI 工具，搜尋與發布更不容易搞錯

你可以用 `gh skills install` 把同一個技能裝進多個 agent host（承載 AI 代理工具的應用程式），不必逐一手動複製。安裝時可選擇允許隱藏資料夾，搜尋也會認得巢狀的 `skills/` 資料夾；若發現某技能曾重新發布，會提供安裝上游版本的選項。預覽會依安裝名稱正確比對，避免看錯技能。`gh` 與 `gh-skill` 的 agent skills 文件也已補上。`skills publish --fix` 也終於只修正問題、不會意外發布，而介面刪除了誤導文字。像替一把萬用鑰匙配多扇門：門更多、標籤更準，修鎖時也不會順手把門打開。

新增遙測控制與紀錄，別名不會被送出

你能用新增的 telemetry 指令查看或調整遙測（telemetry，匿名收集的使用紀錄）設定，也不用再靠環境變數才啟用。GitHub CLI 開始以抽樣方式記錄指令使用情況，並特別記錄技能、官方擴充功能、CI 環境（自動執行測試或部署的環境）與 AI 代理呼叫的脈絡；若沒有可送出的遙測資料，也會留下紀錄說明。相反地，你自訂的別名不會被送出。這些資訊用來了解功能如何被使用，不代表記下每次操作。像商店抽樣統計哪些貨架常被看，但不抄下每位客人替商品取的綽號。

en

Skills install into more AI tools, with safer discovery and publishing

You can now use `gh skills install` to put one skill into multiple agent hosts, the apps that run AI agents, without copying it by hand to each one. Installation can allow hidden directories, and discovery recognizes nested `skills/` directories. When a skill has been republished, the CLI can offer its upstream version for installation. Preview matches by install name, so you see the right skill. Documentation now includes `gh` and `gh-skill` agent skills. `skills publish --fix` now fixes issues without accidentally publishing, and misleading interface text has been removed. It is like fitting one master key to more doors: there are more doors, clearer labels, and fixing a lock does not also open it.

Telemetry gets controls and context, while aliases stay private

You can use the new telemetry command to inspect or adjust telemetry settings, and it no longer needs an environment variable to be enabled. GitHub CLI now samples command use and records context for skills, official extensions, CI environments that run tests or deployments automatically, and AI-agent invocations. When there is no telemetry to send, it records that fact too. Your custom aliases, however, are not sent. This data helps show how features are used; it is not a ledger of every action. It is like a shop sampling which shelves people view, without copying the nicknames each shopper gives its products.

テレメトリに制御と実行状況を追加し、別名は送信しない

新しい telemetry コマンドで、テレメトリ（匿名化して集める利用記録）の設定を確認・調整でき、環境変数がなくても有効にできる。GitHub CLI はコマンド利用を抽出して記録し、スキル、公式拡張機能、CI 環境（テストやデプロイを自動実行する環境）、AI エージェント呼び出しの文脈も記録する。送るテレメトリがない場合も、そのことを記録する。一方、ユーザー定義の別名は送信しない。これは機能の使われ方を把握する情報であり、すべての操作を書き留めるものではない。店がどの棚を見られたかを抽出集計しても、客が商品につけた呼び名までは写さないようなものである。

스킬을 더 많은 AI 도구에 설치하고, 탐색과 공개도 더 정확해진다

`gh skills install`을 쓰면 하나의 스킬을 여러 agent host, 즉 AI 에이전트를 실행하는 앱에 일일이 복사하지 않고 설치할 수 있다. 설치할 때 숨김 디렉터리를 허용할 수 있고 탐색은 중첩된 `skills/` 디렉터리도 알아본다. 다시 공개된 스킬을 발견하면 상위 버전을 설치하는 선택지도 제시한다. 미리보기는 설치 이름으로 정확히 대조해 다른 스킬을 잘못 보지 않는다. `gh`와 `gh-skill` agent skills 문서도 보탰다. `skills publish --fix`는 문제만 고치고 실수로 공개하지 않으며, 오해를 부르는 문구도 없앴다. 만능 열쇠를 더 많은 문에 맞추고 자물쇠를 고쳐도 문까지 열어 버리지는 않는 것과 같다.

원격 측정에 제어와 실행 맥락을 더하고, 별칭은 보내지 않는다

새 telemetry 명령으로 원격 측정(익명으로 모으는 사용 기록) 설정을 확인하거나 조절할 수 있으며, 더는 환경 변수가 있어야만 켜지지 않는다. GitHub CLI는 명령어 사용을 표본으로 기록하고 스킬, 공식 확장 기능, CI 환경(테스트나 배포를 자동으로 실행하는 환경), AI 에이전트 호출의 맥락도 기록한다. 보낼 원격 측정 자료가 없을 때도 그 사실을 남긴다. 반대로 사용자가 만든 별칭은 보내지 않는다. 이는 기능이 어떻게 쓰이는지 알아보기 위한 정보이지 모든 조작을 적는 장부가 아니다. 가게가 어느 진열대를 보는지 표본 조사하되 손님이 물건에 붙인 별명까지 베끼지는 않는 것과 같다.
