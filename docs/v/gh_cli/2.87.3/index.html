en

Fix query variables used when changing project data

When you change Project data—a GitHub board for organizing work—through gh, this release fixes how a mutation query uses its variables. A mutation query is a request telling the server to change data, and its variables are the field values attached to that request. Even with the request written correctly, passing those values incorrectly can keep an update from completing as expected. The release note does not name the affected fields or an error screen, so this is a targeted fix rather than a new feature. It is like sending a completed form in the right envelope: the form is unchanged, but the recipient can now process it correctly.

ko

프로젝트 데이터를 바꿀 때 쓰는 쿼리 변수를 수정한다

gh에서 Project(GitHub에서 작업 항목을 정리하는 보드) 데이터를 바꿀 때 mutation query가 변수를 쓰는 방식을 고쳤다. mutation query는 서버에 데이터를 바꿔 달라고 요청하는 조회이고, 변수는 그 요청에 붙이는 항목 값이다. 요청 자체를 맞게 썼더라도 값을 넘기는 방식이 틀리면 업데이트가 기대대로 끝나지 않을 수 있다. 업데이트 기록에는 영향받은 항목이나 오류 화면이 적혀 있지 않으므로 새 기능으로 부풀리지 않는다. 작성한 서류의 내용은 그대로 두고 올바른 봉투에 넣어 보내 수신처가 처리하게 하는 것과 같다.
