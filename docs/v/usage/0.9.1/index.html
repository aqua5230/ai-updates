發布日期：2026-05-23

zh-TW

終端機介面不再卡死，底層檔案掃描更省力

之前在終端機裡看用量時，因為一個程式迴圈寫錯（不小心跳過了更新步驟），導致畫面永遠凍結在剛打開的那一秒，再也不會自動更新。現在這個迴圈修好了，畫面會乖乖定時刷新。同時，我們也優化了底層的檔案掃描：以前每次更新畫面，系統都會笨笨地把同一個歷史紀錄檔案重複讀 4 次來算不同時間區間的數字；現在它學聰明了，一次讀完最長的區間然後直接往下分派，大幅省下白費力氣的硬碟讀取。另外，舊版的環境變數和過時的內部代號也一併統一了。就像是顧門口的警衛以前一直忘記看手錶導致從來不換班，而且查房時還同一層樓爬上爬下巡了四遍，現在終於準時交接，而且知道從頂樓一次巡下來最省事。

Original changelog
**TUI polling never updated after first fetch**: a `continue` in `poll_usage` caused every timeout to jump back to the loop head, leaving the UI frozen at the initial state. Changed to `pass` so the polling path is actually reached.
**Inconsistent env var name**: `USAG_FORCE_GROUP` (v0.1.x legacy prefix) renamed to `USAGE_FORCE_GROUP` to match all other env vars in the project.
**Redundant filesystem scans per refresh**: `_refresh_in_background` was calling `history_loader.load_entries` four times per cycle (24h × 2, 168h × 1, 720h × 1). Now loads the 720h superset once and passes it down, eliminating the duplicate I/O.
`pricing.py` User-Agent updated from the stale `usage/0.2` to `usage/0.9`.
`--setup` no longer prints a "no migration needed" message on clean installs.

en

Terminal UI stops freezing, and underlying file scans are much more efficient

Previously, when viewing your usage in the terminal, a buggy programming loop (which accidentally skipped the update step) caused the screen to freeze permanently at the exact second you opened it, never automatically refreshing again. This loop is now fixed, and the screen faithfully updates on time. Simultaneously, we've optimized the underlying file scans: before, every time the screen refreshed, the system foolishly read the exact same history file 4 times to calculate numbers for different time ranges; it's now smart enough to read the longest range once and pass the data down, drastically cutting down on wasted hard drive reads. Additionally, legacy environment variables and stale internal names have been unified. It's like a security guard who used to forget to check his watch and never changed shifts, while also climbing up and down the same floor four times during a patrol — now he hands over exactly on time and knows it's easiest to just patrol from the top floor down once.

ja

端末画面がフリーズしなくなり、ファイルのバックグラウンドスキャンも効率化

以前、端末で自身の使用量を見る際、プログラムのループの記述ミス（誤って更新のステップを飛ばしていた）により、開いた瞬間の状態で画面が永久に凍りつき、自動更新されなくなる問題があった。今回このループを修正し、画面がきちんと定期的に更新されるようになった。同時に、裏側のファイルスキャンも最適化した。以前は画面を更新するたびに、システムが愚直に同じ履歴ファイルを4回も読み込んで異なる期間の数値を計算していたが、今では最も長い期間のデータを一度だけ読み込んで振り分けるように賢くなり、無駄なハードディスクの読み込みを大幅に削減した。さらに、旧バージョンの環境変数や古びた内部コードも統一された。門番が以前は時計を見るのを忘れて一向に交代せず、見回りの時も同じ階を4回も上り下りしていたのが、今は時間通りに引き継ぎを行い、最上階から一度に巡回して降りてくるのが一番楽だと学習したようなものである。

ko

터미널 화면이 더 이상 멈추지 않고, 밑바탕 파일 스캔도 훨씬 효율적으로 바뀜

예전엔 터미널에서 사용량을 볼 때 프로그램 반복문에 오류가 있어(실수로 업데이트 단계를 건너뜀) 화면이 갓 열렸던 그 1초에 영원히 얼어붙은 채 다신 알아서 새로고침되지 않았다. 이제 이 반복문을 고쳐서 화면이 얌전히 제때 갱신된다. 이와 함께 밑바탕의 파일 스캔도 다듬었다. 예전엔 화면을 새로고침할 때마다 시스템이 멍청하게 똑같은 대화 기록 파일을 4번씩 다시 읽어가며 여러 시간대의 숫자를 계산했다. 이제는 제일 긴 구간을 한 번에 다 읽고 아래로 나눠 주는 법을 배워서, 헛수고하던 하드디스크 읽기를 크게 줄였다. 또한, 구버전 환경 변수와 낡은 내부 이름도 함께 통일했다. 정문을 지키던 경비원이 예전엔 시계 보는 걸 깜빡해 아예 교대하지도 않고, 순찰할 땐 똑같은 층을 네 번씩 오르락내리락하던 것을, 이제는 제시간에 딱 맞춰 교대하고 꼭층에서부터 한 번에 쭉 순찰하며 내려오는 게 제일 편하다는 걸 알게 된 것과 같다.
