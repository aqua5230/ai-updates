選單列加開關，要 Claude Code 少廢話只講重點

想讓 Claude Code 少廢話、直接講重點，現在選單列多了一個開關能搞定。這個新開關叫「Terse Mode（精簡模式）」，打開後它會要求 Claude Code 在你整個工作階段（session，一段連續的對話期間）裡回答得更精簡——砍掉那些模稜兩可的墊話（hedge，像「可能、也許、我不太確定」這類弱化語氣、沒把話講死的字）、填充廢話（filler，湊字數卻沒訊息量的話），以及每次開頭都重複講一遍的引言（preamble）；但同時會原封不動保留程式碼、指令、檔案路徑跟錯誤訊息，這些技術細節一個字都不會改。它完全跑在你自己電腦上（其實只是一個 SessionStart hook，也就是每次開新對話時自動執行的小設定腳本，不會呼叫任何 API），預設是關閉的。打開後，它會在你第一則回覆的開頭先自我介紹一下——如果同時也開了 Progress Concierge（進度管家，另一個開場打招呼的功能），就併進管家的招呼詞裡一起講，不然就自己單獨一行。要留意的是，跟任何「風格指令」一樣，這是一個請求而非硬性限制：當對話拖得很長時，它會慢慢淡化、漂回 Claude 平常比較囉嗦的樣子，這時你只要再提醒一句就能把它叫回來。就像跟一個話很多的同事說「簡短講重點就好」，他會乖乖精簡，但聊著聊著又會不知不覺越講越長，這時你只要再咳一聲提醒他就行。

Original changelog
**Terse Mode**: a new menu-bar toggle that asks Claude Code to answer more tersely for the whole session — cutting hedging, filler, and repeated preamble while keeping code, commands, file paths, and error messages byte-exact. Fully local (just a SessionStart hook, no API calls), off by default, and announces itself at the start of your first reply — merged into the Progress Concierge's greeting when both are on, or on its own line otherwise. Like any style instruction, it's a request rather than a hard constraint: on a very long conversation it can gradually fade and drift back to Claude's normal verbosity, at which point a one-line reminder brings it back.

en

A menu-bar toggle asks Claude Code to cut the chatter and get to the point

Want Claude Code to cut the chatter and get straight to the point? A new menu-bar toggle handles it. This new switch, "Terse Mode", asks Claude Code to answer more tersely for your entire session (session — one continuous stretch of a conversation) — trimming hedging (hedge — words that soften the claim without committing, like "maybe, perhaps, I'm not entirely sure"), filler (empty words that pad length but carry no information), and the repeated preamble it prints every time it starts; but it leaves code, commands, file paths, and error messages byte-exact, so not a single technical detail is altered. It runs fully on your own machine (it's really just a SessionStart hook — a small setup script that runs automatically whenever a new conversation starts, making no API call), and is off by default. Once on, it announces itself at the start of your first reply — folded into the Progress Concierge's greeting if that other feature (an opener that greets you at the start of a session) is also on, or on its own line otherwise. Note that, like any "style instruction", it's a request rather than a hard constraint: in a very long conversation it can gradually fade and drift back to Claude's normal verbosity, at which point a one-line reminder brings it back. It's like telling a talkative coworker "keep it short, just the point"; they'll trim their words, but as the chat rolls on they'll unconsciously start rambling again — at which point a single cleared throat is enough to rein them in.

メニューバーにトグルを追加し、Claude Code に無駄口を叩かず要点だけ話させる

Claude Code に無駄口を叩かせず要点だけ話させたいなら、メニューバーに新設されたトグルで解決する。この新しいスイッチは「Terse Mode」といい、オンにするとセッション（session、一連の連続した対話期間）全体にわたり Claude Code の回答をより簡潔にするよう要求する。すなわち、曖昧な逃げ言葉（hedge、「たぶん、もしかすると、完全には確信が持てない」のように主張を丸め、断言を避ける言葉）や、中身のない埋め草（filler、文字数を稼ぐだけで情報量のない言葉）、そして毎回冒頭で繰り返される前置き（preamble）を削り落とす。ただしコード、コマンド、ファイルパス、エラーメッセージはバイト単位でそのまま残し、技術的な細部は一文字も変えない。この機能は完全にローカルで動く（実態は SessionStart hook、すなわち新しい対話を始めるたびに自動で走る小さな設定スクリプトであり、いかなる API 呼び出しも行わない）もので、デフォルトはオフである。オンにすると、最初の返答の冒頭で自らを名乗る。別機能の Progress Concierge（セッション開始時に挨拶するオープナー機能）も同時にオンならその挨拶に折り込まれ、そうでなければ単独の一行として示される。留意すべきは、あらゆる「スタイル指示」と同様、これが強制ではなく要求だという点である。対話が長引くと次第に薄れ、Claude 本来の饒舌さへと漂い戻ることがあるが、その際は一行で改めて注意を促せば引き戻せる。まるでよく喋る同僚に「手短に要点だけ話して」と頼むようなもので、彼は大人しく簡潔になるが、話が進むうちにまた知らず知らず長くなり、そこで咳払い一つで再び締められるのと同じである。

메뉴 바에 토글을 추가해, Claude Code에게 잔소리를 줄이고 요점만 말하게 한다.

Claude Code가 잔소리를 줄이고 곧장 요점을 말하게 하고 싶다면, 메뉴 바에 새로 생긴 토글 하나로 해결된다. 이 새 스위치는 'Terse Mode'라 하여, 켜면 세션(session, 이어지는 대화 기간 한 덩어리) 전체에 걸쳐 Claude Code가 더 간결하게 답하도록 요구한다. 즉 얼버무리는 말(hedge, '아마도, 어쩌면, 완전히 확신은 안 됩니다'처럼 말을 부드럽게 하되 단언을 피하는 말)과 알맹이 없는 채움말(filler, 글자 수만 채울 뿐 정보가 없는 말), 그리고 매번 서두에 되풀이되는 서론(preamble)을 잘라 낸다. 다만 코드, 명령어, 파일 경로, 오류 메시지는 한 글자도 건드리지 않고 그대로 둔다. 이 기능은 온전히 내 컴퓨터에서 돌아간다(실체는 SessionStart hook, 새 대화를 시작할 때마다 자동으로 실행되는 작은 설정 스크립트로 어떤 API도 부르지 않는다). 기본값은 꺼짐이다. 켜면 첫 답변 서두에서 스스로를 소개하는데, 다른 기능인 Progress Concierge(세션 시작 때 인사하는 기능)도 함께 켜져 있으면 그 인사말에 아울러 담고, 아니면 따로 한 줄로 알린다. 주의할 점은, 어떤 '문체 지시'와 마찬가지로 이것은 강제가 아니라 요청이라는 것이다. 대화가 길어지면 점차 옅어져 Claude 본래의 말많은 성격으로 다시 흘러갈 수 있는데, 그때 한 줄 다시 일깨워 주면 되돌아온다. 마치 말 많은 동료에게 '짧게 요점만 해'라고 부탁하면 순순히 간결해지지만, 얘기가 흐르다 보면 또 모르는 사이 길어지고, 그때 헛기침 한 번이면 다시 조이는 것과 같다.
