zh-TW

狀態警告變長，Claude 卡片不再被切掉

你打開 Classic popover（浮在畫面上的小面板）查看 Claude 用量時，底部跳出很長的警告也不怕最重要的一列消失。以前像 hook 壞掉（特定事件發生時會觸發的小程式）或需要重新啟動這類狀態訊息一變成兩行，面板固定的高度沒有多留約 30 像素，底部狀態提示就會擠壓 Claude 卡片；卡片又設定成超出範圍不顯示，於是每週重設日期被硬生生切掉。現在面板會替第二行警告留出空間。這像行李箱多塞了一件外套，從前拉鍊一拉就把裡面的行程表夾住；現在箱子會跟著撐開，該看的日期仍完整露在外面。

Original changelog
**Classic popover no longer clips the Claude card when the footer status wraps to two lines.** Long status warnings (e.g. hook broken/restart) push the footer status pill onto a second line, adding roughly 30px the fixed popover height never accounted for; flex then squeezed the overflow-hidden Claude card and clipped its weekly reset row.

5 小時自動工作階段不再錯過額度重設

你把 Auto-start 5-hour Session（每五小時自動維持一次連線的工作階段）交給它跑，現在不會因為時間慢慢飄掉而白白錯過新的額度。原本背後維持連線的 keeper，會用上一次 ping（定時送出的短訊號）的時間開始算五小時；但那個時間可能和真正的額度重設邊界不同步，偶爾便整次略過 ping，下一個使用時段得等你親自跟 Claude 說話才會啟動。現在它改以真正的重設邊界來判斷是否已送過訊號，而不是只看經過多久，所以時鐘漂移不會再漏掉一次。像是每班公車依站牌時刻發車，從前卻拿上一班乘客的下車時間重新計時；現在直接看站牌，下一班便能準時開出。

Original changelog
**Auto-start 5-hour Session could silently skip a reset boundary.** The keeper's fixed 5-hour self-throttle was measured from the last ping's own timestamp, which could drift out of sync with the real quota reset boundary and occasionally skip a ping entirely — leaving the next window to start only once you talked to Claude yourself. It now dedupes by the actual reset boundary instead of elapsed time, so drift can no longer cause a miss.

en

Long status warnings no longer crop the Claude card

When you open the Classic popover to check Claude, a long warning at the bottom will no longer hide the line you came for. Warnings such as a broken hook or a needed restart can wrap onto two lines. The popover's fixed height had not allowed for the extra roughly 30 pixels, so its footer squeezed the Claude card; because overflow was hidden, the weekly reset row got cropped. The panel now makes room for that second line. It is like packing one extra sweater into a suitcase: the old zipper caught the itinerary inside, while the new case expands and keeps the date visible.

Original changelog

Five-hour auto sessions no longer miss a quota reset

If you let Auto-start 5-hour Session keep a session alive, it will no longer miss a fresh quota window because its timing slowly drifts. The keeper used to measure its five-hour throttle from the previous ping's timestamp. That timestamp could fall out of step with the real quota reset boundary, occasionally skipping a ping altogether; the next window then started only after you spoke to Claude yourself. It now deduplicates pings against the actual reset boundary, not simply elapsed time, so drift cannot cause a miss. Think of a bus that used to schedule its next departure from the last passenger's exit time; it now follows the timetable at the stop.
