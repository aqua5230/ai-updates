zh-TW

面板萬一載入失敗，不會再永遠卡在錯誤畫面

你現在切換面板萬一遇到載入失敗，不會再永遠卡在錯誤畫面了。0.22.7 版雖然讓面板切換變快，但如果某個面板剛好讀取失敗顯示了錯誤畫面，它就會把那個錯誤畫面「快取」起來，導致你之後怎麼切換都只看得到那個錯誤，除非整個程式重開。現在系統學乖了，只有成功載入的面板才會被存起來；如果失敗了，它下次會再努力重讀一次。就像一台會記住畫面的電視機，以前連藍畫面死機都記下來天天播給你聽，現在它只記住正常的節目，碰到雜訊下次懂得自己重抓訊號。

Original changelog
**A panel that fails to load is no longer pinned to its error screen until restart**: the panel caching added in 0.22.7 stored whatever `build_view` returned — including the `ErrorPanelView` fallback shown when a panel's HTML can't be read — so once a load failed, switching back kept showing the error even after the file was available again. usage now caches only successfully-built web views; a failed build is shown but left uncached and rebuilt on the next switch, so a transient read failure recovers on its own.

en

Panels no longer get permanently stuck on the error screen if they fail to load

If you switch panels and encounter a loading failure, you will no longer get permanently stuck on the error screen. While version 0.22.7 made switching panels faster, it had a flaw: if a panel failed to load and showed an error screen, it would "cache" that error screen, meaning you'd be stuck looking at that error every time you switched back until you restarted the app. The system has learned its lesson and now only caches successfully loaded panels; if it fails, it will simply try to reload it fresh next time. It's like a TV that records the last channel you watched — it used to record the blue screen of death and replay it for you endlessly, but now it only records actual broadcasts and knows to re-tune the antenna if it hits static.

ja

パネルの読み込みに失敗しても、永遠にエラー画面のまま固まることがなくなった

パネルの切り替え時に読み込みエラーが発生しても、永遠にエラー画面のまま固まることがなくなった。バージョン 0.22.7 でパネルの切り替えは速くなったものの、たまたま読み込みに失敗してエラー画面が出ると、そのエラー画面ごと「キャッシュ」してしまう問題があった。そのため、アプリを再起動しない限り、何度切り替えてもそのエラー画面しか見えなくなっていた。今はシステムが学習し、読み込みに成功したパネルだけを保存するようになった。失敗した場合は、次回もう一度読み込みをやり直してくれる。直前の画面を記憶するテレビが、以前はブルースクリーンまで律儀に録画して毎日流していたのが、今は正常な番組だけを覚え、ノイズが出た時は自ら電波を拾い直してくれるようになったようなものである。

Original changelog
**A panel that fails to load is no longer pinned to its error screen until restart**: the panel caching added in 0.22.7 stored whatever `build_view` returned — including the `ErrorPanelView` fallback shown when a panel's HTML can't be read — so once a load failed, switching back kept showing the error even after the file was available again. usage now caches only successfully-built web views; a failed build is shown but left uncached and rebuilt on the next switch, so a transient read failure recovers on its own.

ko

패널을 불러오다 실패해도 더 이상 영원히 오류 화면에 갇혀 있지 않음

패널을 바꾸다 불러오기 실패를 겪어도, 더 이상 영원히 오류 화면에 갇혀 있지 않게 되었다. 0.22.7 버전에서 패널을 빨리 바꾸게 해 준 건 좋았지만, 어떤 패널이 하필 불러오다 실패해 오류 화면을 띄웠다면 그 오류 화면까지 '캐시(임시 저장)'해 버렸다. 그래서 프로그램 전체를 껐다 켜지 않는 이상 아무리 패널을 바꿔도 그 오류 화면만 봐야 했다. 이제 시스템이 정신을 차려서, 성공적으로 불러온 패널만 저장해 둔다. 실패하면 다음번에 다시 처음부터 읽어오려고 노력한다. 화면을 기억하는 TV가 예전엔 파란 에러 화면까지 녹화해서 매일 틀어줬다면, 이제는 정상적인 방송만 기억하고 잡음이 나오면 다음번에 알아서 신호를 다시 잡는 것과 같다.
