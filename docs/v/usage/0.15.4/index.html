zh-TW

面板萬一當掉不再裝死，會直接告訴你哪裡出錯

萬一選單列的彈出面板（popover，顯示各項數據的小視窗）內嵌的網頁載入失敗，它不會再直接裝死不理你，或是留一片看不懂的空白灰畫面。現在它會換成一個原生的錯誤畫面，把具體的錯誤細節列出來，並附上一個能直接跳轉去回報問題的 GitHub 連結；如果你有另外設定 USAGE_DEBUG=1 這個環境變數（給進階使用者用的除錯開關），導覽失敗、畫面渲染逾時這些狀況還會被記錄下來，方便追查。這就像去銀行辦事，如果號碼牌機器壞了，以前它就直接黑畫面讓你傻站在那邊；現在它會在螢幕上顯示「機器故障，錯誤代碼 E-42」，還附上一張填好資訊、可以直接遞給維修人員的意見反映單。

Original changelog
**Panel load failures no longer degrade to a silent grey window**: when the popover's embedded web panel fails to load, it previously fell back to a blank dark window with no explanation. It now shows a native error view with the error detail and a GitHub report link, and logs navigation failures / render timeouts under `USAGE_DEBUG=1` for easier diagnosis.

en

Panel crashes no longer play dead, telling you exactly what went wrong

If the embedded web panel inside the popover (the small window showing your stats) fails to load, it no longer just silently freezes or leaves a blank grey window with no explanation. Now it shows a native error view listing the actual error detail, plus a GitHub link you can jump straight to for reporting the issue; if you separately set the USAGE_DEBUG=1 environment variable (an advanced debug switch), navigation failures and render timeouts also get logged for easier diagnosis. It's like going to the bank: if the ticket machine breaks, instead of just going dark and leaving you standing there confused, it now displays "Machine error, code E-42" on screen, complete with a pre-filled feedback slip you can hand straight to the technician.

ja

パネルが読み込めない時に沈黙せず、エラーの原因を明示するように

メニューバーから飛び出すパネル（popover、各種データを表示する小窓）に埋め込まれたウェブパネルが読み込めなかった際、フリーズしたり、説明のない灰色の空白画面が残ったりすることはなくなった。今はネイティブのエラー画面に切り替わり、実際のエラー詳細が表示され、問題を報告するための GitHub リンクへすぐ飛べるようになった。さらに USAGE_DEBUG=1 という環境変数（上級者向けのデバッグ切り替え）を別途設定していれば、ナビゲーション失敗や描画タイムアウトも記録され、原因追跡がしやすくなる。銀行の受付発券機が壊れた際、以前は真っ暗な画面のままで客を立ち往生させていたのが、今は「機械故障、エラーコード E-42」と表示し、記入済みの意見カードをそのまま技術者へ渡せるようになったようなものである。

패널이 먹통일 때 입을 꾹 다물지 않고 어디가 아픈지 바로 알려준다

메뉴 바에서 열리는 팝업 패널(popover, 각종 데이터를 보여주는 작은 창)에 내장된 웹 패널이 로딩에 실패했을 때, 이제 패널이 모른 척 먹통이 되거나 설명 없는 회색 빈 화면만 남기지 않는다. 대신 네이티브 오류 화면으로 바뀌어 실제 오류 세부 정보를 보여주고, 문제를 신고할 수 있는 GitHub 링크로 바로 이동할 수 있다. 또한 USAGE_DEBUG=1이라는 환경 변수(고급 사용자용 디버그 스위치)를 따로 설정해 두면, 탐색 실패와 렌더링 시간 초과도 기록되어 원인을 추적하기 쉬워진다. 은행 번호표 기계가 고장 났을 때 예전엔 그냥 까만 화면만 띄워놓고 사람을 바보로 만들었다면, 이제는 "기계 고장, 오류 코드 E-42"라는 안내문과 함께 미리 작성된 의견 카드를 기술자에게 바로 건네주는 것과 같다.
