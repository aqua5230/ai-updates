zh-TW

選單列數字卡住不更新時，現在會主動跳出警告教你修

選單列上釘著的用量百分比，如果因為背後的工具設定跑掉而卡住不更新，現在系統不會再讓你傻傻看著死掉的數字了。當它發現你明明還在聊天（專案紀錄有在動），但狀態檔卻超過 30 分鐘沒更新時，就會主動跳出警告，並教你重新跑一次 `--setup` 或重開程式來修復。它會保留最後一次抓到的真實數字，絕對不會自己亂掰，而且一樣完全不連網。就像你看著時鐘以為還早，現在時鐘沒電不走了，它會自己彈出一個牌子提醒你「我沒電了，快換電池」，而不是讓你錯過開會時間。

Original changelog
**Warns when the status line stops updating while you're still working**: the menu bar percentage comes only from the file the status line hook writes, so if that hook gets unwired (another tool rewrites `settings.json`) or stops firing, the pinned number would silently freeze. usage now detects this — when the status file hasn't updated for 30 minutes but your `~/.claude/projects` logs show recent activity — and shows an actionable warning telling you to re-run `--setup` or restart Claude Code. The last known percentage is kept, never fabricated, and there is still no network call.

en

The app now warns you and helps you fix a frozen menu bar percentage

If the pinned usage percentage on your menu bar gets stuck because the underlying tool's settings got disconnected, the system will no longer let you stare blindly at a dead number. If it detects that you're clearly still chatting (your project logs show activity) but the status file hasn't updated in over 30 minutes, it will proactively pop up an actionable warning teaching you to re-run `--setup` or restart the app to fix it. It preserves the last known real number, never fabricates data, and remains completely offline. It's like looking at a clock thinking you have plenty of time: instead of quietly dying on the wall and making you miss a meeting, the clock now pops up a sign saying "I'm out of battery, change it now."

ja

メニューバーの数字がフリーズした時に、自ら警告して直し方を教えてくれる

メニューバーに固定されている使用量のパーセンテージが、裏で動くツールの設定が外れたせいで更新されずに固まった場合、システムはもはやユーザーに死んだ数字を見せ続けたりしない。明らかにチャットを続けている（プロジェクトの記録が動いている）のに状態ファイルが 30 分以上更新されていないことに気づくと、自ら警告をポップアップさせ、`--setup` を再実行するかプログラムを再起動して直すよう教えてくれる。最後に取得した本物の数値を保持し、でっち上げることは絶対にしないし、通信も一切行わない。まだ時間があると思って時計を見ていたのに電池が切れて止まっていた時、以前は会議に遅刻する羽目になっていたのが、今は時計自ら「電池切れだ、早く交換して」と札を出して知らせてくれるようなものである。

ko

메뉴 바 숫자가 멈췄을 때 이제 알아서 경고를 띄우고 고치는 법을 알려줌

메뉴 바에 박혀 있는 사용량 백분율이, 뒤에서 도는 도구 설정이 풀려버려 멈춘 채 업데이트가 안 될 때 이제 시스템이 바보같이 죽은 숫자만 쳐다보게 내버려 두지 않는다. 당신이 분명히 계속 대화하고 있는데(프로젝트 기록은 움직이는데) 상태 파일이 30분 넘게 바뀌지 않는 걸 발견하면, 알아서 경고 창을 띄우고 `--setup`을 다시 돌리거나 프로그램을 다시 켜서 고치라고 일러준다. 마지막으로 잡았던 진짜 숫자만 그대로 두고 절대 제멋대로 지어내지 않으며, 인터넷 연결도 여전히 아예 없다. 아직 이른 줄 알고 시계를 보는데 건전지가 다 돼서 멈춰 있을 때, 알아서 '나 약 떨어졌어, 얼른 갈아 줘'라고 팻말을 튀어나오게 해 회의 시간을 놓치지 않게 해 주는 것과 같다.
