zh-TW

新增「隱藏 Codex 區塊」開關，不用 Codex 也能讓面板保持清爽

你現在可以在選單列裡找到一個「隱藏 Codex 區塊」的新選項。如果你平常只用 Claude 而不用 Codex，只要打開這個選項，所有 9 款 HTML 面板裡的 Codex 卡片就會全部收起來，彈出視窗（popover）的高度也會自動跟著縮減，幫你省下寶貴的螢幕空間。這個設定會被系統記住，重新開機也不用重調。對於想要自己新增面板的開發者，我們也加強了防呆機制——現在在程式碼裡強制規定必須明確指定 Codex 卡片的高度，如果忘了寫，系統在載入時就會立刻報錯提醒，而不是像以前一樣默默套用預設高度而導致版面錯位。這就像是買便當不吃配菜可以勾選「不要配菜」來換取更小的餐盒；同時廚房的點餐系統也加了防呆，沒填包裝盒尺寸的訂單直接退回，再也不會出現大便當裝小菜的尷尬情況。

Original changelog
**"Hide Codex Section" menu toggle**: the menubar gained a "Hide Codex Section" option that collapses the Codex card across all 9 HTML panels and shrinks popover height per-panel. The preference persists via `NSUserDefaults` so it survives restarts. i18n keys added for all 5 locales. (PR #19, thanks @RayCHWong for the first-time contribution)
**`HTMLPanel.codex_card_height` is now a required keyword-only argument with no default**: previously the parameter had a `192.0` default, so a new panel that forgot to set its height in `panels/__init__.py` would silently fall back to the default — the Codex card would render at a height that doesn't match the rest of the panel without raising any error. Now declared as `*, codex_card_height: float` (keyword-only, no default), so any missing call site raises `TypeError` at import. All 9 existing panels already pass it by keyword and are unaffected; added `test_html_panel_requires_explicit_codex_card_height` to lock the contract.

en

Hide the Codex section for a cleaner panel if you don't use it

You can now find a new "Hide Codex Section" toggle in the menubar. If you only use Claude and not Codex, checking this option will instantly collapse the Codex cards across all 9 HTML panels and automatically shrink the popover windows to match, saving you valuable screen real estate. This preference is saved automatically, so it stays exactly as you left it even after a restart. For developers building new custom panels, we've also added a strict safeguard—the codebase now explicitly requires you to specify the Codex card height. If you forget, it will immediately throw an error at load time instead of silently falling back to a default height that misaligns your layout. It's like ordering a bento box and ticking "no side dishes" so they give you a more compact box; meanwhile, the kitchen's ordering system was upgraded to reject tickets with missing box sizes, preventing tiny sides from awkwardly floating in a massive container.

メニューバーに「Codex 区画を隠す」オプションが追加された。Codex を使わず Claude だけを利用している場合、このオプションをオンにするだけで、9 種類の HTML パネルすべてで Codex カードが折りたたまれ、ポップオーバー（popover）の高さも自動的に縮小して貴重な画面スペースを節約できる。この設定は記憶されるため、再起動しても元に戻ることはない。また、自作パネルを追加したい開発者向けにフェイルセーフ機構も強化した。コード内で Codex カードの高さを明示的に指定することが必須となり、設定を忘れると、以前のようにデフォルトの高さが適用されてレイアウトが崩れるのではなく、読み込み時に即座にエラーを出して警告するようになった。これは、弁当を買う際におかずが不要なら「おかずなし」にチェックを入れてコンパクトな容器に入れてもらえるのと同じである。同時に厨房の注文システムにも、容器サイズの記入漏れがある注文は即座に突き返すフェイルセーフが組み込まれ、大きな容器に小さなおかずがポツンと入れられるような気まずい事態が起こらなくなった。

ko

쓰지 않는 Codex 영역을 숨겨 패널을 더 깔끔하게

메뉴 바에 'Codex 영역 숨기기' 옵션이 새로 생겼다. Codex 없이 Claude만 쓴다면 이 옵션을 켜서 9개의 모든 HTML 패널에서 Codex 카드를 접어버릴 수 있다. 팝업 창(popover) 높이도 알아서 줄어들어 소중한 화면 공간을 넓게 쓸 수 있다. 이 설정은 시스템이 기억하므로 컴퓨터를 껐다 켜도 그대로 유지된다. 패널을 직접 만들어 추가하려는 개발자를 위해 바보 방지(fail-safe) 기능도 강화했다. 이제 코드에서 Codex 카드의 높이를 반드시 지정하도록 강제되며, 깜빡하고 쓰지 않으면 예전처럼 기본 높이가 적용돼 화면 배치가 틀어지는 대신 시스템을 불러올 때 곧바로 오류를 뿜으며 알려준다. 마치 도시락을 살 때 반찬을 안 먹는다고 표시해 더 작고 깔끔한 용기에 받는 것과 같다. 동시에 주방의 주문 시스템에도 포장 상자 크기를 안 적은 주문은 아예 돌려보내는 안전장치가 추가되어, 큰 상자에 조그만 반찬이 나뒹구는 어색한 상황이 더는 생기지 않게 된 셈이다.
