zh-TW

拿掉底部容易誤導的模型標示，還給畫面清爽

你打開面板時，最下面不再顯示 Codex 模型的名稱了。這是因為之前顯示的其實是「上次有留紀錄時」用的模型，可能是好幾個小時前的事了，卻很容易讓人誤以為「現在這一秒」正在用它。因為這個資訊看了也只會讓人混淆，我們決定把它拿掉，讓畫面更乾淨。就像你去銀行抽號碼牌，如果螢幕上一直掛著「上一位客人的辦理項目」，你只會覺得一頭霧水；現在直接把它擦掉，清清爽爽。

Original changelog
**Removed Codex model footer from popover**: the "· model: gpt-5.5" suffix added in v0.11.6 (`menubar.py:868-870`) misled users into thinking the model was being used *right now*, when in fact it reflects the model of the most recent Codex session with rate_limits data — possibly hours old. Without a timestamp context, this information is noise that can't be acted on. TUI model displays (`ui/tables.py:818,857`) are kept since they live inside different contexts (active session block / idle panel). The `model_label` i18n key and `CodexRateLimits.model` field are preserved; only the popover footer concatenation is removed.

en

Removed misleading model labels from the footer to keep things clean

When you open the popover panel, you'll no longer see the Codex model name at the very bottom. This is because the label was actually showing the model used during "your last recorded session"—which could have been hours ago—leading people to mistakenly believe it was actively being used right that second. Since this out-of-context information was more confusing than helpful, we simply removed it to clean up the interface. It's like waiting at the bank: if the digital display kept showing the transaction type of the customer from three hours ago, you'd just be confused; now the board is wiped clean to avoid misunderstandings.
