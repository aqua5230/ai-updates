zh-TW

Antigravity 額度卡不再卡在錯誤百分比

你在 usage 看 Antigravity 額度時，卡片終於會跟官方 CLI 的 /quota 顯示對上。Antigravity CLI 1.1.5 後把 OAuth 憑證（證明你已授權登入的通行證）從硬碟上的普通檔案，搬進 macOS Keychain（Mac 替應用程式保管密碼與登入資料的鑰匙圈）。同時查額度的後端（真正處理查詢的伺服器）也換了地址。卡片卻仍翻舊 token（授權令牌）的檔案；它雖會悄悄更新，拿到的卻是過期授權算出的數字，於是百分比卡住。現在它先讀 Keychain 中 gemini／antigravity 的資料，找不到才回頭讀舊檔，並改連新版後端。像銀行把帳本搬進保險庫又換了櫃台，舊看板仍對著廢櫃台抄數字；現在先到保險庫、再去正確櫃台，額度才會和 CLI 一致。

Original changelog
**Antigravity quota showed a stuck, wrong percentage since the CLI's 1.1.5 update.** The Antigravity CLI moved its OAuth credential from a plain file on disk into the macOS Keychain (`security` service `gemini`, account `antigravity`, `go-keyring-base64:`-wrapped JSON) and switched its quota backend from `cloudcode-pa.googleapis.com` to `daily-cloudcode-pa.googleapis.com`. The card kept reading the now-abandoned token file, which silently refreshed against a stale grant and returned numbers that no longer matched the CLI's own `/quota` output. It now reads the Keychain entry first (falling back to the legacy file), and posts to the current backend.

en

Antigravity quota card no longer gets stuck on a wrong percentage

When you check Antigravity quota in usage, its card should finally agree with the official CLI's /quota result. Since Antigravity CLI 1.1.5, the OAuth credential (proof that you authorized the sign-in) moved from an ordinary disk file into macOS Keychain, where Macs keep passwords and login data. Its quota backend—the server handling the lookup—also moved. The card kept reading the old token file; it could refresh that abandoned credential, but received numbers based on a stale grant, leaving the percentage stuck and wrong. It now checks the gemini / antigravity Keychain entry first, falls back to the legacy file only when needed, and calls the new backend. It is like a bank moving its ledger into a vault and changing counters while an old display board still copies figures from the closed counter; now it visits the vault and the right counter first.

Antigravity のクオータカードが誤った割合で止まらなくなった

usage で Antigravity のクオータを見るとき、カードの表示が公式 CLI の /quota とようやく一致する。Antigravity CLI 1.1.5 以降、OAuth クレデンシャル（ログインを許可した証明）は普通のディスクファイルから、Mac がパスワードやログイン情報を保管する macOS Keychain へ移った。クオータを照会するバックエンド（問い合わせを処理するサーバー）も移転している。それでもカードは古いトークンファイルを読んでいたため、更新できても期限切れの許可に基づく数値を受け取り、割合が誤ったまま止まっていた。今は gemini／antigravity の Keychain 項目を先に読み、なければ旧ファイルへ戻り、新しいバックエンドにも接続する。銀行が帳簿を金庫へ移し窓口も変えたのに、古い掲示板だけが閉じた窓口を見ていた状態から、正しい金庫と窓口を確認する状態への変更である。

ko

Antigravity 할당량 카드가 잘못된 퍼센트에 멈추지 않는다

usage에서 Antigravity 할당량을 볼 때 카드 표시가 이제 공식 CLI의 /quota 결과와 맞는다. Antigravity CLI 1.1.5부터 OAuth 자격 증명(로그인을 허가했다는 증명)은 일반 디스크 파일에서 Mac이 암호와 로그인 정보를 보관하는 macOS Keychain으로 옮겨 갔다. 할당량 조회를 처리하는 백엔드(조회 요청을 처리하는 서버)도 바뀌었다. 하지만 카드는 계속 옛 token 파일을 읽었다. 그 버려진 자격 증명도 갱신은 됐지만 만료된 권한을 바탕으로 한 숫자를 돌려줘 퍼센트가 틀린 채 멈췄다. 이제 gemini／antigravity Keychain 항목을 먼저 읽고, 없을 때만 옛 파일을 읽으며 새 백엔드에 연결한다. 은행이 장부를 금고로 옮기고 창구도 바꿨는데 낡은 전광판만 문 닫은 창구의 숫자를 베끼던 상황에서, 이제 금고와 올바른 창구를 먼저 확인하는 것과 같다.
