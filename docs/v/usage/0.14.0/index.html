zh-TW

報表畫出你一整天什麼時段最常找 AI，而且不偷看內容

想知道自己一天到底都在什麼時段用 AI，現在報表會直接畫給你看。分析報表新增了「Usage Habits（使用習慣）」段落，把一整天 24 小時的活動量畫成一條橫跨整個畫面的直方圖（histogram，一種用高低柱子表示數量多寡的長條圖），並標出你的尖峰時段，旁邊附一句白話摘要，像是「你最常在 HH:00 和 HH:00 這兩個時間點找 AI」。這份資料完全來自你本地 Claude Code 紀錄檔裡訊息的時間戳（timestamp，像 2026-07-14 14:30 這種標記訊息發送時刻的字串），而且只看使用者與 AI 助理訊息的時間點——**絕不碰對話內容**，你們聊了什麼完全不會被讀取。整個功能跑在你自己電腦上，不送任何資料到雲端、不呼叫任何 API。就像一個只看打卡時間、絕不偷瞄你電腦螢幕的書記，幫你整理出你最常在哪些時段上工。

Original changelog
**"Usage Habits" section in the HTML report**: fully local, zero API. The analysis report now shows a full-width 24-hour activity histogram of when you work, highlighting your peak hour with a plain-language summary ("You most often work with AI around HH:00 and HH:00"). Data comes from the message timestamps in your local Claude Code logs (user / assistant messages only) — **never the conversation content**. Parsing lives in a standalone `persona_loader.py` with a 300s TTL cache.

Codex 數字太舊時面板會誠實標「N 分鐘前」，不讓你白等

現在當 Codex 的數字比較舊時，面板會直接告訴你「這是約 N 分鐘前的」，而不是讓你對著過時數字乾著急。Classic 面板的 Codex 卡片，當本地用量快照（snapshot，某個時間點拍下的資料定格照）超過 15 分鐘沒更新，就會顯示一個「about N minutes ago（約 N 分鐘前）」標籤，外加一個 ⓘ 資訊圖示的提示框（tooltip，滑鼠移上去才彈出的小說明）。為什麼會落後？因為跟 Claude Code 不同，Codex 沒有即時的狀態列掛鉤（status-line hook，一種讓程式把最新狀態即時塞進選單列的接掛點），它的用量數字只能靠它自己間歇寫進紀錄檔的工作階段（session）資料，自然會比你真實帳戶慢半拍。那個提示框也會說清楚：刻意保持離線、不上線查詢是設計選擇，為的就是絕不消耗你的 token（AI 計量的單位，回答越長花越多）。而這整個提示只用既有的 rate_limits.updated_at 欄位就能算出來——不用連網、不用 API。就像油耗表上如果數字是早上加油時記的，現在會貼一張「約 3 小時前」的便條提醒你別太相信，並附上說明：這台車刻意不連線回報，是為了不花你的通訊費。

Original changelog
**"Stale data" hint on the Codex card**: when the local Codex usage snapshot is older than 15 minutes, the classic panel's Codex card shows an "about N minutes ago" tag plus an info (ⓘ) tooltip. Unlike Claude Code, Codex has no live status-line hook, so its usage numbers come from session logs it writes only intermittently and can lag your real account; the tooltip also explains that staying offline is a deliberate choice so it never burns your tokens. Built from the existing `rate_limits.updated_at` — **no network, no API**.

Want to know which hours of the day you actually lean on AI? The report now charts it directly. The analysis report gained a new "Usage Habits" section that plots a full-width, 24-hour activity histogram (a chart of high and low bars showing how much activity falls in each slot), highlights your peak hour, and prints a plain-language summary beside it like "You most often work with AI around HH:00 and HH:00". The data comes entirely from the message timestamps (strings marking when each message was sent, like 2026-07-14 14:30) in your local Claude Code logs, looking only at the timing of user and assistant messages — **never the conversation content**; what you actually discussed is never read. The whole feature runs on your own machine, sending nothing to the cloud and calling no API. It's like a clerk who watches only the clock when you badge in and never glances at your screen, summing up which hours you work most.

When Codex's number goes stale, the panel honestly flags it "N minutes ago" so you don't wait in vain

When Codex's number goes stale, the panel now tells you straight up "this is about N minutes old" instead of leaving you staring at an outdated figure. On the classic panel's Codex card, once the local usage snapshot (a frozen data snapshot taken at one point in time) is older than 15 minutes, an "about N minutes ago" tag appears along with an ⓘ info tooltip (a small explanation that pops up when you hover). Why does it lag? Unlike Claude Code, Codex has no live status-line hook (a hookup point that lets a program push its latest state into the menu bar in real time), so its usage figures can only come from the session data it writes into its logs intermittently, naturally trailing your real account. The tooltip also explains that staying deliberately offline — not querying online — is a design choice so it never burns your token (the unit AI usage is metered in; longer answers cost more). And this entire hint is computed from the existing rate_limits.updated_at field — no network, no API. It's like a fuel gauge whose reading was taken at the morning fill-up now wearing a sticky note saying "about 3 hours ago" so you don't trust it blindly, with an attached note explaining: this car deliberately doesn't report in remotely, so it won't rack up your data charges.

보고서가 하루 중 언제 AI에 가장 많이 기대는지 그려주되, 내용은 훔쳐보지 않는다.

하루 중 언제 AI에 기대는지, 이제 보고서가 직접 그려 보여 준다. 분석 보고서에 'Usage Habits(사용 습관)' 구간이 새로 들었는데, 하루 24시간 활동량을 화면을 가로지르는 히스토그램(histogram, 간격마다 양을 막대 높이로 나타내는 막대그래프)으로 그리고, 가장 붐비는 시간대를 짚어 주며 옆에 알기 쉬운 한 줄 요약을 붙인다. 예를 들어 '당신은 보통 HH:00와 HH:00, 두 시각에 AI와 작업한다'처럼. 이 자료는 온전히 로컬 Claude Code 로그 안 메시지의 타임스탬프(timestamp, 2026-07-14 14:30처럼 메시지를 보낸 순간을 표시한 문자열)에서 나오며, 사용자와 AI 도우미 메시지의 시각만 본다. **대화 내용은 결코 들여다보지 않으므로** 무슨 말을 나누었는지는 전혀 읽히지 않는다. 기능 전체가 내 컴퓨터에서 돌아가고, 어디에도 자료를 보내지 않으며 그 어떤 API도 부르지 않는다. 마치 출퇴근 시각만 보고 절대 화면을 훔쳐보지 않는 서기가 당신이 주로 어떤 시간대에 일하는지 정리해 주는 것과 같다.

Codex 숫자가 오래되면 패널이 솔직하게 'N분 전'을 표시해 헛기다리지 않게 한다.

Codex의 숫자가 오래되면 패널이 '이건 약 N분 전 것'이라 솔직히 알려 주어, 낡은 숫자를 보며 헛되이 애태우지 않게 했다. Classic 패널의 Codex 카드는 로컬 사용량 스냅샷(snapshot, 어느 시점에 찍어 낸 자료 정지 사진)이 15분 넘게 갱신되지 않으면 'about N minutes ago(약 N분 전)' 표지와, 마우스를 올려야 뜨는 ⓘ 정보 도구 설명(tooltip, 올렸을 때 비로소 나타나는 짧은 해명)을 보여 준다. 왜 늦는가? Claude Code와 달리 Codex에는 실시간 상태 줄 걸이(status-line hook, 프로그램이 최신 상태를 메뉴 바에 즉시 밀어 넣는 연결점)가 없어, 사용량 숫자는 Codex이 간헐적으로 로그에 적어 넣는 세션(session) 자료에 기댈 수밖에 없고 당연히 실제 계정보다 반 박자 늦는다. 그 도구 설명은 또 분명히 적어 둔다. 일부러 오프라인을 유지하고 온라인으로 조회하지 않는 것은 설계상 선택이며, 그래야 당신의 토큰(token, AI 사용량을 재는 단위. 답이 길수록 더 든다)을 절대 태우지 않기 때문이라고. 그리고 이 안내 전체는 기존 rate_limits.updated_at 필드만으로 계산되어, 네트워크도 API도 필요 없다. 마치 연비계 수치가 아침 주유 시에 기록된 것이라면 '약 3시간 전' 메모지를 붙여 무작정 믿지 말라 일러 주고, '이 차는 통신비를 들이지 않으려 일부러 원격 보고를 하지 않는다'는 설명을 덧붙인 것과 같다.
