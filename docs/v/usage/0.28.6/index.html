Windows 控制台不再亂碼，進度條與設定順利顯示

在老舊的 Windows 控制台裡執行指令時，你不會再被莫名其妙的亂碼或錯誤中斷了。以前在傳統的 cp950（繁體中文的舊編碼）控制台環境下，跑 `--setup` 和 `--unsetup` 時若遇到像「✓」這樣的在地化符號，系統會因為讀不懂這些 Unicode（萬國碼）字元而直接失敗中斷。現在這些 hook（用來攔截並回報狀態的隱藏外掛）與設定輸出都全面改用 UTF-8（現代通用編碼），因此 `--setup` 系列指令不再報錯，而且當 Claude Code 從 pipe（用來傳遞資料的通道）讀取時，statusLine（顯示在最底下的狀態列）進度條也能順利保留原本的圖形字元。就像你拿著一本現代的表情符號字典去給只懂古文的老爺爺看，以前他會直接罷工，現在系統幫你請了個翻譯，爺爺也能看懂並笑著幫你辦事了。

Original changelog
**Windows hook and setup output now uses UTF-8**: statusLine bars keep their intended Unicode glyphs when Claude Code reads a pipe, and `--setup` / `--unsetup` no longer fail in legacy cp950 consoles when localized messages contain characters such as ✓.

Windows 面板支援自由拖曳，再也不會誤觸卡片排序

你現在可以隨心所欲地在螢幕上拖動 Windows 的系統匣面板了。只要按住無邊框面板頂端不顯眼的把手，或是額度卡片的空白處，游標就會變成抓取的圖示，讓你輕鬆移動視窗；系統也會記住你放的位置，下次打開還在原地，並確保它不會跑出螢幕外。更棒的是，以前在額度卡片空白處按壓可能會不小心觸發重新排列卡片的功能，現在 Windows 墊片（shim，用來轉換不同系統行為的中介層）會暫時把這些非互動區域標記為純拖曳區。這表示你可以放心用它來移動整個面板，不會打亂你原本排好的卡片順序，而卡片上的按鈕和連結依然可以正常點擊。就像一張原本黏死在桌布上的佈告欄，現在不只裝了滑輪讓你隨意推動，而且你推著板子邊緣時，也不用怕不小心把上面的便利貼給撕下來。

Windows console stops showing gibberish, progress bars and setup output display perfectly

You will no longer be interrupted by bizarre gibberish errors when running commands in legacy Windows consoles. Previously, in traditional cp950 (legacy text encoding) console environments, encountering localized symbols like "✓" during `--setup` and `--unsetup` would cause the system to fail because it couldn't understand those Unicode (universal character standard) characters. Now, these hooks (hidden plugins used to intercept and report status) and setup outputs strictly use UTF-8 (modern text encoding). As a result, setup commands no longer fail, and when Claude Code reads a pipe for the statusLine (the progress bar at the bottom), it successfully keeps its intended glyphs. It's like handing a modern emoji dictionary to a grandfather who only reads ancient text; he used to just walk away, but now the system provides a translator so he can understand and happily do his job.

Windows 사용자가 시스템 트레이나 패널 메뉴를 누르면 이 기능들이 깔끔하게 정리되어 있어 버튼을 못 찾는 일이 없다. 예전엔 Windows에서 패널 제어 메뉴를 열면 하위 메뉴가 창 가장자리에 무참히 잘리곤 했는데, 이제 '패널 변경'과 '섹션 숨기기' 같은 옵션이 스크롤 가능한 메뉴 안에서 제자리에 펼쳐진다. 그뿐만 아니라 Windows UI가 일일 업데이트와 워크플로 제어 측면에서 macOS와 기능이 같아졌다. 패널에 내장된 버튼을 누르면 현지화된 HTML(웹 마크업 언어) 오버레이가 떠서 각종 제어 기능을 제공하고, 트레이 메뉴에는 AI 일일 업데이트, Claude, Codex, Antigravity의 '섹션 숨기기' 하위 메뉴는 물론 '마지막 세션 이어서 하기'와 'token(AI 처리 단위) 절약 모드' 스위치까지 채워졌다. 화면 섹션 변경 사항은 열려 있는 패널에 즉시 적용되며, 할당량 경고는 macOS와 동일한 기준점 바탕으로 Windows 기본 알림을 제대로 쓴다. 패널 메뉴 맨 아래에 중복되던 업데이트, 위치 초기화, 종료 버튼도 싹 치워졌다. 새로 단장한 중앙 통제실과 같다. 예전엔 모니터가 좁아서 버튼이 잘려 누르지도 못하고 옆집 애플 부서보다 기능도 적었지만, 이제 위아래로 굴릴 수 있는 대형 스크린으로 바꾸고 이런 좋은 제어 버튼들까지 가져와 말끔하게 정리한 셈이다. Windows 패널을 자유롭게 끌고 다닐 수 있으며, 카드 순서를 실수로 섞는 일도 없다.

이제 화면에서 Windows 시스템 트레이 패널을 마음대로 끌고 다닐 수 있다. 테두리 없는 패널 맨 위의 눈에 잘 띄지 않는 손잡이나 할당량 카드의 빈 곳을 누르면 커서가 움켜쥐는 모양으로 변하며 창을 쉽게 옮길 수 있다. 시스템은 당신이 창을 둔 위치를 기억해 다음 번에도 그 자리에 열어주며, 화면 밖으로 벗어나지 않게 잡아준다. 더 반가운 소식은, 예전엔 할당량 카드 빈 곳을 누르면 실수로 카드 순서가 뒤섞이곤 했는데, 이제 Windows 심(shim, 서로 다른 시스템의 동작을 맞춰주는 중간 계층)이 이 반응 없는 공간을 임시로 '드래그 전용 구역'으로 지정한다. 즉, 애써 정렬해 둔 카드를 망가뜨릴 걱정 없이 패널 전체를 안심하고 옮길 수 있고, 카드 위 버튼과 링크는 여전히 잘 눌린다. 벽에 못 박혀 있던 게시판에 바퀴를 달아 마음껏 밀고 다닐 수 있게 되었고, 널빤지 가장자리를 잡고 밀 때 행여나 붙어 있던 포스트잇이 떨어질까 노심초사하지 않아도 되는 것과 같다.
