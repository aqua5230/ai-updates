zh-TW

獨立 App 產出報告時不再因為找不到神獸圖片而崩潰

如果你是直接執行打包好的獨立 App（.app），現在產出年度報告時不會再直接當掉了。以前在算年度回顧時，系統會去抓鳳凰或飛龍的圖片，但它找圖片的路徑是用給開發者看的原始碼路徑，所以在打包好的 App 裡根本找不到，導致整個報告出不來。現在它懂得在打包好的資源資料夾裡找圖片了，讓報告能順利生出來。就像你把食譜帶去別人家廚房煮菜，以前一直去原本家裡的位置找鹽巴所以煮不出來，現在懂得在別人家的調味料櫃裡找了。

Original changelog
**HTML report no longer fails inside the packaged .app**: the year-in-review spirit images (phoenix/dragon) were resolved with a source-tree path that doesn't exist in the py2app bundle, so generating any report raised `FileNotFoundError`. They are now resolved through the bundle's resource path, with the source-tree path kept as a fallback for source/CLI runs.

en

Packaged app stops crashing over missing spirit images when generating reports

If you're running the standalone packaged App (.app), generating the annual report will no longer cause it to crash outright. Previously, when calculating the year-in-review, the system tried to fetch the phoenix or dragon images using a source code path meant for developers. This path didn't exist inside the packaged app, causing the entire report generation to fail. Now, it knows to look for the images inside the bundled resources folder, ensuring the report generates smoothly. It's like taking a recipe to cook in someone else's kitchen: you used to fail because you kept reaching for salt in the exact spot it is in your own house, but now you know to look in the host's actual spice cabinet.

Original changelog
**HTML report no longer fails inside the packaged .app**: the year-in-review spirit images (phoenix/dragon) were resolved with a source-tree path that doesn't exist in the py2app bundle, so generating any report raised `FileNotFoundError`. They are now resolved through the bundle's resource path, with the source-tree path kept as a fallback for source/CLI runs.

ja

パッケージ化されたアプリが、神獣の画像が見つからないせいでレポート作成時にクラッシュしなくなった

パッケージ化された独立アプリ（.app）を直接実行している場合、年間レポートの作成時にいきなりクラッシュすることがなくなった。以前は年間振り返りを計算する際、システムが不死鳥や飛竜の画像を開発者用のソースコードパスで探しに行っていた。そのため、パッケージ化されたアプリ内では画像が見つからず、レポート全体が出力できなくなっていた。今はパッケージ内のリソースフォルダから画像を正しく探し出せるようになり、レポートがスムーズに作成される。他人の家のキッチンで料理をする際、以前は自分の家の定位置で塩を探し続けて料理が作れなかったのが、今は他人の家の調味料棚を探すことを覚えたようなものである。

ko

독립 앱이 보고서를 만들 때 신수 이미지를 못 찾아서 죽는 일이 없어짐

포장된 독립 앱(.app)을 바로 실행하는 경우, 이제 연간 보고서를 뽑아낼 때 곧바로 죽어버리지 않는다. 예전엔 한 해 돌아보기를 계산할 때 시스템이 불사조나 비룡 이미지를 가져오려 했지만, 개발자들이나 보는 소스 코드 경로를 써서 찾으려 했기 때문에 포장된 앱 안에서는 아예 찾지를 못해 보고서 전체가 나오지 못했다. 이제는 앱 안의 자원 폴더에서 이미지를 찾을 줄 알아서 보고서가 무사히 만들어진다. 마치 남의 집 주방에 요리법을 들고 가 요리할 때, 예전엔 원래 우리 집에서 소금 있던 자리만 뒤지다가 요리를 망쳤다면 이제는 남의 집 조미료장 안을 뒤질 줄 아는 것과 같다.
