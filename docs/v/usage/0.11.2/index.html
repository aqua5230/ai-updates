zh-TW

選單列反應快如閃電，背後運作更省力

這次更新讓選單列上的數字更新變得瞬間到位。以前系統是靠一個呆呆的計時器，每隔一分鐘才去查一次你的用量有沒有變，所以常常會慢半拍；現在它學會了直接監聽作業系統的檔案變動事件（FSEvents），你一燒掉額度，畫面在幾毫秒內就會立刻跳動。不僅反應變快，背後讀資料也變得更省力：現在它只會去解析真正有變動的歷史紀錄，不再每次都把沒變的舊檔案重讀一遍，大幅減輕你硬碟的負擔。另外，如果你同時開了好幾個終端機視窗在跑 Claude，現在存檔時會自動排隊鎖上檔案（檔案鎖），不會再發生兩邊同時寫入把用量互相覆蓋的慘劇。這就像是餐廳的出菜系統，以前服務生每隔一分鐘才慢吞吞地進廚房看菜好了沒，現在廚房一出菜就會按鈴，服務生馬上把菜端走，而且大家端菜時絕不會再撞在一起把盤子摔碎了。

Original changelog
**`usage_cli.py` crashed on every first run** (thanks @will30-blockchain — [#7](https://github.com/aqua5230/usage/pull/7)): `setup(auto=True)` passed a non-existent keyword argument to `setup_hook.setup()`, causing a `TypeError` on any fresh install or after `unsetup`. Users who already had the hook installed were unaffected. Fix: drop the stale `auto=True` kwarg.
**Incremental JSONL parsing**: `history_loader` and `codex_loader` now maintain module-level mtime+size caches and skip re-parsing files whose content hasn't changed, significantly reducing per-refresh disk I/O.
**Parallel hook forwarding**: `usage_statusline_forwarder` now dispatches all hooks concurrently via `ThreadPoolExecutor`; a single slow or timing-out hook no longer stalls the others. Worst-case latency drops from `n × 5s` to `5s`.
**Multi-session write protection**: `usage_statusline.py`'s `save()` now acquires `fcntl.LOCK_EX` before writing, preventing concurrent Claude Code sessions from clobbering each other's data.
**Python path resolution**: `setup_hook` now uses `_find_system_python()` when building hook commands — preferring the bundled `.app` Python, then `/usr/bin/python3`, avoiding the broken Xcode stub that `shutil.which("python3")` can resolve to after an Xcode update.
**FSEvents-driven UI refresh**: `menubar` now uses a CoreServices `FSEventStream` (via ctypes) to watch `~/.claude/`. Changes to `usage-status.json` trigger `_refresh()` immediately, cutting update latency from up to 60 seconds to milliseconds. `NSTimer` is demoted to a 300-second fallback; silently degrades to timer-only mode if CoreServices is unavailable.

en

Lightning-fast menu bar updates with drastically reduced disk work

The numbers on your menu bar will now update instantaneously. Previously, the system relied on a dumb timer that polled your usage once a minute, often lagging far behind reality. Now, it directly listens to your operating system's file events (FSEvents), so the moment you burn through some quota, the on-screen numbers twitch in milliseconds. Not only is it faster, but reading data behind the scenes is much lighter too: it now incrementally parses only the history files that have actually changed, saving your hard drive from redundantly reading identical old files over and over. Additionally, if you run multiple Claude Code sessions in different terminal windows, the system now uses file locks to make them line up politely when saving data, completely preventing them from clobbering each other's usage totals. It's like a restaurant's kitchen flow: waiters used to slowly poke their heads into the kitchen every minute to check for food, but now the kitchen rings a bell the millisecond a dish is ready — and the waiters no longer crash into each other dropping plates on their way out.

zh-CN

菜单栏反应快如闪电，背后运作更省力

这次更新让菜单栏上的数字更新变得瞬间到位。以前系统是靠一个呆呆的计时器，每隔一分钟才去查一次你的用量有没有变，所以常常会慢半拍；现在它学会了直接监听操作系统的文件变动事件（FSEvents），你一烧掉额度，画面在几毫秒内就会立刻跳动。不仅反应变快，背后读数据也变得更省力：现在它只会去解析真正有变动的历史记录，不再每次都把没变的旧文件重读一遍，大幅减轻你硬盘的负担。另外，如果你同时开了好几个终端机窗口在跑 Claude，现在存档时会自动排队锁上文件（文件锁），不会再发生两边同时写入把用量互相覆盖的惨剧。这就像是餐厅的出菜系统，以前服务生每隔一分钟才慢吞吞地进厨房看菜好了没，现在厨房一出菜就会按铃，服务生马上把菜端走，而且大家端菜时绝不会再撞在一起把盘子摔碎了。

ja

メニューバーの更新が電光石火に、裏の処理も省エネ化

今回の更新で、メニューバーの数字の反映が瞬時に行われるようになった。以前はシステムが鈍感なタイマーに頼り、1分ごとにしか使用量の変化を見に行かなかったため、表示が遅れがちだった。今は OS のファイル変更イベント（FSEvents）を直接監視するようになり、制限枠を消費した数ミリ秒後には画面がすぐに反応する。反応が速くなっただけでなく、裏でのデータ読み込みも軽くなった。本当に変更があった履歴ファイルだけを解析し、変化のない古いファイルを毎回読み直す無駄を省き、ハードディスクの負担を大幅に減らした。また、複数の端末ウィンドウで Claude を同時に走らせている場合でも、保存時に自動で順番待ちをするファイルロックがかかるようになり、両者が同時に書き込んでデータを上書きし合う惨事も防げる。レストランの配膳システムが、以前は1分おきにウェイターがのんびり厨房を覗き込んでいたのが、今は料理ができたら即座にベルが鳴り、しかもウェイター同士がぶつかって皿を割ることもなくなったようなものである。

ko

메뉴 바 반응 속도는 번개처럼 빨라지고, 보이지 않는 작업은 가벼워졌다

이번 업데이트로 메뉴 바의 숫자가 눈 깜짝할 새 즉각 반영된다. 예전엔 시스템이 멍청한 타이머에 기대어 1분마다 한 번씩 당신의 사용량이 바뀌었는지 들여다보는 바람에 한 박자씩 늦기 일쑤였다. 이제는 운영체제의 파일 변동 이벤트(FSEvents)를 직접 엿듣는 법을 배워, 당신이 한도를 태워버리면 몇 밀리초 안에 화면이 곧바로 꿈틀거린다. 반응만 빨라진 게 아니라, 뒤에서 데이터를 읽는 작업도 훨씬 수월해졌다. 이제는 정말 내용이 바뀐 역사 기록만 파악하며, 똑같은 옛날 파일을 매번 처음부터 다시 읽어대던 낭비를 없애 하드디스크의 부담을 확 줄였다. 또한, 터미널 창 여러 개를 띄워 놓고 Claude를 동시에 돌리더라도 이제 저장할 때 파일 잠금을 통해 알아서 줄을 서게 만들어, 양쪽에서 동시에 글을 쓰다 서로의 사용량을 덮어쓰는 대참사를 막았다. 식당의 서빙 방식이, 예전엔 웨이터가 1분마다 느릿느릿 주방에 얼굴을 내밀며 요리가 나왔나 확인하던 것에서, 이제 주방장이 요리를 내놓자마자 벨을 울리면 바로바로 음식을 나르고 웨이터들끼리 부딪혀 접시를 깨뜨릴 일도 없어진 것과 같다.
