zh-TW

系統匣不被舊額度混淆，刷新歷史紀錄不再卡頓

現在你可以在 Windows 系統匣中即時看到最準確的 AI 使用額度，而且檢視歷史紀錄時介面也變得滑順無比。以前當 hook（掛鉤，特定事件發生時被觸發的小程式）傳回全新的 statusLine（顯示在終端機最底下的狀態列）資料時，系統匣卻常被備援的舊 cache（快取，把算過或查過的結果暫存起來直接用）覆蓋，導致顯示過期或錯誤的額度；現在系統匣會優先採用最新的掛鉤資料，並嚴格把關防範錯誤格式。同時，系統匣掃描歷史工作階段的頻率也加上了 30 秒的暫存限制，免去每次重新掃描所導致的卡頓。就像看銀行的臨櫃螢幕，以前網路一抖就顯示昨天餘額，而且每次刷新都重讀交易明細害螢幕死當；現在能鎖定即時餘額，明細也改為定期刷新，省電又順暢。

Original changelog
**The Windows tray no longer falls back to a stale quota cache when the hook already has live data**: a complete statusLine payload from the hook could still be overridden by the `.claude.json` fallback snapshot whenever its cached timestamp looked newer, even though that cache might describe a different or expired session; the tray now always prefers a complete hook payload, and the "complete" check itself is now stricter — malformed non-numeric percentages in the hook payload no longer count as complete, so a broken payload can't silently blank the quota by skipping the cache.
**Windows tray history scanning no longer re-scans every Codex/Claude session log on each refresh tick**: `history_source_scan()` results are now cached for 30 seconds, removing the main source of UI jank in the tray.

en

Windows tray stops showing stale quota and speeds up history scans

You can now see your most accurate AI quota right in the Windows tray, with history scans running completely lag-free. Previously, the tray could override a fresh statusLine payload with a stale .claude.json fallback cache if its timestamp seemed newer, displaying outdated or expired limits; the tray now always prioritizes complete live data and rejects malformed values. Additionally, history source scanning is now cached for 30 seconds instead of checking logs on every refresh tick, eliminating UI jank. It is like a bank screen that used to show yesterday's balance on a network glitch and froze because it re-read every transaction on every refresh; it now locks onto real-time balances and updates your statement list on a smart delay.

ja

システムトレイの制限枠表示が正確になり、履歴スキャンも軽快に

Windowsのシステムトレイで最も正確なAIの利用制限枠をリアルタイムに確認できるようになり、履歴スキャン時のUIの引っ掛かりも解消された。以前は、フックが最新のステータスラインのデータを返しても、タイムスタンプが新しく見えると代替の古いキャッシュで上書きされ、期限切れの利用制限枠が表示されることがあった。今後は常に完全な生データを優先し、不正な値も厳格に排除する。また、履歴スキャンの実行結果を30秒間キャッシュすることで、毎回の更新時のログ再読み込みによる画面の引っ掛かり（jank）をなくした。これは、銀行の窓口モニターが、ネットワークの乱れで昨日の残高を表示し、更新のたびに取引明細を読み直してフリーズしていたのが、リアルタイム残高を固定し、明細は定期更新にしてスムーズに動くようになったようなものである。

시스템 트레이의 한도 표시가 정확해지고, 기록 스캔 속도도 빨라졌다

이제 Windows 시스템 트레이에서 가장 정확한 AI 사용 한도를 실시간으로 확인할 수 있으며, 기록을 조회할 때 화면 끊김 현상도 사라졌다. 이전에는 훅이 최신 상태 표시줄 데이터를 반환하더라도 타임스탬프가 더 최신인 것처럼 보이면 백업용 이전 캐시로 덮어씌워져 만료된 한도가 표시되곤 했다. 앞으로는 항상 온전한 실시간 데이터를 우선시하고 잘못된 값은 엄격히 차단한다. 또한 기록 스캔 결과를 30초 동안 캐싱하여 매번 업데이트할 때마다 로그를 재검색하면서 발생하던 화면의 끊김（jank）을 없앴다. 이는 은행 창구의 모니터가 네트워크 오류로 어제 잔액을 보여주고 업데이트할 때마다 거래 내역을 다시 읽느라 멈추던 것에서, 실시간 잔액을 고정하고 내역은 주기적으로 새로고침하여 부드럽게 작동하도록 개선된 것과 같다.
