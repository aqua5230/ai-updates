zh-TW

App 學會自己檢查更新了，且不會煩死你

你現在不用再時不時跑去 GitHub 網頁看有沒有新版本了。App 啟動時會自動去檢查有沒有新版，如果發現更新，會直接跳出視窗告訴你新版本的改動內容，讓你一鍵連過去下載。為了不打擾你，這個自動檢查一天最多只會跑一次；如果你覺得這個版本不重要，也可以點「跳過此版本」，它就不會再拿同一個版本煩你。當然，如果你想要手動控制，可以在「更換面板」選單裡關掉「自動檢查更新」，或者隨時點「立刻檢查更新」來強迫它查一次。就像你的汽車導航，現在懂自己在背景偷偷查圖資有沒有更新，有重大改變才問你要不要更新，而不是每次一發動就逼著你按確認。

Original changelog
**In-app update check (Stage 1)**: On launch, usage pings GitHub Releases for a newer version (rate-limited to once per 24h so you're not nagged every time you open the app). When a newer version is found, an NSAlert shows the version + release notes with three buttons: **Download**, **Later**, **Skip this version**. "Download" opens the Release page in your default browser — manually replace the old `.app` with the new one. (Stage 2 will bring Sparkle-style auto download + replace.)
**Two new entries in the "Switch panel" menu**:
- **Automatically Check for Updates** (toggleable): unchecking it disables the launch-time auto check entirely; the manual entry below still works.
- **Check for Updates Now**: manually triggers a check, bypassing the 24h cooldown and skip-version preference. If you're already up to date, an alert says so; on network error you see "Update check failed".
Preferences are stored in the existing `~/.claude/usage-preferences.json`, with three new keys: `auto_update_check` (default true), `update_dismissed_at` (Unix timestamp), `update_skipped_version` (skipped version string).
`setup_app.py` now bundles `pyproject.toml` and `update_checker` into the py2app build — so the packaged `.app` can fall back to reading `pyproject.toml` when `importlib.metadata` can't resolve the version.

en

The app now checks for updates automatically without annoying you

You no longer have to manually check the GitHub repository for new releases. On launch, the app now automatically checks for new versions and pops up a dialog with the release notes and a one-click button to download it. To respect your focus, this auto-check runs at most once every 24 hours. If a particular update doesn't interest you, you can hit "Skip this version" and it won't bother you about that specific release again. You are in full control — you can toggle off "Automatically Check for Updates" in the Switch Panel menu, or manually trigger a "Check for Updates Now" whenever you want. It's like your car's GPS silently checking for new maps in the background and only asking if you want to upgrade when there's actually a major new route, instead of forcing you to click OK every time you start the engine.

ja

アプリが自分で更新をチェックするようになり、しかも煩わしくない

今後、新しいバージョンが出ているか GitHub を逐一見に行く必要はなくなった。アプリ起動時に自動で最新版を確認し、更新があればウィンドウを出して変更内容を知らせ、ワンクリックでダウンロード画面へ案内する。作業の邪魔にならないよう、この自動チェックは1日に最大1回しか行われない。もしその更新が不要だと思えば「このバージョンをスキップ」を押せば、同じバージョンで二度と煩わされることはない。もちろん手動で管理したい場合は、「パネルを変更」メニューから「自動で更新を確認」をオフにしたり、いつでも「今すぐ更新を確認」をクリックして強制的に調べさせたりできる。カーナビが裏でこっそり地図の更新を確認し、本当に大きな変更があった時だけアップデートするか聞いてくるようになり、エンジンをかけるたびに確認ボタンを強要しなくなったようなものである。

ko

앱이 알아서 업데이트를 확인하며, 당신을 귀찮게 하지 않는다

이제 새 버전이 나왔는지 확인하러 GitHub 페이지를 들락거릴 필요가 없다. 앱을 켜면 알아서 새 버전을 확인한 뒤, 업데이트가 있으면 창을 띄워 변경 사항을 알려주고 버튼 한 번으로 다운로드 페이지로 넘어갈 수 있게 해준다. 작업에 방해되지 않도록 이 자동 확인은 하루에 딱 한 번만 실행되며, 당장 필요 없는 버전이라면 '이 버전 건너뛰기'를 눌러 다시는 알림을 받지 않을 수 있다. 이 모든 것은 당신이 직접 제어할 수 있다. '패널 변경' 메뉴에서 '자동 업데이트 확인'을 끄거나, 원할 때 언제든 '지금 업데이트 확인'을 눌러 억지로 한 번 확인하게 만들 수도 있다. 자동차 내비게이션이 이제 뒤에서 조용히 새 지도가 나왔는지 확인하다가 정말 중요한 길이 새로 뚫렸을 때만 업데이트할지 묻고, 시동을 걸 때마다 억지로 확인 버튼을 누르게 하지 않는 것과 같다.
