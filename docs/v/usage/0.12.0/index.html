報告大升級：幫你抓出訂閱方案，新增專案圓環圖

HTML 分析報告現在會顯示更多有用的資訊。它新增了「你的訂閱」區塊，會自動從你電腦裡的授權檔讀出你買的是哪種 Claude 方案或 Codex 的 ChatGPT 方案，以及訂閱開始日期（放心，它絕不會偷看你的密碼或 email），而且當你把報告分享給別人並開啟「隱藏專案名稱」時，訂閱日也會自動幫你打上馬賽克。報告裡還多了一張精美的「專案佔比圓環圖」，把最吃資源的前 6 大專案塗上不同顏色，剩下的歸類為「其他」，正中央大字寫著總用量，一目了然。此外，以前系統其實早就默默算好了 Claude 跟 Codex 兩邊的用量對決（花了多少 token、佔比與成本），現在這份「Claude vs Codex」的對比戰報終於解禁顯示出來了。就像你去體檢，以前只給你一張總結數字，現在不僅在開頭幫你印出你買的是哪種等級的健檢套餐，還附上精緻的彩色圓餅圖告訴你哪裡脂肪最多，甚至把你兩隻手的肌肉量拉出來做對比。

Original changelog
**"Your subscription" section in the HTML report**: auto-detects Claude (plan + subscription start date) and Codex (ChatGPT plan + subscription start date) from the local OAuth account files. Only the plan name and start date are read — tokens, emails and other secrets are never touched. When sharing the report, the subscription date is masked together with the "Hide project names" toggle. Adds the `subscription.py` module and its tests.
**Project-share donut chart in the HTML report**: pure-SVG (zero external deps) breakdown of token share per project; the top 6 projects get their own colour, the rest fold into "Other", and the centre shows the total.
**"Claude vs Codex" comparison section in the HTML report**: surfaces the per-agent usage (tokens / share / cost) that `build_report_data` already computed but never displayed.

整理底層管線，除錯也不再讓電腦做白工

產出報告的速度變得更快，因為系統改掉了背後的愚蠢計算方式。在計算報告的花費時，系統以前很糊塗，會先把所有數字加總一次，然後又在小圈圈裡一筆一筆重新算，如果你的資料量很大，它等於是在做雙倍的白工；現在改為只在小圈圈裡算一次，速度順暢多了。另外，報告裡「複製指令」按鈕的程式碼也被清理過了，不再疊床架屋寫兩份一樣的複製邏輯；而報告裡用來估算的台幣匯率，也標上了清楚的內部註解，說明這只是個固定的預估值、不是即時連網抓的匯率。就像餐廳的結帳櫃檯，以前算帳時會先把所有菜單全拿出來用計算機按一次，然後又翻開點菜單一筆一筆再加一次，客人排隊排到天荒地老；現在收銀員變聰明了，一邊點菜一邊就算好總金額。而且點餐機裡的程式碼也整理過了，上面還貼了張便條紙提醒新來的員工「這裡的匯率是寫死的參考值，不要傻傻去對當天的新聞」。

Original changelog
**Double-counted report cost**: `build_report_data` summed cost once over all entries and then recomputed it per entry inside the loop — effectively doubling the work on large datasets. Now accumulated once inside the loop.
**Duplicated clipboard code in the report's "copy command" button**: the tip-copy button now reuses the shared `copyText()` helper instead of re-implementing the legacy-browser fallback.
**Hard-coded TWD rate**: the USD→TWD estimate in the report is now a named `_USD_TO_TWD` constant with a note that it is a display estimate, not a live FX rate.

It's like getting a medical checkup: they used to just hand you a single total score, but now the report spells out exactly which premium health package you bought, gives you a vivid color chart of exactly where you're storing the most fat, and even explicitly compares the muscle mass of your left arm versus your right.

Internal pipelines polished, saving your computer from doing double duty

Your reports now generate faster because the system dropped some silly internal calculation habits. When calculating your report costs, the system used to be deeply confused: it would sum up all the numbers once, and then redundantly re-calculate everything piece by piece inside a loop. If you had massive amounts of data, it was effectively doing double the work for nothing; it now smartly accumulates the total in a single pass, saving processing time. We also cleaned up the clipboard code for the "copy command" buttons in the report so it no longer relies on two duplicated chunks of logic, and we added a clear internal note to the hard-coded TWD exchange rate explaining it's just a static display estimate rather than a live network fetch. It's like a restaurant checkout counter: the cashier used to punch every single receipt into the calculator once, and then painstakingly re-add every itemized dish again while the line grew out the door; now they simply tally the total in one smooth motion. The cash register's software is much cleaner now, too, sporting a sticky note that reminds the new hires, "The exchange rate here is just an estimate, don't try matching it against the morning news."

システムの内部パイプラインを整理し、パソコンの二度手間を解消

システムが裏側の馬鹿げた計算方法を改めたため、レポート出力の速度が向上した。レポートのコストを計算する際、システムは以前、全体の数字を一度合計してから、再びループの中で1件ずつ計算し直すというひどい勘違いをしており、データ量が多いと完全に二度手間を踏んでいたが、今はループ内で1回だけ計算するように改められ、処理がスムーズになった。また、レポート内の「コマンドをコピー」ボタンのクリップボード用コードも整理され、同じコピーの仕組みを2重に書く無駄が省かれた。レポートの推定に使われる台湾ドルの為替レートにも明確な内部注釈がつき、これがリアルタイム通信で取得したレートではなく固定の目安値であることが明記された。レストランのレジで、以前はすべての伝票を電卓で一度足してから、また一品一品を計算し直して客を大行列させていたのが、今はスムーズに一度の計算で総額を出せるようになったようなものである。さらにレジのプログラム自体もすっきりと整理され、「ここの為替レートは固定の目安だから、今日のニュースの数字と真面目に比べないように」という新人向けの付箋まで貼られたわけだ。

보고서 대폭 업그레이드: 구독 요금제 자동 감지 및 프로젝트 도넛 차트 추가.

HTML 분석 보고서가 이제 훨씬 유용한 정보를 많이 보여준다. 새로 생긴 '당신의 구독' 구역은 당신 컴퓨터의 인증 파일에서 어떤 Claude 요금제나 Codex의 ChatGPT 요금제를 쓰고 있는지와 구독 시작일을 알아서 읽어온다(비밀번호나 이메일은 절대 훔쳐보지 않으니 안심하라). 게다가 보고서를 남에게 공유할 때 '프로젝트 이름 숨기기'를 켜면, 구독 시작일도 알아서 모자이크 처리해 준다. 보고서에는 예쁜 '프로젝트 비중 도넛 차트'도 추가되어, 자원을 가장 많이 잡아먹은 상위 6개 프로젝트에 서로 다른 색을 칠하고 나머지는 '기타'로 묶어, 한가운데 총 사용량을 큼직하게 띄워 한눈에 알아볼 수 있다. 그뿐만 아니라, 예전부터 시스템은 속으로 Claude와 Codex 두 도구의 사용량 대결(각각 토큰(token), 비중, 비용을 얼마나 썼는지)을 계산하고 있었는데, 이제 이 'Claude vs Codex' 비교 전적판이 봉인을 풀고 화면에 당당히 나타났다. 건강검진을 받으러 갔을 때, 예전엔 달랑 총점 숫자 하나만 주던 것이 이제는 첫 장에 당신이 산 프리미엄 검진 패키지 이름이 적혀 있고, 어디에 지방이 제일 많은지 예쁜 컬러 도넛 차트로 보여주며, 양팔의 근육량까지 적나라하게 비교해서 보여주는 것과 같다.

시스템 내부 배관을 정리해, 컴퓨터가 헛수고하는 일을 없앴다.

시스템이 뒤에서 벌이던 바보 같은 계산 방식을 고쳐서 보고서 나오는 속도가 더 빨라졌다. 보고서 비용을 계산할 때, 예전엔 시스템이 얼빠진 짓을 해서 전체 숫자를 한 번 다 더해 놓고는 다시 빙글빙글 도는 루프 안에서 한 건씩 처음부터 계산하곤 했다. 데이터가 많으면 완전히 헛고생을 두 번 하는 꼴이었는데, 이제는 한 번에 깔끔하게 누적해서 계산하도록 고쳐 속도가 매끄러워졌다. 또한 보고서 안의 '명령어 복사' 버튼에 들어간 복사 코드도 깔끔하게 치워, 똑같은 복사 논리를 두 번 겹쳐 쓰는 일이 없어졌다. 보고서에 쓰이는 대만 달러 환율 역시 명확한 내부 주석을 달아, 실시간 인터넷 환율이 아니라 단순 참고용 고정값이라는 점을 분명히 했다. 식당 계산대에서 예전엔 계산원이 전체 영수증을 계산기로 한 번 다 더하고 나서, 다시 주문서를 펴고 메뉴 하나하나를 일일이 다시 더하느라 손님들을 문밖까지 줄 세웠다면, 이제는 부드럽게 한 번에 싹 계산해 내는 것과 같다. 게다가 계산기 안의 프로그램도 말끔히 청소했고, 그 위에 '여기 환율은 고정된 참고값이니 오늘 아침 뉴스랑 비교하지 말라'고 신입 사원에게 남기는 포스트잇까지 붙여 둔 셈이다.
