zh-TW

Windows 的 Claude Code 狀態列終於正常運作

Windows 的 Claude Code 狀態列（statusLine，顯示在終端機最底下的狀態列）終於正常運作。這次修好四處：掛鉤（hook，特定事件發生時被觸發的小程式）優先用全 ASCII 的 Python 路徑；虛擬環境路徑不合就改用系統 PATH，`--setup` 也會遷移舊指令。五支掛鉤用 UTF-8 讀標準輸入（stdin，程式接收外來資料的入口），轉發器啟動子行程時也固定 UTF-8。Windows 不會再以系統語言的編碼頁誤解工作階段 JSON，將 `GitHub專案` 這類工作資料夾變成亂碼、寫壞 `usage-status.json`。沒有 `LANG` 類環境變數時，改查系統介面語言，和系統匣相同；主控台量實際寬度，寬終端機會重現 `(left)` 重設時間。像校正列車資訊牌的線路、語言和尺規，班次不再中途消失。非 Windows 不變。

Original changelog
**The Claude Code statusLine hook now works end-to-end on Windows**: four fixes land together. `setup_hook` now prefers an all-ASCII `python.exe` path (falling back to system PATH when the venv path isn't ASCII), since Claude Code on Windows fails to spawn statusLine commands whose path contains non-ASCII characters; `--setup` migrates existing non-ASCII commands. All five hook scripts now read stdin via `sys.stdin.buffer` and decode UTF-8 explicitly (and the forwarder pins `encoding=utf-8` on its subprocess fan-out), since Windows otherwise decodes the piped session JSON with the locale codepage, turning a cwd like `GitHub專案` into mojibake and silently breaking `usage-status.json` writes. Hooks now fall back to `GetUserDefaultUILanguage` when no `LANG`-style env var is set (the Windows norm), matching the tray's existing language detection. `get_width()` now probes the real console width via `CONOUT$` + `GetConsoleScreenBufferInfo` instead of always falling back to a fixed 116 columns, restoring the `(left)` reset-time suffix on wide terminals. Non-Windows behavior is unchanged.

en

Claude Code's status line finally works end-to-end on Windows

On Windows, Claude Code's status line at the bottom of the terminal could fail somewhere along the way. This release fixes four links in that chain. Hooks (small programs triggered by particular events) now avoid Python paths containing non-ASCII characters, and `--setup` migrates older commands. All five hook scripts now read standard input as UTF-8, so a working-folder name like `GitHub專案` is no longer turned into mojibake and prevented from writing `usage-status.json`. If Windows has no `LANG`-style environment variable, the hooks now use the system UI language instead. Wide terminals also measure their actual width again, restoring the `(left)` reset-time suffix. It is like repairing the wiring, language setting, and ruler behind a train-information board so its timetable does not vanish mid-journey. Non-Windows behavior is unchanged.

ko

Windows에서 Claude Code 상태 표시줄이 마침내 끝까지 제대로 작동한다

Windows에서는 터미널 맨 아래 Claude Code 상태 표시줄이 중간에 멈추는 일이 있었다. 이번에는 그 길목의 네 문제를 한꺼번에 고쳤다. 훅(hook, 특정 일이 일어날 때 실행되는 작은 프로그램)은 ASCII 밖의 문자가 든 Python 경로를 피하고, `--setup`은 기존 명령도 옮긴다. 다섯 훅 스크립트는 표준 입력(stdin, 프로그램이 바깥 데이터를 받는 입구)을 UTF-8로 읽으므로 `GitHub專案` 같은 작업 폴더 이름이 깨져 `usage-status.json`을 쓰지 못하던 일도 막는다. `LANG` 계열 환경 변수가 없는 Windows에서는 시스템 화면 언어를 사용한다. 넓은 터미널은 실제 너비를 다시 재서 `(left)` 재설정 시간 표시도 되살린다. 열차 안내판의 전선, 언어 설정, 자를 함께 고쳐 운행표가 중간에 사라지지 않게 한 셈이다. Windows 밖의 동작은 바뀌지 않는다.
