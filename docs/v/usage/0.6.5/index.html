zh-TW

新增「開機自動啟動」開關

現在你可以在「更換面板」的選單裡，找到一個新的「開機自動啟動」選項。只要打個勾，下次登入電腦時這款 App 就會自己啟動，不用再每次手動打開了。如果你取消勾選，系統只會把自動啟動的設定檔（plist）拿掉，並不會把正在執行中的程式關掉。就像是幫每天早上都要用到的烤麵包機插上定時插座，起床就能直接用；拔掉插座也只是明天不會自動烤，現在正在烤的麵包還是會乖乖烤完。

Original changelog
**Launch at Login toggle**: the panel-switcher menu (opened from the "Switch Panel" button) gains a checkable "Launch at Login" item. Ticking it makes usage start automatically at next login, so you don't have to relaunch it manually. The .app and source builds each generate the matching LaunchAgent plist; unticking only removes the plist — it never quits a running app.
README "Auto-start on login" section now documents the popover toggle (Traditional Chinese / English).

en

Added a "Launch at Login" toggle

You can now find a new "Launch at Login" option in the panel-switcher menu. Just tick the checkbox, and the app will start itself automatically the next time you log into your computer, saving you from opening it manually every time. If you untick it, the system simply removes the auto-start config file (plist) and will never force-quit the running app. It's like plugging your daily toaster into a smart timer so it's ready when you wake up; unplugging the timer just means it won't start on its own tomorrow, but the bread currently toasting will still finish perfectly.

ja

「ログイン時に自動起動」のスイッチを追加

「パネルの切り替え」メニュー内に、新たに「ログイン時に自動起動」のオプションが追加された。チェックを入れるだけで、次回パソコンにログインした際にこのアプリが自動的に起動するようになり、毎回手動で開く手間が省ける。チェックを外した場合、システムは自動起動の設定ファイル（plist）を削除するだけであり、実行中のプログラムが勝手に終了することはない。毎朝使うトースターをタイマー付きコンセントに繋いでおき、起きたらすぐ使えるようにするのと同じである。コンセントを抜いても明日から自動で焼けなくなるだけで、今焼いているパンはちゃんと最後まで焼き上がる。

ko

'로그인 시 자동 실행' 스위치 추가

이제 '패널 바꾸기' 메뉴에서 '로그인 시 자동 실행'이라는 새로운 옵션을 찾을 수 있다. 체크 표시만 해두면 다음번 컴퓨터에 로그인할 때 이 앱이 알아서 켜지니 매번 직접 켤 필요가 없다. 체크를 해제하면 시스템은 자동 실행 설정 파일(plist)만 치울 뿐, 지금 켜져 있는 프로그램까지 꺼버리진 않는다. 매일 아침 쓰는 토스터에 타이머 콘센트를 꽂아 두고 일어나자마자 곧바로 쓰는 것과 같다. 콘센트를 뽑는다고 해도 내일부터 알아서 구워지지 않을 뿐이지, 지금 굽고 있는 빵은 얌전히 끝까지 구워낸다.
