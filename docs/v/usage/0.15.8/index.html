zh-TW

Codex 額度過期不再留白，指令與選單列數字終於同步

這次讓 Codex 的額度顯示更精準。當它的 5 小時額度窗重置時，選單列以前會直接變成空蕩蕩的「--」，現在會比照 Claude 的做法，明確標示為「0%」。同時，我們把終端機指令（CLI）跟選單列查額度的管線打通了，兩邊現在看的是同一個來源，不會再出現指令查是一個數字、選單列又是另一個數字的窘境。就像你原本看儀表板，油箱空了指針卻直接消失，而且車內跟車外讀到的油量還不一樣；現在空了會老實指在零，且兩邊的油表終於統一了。

Original changelog
**Codex "Session (5h)" quota no longer blanks out when the window expires**: after the 5-hour window resets, the session used to show blank (`--`), inconsistent with the Claude side; it now shows 0% like Claude. The CLI and menu bar now read rate limits from the same source, so their numbers no longer disagree.

終端機診斷指令升級，幫你秒抓 Codex 額度問題

現在你可以秒抓 Codex 額度抓不到的具體原因了。以前當你發現系統抓不到 Codex（另一款 AI 工具）的額度時，只能像無頭蒼蠅一樣自己亂找原因。現在我們幫終端機裡的 doctor（健檢）指令加入了專門針對 Codex 的診斷功能。它會幫你檢查最近的紀錄檔時間、資料庫裡面有沒有配額資料，並告訴你 5 小時或每週的額度目前到底抓不抓得到。這就像是車子儀表板不亮時，以前你只能把車蓋打開亂看一通；現在只要插上診斷電腦，它就會直接告訴你是感測器壞了還是線沒接好，讓你一秒就知道問題出在哪裡。

Original changelog
`doctor` now reports Codex diagnostics: latest session-log age, `logs_2.sqlite` rate-limit row count, `state_5.sqlite` status, and whether 5h / weekly quota data is currently available — making "why isn't it detected" easy to diagnose.

en

Expired Codex quotas no longer show blank, and CLI numbers finally sync with the menu bar

Codex quota readouts are now far more precise. When the 5-hour Codex window resets, the menu bar used to drop to a blank "--"; it now explicitly displays "0%" to match Claude's behavior. Concurrently, the terminal command (CLI) and the menu bar now share the exact same quota reader pipeline, completely eliminating the confusion of seeing one number in the terminal and a different one on the menu bar. It's like a car dashboard where the fuel gauge used to vanish entirely when the tank was empty, while the internal and external displays reported different fuel levels; now it properly points to zero, and both gauges finally agree.

Original changelog
**Codex "Session (5h)" quota no longer blanks out when the window expires**: after the 5-hour window resets, the session used to show blank (`--`), inconsistent with the Claude side; it now shows 0% like Claude. The CLI and menu bar now read rate limits from the same source, so their numbers no longer disagree.

Doctor CLI upgraded to instantly diagnose Codex quota issues

You can now instantly pinpoint exactly why your Codex quota isn't showing up. Whenever the system failed to fetch your Codex (another AI tool) quota, you previously had to guess the root cause yourself. We've now upgraded the terminal's doctor (diagnostic) command to specifically troubleshoot Codex issues. It automatically checks the timestamp of your latest session log, counts the quota entries in the database, and clearly reports whether your 5-hour and weekly quota data are currently available. It's like a car dashboard going dark: you used to have to pop the hood and guess what's wrong, but now you can just plug in an OBD scanner and it instantly tells you whether a sensor broke or a wire just came loose.

Original changelog
`doctor` now reports Codex diagnostics: latest session-log age, `logs_2.sqlite` rate-limit row count, `state_5.sqlite` status, and whether 5h / weekly quota data is currently available — making "why isn't it detected" easy to diagnose.

診断コマンドがアップグレードされ、Codex クオータの問題を即座に特定

これで Codex クオータが取得できない具体的な原因を瞬時に特定できるようになった。システムが Codex（もう一つの AI ツール）のクオータを取得できなかった時、以前は原因を自分で当てずっぽうに探すしかなかった。今回、端末の doctor（健康診断）コマンドに Codex 専用の診断機能を追加した。最近のログの日時や、データベース内のクオータデータの有無を自動で確認し、5時間枠と週間枠のデータが現在取得可能な状態かどうかを教えてくれる。車のメーターが消えた時、以前はボンネットを開けて適当に調べるしかなかったのが、今は診断用のコンピューターを繋げば、センサーの故障なのか配線の抜けなのかを一瞬で教えてくれるようなものである。

Codex 할당량 만료 시 여백을 남기지 않고, 명령어와 메뉴 바의 숫자가 마침내 동기화되었다

이번 업데이트로 Codex 할당량 표시가 훨씬 정확해졌다. 5시간 할당량 창이 초기화될 때 메뉴 바가 텅 빈 '--'로 변하던 것을, 이제는 Claude처럼 확실하게 '0%'로 띄워 준다. 또한 터미널 명령어(CLI)와 메뉴 바가 할당량을 조회하는 파이프라인을 하나로 합쳐, 터미널에서 조회한 숫자와 메뉴 바 숫자가 서로 따로 노는 일도 없앴다. 자동차 계기판에서 기름이 떨어지면 바늘이 아예 사라져 버리고 실내와 실외 연료 표시가 서로 달랐다면, 이제는 바닥나면 정직하게 0을 가리키고 양쪽 계기판이 드디어 딱 맞아떨어지는 것과 같다.

진단 명령어가 업그레이드되어 Codex 할당량 문제를 바로 짚어준다

이제 Codex 할당량을 가져오지 못하는 정확한 원인을 즉시 찾아낼 수 있다. 시스템이 Codex(또 다른 AI 도구) 할당량을 못 가져올 때 예전엔 원인을 몰라 헤매야만 했다. 이제 터미널의 doctor (진단) 명령어에 Codex 전용 진단 기능을 추가했다. 최근 기록 파일의 시간과 데이터베이스 안에 할당량 정보가 있는지 확인하고, 5시간 및 주간 할당량을 지금 가져올 수 있는 상태인지 정확히 알려준다. 자동차 계기판이 꺼졌을 때 예전엔 보닛을 열고 대충 살펴봐야 했다면, 이제는 진단기를 꽂기만 하면 센서가 고장 났는지 선이 빠졌는지 곧바로 알려주어 1초 만에 문제를 파악할 수 있는 것과 같다.
