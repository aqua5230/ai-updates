zh-TW

Windows 掛鉤與中日韓路徑修復，額度不留白、重啟不重複

現在你在 Windows 上使用中日韓字元路徑時，不用再擔心專案路徑會讓工具掛掉，或額度卡片一直顯示空白。以前有些 Windows 電腦會因為 statusLine（顯示在終端機最底下的狀態列）的 hook（掛鉤，特定事件發生時被觸發的小程式）沒有被正常叫起，導致額度卡片空白；現在當系統偵測不到狀態檔時，會自動採取備援（fallback）方案，直接讀取設定檔填補資訊。同時，如果你的工作階段（session）掛鉤路徑中含有中文等非英文字元，自我修復功能會自動將它們導向標準路徑，而且這個修正過程是冪等（idempotent）的，重複開機也不會多寫無用的紀錄。這就像是水管工人在裝水管時，不僅幫你把路徑上的石子清乾淨，還預備了第二條水管，就算主水管不通也能自動切換，而且水管鋪好一次就搞定，不會每次開水龍頭都重新拆裝一次。

Original changelog
**Claude quota now shows on Windows even when the statusLine hook never fires**: a Claude Code regression stops the hook from being invoked on some Windows setups, leaving the Claude quota card permanently blank. `usage_client.py` now falls back to reading `cachedUsageUtilization` straight out of Claude Code's own `~/.claude.json` when no `usage-status.json` exists, so the card still populates.
**Session hooks no longer break on non-ASCII (CJK) project paths on Windows**: the resume, terse-mode, and terse-reminder hook commands could point into the project or app-bundle source path, which failed to execute on Windows when that path contained Chinese/Japanese/Korean characters. Self-heal now normalizes these commands to their canonical `~/.claude/` targets, and the migration is idempotent — it no longer rewrites `settings.json` and appends a duplicate self-heal log entry on every restart once the command is already correct.

en

Windows hooks and CJK path fixes: quotas display correctly and restarts stay clean

You can now use Chinese, Japanese, or Korean characters in your project paths on Windows without fearing that tool commands will crash, or that your quota card will stay stubbornly blank. Previously on some Windows setups, the statusLine hook would fail to fire, causing the usage quota card to remain blank; now the tool automatically falls back to reading the configurations directly to populate the info. At the same time, if CJK characters in your project paths caused session hooks to error out, the self-heal process now normalizes them to their standard directory targets, and this migration is idempotent so it won't repeatedly write redundant logs on restarts. It is like a plumber who not only clears out pebbles blocking your pipe but also routes a backup water line in case the main line clogs, ensuring the faucet works smoothly and the configuration is set once and for all.

ja

WindowsフックとCJKパスの修正：クォータが表示され、再起動時の重複も解消

Windows上で日本語や中国語などの文字（CJK文字）を含むプロジェクトパスを使っている場合でも、コマンドがクラッシュしたり使用量カードが空白のままになったりする心配がなくなった。以前は一部のWindows環境においてstatusLine（端末の最下部に表示されるステータスバー）のhook（フック、特定のイベントが発生したときに実行されるプログラム）が呼び出されず、クォータ表示が空欄になる問題があったが、現在は状態ファイルが見つからない場合に設定ファイルから直接データを読み込むフォールバック（代替）処理が行われるようになった。また、パスに日本語などの文字が含まれることでセッションのフックコマンドが失敗していた問題も、自動修復機能により標準のパスへと正規化されるようになり、この処理は冪等（ベキトウ、何度実行しても同じ結果になる性質）であるため再起動のたびに不要なログが書き込まれることもない。これは、水道管の通り道を塞ぐ石を取り除くだけでなく、本管が詰まったときに備えて予備のバイパス管を用意しておくようなものである。一度しっかり配管すれば、蛇口をひねるたびにやり直す必要もない。

ko

Windows 훅 및 CJK 경로 수정: 할당량 표시 정상화 및 중복 기록 방지

Windows에서 한국어 등 아시아 문자(CJK)가 포함된 프로젝트 경로를 사용하더라도 명령어가 충돌하거나 사용량(quota) 카드가 계속 빈칸으로 남는 걱정을 하지 않아도 된다. 이전에는 일부 Windows 환경에서 statusLine(터미널 가장 아래에 표시되는 상태 표시줄)의 hook(훅, 특정 이벤트 발생 시 트리거되는 프로그램)이 정상적으로 호출되지 않아 할당량 카드가 공란으로 표시되었으나, 이제는 상태 파일이 없을 때 설정 파일에서 직접 데이터를 읽어오는 백업 기능이 작동한다. 또한 프로젝트 경로에 한글 등이 포함되어 세션 훅 명령어가 실행 실패하던 문제도 자체 복구 기능을 통해 표준 경로로 규격화되며, 이 마이그레이션 작업은 멱등성(동일한 작업을 여러 번 수행해도 결과가 같은 성질)을 지녀 재부팅할 때마다 무의미한 로그를 중복해서 기록하지 않는다. 이는 배관공이 파이프 안의 돌멩이를 치워줄 뿐만 아니라, 메인 관이 막힐 때를 대비해 보조 파이프를 마련해 두는 것과 같다. 한 번 제대로 설치하면 수도꼭지를 틀 때마다 매번 새로 공사할 필요가 없는 셈이다.
