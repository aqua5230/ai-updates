zh-TW

選單列不再因為無關的檔案變動而白忙一場

你的選單列現在不會再因為無關的檔案變動而莫名其妙重新排版了。以前系統在背後監聽時會盯著整個資料夾看，只要有任何代理（agent）寫入無關的狀態檔，選單列就會傻傻地跟著刷新；現在它只會緊盯 Claude 與 Codex 真正用來記錄歷史的特定資料夾，而且如果標題根本沒變，它就不會浪費力氣重新排版。就像警衛以前只要看到大樓裡有人走動就全棟廣播，現在只盯著大門看，沒人進出就不會隨便拉警報。

Original changelog
**Idle menu-bar refreshes no longer react to unrelated agent state writes**: FSEvents now watches Claude and Codex usage-history directories instead of their entire data roots, and unchanged status titles no longer trigger redundant AppKit layout work.

en

Menu bar stops useless refreshes on unrelated file changes

Your menu bar will no longer inexplicably re-render its layout due to unrelated file changes. Previously, when the system listened in the background, it watched the entire data directory, meaning any agent writing to an unrelated status file would cause the menu bar to foolishly trigger a full visual refresh. Now, it strictly watches only the specific directories where Claude and Codex save their usage history, and if the status title hasn't actually changed, it won't waste effort re-rendering the layout. It's like a security guard who used to trigger a building-wide alarm every time they saw movement inside, but now strictly watches the front door and only sounds the alarm when someone actually enters or leaves.

Original changelog
**Idle menu-bar refreshes no longer react to unrelated agent state writes**: FSEvents now watches Claude and Codex usage-history directories instead of their entire data roots, and unchanged status titles no longer trigger redundant AppKit layout work.

ko

메뉴 바가 상관없는 파일 변경 때문에 헛고생하지 않음

당신의 메뉴 바가 상관없는 파일 변경 때문에 뜬금없이 화면을 다시 짜 맞추는 일이 없어졌다. 예전엔 시스템이 뒤에서 엿들을 때 전체 폴더를 다 지켜보고 있어서, 어떤 에이전트(agent)가 상관없는 상태 파일에 글을 쓰기만 해도 메뉴 바가 바보처럼 새로 고침을 했다. 이제는 Claude와 Codex가 진짜로 역사를 기록하는 특정 폴더만 뚫어져라 쳐다보고, 제목이 전혀 바뀌지 않았다면 헛수고하며 다시 화면을 짜 맞추지 않는다. 경비원이 예전엔 건물 안에서 사람만 움직여도 온 건물에 방송을 틀어대다가, 이제는 정문만 지켜보고 아무도 안 들락거리면 쓸데없이 사이렌을 울리지 않는 것과 같다.
