zh-TW

網路斷線面板不再變白，三十天報告還你真實天數

現在就算網路斷線，你的選單列面板也不會突然變成一片空白了。以前如果遠端額度抓取失敗，選單列（menu bar，螢幕頂端那一排系統圖示區）的面板會整片變成空白，把你本來就存在電腦裡的專案用量也一併藏起來；現在它懂得在斷線時保留已經讀到的本機資料，畫面不再短暫變空。另外，以前在專案用量裡點「30d」產生的報告，其實只會從這個月的 1 號開始算，現在終於修正回真正的「滾動近 30 天」了。就像是原本你看報紙，只要外頭一停網，連你手上印好的半份都會被強行抽走；現在它不但讓你好好看完手邊的進度，而且連載的三十天回顧也不再偷工減料只給這一個月的份。

Original changelog
**Menu bar no longer blanks out when a refresh fails (#27)**: follow-up to #25. Local project usage / today's stats / the status line are now loaded *before* the remote quota fetch, and preserved when that fetch fails, so the view no longer flashes empty. Alert (NSAlert) creation or icon-setup failures now fall back to a safe no-op instead of interrupting the menu bar update.
**Project Usage "30d" report aligns with a rolling 30 days (#28)**: generating a report from the menu bar's "30d" Project Usage range previously mapped to "this month" (1st of the month to today), which didn't match the labeled rolling 30-day range. It now maps to the report pipeline's `last30` (the last 30 days).

en

Menu bar no longer blanks out on refresh failures, and 30-day reports actually roll 30 days

You can now check your menu bar panel without worrying about it going blank during a network disconnect. Previously, if the remote quota fetch failed, the menu bar (the row of system icons at the top of your screen) panel would instantly go completely blank, taking down your locally stored project usage with it; now it smartly retains the local data it already loaded, so the screen no longer flashes empty. Additionally, selecting "30d" for a project usage report used to stubbornly only calculate from the 1st of the current month; it has now been fixed to properly roll back a full 30 days. It's like reading a newspaper where a sudden internet outage used to magically erase the printed pages already in your hands; now you can finish reading what you have, and the 30-day retrospective actually covers 30 days instead of shortchanging you for just the current month.

ja

ネットワーク切断でパネルが白くならなくなり、30日レポートの日数も正確になった

これでネットワークが切断されても、メニューバーのパネルが突然真っ白になることはなくなった。以前は遠隔のクオータ取得に失敗すると、メニューバー（画面上部に並ぶシステムアイコン群）のパネルが真っ白になり、パソコン内に保存されていたプロジェクトごとの使用量まで一緒に消えていたが、今は取得済みのローカルデータを保持し、画面が一瞬空になることはなくなった。また、プロジェクトの使用量で「30d」を選んでレポートを作成した際、以前は今月の1日からしか計算されていなかったが、今後は正しく「直近30日間」で遡って計算される。以前はネットが切れた途端に手元の新聞まで強制的に白紙にされていたのが、今は手元にある分は読めるようになり、30日間の振り返り記事も今月分だけでごまかされなくなったようなものである。

ko

인터넷이 끊겨도 패널이 하얗게 변하지 않으며, 30일 보고서의 날짜 계산도 정확해졌다

이제 인터넷이 끊겨도 메뉴 바 패널이 갑자기 하얗게 변하는 일을 겪지 않아도 된다. 예전엔 원격으로 할당량을 가져오는 데 실패하면 메뉴 바(화면 꼭대기에 있는 시스템 아이콘 줄) 패널이 하얗게 변하며 컴퓨터에 저장돼 있던 프로젝트 사용량까지 싹 가려버렸는데, 이제는 인터넷이 끊겨도 이미 읽어둔 로컬 데이터를 유지해 화면이 잠깐 비워지는 일이 없다. 또한 프로젝트 사용량에서 '30d'를 눌러 보고서를 뽑을 때, 예전엔 무조건 이번 달 1일부터만 계산했지만 이제는 진짜 '최근 30일'을 제대로 거슬러 올라가 계산한다. 인터넷이 끊기면 손에 쥐고 있던 신문지마저 억지로 빼앗기던 것에서, 이제는 손에 있는 건 끝까지 읽을 수 있고 30일짜리 회고록도 이번 달치만 대충 주지 않고 꽉 채워주는 것과 같다.
