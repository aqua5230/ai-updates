zh-TW

換上專屬圖示，還能自動幫你接續上次的對話

開啟 App 時，你不再看到公版的火箭圖案，而是專屬的品牌圖示，而且它還學會了超強的「進度管家」功能。以前你每次開啟新的 Claude Code 對話（例如啟動或輸入 `/clear` 指令）時，都要苦命地跟 AI 重新交代「我上次做到哪裡、我接下來要幹嘛」；現在只要在選單列打開「接著上次做」開關，它會自動把你前一次的最後要求、寫完的程式碼紀錄（commit）和未完成的待辦事項打包，趁新對話一開始偷偷塞給 AI，並讓 AI 跟你打招呼說「🐾 已接回上次進度，繼續吧！」。同時，原本選單裡佔掉一大片版面的 9 個面板主題，現在整齊地收進了「面板主題」子選單裡，不再滑不到底了。就像去髮廊，以前每次都要跟新來的設計師重新解釋你要剪什麼髮型，現在櫃檯管家會先把你的修剪紀錄跟未完成的染髮卡交給設計師，你一坐下他就會說「我們繼續上次的進度吧」；而這家髮廊的招牌也終於從公版圖案換成了自家的專屬 Logo。

Original changelog
**"Progress Concierge" feature** (menu label: "Resume Last Session"): fully local, zero API. When you open a new Claude Code session (`startup` / `/clear`), it automatically hands your last progress to the AI — no need to re-explain. A single menu toggle (off by default, opt-in) installs a Claude Code SessionStart hook (`usage_session_resume.py`, stdlib-only so it runs under macOS's bundled Python 3.9) that reads the project's previous session for **your last request + the commits made + any unfinished todos (if TodoWrite was used)**, assembles a resume prompt, injects it at the start of the new session, and asks Claude to open with "🐾 Picked up where you left off — let's keep going!" so you know it took effect. Wording lives in `i18n.json` (written to a sidecar at install time so the hook stays single-sourced); `setup_hook` handles install/remove/backup/self-heal. The menu item carries a tooltip with the full explanation.
**Dedicated app icon**: replaces py2app's default rocket; NSAlert dialogs now use the brand icon too (via `setIcon_`).
**Slimmer menu**: the 9 panel themes are collapsed into a "Panel theme" submenu, so the menu is no longer dominated by a long inline list.

嚴密防堵壞檔案，面板載入不再意外死當

讀取本機資料時變得更「耐摔」了。以前只要你的硬碟裡躺著一兩個亂碼檔（壞掉的 UTF-8 編碼）、格式爛掉的設定檔（壞 JSON）或是原本該是文字卻變成數字的詭異欄位，一讀進去就會當場嚇到死當；現在所有「讀檔入口」都加上了防護，遇到這種壞檔案會冷靜過濾掉，不會讓整個程式崩潰。另外，在載入 WebKit 面板的備用路線（loadBundle fallback）上，原本漏註冊了 evaluateJavaScript 需要的 block 簽名（signature），導致面板可能一片空白，現在也把它補上了。就像資源回收場的機器，以前不小心吞進一塊石頭整台機器就卡死冒煙，現在裝了強大的過濾網，不是鐵鋁罐的東西都會被自動彈開，機器依然順暢運轉。

Original changelog
**Broad robustness hardening**: systematically hardened every entry point that reads user files on disk against bad UTF-8, bad JSON, and type drift (numeric strings, non-dict, non-str fields) — covering `setup_hook`, `codex_loader`, the Codex / Claude / rate-limit adapters, the statusline, the history loader, subscription reads and JWT decoding, and the tips loader.
**WebKit panel fallback**: registered the missing `evaluateJavaScript` block signature on the `loadBundle` fallback path.

When launching the app, you'll see a dedicated brand icon instead of the generic rocket, and it now comes with an incredibly handy "Progress Concierge". Previously, whenever you started a new Claude Code session (like launching or typing `/clear`), you had to painstakingly re-explain to the AI where you left off and what your next goal was. Now, if you toggle on "Resume Last Session" in the menu, it automatically bundles your last request, recent commits, and unfinished to-dos, then quietly hands them to the AI as the new session opens. The AI will even greet you with "🐾 Picked up where you left off — let's keep going!" so you know it worked. In addition, the long list of 9 panel themes that used to hog the menu has been neatly tucked into a "Panel theme" submenu. It's like going to a hair salon: you used to have to re-explain your desired haircut to a new stylist every time, but now the receptionist hands your styling history and pending coloring notes to the stylist before you even sit down, and they just say "Let's continue where we left off"; plus, the salon finally replaced its generic stock-image sign with a proper custom logo.

Bad files are blocked securely, and the panel loads without unexpected crashes

Reading your local data is now much more "drop-proof". Previously, if your hard drive contained a few garbled files (bad UTF-8 encodings), corrupted configurations (bad JSON), or weird fields where numbers appeared instead of text, reading them would immediately panic and crash the application. Every single "file reading entry point" is now armored: when it encounters these toxic files, it calmly filters them out instead of bringing down the whole app. Additionally, on the WebKit panel's fallback loading route (loadBundle), the missing block signature for evaluateJavaScript has been registered to prevent the screen from rendering completely blank. It's like a recycling plant sorting machine: it used to jam and catch fire if it accidentally swallowed a rock, but now it has a heavy-duty filter that automatically bounces out anything that isn't an aluminum can, letting the machine hum along uninterrupted.

전용 아이콘으로 갈아입고, 저번 대화를 자동으로 이어받는다.

앱을 켤 때 흔해 빠진 로켓 대신 전용 브랜드 아이콘이 보이며, 엄청나게 편리한 '진행 상태 관리자(Progress Concierge)' 기능도 쓸 수 있다. 예전엔 새로운 Claude Code 세션을 시작할 때(앱을 켜거나 `/clear` 명령어를 입력할 때)마다 AI에게 '저번에 어디까지 했고 이제 뭘 할 건지'를 구구절절 다시 설명해야 했다. 이제 메뉴에서 '이전 작업 이어서 하기' 스위치를 켜면, 직전의 마지막 요청과 완료된 코드 기록(commit), 아직 끝내지 못한 할 일 목록을 자동으로 묶어 새 대화가 시작될 때 AI에게 몰래 건네준다. 그러면 AI가 "🐾 이전 진행 상황을 이어받았습니다. 계속합시다!"라고 인사를 건네 적용되었음을 알려준다. 이와 함께, 예전엔 메뉴의 넓은 자리를 차지하던 9개의 패널 테마가 '패널 테마' 하위 메뉴 안으로 깔끔하게 들어가 더 이상 끝없이 스크롤할 일이 없어졌다. 미용실에 갈 때 예전엔 만나는 디자이너마다 어떤 머리를 하고 싶은지 처음부터 다시 설명해야 했다면, 이제는 안내 데스크의 매니저가 당신의 커트 기록과 아직 덜 끝난 염색 카드를 미리 디자이너에게 넘겨주어 자리에 앉자마자 "저번에 하던 거 이어서 할게요"라고 말해 주는 것과 같다. 게다가 이 미용실의 간판도 마침내 흔해 빠진 기본 그림에서 가게 전용 로고로 바뀌었다.
