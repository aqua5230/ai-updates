zh-TW

Codex 額度顯示更精準，不再被舊資料吃掉

你選單列上的 Codex 額度顯示現在會更新得更快也更準了。以前系統在匯整兩種不同來源的紀錄時，只要看到一份舊資料就會直接覆蓋掉剛出爐的新資料，導致你明明剛衝破 100% 額度，卻突然又倒退回 80%。現在它學會了分開核對每種額度的最新狀態，而且微小用量會精準顯示小數點，也不必再苦等歷史紀錄跑完才更新畫面。同時，系統改用可設定的重新整理間隔來取代寫死的 300 秒；檔案變動（FSEvents）觸發的重新整理現在會乖乖排隊而不是互相打斷；就算中途 Claude Code 讀取失敗，也會保留已經載入的 Codex 百分比而不是讓畫面閃一下又清空。就像你的悠遊卡原本每次感應都會被昨天搭車的舊紀錄覆蓋餘額，現在它終於學會以最新的扣款時間為準，連幾毛錢都能算得清清楚楚，而且逼卡瞬間就顯示餘額，不用再等機器跑半天。

Original changelog
**Codex quota shows fresher, more accurate numbers (#24)**: (1) the menu-bar Codex quota now updates at the very start of each refresh instead of waiting for the slower history pass; (2) SQLite and JSONL sources are merged per window (5-hour / weekly) instead of picking one whole source, so a just-hit 100% limit is no longer overwritten by an older 80% snapshot; (3) small usage shows a fractional percentage instead of rounding to 0%; (4) the refresh timer uses the configured interval instead of a hard-coded 300s; (5) FSEvents-triggered refreshes queue instead of being dropped while one is in flight; (6) if the Claude Code read fails mid-refresh, the already-loaded Codex percentage is preserved instead of flickering away.

更新完畢後，「可更新」標籤懂得乖乖消失

當你升級完這套工具後，選單列上煩人的「🆕 可更新」小標籤終於會自己消失了。以前因為系統只有在跑「檢查更新」那個動作時才會順便清掃狀態，導致你就算已經裝好最新版，那個標籤還是會死皮賴臉地掛在上面，直到你徹底把 App 關掉重開。現在系統每次例行重新整理時都會順便檢查一下你當前安裝的版本，只要發現你已經跟上最新進度，標籤就會立刻退場。就像包裹已經領走了，信箱上的「有掛號信」小紅燈就不會再一直亮著逼你去收信。

Original changelog
**Stale "🆕 update available" badge no longer lingers after upgrading**: the cache cleanup previously ran only inside the update-check path, so the badge stuck until the app restarted; it now compares the installed version on every timer refresh and clears as soon as you're current.

en

Codex quota updates are instantly precise and stop overwriting themselves with stale data

The Codex quota shown on your menu bar is now much fresher and more accurate. Previously, when the system merged two different data sources, it could accidentally let an older snapshot overwrite a brand-new update—meaning you might hit a 100% limit only to see it jump bizarrely back down to 80%. Now, it individually tracks the latest state for each quota, displays precise fractional percentages for small usage, and updates the UI instantly without waiting for the heavier history scans to finish. It also uses your configured refresh interval instead of a hard-coded 300 seconds. File-system (FSEvents) triggered refreshes now neatly queue up instead of dropping when one is in flight, and if reading Claude Code fails mid-refresh, the already-loaded Codex percentage is safely preserved instead of flickering away. It's like your subway card suddenly forgetting your last ride and showing yesterday's balance; now it strictly respects the newest timestamp, calculates every single cent accurately, and flashes the balance the moment you tap without making you wait for the machine to grind.

The "update available" badge cleanly disappears the moment you upgrade

After you upgrade the tool, that nagging "🆕 update available" badge on your menu bar will finally go away on its own. Previously, because the system only swept out the cache while actively checking the internet for new releases, the badge would stubbornly cling to your screen even after you had installed the latest version, forcing you to completely restart the app to clear it. Now, the system double-checks your currently installed version during every routine background tick, and drops the badge the moment it sees you're fully up to date. It's like the "you have mail" light on your mailbox finally knowing to turn off the moment you actually take the package out, instead of glaring at you all day.

Codex の利用枠表示がより正確になり、古いデータで上書きされる不具合を解消

Codex 한도 표시가 더 정확해지고, 낡은 데이터에 덮어씌워지지 않는다

메뉴 바에 나타나는 Codex 한도가 이제 훨씬 빠르고 정확하게 업데이트된다. 예전에는 시스템이 두 가지 다른 출처의 기록을 합칠 때 옛날 데이터로 방금 들어온 새 데이터를 덮어써 버리는 바람에, 분명 한도의 100%를 다 썼는데 갑자기 80%로 되돌아가는 일이 있었다. 이제는 각 한도별로 최신 상태를 따로따로 꼼꼼히 대조할 줄 알고, 적은 사용량도 소수점까지 정확하게 보여주며, 과거 기록을 다 읽어올 때까지 기다릴 필요 없이 화면을 바로 업데이트한다. 또한, 고정된 300초 대신 설정된 새로 고침 간격을 사용하게 바뀌었고, 파일 변경(FSEvents)으로 인해 촉발된 새로 고침은 중간에 버려지지 않고 차곡차곡 대기열에 쌓인다. 만약 도중에 Claude Code 읽기에 실패하더라도, 이미 불러온 Codex 백분율은 그대로 유지되어 화면이 깜빡이며 사라지지 않는다. 교통카드를 찍을 때마다 어제 탔던 옛날 기록이 잔액을 덮어쓰던 상황에서, 이제는 가장 최근에 돈이 빠져나간 시간을 기준으로 1원 단위까지 깔끔하게 계산하고, 카드를 대는 순간 기계가 버벅댈 필요 없이 바로 잔액을 띄워주는 것과 같다.
