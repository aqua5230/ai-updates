zh-TW

算錢與模型統計不再漏接 Codex 的資料

現在查看帳單或模型使用比例時，你不會再被 0 元或未知的 Codex 估算數字誤導了。新版的 Codex 有時候把用哪個模型這件事記在別的地方（turn context），如果系統在原本習慣的地方找不到，現在會懂得去新的地方挖出來，讓你的帳單跟統計圖表保持完整。就像結帳時如果收銀機找不到條碼，店員現在懂得自己去翻包裝盒底下的備用標籤一樣。

Original changelog
**Codex model attribution now falls back to turn context**: newer Codex sessions can store the model in `turn_context.payload.model`, while `state_5.sqlite` may not have a matching thread row yet. The reader still prefers SQLite when available, but now uses the turn context as a fallback so cost estimates and model distribution no longer collapse to unknown or $0. (#38, by @ericweichun)

動畫面板更新時不再閃爍

那些額度進度條有動畫的面板（例如 Prism Arcade、黑洞、水族箱等），現在更新數字時畫面不會再閃一下了。以前每次狀態更新，系統都會把整個動畫軌道打掉重做，導致動畫會重新播放而產生閃爍；現在它只會在原地更新數字，不會中斷正在跑的動畫。就像換球場上的記分板數字，以前是把整個板子拆下來重裝，現在只換數字卡片，比賽照常進行。

Original changelog
**Animated quota rows no longer restart on every panel refresh**: panels with animated quota tracks, including Prism Arcade, Black Hole, and Aquarium, now mount each quota row once and update it in place instead of rebuilding the markup on every status update. This prevents the CSS animation flicker during normal refreshes. (#39, by @ericweichun)

en

Cost and model tracking no longer miss Codex data

When checking your bills or model usage, you will no longer be misled by $0 or "unknown" estimates just because Codex logs are missing. Newer versions of Codex sometimes stash which model was used in a different spot (turn context); if the system can't find it in the usual place, it now knows to dig it out from the new location, keeping your bills and charts perfectly intact. It's like a cashier who, unable to find the main barcode, now knows to check under the box for the backup sticker.

Original changelog
**Codex model attribution now falls back to turn context**: newer Codex sessions can store the model in `turn_context.payload.model`, while `state_5.sqlite` may not have a matching thread row yet. The reader still prefers SQLite when available, but now uses the turn context as a fallback so cost estimates and model distribution no longer collapse to unknown or $0. (#38, by @ericweichun)

Animated panels stop flickering on refresh

Panels that feature animated quota progress bars (like the Prism Arcade, Black Hole or Aquarium themes) no longer flash when their numbers update. Previously, on every status refresh, the system would tear down and rebuild the entire animation track, causing the animation to restart and flicker; it now just updates the digits in place without interrupting the moving parts. Like changing the score at a stadium: instead of tearing down the entire scoreboard and installing a new one, they just flip the number cards while the game plays on.

This prevents the CSS animation flicker during normal refreshes. (#39, by @ericweichun)

ja

コスト計算とモデル統計で Codex のデータを取りこぼさない

請求書やモデルの使用割合を確認する際、Codex の記録が見つからないせいで 0 ドルや「不明」と見積もられた数字に惑わされることがなくなった。新版の Codex は、どのモデルを使ったかを別の場所（turn context）に書き込むことがあるが、従来見に行っていた場所で見つからなくても、今は新しい場所を探り当ててくれるため、請求書や統計グラフに穴が開かない。レジでバーコードが見つからない時、店員が箱の底にある予備のラベルを自ら探して読み取ってくれるようになったようなものである。

アニメーションパネルが更新時にちらつかなくなる

枠のプログレスバーにアニメーションがあるパネル（Prism Arcade やブラックホール、水族館など）において、数値を更新する際に画面がチラッと光ることがなくなった。以前は状態が更新されるたびに、システムがアニメーションの軌道全体を壊して作り直していたため、アニメーションが最初から再生され、ちらつきが発生していた。今はその場で数字だけを書き換え、動いているアニメーションを途切らせない。球場のスコアボードの数字を変える際、以前はボードごと取り外して付け直していたのが、今は数字のパネルだけを差し替え、試合の進行を邪魔しなくなったようなものである。

This prevents the CSS animation flicker during normal refreshes. (#39, by @ericweichun)
