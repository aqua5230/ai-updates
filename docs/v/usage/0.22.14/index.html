zh-TW

歷史讀取更穩更快，壞掉時會坦白告訴你

你現在會發現歷史讀取更穩更快，而且壞掉時也會坦白告訴你了。背後整理歷史紀錄的機制做了大掃除，現在只要掃描一次目錄，就能同時算好數字並找出有變動的地方，不用重複多跑好幾趟；而且如果你把 Codex 對話封存（archive）了，它也能即時跟上變化，不會再顯示舊數字。此外，萬一系統讀不到你的對話紀錄，以前它會悄悄拿上次的舊資料充數、讓你以為沒人在講話，現在它會在卡片旁邊掛一個小徽章，老實告訴你「我讀不到資料」；如果背後負責更新狀態的程式（hook）壞了，它也不會再像無頭蒼蠅一樣瘋狂重掃整個資料夾。就像以前計分板壞了會一直顯示 0 分讓你以為沒人得分，現在它會掛個牌子說「機器故障」，而且修理工人不會再為了找問題把整個球場的地板掀開。

Original changelog
**Archived Codex sessions could show stale data**: usage counted `~/.codex/archived_sessions/` when computing totals but didn't watch it for changes, so edits there might not refresh the menu bar until an unrelated refresh happened; the archived folder is now included in the same change-detection that drives live updates.
**A broken usage-data hook no longer causes constant background rescanning**: when Claude Code's status hook goes stale, usage now caches the "is there recent activity" check briefly instead of rescanning your entire `~/.claude/projects` tree on every poll.
**A small warning badge appears if local usage history can't be read**: previously, a failed read silently fell back to the last known data with no visible sign anything was wrong; the Project Usage card now shows a brief note (hover for detail) so you can tell "no new data" apart from "something broke."
**Faster per-refresh history scans**: each refresh previously walked the Claude/Codex session directories multiple times (once to detect changes, again per data source); it's now scanned once and reused.

en

History reads are faster, stabler, and tell you honestly when they break

You'll notice that reading your history is now faster, more stable, and more honest when things break. The engine that crunches your history in the background has been completely overhauled. It now scans your directories just once to both calculate the numbers and spot any changes, rather than making multiple redundant trips; also, if you archive your Codex chats, the system now tracks those edits instantly instead of showing stale data. Additionally, if the system fails to read your chat logs, it used to quietly fall back to the last known data, making you think you simply hadn't typed anything; now, it hangs a small warning badge on the card, honestly confessing that it can't read the data. And if the background updater program (the hook) breaks, it will no longer frantically rescan your entire folder tree over and over. It's like a broken stadium scoreboard that used to freeze at zero and make you think no one was scoring, but now it hangs a "Broken" sign — and the repair crew no longer rips up the entire field trying to find the problem.

履歴の読み込みが速く安定し、壊れた時は正直に教えてくれる

履歴の読み込みがより速く安定し、万が一壊れた時も正直に教えてくれるようになった。裏で履歴を整理する仕組みが大掃除され、ディレクトリを一回スキャンするだけで、計算も変更箇所の特定も同時に行えるようになり、何度も無駄走りしなくなった。また Codex の対話をアーカイブ（archive）した場合も、その変更に即座に追従し、古い数字を表示したままにしない。さらに、万が一対話記録が読み取れなくなった際、以前はこっそり前回データを流用して「誰も話していない」と勘違いさせていたのが、今はカードの横に小さなバッジを付け、「データが読めない」と正直に白状する。状態を更新するプログラム（hook）が壊れた際も、頭の無いハエのように全フォルダを狂ったように再スキャンし続けることはなくなった。以前はスコアボードが壊れると 0 点のまま表示され、誰も得点していないように見えていたのが、今は「故障中」の札を掲げるようになり、修理工も原因探しのために球場の床を全部ひっぺがすような真似はしなくなったようなものである。

ko

기록 읽기가 더 빠르고 안정적이며, 고장 났을 때 솔직하게 알려줌

이제 기록을 불러오는 속도가 훨씬 빠르고 안정적이며, 무언가 고장 났을 때도 솔직하게 알려준다는 걸 알게 될 것이다. 뒤에서 역사 기록을 정리하는 방식이 싹 청소됐다. 이제는 폴더를 딱 한 번만 훑어봐도 숫자 계산과 바뀐 부분 찾기를 동시에 해내, 여러 번 헛걸음할 필요가 없다. 당신이 Codex 대화를 보관(archive)해 두었을 때도, 바뀐 걸 즉시 알아채고 낡은 숫자를 띄우지 않는다. 또한, 시스템이 대화 기록을 읽지 못할 때 예전엔 몰래 지난번 옛날 자료로 때우며 아무도 말 안 하는 척했지만, 이제는 카드 옆에 작은 배지를 달아 '자료를 못 읽겠어'라고 솔직하게 고백한다. 뒤에서 상태를 업데이트하는 프로그램(hook)이 고장 났을 때도 머리 없는 파리처럼 폴더 전체를 미친 듯이 다시 뒤지지 않는다. 예전엔 점수판이 고장 나면 0점만 띄워 아무도 점수를 못 낸 줄 알게 했지만, 이제는 '고장 났음' 팻말을 걸어 두고, 수리공이 문제를 찾겠다고 경기장 바닥을 다 뜯어내지도 않는 것과 같다.
