zh-TW

Windows 視窗切換不留白，也不再亮瞎眼

在 Windows 上快速切換面板時，畫面不會再莫名其妙留白，也不會再閃現刺眼的白光了。以前在切換面板時，系統會在 pywebview（讓桌面程式內嵌顯示網頁畫面的套件）的 JS-API 回呼函式（callback，工作做完後系統自動呼叫回來的那段程式碼）還沒跑完前就急著重新載入畫面，這會把用來等候結果的 Promise（JavaScript 裡用來處理未來才會完成的工作的排隊號碼牌）給摧毀掉，導致 Edge WebView（在桌面程式裡嵌入網頁畫面的瀏覽器元件）卡在一片空白。而且，原本 pywebview 視窗的預設背景色死板地設為白色，就算你開了深色模式，在載入瞬間或圓角邊緣還是會閃瞎你的眼睛。現在它懂等橋樑溝通完畢才載入，還加上了防禦機制，就算你急躁地連點兩下，系統也會重新算出你最後想切去的那個目標面板，不會傻傻沿用已經過期的舊面板；同時視窗背景也會自動讀取你的系統深淺色主題。此外，Windows 的打包檔（PyInstaller bundle，把 Python 程式連同工具包壓成單一執行檔的打包方式）現在也補齊了先前遺漏的 status-line 與 session-hook 原始檔，讓安裝與自我修復流程在執行期能順利複製這些檔案。就像電視換台，以前轉台太快會卡在沒有訊號的白畫面，或者看恐怖片看到一半切台被突然閃現的白光亮瞎；現在不管你怎麼瘋狂轉台，畫面都順滑無比，並且保持在不刺眼的電影院模式。

Original changelog
**Switching panels on Windows no longer risks a blank white window**: the panel-switch action reloaded the document synchronously from inside the pywebview JS-API callback, which could destroy the in-flight JavaScript Promise callback for that call and leave Edge WebView blank. The reload is now deferred until the bridge call has returned to JavaScript, with a pending-switch guard so rapid double-clicks recompute the target panel instead of reusing a stale one. The Windows PyInstaller bundle also now includes the status-line and session-hook source files that the setup and self-heal paths copy at runtime, which it previously omitted.
**The Windows tray window no longer flashes white regardless of theme**: pywebview's window background defaults to white, which bled through the frameless window's rounded corners (and any unpainted first frame) even when the active panel's CSS was rendering its dark variant. The window background now reads the system's light/dark theme setting and matches the panel's background color.

Windows panel switching avoids blank screens and white flashes

Switching panels rapidly on Windows won't leave you stranded on a blank white screen anymore, and the jarring white flashes are gone. Previously, a panel-switch action rudely reloaded the document before pywebview (a package that embeds web content within desktop apps) finished its JS-API callback (code automatically executed after a task completes), completely destroying the in-flight Promise (a JavaScript placeholder for a task that will finish in the future) and leaving the Edge WebView (the browser component embedded inside the desktop app) entirely blank. Furthermore, pywebview's default window background was hardcoded to white, bleeding blinding flashes through rounded corners or the first frame even if you were using a dark theme. Now, the reload patiently waits until the communication bridge has safely returned, with an extra pending-switch guard so rapid double-clicks correctly recalculate the final target panel instead of reusing an outdated locked one; additionally, the window properly reads your system's light/dark theme. Furthermore, the Windows package (PyInstaller bundle, a way to compress the Python program and tools into a single executable file) now includes the previously omitted status-line and session-hook source files, allowing the setup and self-heal paths to copy them correctly at runtime. It's like flipping channels on a TV—it used to get stuck on blinding static if you mashed the remote, or flash a bright white light before switching to your dark movie; now it changes channels buttery smooth and stays comfortably in dark theater mode.

Windows 画面切り替え時の白紙化と眩しい発光を解消

Windows でパネルをすばやく切り替える際、画面が真っ白のまま固まることも、眩しい白光がフラッシュすることもなくなった。以前は、パネル切り替え操作の際、pywebview（デスクトップアプリ内にウェブ画面を埋め込むためのパッケージ）の JS-API コールバック（処理が終わった後にシステムが自動的に呼び出すコード）が終わる前にシステムが急いでドキュメントを再読み込みしていたため、結果を待つための Promise（JavaScript において未来に完了する作業を待つための整理券）が破壊され、Edge WebView（デスクトップアプリの中にウェブ画面を埋め込むためのブラウザ部品）が白紙状態で止まってしまっていた。その上、pywebview ウィンドウのデフォルトの背景色が頑なに白色とされていたため、ダークモードを設定していても、読み込みの瞬間や角丸の隙間から眩しい白光が漏れ出していた。今は、通信の架け橋が安全に戻ってくるまで再読み込みを待ち、ユーザーが焦ってダブルクリックした際にも古い画面を使い回すのではなく、最終的な切り替え先のパネルを正確に再計算するガード機能を追加した。さらに、ウィンドウの背景はシステムのライト／ダークテーマを正しく読み取るようになった。加えて、Windows 用のパッケージ（PyInstaller bundle、Python プログラムやツールを単一の実行ファイルに圧縮する方式）には、これまで漏れていた status-line と session-hook のソースファイルが含まれるようになり、インストールや自己修復処理が実行時に正しくファイルをコピーできるようになった。テレビのチャンネルを切り替える際、以前はリモコンを連打すると信号のない真っ白な画面で固まったり、暗い映画に変える直前に眩しい白い光がフラッシュしたりしていたのが、今は何度切り替えてもスムーズで、目に優しい暗い映画館モードのまま保たれるようなものである。

Windows 창 전환 시 하얀 화면과 눈부심 방지

Windows에서 패널을 빠르게 전환할 때 화면이 뜬금없이 하얗게 굳어버리거나 눈부신 흰빛이 번쩍이는 일이 사라졌다. 예전엔 패널을 바꿀 때 pywebview(데스크톱 앱 안에 웹 화면을 표시해 주는 패키지)의 JS-API 콜백(작업이 끝난 뒤 시스템이 자동으로 호출하는 코드)이 일을 채 마치기도 전에 시스템이 급하게 화면을 다시 불러오는 바람에, 결과를 기다리던 Promise(JavaScript에서 미래에 끝날 작업을 위해 쥐어주는 대기표)가 파괴되어 Edge WebView(데스크톱 앱 안에 웹 화면을 끼워 넣는 브라우저 부품)가 하얀 백지로 멈춰버리곤 했다. 게다가 pywebview 창의 기본 배경색이 무식하게 흰색으로 고정되어 있어서 다크 모드를 켜더라도 로딩되는 순간이나 둥근 모서리 틈새로 눈을 찌르는 흰빛이 새어 나왔다. 이제는 통신 다리가 무사히 대답을 돌려줄 때까지 재장전을 얌전히 기다리며, 조급하게 더블 클릭을 하더라도 만료된 예전 화면을 재사용하지 않고 최종적으로 넘어가야 할 목표 패널을 다시 계산하도록 방어막을 추가했다. 또한 창 배경이 시스템의 라이트/다크 테마 설정을 올바르게 읽어온다. 게다가 Windows 패키지(PyInstaller bundle, Python 프로그램과 도구를 하나의 실행 파일로 압축하는 방식)에 그동안 빠져 있던 status-line 및 session-hook 소스 파일이 추가되어, 설치 및 자가 복구 과정에서 실행 시 이 파일들을 무사히 복사할 수 있게 되었다. 텔레비전 채널을 돌릴 때, 예전엔 리모컨을 마구 누르면 신호가 끊긴 하얀 화면에서 먹통이 되거나 어두운 영화로 넘어가기 전 눈부신 백색광이 번쩍였지만, 이제는 아무리 채널을 돌려도 매끄럽게 넘어가고 눈이 편안한 어두운 극장 모드를 그대로 유지하는 것과 같다.
