zh-TW

Windows 上的額度收集不再罷工，路徑與憑證抓取更聰明

如果你是用 Windows 跑這些 AI 工具，現在統計使用額度時，終於不會卡住或抓不到你的身分了。以前 Antigravity 的額度探測（quota probing，去查你剩下多少可用次數的動作）只認得舊版的 token 檔案，這個檔案不見或壞掉時就會直接罷工；現在它多了一條退路，會退而求其次去唯讀 Windows CLI 目前的 OAuth 憑證（OAuth credential，一種讓程式證明這是你本人授權的通行證，存在 Windows Credential Manager 的 `gemini:antigravity` 項目裡），而且回報資料時能正確說出你是哪種主機平台，不再一律宣稱大家都是 Mac（Darwin/arm64）。另外，當你在 Windows 搭配 Git Bash 收集 Claude Code 額度時，以前狀態列指令（status-line commands，顯示在終端機底部的狀態更新指令）用的是 Windows 的反斜線路徑，這在 Git Bash 裡會被當成跳脫字元（escape character，用來標記特殊符號的寫法）把路徑吃掉，害背後的 Python 程式無法啟動。現在全部改用正斜線，啟動時的自我修復（self-heal，程式自己動手更新舊設定的機制）會順手幫你把舊指令升級，用 `--doctor` 也能幫你抓出舊格式並提示復原步驟。就像員工去外地出差報帳，以前只認一種舊版的紙本通行證，弄丟了就直接卡關，填表還會因為斜線打反導致印表機當機；現在多了備案，只要翻出目前有效的電子授權證明就能確認身分，並把報表格式統一，甚至連你以前寫錯的舊單據都順手改對了。

Original changelog
**Antigravity quota probing now reads the current Windows CLI OAuth credential**: when the legacy token file is missing or unusable, usage falls back read-only to the `gemini:antigravity` Windows Credential Manager entry. The quota request's user agent now also identifies the actual host platform instead of always claiming Darwin/arm64.
**Claude Code quota collection now works with Git Bash on Windows**: status-line commands were written with Windows backslashes, but Claude Code runs them through Git Bash when it is installed, where those backslashes escape the path and prevent the Python hook from launching. New commands use portable forward slashes; startup self-heal migrates existing usage-owned commands, and `--doctor` identifies the legacy form and the recovery step.

en

Quota collection stops failing on Windows with smarter paths and credentials

If you're running these AI tools on Windows, tallying your usage quota will finally stop failing or losing track of your identity. Previously, Antigravity's quota probing (the action that checks how many requests you have left) only recognized the legacy token file, and would simply give up if it was missing or corrupted; now it has a fallback, reading the current Windows CLI OAuth credential (a pass proving you authorized the program, stored in the Windows Credential Manager's `gemini:antigravity` entry) in read-only mode, and it correctly identifies your actual host platform instead of blindly claiming everyone is on a Mac (Darwin/arm64). Additionally, when collecting Claude Code quota via Git Bash on Windows, status-line commands (instructions that update the state at the bottom of the terminal) used to be written with Windows backslashes, which Git Bash treated as escape characters (syntax used to mark special symbols), ruining the path and preventing the background Python hook from launching. Everything now uses portable forward slashes; a startup self-heal (a mechanism where the program automatically upgrades its own old configurations) process silently migrates your old commands, and running `--doctor` will also catch legacy formats to guide you through recovery. It's like an employee filing an expense report on a business trip — previously, only an old paper pass was accepted, and losing it would instantly halt the process, while writing slashes backward would jam the printer; now there's a backup option to verify identity using your current valid digital authorization, standardizes the form formatting, and automatically corrects any incorrectly filled old forms on the spot.

Windows でのクオータ収集が機能停止しなくなり、パスや認証情報の取得がより賢明に

Windows でこれらの AI ツールを使っている場合、使用量（クオータ）を集計する際にエラーで止まったり身元を見失ったりすることがついに解消された。以前は Antigravity のクオータ探査（quota probing、残り利用可能回数を調べる動作）において、古いトークンファイルしか認識せず、それが見つからないか破損していると即座に処理を放棄していたが、現在は代替経路が追加され、現在の Windows CLI の OAuth クレデンシャル（OAuth credential、本人が許可したことを証明するパスであり、Windows クレデンシャルマネージャーの `gemini:antigravity` 項目に保存される）を読み取り専用で参照するようになり、皆が Mac（Darwin/arm64）を使っていると盲目的に申告するのをやめ、実際のホストプラットフォームを正確に報告する。また、Windows 上の Git Bash で Claude Code のクオータを収集する際、以前のステータスラインコマンド（status-line commands、端末下部の状態を更新する指示）には Windows 特有のバックスラッシュが使われていたため、Git Bash がそれをエスケープ文字（escape character、特殊記号を示すための記法）として扱ってパスを破壊し、裏側の Python フックが起動できなくなっていた。現在はすべてポータブルなスラッシュに改められ、起動時の自己修復（self-heal、プログラムが自ら古い設定を更新する仕組み）機能が古いコマンドを自動的に移行し、`--doctor` を使えば古い形式を検知して復旧手順を教えてくれる。以前は出張先で経費精算をする社員が、古い紙の通行証しか認めてもらえず、なくしただけで手続きが止まり、斜線の向きを間違えただけでプリンターを詰まらせていたのが、今は現在有効な電子承認を用いて身分証明できる代替手段ができ、書類の書式を統一し、過去の書き間違えた伝票までついでに直してくれるようなものである。

ko

Windows에서 할당량 수집이 더 이상 멈추지 않고, 경로와 자격 증명 확인이 똑똑해진다.

Windows에서 이 AI 도구들을 쓴다면, 사용량(할당량)을 집계할 때 더 이상 멈추거나 당신의 신원을 놓치는 일이 사라진다. 예전엔 Antigravity의 할당량 탐색(quota probing, 남은 사용 가능 횟수를 확인하는 동작) 시 오래된 토큰 파일만 인식했고, 이 파일이 없거나 망가지면 바로 파업해 버렸다. 이제는 퇴로가 생겨, 현재의 Windows CLI OAuth 자격 증명(OAuth credential, 본인이 권한을 주었음을 증명하는 통행증으로 Windows 자격 증명 관리자의 `gemini:antigravity` 항목에 저장됨)을 읽기 전용으로 돌아보며, 모두가 Mac(Darwin/arm64)을 쓴다고 맹목적으로 우기는 대신 실제 호스트 플랫폼을 정확히 보고한다. 또한, Windows의 Git Bash에서 Claude Code 할당량을 수집할 때, 예전엔 상태 표시줄 명령어(status-line commands, 터미널 밑부분의 상태를 새로 고치는 지시)에 Windows식 백슬래시를 썼는데 Git Bash가 이를 이스케이프 문자(escape character, 특수 기호를 나타내는 표기법)로 여겨 경로를 망가뜨리고 뒤에 있는 Python 프로그램 실행을 막았다. 이제는 모두 호환성 좋은 슬래시로 바뀌었고, 시작할 때 자가 복구(self-heal, 프로그램이 스스로 낡은 설정을 고치는 기능)가 기존 명령어를 알아서 업그레이드해주며, `--doctor`를 쓰면 예전 형식을 찾아내 복구 방법도 알려준다. 예전엔 출장 간 직원이 경비 정산을 할 때 낡은 종이 출입증만 인정받아 하나 잃어버렸다고 절차가 멈추고, 빗금을 반대로 그었다는 이유로 프린터가 멈춰 섰다면, 이제는 현재 유효한 전자 승인을 통해 신분을 증명하는 대안이 생겼고 서식 방향을 통일하며, 예전에 잘못 쓴 서류까지 덤으로 고쳐 주는 것과 같다.
