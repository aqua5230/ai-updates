zh-TW

「開機自動啟動」開關立刻生效，不用等到下次重開機

以前你在選單裡點了「開機自動啟動」的開關，系統只會默默記在筆記本裡，你得等到下一次重開機，電腦才會真的照做；要是你把它關掉，背景還會留下一支不會停的舊程式。現在只要你一點開關，系統就會馬上拍拍電腦的肩膀跟它說「現在就開始」或「馬上停下來」，不用再等重開機了。就像你跟大樓管理員說明天開始不要送報紙了，以前他只會寫在明天的交接本上，今天還是照送；現在他會立刻用對講機跟派報生說停送。

Original changelog
**"Launch at login" toggle now takes effect immediately, no reboot needed**: `login_item.enable()` / `disable()` now invoke `launchctl bootstrap gui/<uid> <plist>` / `launchctl bootout gui/<uid>/<label>` in addition to writing/removing `~/Library/LaunchAgents/com.lollapalooza.usage.plist`, so launchd learns about the change right away. Previously only the plist file was touched, so the toggle did nothing until the next reboot, and disabling left a KeepAlive orphan process behind. `launchctl` "already bootstrapped" (exit 17) and "not bootstrapped" (exit 113) are treated as success; other failures log a warning without affecting the plist operation (signatures stay `() -> None`).

en

The "Launch at login" toggle now takes effect instantly without a reboot

When you toggled "Launch at login" from the menu, the system previously just jotted it down in a config file and waited for your next reboot to actually do anything about it; and turning it off left a zombie background process running. Now, the moment you flip the switch, the system instantly taps macOS on the shoulder to say "start doing this now" or "stop right now", with zero reboots required. It's like telling the building manager to cancel your newspaper delivery: they used to just write it in tomorrow's log while still delivering today's paper, but now they immediately radio the delivery person to stop.

Original changelog

Previously only the plist file was touched, so the toggle did nothing until the next reboot, and disabling left a KeepAlive orphan process behind. `launchctl` "already bootstrapped" (exit 17) and "not bootstrapped" (exit 113) are treated as success; other failures log a warning without affecting the plist operation (signatures stay `() -> None`).

ja

「起動時に開く」スイッチが即座に反映され、再起動が不要に

以前メニューの「起動時に開く」スイッチを切り替えても、システムは設定ファイルにこっそり書き込むだけで、次に再起動するまでパソコンは何もしてくれなかった。しかもオフにした時には、裏で古いプログラムが止まらずに居座り続けていた。今はスイッチを押した瞬間に、システムがパソコンの肩を叩いて「今すぐ始めろ」または「今すぐ止まれ」と伝えるため、もう再起動を待つ必要はない。マンションの管理人に明日から新聞配達を止めるよう頼んだ際、以前は明日の引き継ぎノートに書くだけで今日はまだ配達されていたのが、今はすぐにトランシーバーで配達員に中止を伝えてくれるようなものである。

ko

'부팅 시 자동 시작' 스위치가 재부팅 없이 즉시 적용된다.

예전엔 메뉴에서 '부팅 시 자동 시작' 스위치를 켜고 꺼도 시스템이 설정 파일에 조용히 적어 두기만 해서 다음 번에 컴퓨터를 껐다 켜야만 진짜로 작동했다. 게다가 끌 때는 뒤에서 멈추지 않는 옛날 프로그램이 좀비처럼 남아 있었다. 이제는 스위치를 누르는 순간 시스템이 컴퓨터의 어깨를 툭툭 치며 '지금 바로 시작해' 혹은 '당장 멈춰'라고 말해주므로 재부팅을 기다릴 필요가 없다. 아파트 관리인에게 내일부터 신문 배달을 끊어달라고 했을 때, 예전엔 내일 자 인수인계 장부에만 적어둬서 오늘은 기어이 신문이 날아왔다면, 이제는 그 자리에서 바로 무전기를 들어 배달원에게 중지하라고 말해주는 것과 같다.
