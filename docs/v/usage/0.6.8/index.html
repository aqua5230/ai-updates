zh-TW

修正打包版 App 找不到多語系檔而崩潰的問題

之前直接點擊安裝好的 App 啟動時，因為打包軟體漏掉了一份多語系檔（i18n.json），導致系統一開機就找不到檔案而直接閃退崩潰。現在我們把這份負責翻譯的檔案正確包進去了，而且系統也學會優先從 App 自己的肚子裡找檔案。就像是買了進口家電，以前箱子裡漏放了翻譯說明書，你一開機就完全無法使用；現在原廠已經把說明書乖乖塞進包裝盒，拿出來就能直接懂它的語言了。

Original changelog
**Fix .app launch failure when i18n.json is missing**: py2app now includes `i18n.json` in the resource list, and the menu bar / Web panel loaders prefer the `.app` bundle's `Contents/Resources/i18n.json` before falling back to source-tree paths, preventing the `FileNotFoundError` that broke v0.6.0+ launches.

en

Fixed bundled app crashing due to a missing translation file

Previously, when launching the compiled App directly, a missing translation file (i18n.json) caused the system to panic and crash immediately on startup. We've now properly packed this responsible language file inside, and the system has learned to prioritize looking inside its own app bundle for it. It's like buying an imported appliance where the box used to be missing the translated manual, leaving you completely stuck on power-up; now the manufacturer has properly tucked the manual into the packaging so you can understand its language right out of the box.

Original changelog
**Fix .app launch failure when i18n.json is missing**: py2app now includes `i18n.json` in the resource list, and the menu bar / Web panel loaders prefer the `.app` bundle's `Contents/Resources/i18n.json` before falling back to source-tree paths, preventing the `FileNotFoundError` that broke v0.6.0+ launches.

ja

パッケージ版アプリが多言語ファイルを見つけられずにクラッシュする問題を修正

以前はインストール済みのアプリを直接クリックして起動した際、パッケージソフトが多言語ファイル（i18n.json）を入れ忘れていたため、システムが起動直後にファイルを見つけられず、即座にクラッシュして強制終了していた。今回、この翻訳を担当するファイルを正しく梱包し、システムも優先的にアプリ自身の中身からファイルを探すように改善された。輸入家電を買った時、以前は箱の中に翻訳済みの説明書が入っておらず、電源を入れても全く使えなかったのが、今ではメーカーがきちんと箱に説明書を入れており、取り出してすぐにその言語が理解できるようになったようなものである。

Original changelog
**Fix .app launch failure when i18n.json is missing**: py2app now includes `i18n.json` in the resource list, and the menu bar / Web panel loaders prefer the `.app` bundle's `Contents/Resources/i18n.json` before falling back to source-tree paths, preventing the `FileNotFoundError` that broke v0.6.0+ launches.

패키지 앱이 다국어 파일을 찾지 못해 튕기던 문제 수정

예전엔 설치된 앱을 직접 눌러 실행할 때, 패키징 소프트웨어가 다국어 파일(i18n.json)을 빼먹는 바람에 시스템이 켜지자마자 파일을 찾지 못하고 곧바로 튕기며 꺼져버렸다. 이제는 이 번역을 맡은 파일을 제대로 포장해 넣었고, 시스템도 앱 자신의 배 속에서 먼저 파일을 찾도록 배웠다. 수입 가전제품을 샀는데 예전엔 상자에 번역된 설명서가 빠져 있어 전원을 켜도 도무지 쓸 수 없었던 것을, 이제는 제조사가 포장 상자에 설명서를 얌전히 넣어 두어 꺼내자마자 그 언어를 알아들을 수 있게 된 것과 같다.
