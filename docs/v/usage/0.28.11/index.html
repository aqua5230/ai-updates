zh-TW

用量報表不再像套版，重點更好找也更好看

月底打開 HTML 用量報表（可在瀏覽器看的網頁格式）時，重要數字終於不必從一片金色裡挖。金色現在只留給標題尾端、主要 KPI 數字（最關鍵的指標）、Wrapped 小標與終端機游標；貢獻熱圖和甜甜圈圖則改用一致的藍綠與藕紫。這像把舞台追光只打在主角身上，其餘演員穿協調的服裝，目光自然會落到最該看的地方。六張 KPI 卡片的排列距離已對齊，Wrapped 的總 token 數（AI 處理文字的計量單位）再長也不會壓住鳳凰插圖，最後一張指標卡也不會孤零零掉到下一列。卡片和區塊邊框在兩套配色中都更清楚；按鈕的停用、啟用狀態也更好認。動畫會尊重「減少動態效果」偏好，分享視窗三個圖示換成筆畫一致的 SVG 圖檔。連 CSS 的 transition: all 都改成只列會動的屬性，少掉模板常見的多餘手勢。

Original changelog
**The HTML usage report looks less like a template and reads more cleanly**: gold is no longer a blanket accent — it's now reserved for four focal points (headline tail, primary KPI number, the Wrapped kicker badge, the terminal cursor), with the contribution heatmap and donut chart moved to a cohesive teal/mauve palette instead. Fixed three alignment bugs: the KPI card grid now keeps a consistent rhythm across all six cards, the Wrapped section's total-token figure no longer overlaps the phoenix illustration at high digit counts, and the trailing metric card in that section no longer orphans onto its own row. Also cleans up several small "obviously AI-generated" tells: `transition: all` rules now list only the properties that actually animate, card/section borders read at a clearer contrast in both color schemes, disabled/active button states are now styled instead of falling back to defaults, motion respects `prefers-reduced-motion`, and the three share-dialog emoji icons are now matched-stroke SVGs.

en

Usage reports look less templated and put the important numbers in focus

When you open your HTML usage report (the web-page version you view in a browser) at the end of the month, the important numbers no longer have to fight through a sea of gold. Gold is now reserved for the headline tail, main KPI number (the key performance indicator), Wrapped badge, and terminal cursor; the contribution heatmap and donut chart use a coordinated teal-and-mauve palette instead. It is like saving the spotlight for the lead actor while the rest of the cast wears matching costumes. The six KPI cards now keep even spacing, a long Wrapped total-token count no longer overlaps the phoenix, and the last metric card no longer drops onto a row by itself. Card and section borders are clearer in both color schemes, while button states are easier to recognize. Motion follows reduced-motion preferences, and the share dialog's three icons now use matching-stroke SVG artwork. Even CSS transition: all rules now name only properties that move, removing a telltale template habit.

ko

사용량 보고서가 틀에 박힌 느낌을 벗고 중요한 숫자를 더 쉽게 찾게 한다

월말에 HTML 사용량 보고서(브라우저에서 보는 웹페이지 형식)를 열면 중요한 숫자를 금빛 화면 속에서 찾아 헤맬 필요가 없다. 금색은 이제 제목 끝부분, 주요 KPI 숫자(가장 중요한 지표), Wrapped 배지, 터미널 커서에만 쓰고 기여도 히트맵과 도넛 차트는 조화로운 청록·연보라 색으로 바꿨다. 무대 조명은 주인공에게만 비추고 다른 배우에게는 어울리는 의상을 입히는 것과 같다. KPI 카드 여섯 장의 간격이 고르게 맞고, Wrapped의 총 token 수(AI가 글자를 처리하는 계량 단위)가 길어도 불사조 그림을 덮지 않으며 마지막 지표 카드만 다음 줄에 홀로 남지 않는다. 카드와 구역의 테두리는 두 색 구성 모두에서 더 또렷하고, 버튼의 비활성·활성 상태도 더 분명하다. 움직임은 동작 줄이기 설정을 따른다. 공유 창의 아이콘 세 개는 선 굵기가 맞는 SVG 그림으로 바뀌었으며 CSS의 transition: all도 실제로 움직이는 속성만 적는다.
