zh-TW

Windows 也能一鍵保住五小時工作階段

你在 Windows 上盯著 Usage 的額度面板，不必再每隔一陣子手動續住工作。系統匣（tray，Windows 右下角收納常駐程式圖示的地方）現在也有 macOS 已有的「自動啟動 5 小時工作階段」開關；在系統匣選單或 HTML 面板選單打開它，程式每次重新整理後都會接上既有的 window_keeper 自動 ping 機制，替這段 5 小時工作階段續著。像替停車計時器開啟自動續費，不用守在旁邊反覆按鈕。若 Windows 阻擋背景運作，畫面也會指向「設定 → 電源與電池」的啟用方法。另補上 claude 與 agy 執行檔（讓指令能啟動程式的檔案）在 Windows 常見安裝位置的備援路徑，找程式時更不易撲空。

Original changelog
**Windows tray now has the same "Auto-start 5-hour session" toggle macOS has had.** Wires the existing `window_keeper` auto-ping logic into `wintray.py`, firing after each refresh; the toggle is in both the tray menu and the HTML panel menu, with Windows-specific enable instructions (Settings → Power & battery) across all five languages. Also adds real-world Windows install-path fallbacks for the `claude`/`agy` binaries.

en

Windows can now keep a five-hour session going automatically

If you keep an eye on Usage's quota panel on Windows, you no longer need to keep a session alive by hand. The system tray now gets the same "Auto-start 5-hour session" switch macOS already had. Turn it on in either the tray menu or HTML panel menu, and each refresh reconnects the existing window_keeper auto-ping logic to keep that five-hour session going. It is like enabling automatic top-ups on a parking meter instead of standing beside it to press a button again and again. If Windows blocks background operation, the app points you to Settings → Power & battery for the enablement steps. It also adds fallback paths for the claude and agy binaries (files that let a command start a program) in real-world Windows install locations, so the app is less likely to come up empty when looking for them.

ko

Windows에서도 5시간 세션을 자동으로 유지할 수 있다

Windows에서 Usage의 할당량 패널을 지켜본다면, 세션을 유지하려고 틈틈이 손으로 조작할 필요가 없어졌다. 시스템 트레이(tray, Windows 오른쪽 아래에 상주 프로그램 아이콘을 모아 두는 곳)에도 macOS에 있던 ‘5시간 세션 자동 시작’ 스위치가 생겼다. 트레이 메뉴나 HTML 패널 메뉴에서 켜면, 새로 고칠 때마다 기존 window_keeper 자동 ping 기능이 이어져 그 5시간 세션을 유지한다. 주차 미터기에 자동 연장을 켜 두고 옆에서 버튼을 계속 누르지 않아도 되는 것과 같다. Windows가 백그라운드 실행을 막으면 화면에서 ‘설정 → 전원 및 배터리’로 가는 활성화 방법도 안내한다. 또 claude와 agy 실행 파일(명령어로 프로그램을 시작하게 하는 파일)의 Windows에서 흔한 설치 위치를 위한 대체 경로도 더해, 프로그램을 찾지 못할 일이 줄었다.
