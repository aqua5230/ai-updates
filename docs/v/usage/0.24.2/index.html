zh-TW

恢復工作對話不再被指令碼洗版，也不會重複抓取任務

當你接續之前的對話紀錄時，系統提議的「你最近在做什麼」清單，不再會被那些落落長的工具說明書洗版了。以前系統在整理你的對話紀錄時，只要看到是標示為「使用者發出」的內容就全抓進來，根本不管那是不是系統自己為了擴充技能（skill，輔助執行特定任務的指令檔）而偷塞的冗長說明，結果把這些雜訊也當成你的工作進度；而且因為它只跟上一句話比對，同一句話如果被這些雜訊隔開，就會被重複計算，白白浪費了最多只能顯示三筆的珍貴名額。現在系統學會了跳過這些標記為內部後設資料（isMeta）的雜訊，並且會跟整份清單比對來剔除重複。就像你請秘書整理昨天開會的重點，以前他會連你在會上念的一大本操作手冊都逐字抄進去當作討論結論，而且你重複講了兩次的話他還會當成兩件不同的事記下來；現在他懂得只挑出真正的行動方案，並且把重複講的話自動合併成一條。

Original changelog
**Session-resume handoff could surface skill-expansion noise and duplicate the same request**: `usage_session_resume.py`'s `_parse_session` collected every `type: "user"` transcript entry as a candidate "recently working on" request without checking Claude Code's `isMeta` flag, so skill/command expansions injected into the transcript (e.g. a full `SKILL.md` body) could show up as if they were a real request. Dedup also only compared against the immediately preceding request, so the same request separated by one of these injected entries counted twice, wasting handoff slots that are capped at 3. `isMeta: true` entries are now skipped, and dedup checks against every request seen so far, not just the last one. (Reported in #46 by @apple8409.)

en

Session handoffs filter out noisy tool injections and correctly deduplicate task history

When you resume a previous conversation, the "what you were recently working on" suggestions will no longer be overwhelmed by giant walls of tool documentation. Previously, the system blindly scooped up every transcript entry marked as a "user" message, failing to realize that some of these were actually long instructions injected by the system itself to expand skills (skill files that help execute specific tasks). This allowed noisy text to masquerade as your real work progress. Furthermore, because the system only checked for duplicates against the very last message, a single real request interrupted by one of these noise entries would get counted twice, wasting your precious three handoff slots. The system now correctly skips these system-injected metadata entries (isMeta) and checks for duplicates against the entire list. It's like asking an assistant to summarize yesterday's meeting: they used to copy down an entire 100-page manual you happened to read aloud as if it were a decision, and if you repeated a point twice they logged it as two separate tasks. Now, they know to extract only the actual action items and merge repeats into a single line.

作業再開時にコマンドの出力ログが混ざらなくなり、タスクの重複抽出も解消

以前の会話記録から作業を再開する際、システムが提案する「最近の作業内容」のリストが、長大なツールの説明書で埋め尽くされることはなくなった。これまでは、システムが対話履歴を整理する際、「ユーザーの発言」とマークされたものを手当たり次第に拾い上げ、それがシステム自身がスキル（skill、特定のタスク実行を補助するコマンドファイル）を拡張するためにこっそり差し込んだ長い説明であることには全く気付かず、ただのノイズをあなたの作業進捗として扱ってしまっていた。さらに、直前の一文としか重複チェックをしていなかったため、同じ言葉の間にこれらのノイズが挟まると2回カウントされ、最大3つしかない貴重な引き継ぎ枠を無駄に消費していた。現在は、内部のメタデータ（isMeta）とマークされたノイズを正しくスキップし、リスト全体と照合して重複を弾くようになった。秘書に昨日の会議の要点をまとめさせた時、以前はあなたが会議中に読み上げた分厚いマニュアルまで一言一句書き写して結論だと言い張り、二度繰り返した発言は別のタスクとして二重に記録していたのが、今は真の行動目標だけを抜き出し、重複した発言は自動で一つにまとめてくれるようになったようなものである。

ko

작업 재개 시 명령어 로그로 도배되지 않고, 같은 작업을 중복으로 가져오지 않는다.

이전 대화 기록을 이어갈 때, 시스템이 '최근에 하던 작업'이라며 띄워주는 목록이 더 이상 장황한 도구 설명서로 도배되지 않는다. 예전엔 시스템이 당신의 대화 기록을 정리할 때 '사용자가 보낸 것'이라고 표시된 내용은 덮어놓고 전부 긁어왔다. 그것이 시스템 스스로 특정 작업을 돕는 스킬(skill, 명령어가 담긴 파일)을 넓히려고 몰래 끼워 넣은 기나긴 설명인지도 모른 채, 이런 잡동사니를 당신의 작업 진척도로 착각한 것이다. 게다가 바로 앞의 한 문장과만 비교했기 때문에, 똑같은 말을 했는데 그사이에 이 잡동사니가 끼어들면 두 번 말한 것으로 계산되어 최대 3개밖에 안 되는 귀중한 이어받기 자리를 낭비했다. 이제 시스템은 내부 메타데이터(isMeta)로 표시된 이런 잡동사니를 건너뛸 줄 알고, 전체 목록과 꼼꼼히 비교해 중복을 걸러낸다. 비서에게 어제 회의의 요점 정리를 시켰더니, 예전엔 당신이 회의에서 소리 내어 읽은 두꺼운 매뉴얼까지 토씨 하나 안 틀리고 베껴서 결론이랍시고 내놓고 똑같은 말을 두 번 하면 다른 일 두 가지로 적어두던 것을, 이제는 진짜 행동 계획만 쏙쏙 뽑아내고 중복된 말은 한 줄로 알아서 합쳐주는 것과 같다.
