zh-TW

脈絡視窗太重時會跳出提醒，教你怎麼清空

當 Claude Code 對話把脈絡視窗（context，AI 能記住的前後文範圍）塞到 80% 以上的紅區時，狀態列現在會補上一行提醒。因為過了這個點，每一輪對話都會把沉重的紀錄重送一次，讓每輪成本更貴，額度也燒得更快；這時候輸入 /clear 就能把包袱清掉重置。提醒還會顯示脈絡百分比與本次對話花費。就像你的背包快塞滿時，系統會拍拍你的肩膀，建議你把用不到的東西清掉再繼續走。

Original changelog
**Status line nudges `/clear` when the context window goes heavy (≥80%)**: once a Claude Code conversation fills its context window past the red zone, the status line appends a one-line reminder. Past that point every turn resends a heavy context — pricier turns and a faster rate-limit burn, both of which `/clear` resets. The nudge shows the context % and, when available, the session cost, in all five languages.

長時間的 Codex 對話，5 小時額度數字不再抓錯

以前如果一個 Codex 工作階段（session，一段連續的對話期間）聊太久，系統在抓 5 小時額度數字時，可能會漏看真正最新被修改的檔案，導致選單列顯示過期的數字。現在系統會把所有看得到的檔案照最後修改時間排序，確保選單列永遠反映最新狀況。就像看行事曆，以前只照建立時間排、漏了後來才更新的記事，現在直接照最後更新時間排，保證你看到的是最新進度。

Original changelog
**Codex 5h quota no longer goes stale on long-lived sessions**: the rate-limit reader scanned Codex session files newest-date-directory first and stopped at a scan limit, which could skip the file that was *actually* modified most recently when a long session keeps appending to an older creation-date directory. It now sorts all visible session files by modification time, so the menu bar always reflects the newest snapshot. (#37, by @ericweichun)

en

Status line nudges when context is too heavy and teaches you how to clear it

When a Claude Code conversation packs its context window (context — the range of previous messages the AI remembers) past the 80% red zone, the status line now appends a one-line reminder. Past this point, every turn resends a heavy log, making each turn more expensive and causing your quota to burn faster; typing /clear lets you drop the baggage and reset. The nudge also shows the context percentage and your session cost. It's like having a system tap you on the shoulder when your backpack is nearly full, suggesting you empty out what you don't need before hiking further.

Long Codex sessions no longer show stale 5-hour quotas

Previously, if a Codex session (session — one continuous stretch of a conversation) went on for too long, the system fetching the 5-hour quota could overlook the file that was actually modified most recently, causing the menu bar to show stale numbers. Now, the system sorts all visible files by their last modified time, ensuring the menu bar always reflects the latest snapshot. It's like checking a planner: you used to sort entries only by when they were created and missed ones updated later, but now you sort straight by last updated, ensuring you always see the freshest progress.
