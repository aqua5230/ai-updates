zh-TW

不小心按到右鍵「重新載入」不會再讓面板變成一片死白

以前在面板上按滑鼠右鍵選「重新載入（Reload）」，面板會永遠變成一片空白，除非整個程式重開不然救不回來。現在系統直接把右鍵選單裡那些沒用的網頁導航按鈕（重新載入、上一頁、下一頁）拿掉了，而且內部的重載機制也改成重新把畫面畫出來，不會再傻傻地載入空白網頁。這也順便修好了一個問題：當系統在背後把網頁畫面偷偷關掉時，現在它懂得自己重新畫回來。就像以前你不小心按到電視遙控器的重設鍵，畫面就全黑再也打不開，現在它把那顆危險的按鈕藏起來了。

Original changelog
**Panel no longer goes permanently blank after a context-menu "Reload"**: the popover panel loads its HTML via `loadHTMLString` with no base URL, so the WKWebView system context-menu Reload reloaded `about:blank` and left the panel blank with no obvious way to recover (#42). usage now strips navigation items (Reload/back/forward/open/download) from the panel's context menu, and internal reloads re-inject the original HTML instead of calling `reload()` — which also fixes panel recovery after the web-content process is terminated.

en

Accidental right-click "Reload" no longer breaks the panel into a permanent blank screen

If you accidentally right-clicked the panel and selected "Reload", the panel used to turn permanently blank, leaving no way to recover short of restarting the entire app. The system has now stripped those useless web-navigation buttons (Reload, Back, Forward) from the context menu entirely, and the internal reloading mechanism now redraws the actual screen instead of foolishly loading a blank page. This also fixed a related issue: if the system silently killed the web view in the background, it now knows how to redraw itself automatically. It's like accidentally hitting the factory-reset button on your TV remote and having the screen go black forever — the system now hides that dangerous button away.

ja

右クリックの「再読み込み」をうっかり押しても、パネルが永久に真っ白にならなくなった

以前はパネル上で右クリックして「再読み込み（Reload）」を選ぶと、パネルが永久に真っ白になり、プログラム全体を再起動しない限り二度と元に戻せなかった。今は右クリックメニューから役に立たないウェブのナビゲーションボタン（再読み込み、戻る、進む）を丸ごと削除した。さらに内部の再読み込みの仕組みも、馬鹿正直に空白ページを読むのではなく、画面を正しく描画し直すように変更した。これにより、システムが裏でこっそりウェブ画面を強制終了させた際にも、自力で画面を復元できるようになった。誤ってテレビのリモコンのリセットボタンを押すと画面が真っ暗になって二度とつかなくなっていたのが、今はその危険なボタンを隠してくれたようなものである。

ko

실수로 우클릭 '새로 고침'을 눌러도 패널이 영영 하얗게 죽지 않음

예전엔 패널에서 마우스 오른쪽 버튼을 눌러 '새로 고침(Reload)'을 선택하면 패널이 영원히 하얗게 변해버려, 프로그램 전체를 다시 켜지 않는 한 살려낼 방법이 없었다. 이제는 시스템이 우클릭 메뉴에서 쓸모없는 웹 탐색 버튼(새로 고침, 뒤로 가기, 앞으로 가기)을 아예 빼버렸다. 또한 내부의 새로 고침 방식도 멍청하게 빈 웹페이지를 불러오는 대신 화면을 다시 제대로 그리도록 바꿨다. 덕분에 시스템이 뒤에서 몰래 웹 화면을 꺼버렸을 때도 이제는 알아서 다시 그릴 줄 안다. 예전엔 실수로 TV 리모컨의 초기화 버튼을 누르면 화면이 까매져서 다시는 안 켜졌는데, 이제는 그 위험한 버튼을 아예 숨겨버린 것과 같다.
