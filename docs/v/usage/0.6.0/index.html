zh-TW

介面自動說你的語言，五種語言隨系統切換

你的面板會自動變成你電腦的語言。usage 現在會偵測 macOS 系統語言，自動把整個介面切成繁體中文、簡體中文、英文、日文或韓文，不用設定——系統是什麼語言就顯示什麼。以前介面的狀態文字（像「✓ 已同步」）是硬寫死的中文，現在全改交給一套多語系系統（i18n，把同一套介面翻成多種語言、依系統自動切換的機制）統一管理，所以五種語言都顯示得出來。如果你是開發者，想強制測試某個語言，可以設環境變數（開程式之前預先塞給它的隱藏開關，一般使用者看不到）USAGE_LANG=ja 之類來釘住語言。就像一台跨國販賣機，以前只會講中文，現在進到哪一國就自動切那一國的語言顯示。

Original changelog
**Multi-language UI (i18n)**: automatically detects the macOS system language and displays the interface in Traditional Chinese, Simplified Chinese, English, Japanese, or Korean. No configuration needed.
**`USAGE_LANG` environment variable**: force a specific language (e.g. `USAGE_LANG=ja`) for development and testing.
Removed hardcoded Chinese status strings (e.g. `✓ 已同步`) from `usage_client.py`; all status text now goes through the i18n system.

開源授權改採 AGPL，改作發佈也須公開原始碼

如果你打算拿 usage 的原始碼去改、然後發佈出去，要注意授權變嚴了。授權從 MIT（幾乎隨你用的寬鬆條款）改成 AGPL-3.0（修改後發佈出去的版本必須同樣開源、把原始碼公開的條款）——目的是保護原作者，讓別人不能把改過的版本藏起來不還給社群。另外，面板底部現在會顯示一行小字「based on usage by lollapalooza」，標明出處。就像一份開放讓大家免費用的食譜，本來隨你改隨你賣，現在改成「你可以改、但端出去的菜也得公開你的配方」，而且盤子底下印著原廚師的名字。

Original changelog
**License changed from MIT to AGPL-3.0**: modified versions that are distributed must be open-sourced.
**Attribution footer in popover**: `based on usage by lollapalooza` shown at the bottom of the panel.

en

The interface now speaks your language, switching between five languages with your system

Your panel now shows up in your computer's language. usage detects your macOS system language and automatically switches the entire interface between Traditional Chinese, Simplified Chinese, English, Japanese, or Korean — no setup needed, whatever your system uses is what you get. Previously the status text (like "✓ 已同步", meaning "synced") was hardcoded in Chinese; it's all now routed through an i18n system (internationalization — the machinery that translates one interface into many languages and switches them based on your system), so all five languages actually display. If you're a developer who wants to force a specific language for testing, you can set an environment variable (a hidden switch stuffed into the program before it opens, invisible to regular users) like USAGE_LANG=ja to pin the language. It's like a vending machine that used to only speak Chinese but now switches to whatever country it's sitting in.

Original changelog
**Multi-language UI (i18n)**: automatically detects the macOS system language and displays the interface in Traditional Chinese, Simplified Chinese, English, Japanese, or Korean. No configuration needed.
**`USAGE_LANG` environment variable**: force a specific language (e.g. `USAGE_LANG=ja`) for development and testing.
Removed hardcoded Chinese status strings (e.g. `✓ 已同步`) from `usage_client.py`; all status text now goes through the i18n system.

License switches to AGPL — forked releases must stay open-source

If you plan to take usage's source code, modify it, and release it, note that the license got stricter. It changed from MIT (a permissive license that lets you do almost anything) to AGPL-3.0 (which requires that any modified version you distribute must also be open-sourced) — the goal is to protect the original author so nobody can hoard a modified version without giving back to the community. Also, the bottom of the panel now shows a small line, "based on usage by lollapalooza," crediting the source. It's like a recipe that was free for anyone to use, tweak, and sell, but now says "you may modify it, but the dish you serve must come with your recipe published" — and the plate is stamped with the original chef's name.

Original changelog
**License changed from MIT to AGPL-3.0**: modified versions that are distributed must be open-sourced.
**Attribution footer in popover**: `based on usage by lollapalooza` shown at the bottom of the panel.

ja

インターフェースが自動であなたの言語を話す、システムに合わせて五言語を切り替える

パネルがあなたのパソコンの言語で表示されるようになった。usage は macOS のシステム言語を検出し、インターフェース全体を繁体字中国語・簡体字中国語・英語・日本語・韓国語のいずれかに自動で切り替える。設定は不要で、システムが使っている言語がそのまま表示される。以前はステータス文字（「✓ 已同步」（中国語で「同期済み」を意味する）など）が中国語で固定されていたが、今回から i18n（国際化、一つの画面を多数の言語に翻訳しシステムに合わせて切り替える仕組み）で一元管理されるようになり、五言語すべてが正しく表示される。開発者で特定の言語を強制的に試したい場合は、環境変数（プログラム起動前にあらかじめ仕込む隠しスイッチで一般ユーザーの目には触れない）に USAGE_LANG=ja などを設定すれば言語を固定できる。中国語しか話せなかった自動販売機が、置かれた国に合わせてその国の言語に自動切り替えするようになったようなものである。

Original changelog
**Multi-language UI (i18n)**: automatically detects the macOS system language and displays the interface in Traditional Chinese, Simplified Chinese, English, Japanese, or Korean. No configuration needed.
**`USAGE_LANG` environment variable**: force a specific language (e.g. `USAGE_LANG=ja`) for development and testing.
Removed hardcoded Chinese status strings (e.g. `✓ 已同步`) from `usage_client.py`; all status text now goes through the i18n system.

ライセンスを AGPL に変更、改造して公開する場合はソース公開が必須

usage のソースコードを改造して公開しようとするなら、ライセンスが厳しくなったことに注意したい。MIT（ほぼ自由に使える緩い条項）から AGPL-3.0（改造して配布する版も同様にオープンソース化し、ソースを公開しなければならない条項）へ変更された。目的は原作者を守り、改造版をこっそり隠してコミュニティに還元しない使い方を防ぐことだ。また、パネル下部に「based on usage by lollapalooza」という出典を示す小さな一行が表示されるようになった。誰でも自由に使えたレシピが、「改変してよいが、出す料理には必ずレシピも公開せよ」と変わり、さらに皿の底に元シェフの名前が刻まれたようなものである。

Original changelog
**License changed from MIT to AGPL-3.0**: modified versions that are distributed must be open-sourced.
**Attribution footer in popover**: `based on usage by lollapalooza` shown at the bottom of the panel.

인터페이스가 자동으로 당신의 언어를 쓴다, 시스템에 맞춰 5개 언어를 전환한다

패널이 컴퓨터의 언어를 따라 바뀐다. usage가 macOS 시스템 언어를 감지해 인터페이스 전체를 번체 중국어·간체 중국어·영어·일본어·한국어 중 하나로 자동으로 바꾼다. 설정은 필요 없고, 시스템이 쓰는 언어가 그대로 뜬다. 예전에는 상태 문구(예: '✓ 已同步', 즉 중국어로 '동기화 완료'를 뜻함)가 중국어로 고정돼 있었지만, 이제는 i18n(국제화, 하나의 화면을 여러 언어로 번역해 시스템에 맞춰 갈아끼는 장치)로 한곳에서 관리되어 5개 언어 모두 제대로 표시된다. 개발자라서 특정 언어를 강제로 테스트하고 싶다면 환경 변수(프로그램을 켜기 전에 미리 끼워 넣는 숨겨진 스위치로 일반 사용자는 볼 수 없음)에 USAGE_LANG=ja 따위를 설정해 언어를 고정할 수 있다. 중국어만 할 줄 알던 자판기가, 놓인 나라에 따라 그 나라 말로 자동으로 바뀌는 것과 같다.

Original changelog
**Multi-language UI (i18n)**: automatically detects the macOS system language and displays the interface in Traditional Chinese, Simplified Chinese, English, Japanese, or Korean. No configuration needed.
**`USAGE_LANG` environment variable**: force a specific language (e.g. `USAGE_LANG=ja`) for development and testing.
Removed hardcoded Chinese status strings (e.g. `✓ 已同步`) from `usage_client.py`; all status text now goes through the i18n system.

라이선스를 AGPL로 바꿨다, 고쳐 배포하려면 소스 코드 공개 필수

usage의 소스 코드를 뜯어고쳐 배포하려 한다면 라이선스가 엄격해졌음을 알아둬야 한다. MIT(거의 마음대로 써도 되는 관대한 조항)에서 AGPL-3.0(고쳐서 배포하는 판도 똑같이 오픈소스로 공개해야 하는 조항)로 바뀌었다. 목적은 원작자를 지키고, 누군가 고친 판을 몰래 숨겨 커뮤니티에 돌려주지 않는 일을 막는 것이다. 또한 패널 맨 아래에 'based on usage by lollapalooza'라는 출처를 표시하는 작은 글줄이 뜬다. 누구나 자유롭게 쓰던 레시피가 '고쳐도 좋지만 내놓는 요리에는 레시피도 공개해야 한다'로 바뀌고, 게다가 접시 밑에 원래 요리사의 이름이 찍힌 것과 같다.

Original changelog
**License changed from MIT to AGPL-3.0**: modified versions that are distributed must be open-sourced.
**Attribution footer in popover**: `based on usage by lollapalooza` shown at the bottom of the panel.
