zh-TW

支援所有語系的專案路徑，報告不再因為中文檔名當機

你的專案資料夾現在可以安心使用任何語言命名了。以前如果你直接雙擊點開打包好的 App，它在背景用 git 抓專案名稱時會預設用 ASCII 解碼，只要路徑裡有中文、日文、韓文或特殊符號，就會丟出解碼錯誤，連帶讓你的「使用習慣」報告在今天以外的區間一片空白。現在系統會統一用支援所有語言的 UTF-8 格式來解讀路徑。就像是包裹配送員以前只看得懂英文地址，看到中文標籤就把整箱退件不送，現在他終於配備了萬國語言翻譯機，什麼地址都能準確送達。

Original changelog
**Menu bar / report no longer fail on non-ASCII (e.g. Chinese) project paths**: a .app launched by double-click has no locale set, so resolving project names via `git` decoded its output as ASCII and raised `UnicodeDecodeError` on paths containing Chinese/Japanese/Korean/accented characters. This affected `history_loader`/`codex_loader` (live menu bar) and `persona_loader` (Usage Habits), leaving the report's "Usage Habits" section blank for non-today ranges. `git` output is now always decoded as UTF-8, so paths in any language work.

en

Universal language support for project paths means reports no longer crash on CJK names

You can now safely use any language to name your project folders. Previously, if you launched the packed App by double-clicking it, the background git command trying to fetch your project name would default to ASCII decoding; if the path contained Chinese, Japanese, Korean, or accented characters, it would raise a decode error, leaving your "Habits" report blank for non-today ranges. The system now explicitly decodes all paths using the universal UTF-8 format. Like a delivery driver who used to only understand English addresses and tossed out any parcel with a foreign label; now they're equipped with a universal translator and can accurately deliver to any address.

ko

모든 언어의 프로젝트 경로를 지원해, 더 이상 한국어나 중국어 파일명 때문에 보고서가 죽지 않는다

이제 프로젝트 폴더 이름에 어떤 언어든 마음 놓고 쓸 수 있다. 예전엔 패키징된 App을 더블 클릭해 바로 켜면, 백그라운드에서 git으로 프로젝트 이름을 가져올 때 기본적으로 ASCII로 디코딩하려 들어, 경로에 한국어나 중국어, 일본어, 특수 기호가 섞여 있으면 디코딩 오류가 발생해, 그 바람에 '사용 습관' 보고서의 오늘 외의 기간이 하얗게 비어버렸다. 이제는 시스템이 모든 언어를 지원하는 UTF-8 형식으로 경로를 해석하도록 통일했다. 택배 기사가 예전엔 영어 주소만 읽을 줄 알아서 한국어 라벨이 붙은 상자는 아예 반송해 버렸는데, 이제 만국어 번역기를 달고 어떤 주소든 정확하게 배달하는 것과 같다.
