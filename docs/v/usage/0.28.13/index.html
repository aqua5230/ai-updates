zh-TW

離席自動起跑，現在真的會啟動也不會誤響

你離開電腦前開啟 Auto-start 5-hour Session（下個 5 小時階段自動啟動），它現在就會準時啟動。Mac App 的 `PATH` 很窄，找指令的名單漏掉原生安裝器放置 `claude` CLI（從終端機下指令操作 Claude 的工具）的 `~/.local/bin/claude`，背景 ping 找不到可執行項目便放棄；已補上。它也只信任 statusLine hook（會提供終端機底部狀態列資料、在特定事件觸發的小程式）的即時資料來判斷重設時間，避免備援來源把缺少的 `resets_at` 當成剛解析時間，下一次刷新又誤認額度已過期。到期滿兩分鐘才 ping，濾掉誤報，離席自動啟動不會明顯變慢。像定時洗衣機終於找得到插座，也不會把剛按下的按鈕誤當成洗完。說明視窗改成先講睡眠會暫停、醒來會恢復；睡眠設定改用 System Settings 搜尋。

Original changelog
**Auto-start 5-hour Session now actually fires**: the `claude` CLI resolution list was missing `~/.local/bin/claude` (where the native installer puts it), so inside the `.app` bundle's narrow `PATH` the ping silently found nothing to run and gave up. Also hardened against a false "expired" read: the reset-time check now only trusts the live statusLine hook data source (fallback sources can default a missing `resets_at` to parse time, which the next refresh would misread as already-expired) and requires the expiry to be at least 2 minutes old before firing, filtering out that false positive without meaningfully delaying a real away-from-keyboard ping. Dialog copy from 0.28.12 also simplified — leads with "pauses on sleep, resumes on wake, nothing to manage" and routes the sleep-setting instructions through System Settings search (the exact pane/wording varies by Mac model and macOS version) with a fallback suggestion to ask Claude Code to walk through it.

en

Away-time auto-start now runs reliably without false alarms

Turn on Auto-start 5-hour Session before stepping away, and it will now fire on time. The Mac app's `.app` bundle has a narrow `PATH`, and its command lookup list had omitted `~/.local/bin/claude`, where the native installer puts the `claude` CLI. The background ping could find nothing to run and quietly give up; that location is now included. The reset-time check also now trusts only live statusLine hook data, supplied by a small program that is triggered by particular events and feeds the terminal's bottom status bar, rather than fallback data that can treat a missing `resets_at` value as the parsing time and later mistake it for an expired session. It waits until expiry is at least two minutes old before pinging, filtering that false alarm without meaningfully delaying a real away-time auto-start. Think of a timed washing machine that can finally find its outlet and no longer mistakes a freshly pressed button for a completed wash. The dialog now leads with the important bit: it pauses during sleep and resumes on wake. Sleep instructions send you to System Settings search, so you do not have to hunt through differently named panes on different Macs or macOS versions.

ko

자리 비운 동안 자동 시작이 제대로 작동하고 오작동도 줄었다

자리를 비우기 전에 Auto-start 5-hour Session을 켜면, 이제 제시간에 시작한다. Mac App의 `.app` bundle은 `PATH`가 좁은데, 명령어를 찾는 목록에서 네이티브 설치기가 `claude` CLI를 두는 `~/.local/bin/claude`가 빠져 있었다. 그래서 백그라운드 ping은 실행할 것을 찾지 못한 채 조용히 포기했지만, 이제 그 위치도 찾는다. 재설정 시간 판단도 statusLine hook(터미널 맨 아래 상태 표시줄 데이터를 제공하고 특정 이벤트에 반응해 실행되는 작은 프로그램)의 실시간 데이터만 믿도록 바뀌었다. 다른 데이터는 `resets_at`가 없을 때 분석 시간을 넣을 수 있고, 다음 새로고침에서 이를 이미 만료된 것으로 잘못 읽을 수 있기 때문이다. 만료 뒤 적어도 2분이 지나야 ping을 보내므로 이런 오작동은 걸러 내면서도 실제 자리 비움 자동 시작은 눈에 띄게 늦어지지 않는다. 예약 세탁기가 마침내 콘센트를 찾고, 방금 누른 버튼을 세탁 완료로 착각하지 않는 것과 같다. 안내 창은 잠자기 중에는 멈추고 깨어나면 다시 시작한다는 점을 먼저 알려 준다. 수면 설정은 System Settings 검색으로 열 수 있어 Mac이나 macOS마다 다른 메뉴를 헤맬 필요도 없다.
