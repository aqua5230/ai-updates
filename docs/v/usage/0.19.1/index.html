zh-TW

隱藏 Claude Code 區塊後，底部的錯誤提示不會再出來嚇人

當你把 Claude Code 區塊隱藏起來時，系統不會再莫名其妙跳出錯誤訊息了。以前如果你只用 Codex，把 Claude Code 隱藏掉，面板最底下還是會顯示找不到狀態檔的錯誤，甚至還會冒出一顆叫你「安裝 Hook」的按鈕，讓人一頭霧水；現在系統學乖了，只要區塊被隱藏，這些專屬於 Claude Code 的警告也會一起乖乖消失，底部只會顯示乾淨的中性同步狀態。就像你已經把一輛破車報廢了，車庫的儀表板就不該還在天天亮紅燈叫你換機油，現在它終於懂得閉嘴了。

Original changelog
**Hidden Claude Code section no longer leaks a setup error**: Codex-only users who hid the Claude Code section still saw a "status file not found — run `python3 main.py --setup`" message in the popover footer, plus an "Install Hook" button. Both are Claude Code-specific and are now suppressed while the section is hidden; the footer falls back to a neutral synced status. (#36, reported by @ilss0902)

en

Hiding the Claude Code section finally silences its leftover setup errors

When you hide the Claude Code section, the system will no longer pop up confusing error messages. Previously, if you only used Codex and hid Claude Code, the bottom of the panel would still panic about a missing status file and even surface an "Install Hook" button, leaving users scratching their heads; the system has wised up, so when the section is hidden, these Claude Code-specific warnings are fully suppressed alongside it, and the footer smoothly falls back to a clean, neutral sync status. Like scrapping a broken car but the garage's master dashboard still flashes a red light begging you to change the oil every day — it finally knows to stay quiet.
