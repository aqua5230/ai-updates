zh-TW

成本估算在價格更新後自動校正，網頁面板不再無限重新整理

現在成本估算會自動追蹤最新價格，面板也不會再卡死在無限重整裡了。以前當系統剛啟動還沒抓到最新價格表時，會先用本機的預設價格幫你估算成本，但算完就把這個暫時的數字「寫死」在紀錄上，導致後來真正最新的價格表抓回來了，它也不會重算（這對 Codex 等不自帶價格的紀錄影響特別大）；現在它學會不寫死數字了，一拿到新價格就會立刻校正。另外，網頁面板在更新資料（state injection，也就是把最新數據塞進畫面的過程）時如果一直失敗，以前會陷入「失敗、重整、又失敗」的無限迴圈卡死；現在它知道同一個資料重試幾次後就該放棄，不會再死心眼地一直重整（但如果是負責畫面的程序真的死掉，它還是會幫你重新啟動）。就像你點菜時菜單還沒印好，店員先用口頭報價並直接寫死在帳單上，後來發現降價了也不改；現在他只會先用鉛筆寫，拿到最新菜單就馬上幫你重算。而且遇到打不開的抽屜，他也不會再傻傻地一直狂拉到天黑了。

Original changelog
**Estimated cost now recomputes after a pricing update**: a cost computed with fallback prices was written back and cached onto usage entries, so it was never recomputed once real prices loaded — leaving cost figures persistently off (mainly for entries without a source cost, e.g. Codex). The estimate is no longer written back, so it reflects updated prices immediately.
**Web panel no longer reloads endlessly when injection keeps failing**: if state injection failed repeatedly the panel would loop reloading; reloads per payload are now capped (WebContent-process crash recovery is unaffected).

en

Cost estimates correctly recalculate when prices update, and web panels avoid infinite reload loops

Cost estimates now automatically track live prices, and the web panels will no longer trap themselves in infinite reload loops. Previously, if the system estimated your cost using fallback prices before the live pricelist could be fetched, it would hard-code that temporary estimate into your local usage item. Even after the real prices downloaded, it refused to recalculate, causing chronically inaccurate cost estimates (especially for sources that don't embed their own costs, like Codex); it now smartly leaves the item alone, recalculating immediately whenever fresh prices arrive. Also, if the web panel repeatedly failed to update its data (state injection, the process of feeding fresh data into the UI), it used to trap itself in an infinite reload loop; it now gives up gracefully after a few tries with the same dataset (though it still auto-recovers if the actual rendering process crashes). It's like a waiter who jotted down an estimated price in pen while waiting for the new menu to be printed, and refused to adjust your bill when the actual prices came in lower; now they use a pencil and update it instantly. And when they face a jammed drawer, they no longer yank at it endlessly until the restaurant closes.

The estimate is no longer written back, so it reflects updated prices immediately.

ja

コスト計算が価格更新後に自動補正され、ウェブパネルが無限リロードを繰り返さなくなった

コスト計算が最新の価格を自動で追跡するようになり、パネルが無限リロードのループでフリーズすることもなくなった。以前は、システムが最新の価格表を取得する前にローカルのデフォルト価格でコストを仮計算した場合、その数値を記録に「ベタ書き」してしまい、後から本物の価格表を取得しても再計算されなかった（これは Codex のように記録自体にコストを持たないもので特に問題になっていた）。今は数値を固定せず、新しい価格が届き次第すぐに再計算する。また、ウェブパネルがデータの更新（state injection：最新データを画面に注入するプロセス）に連続して失敗した際、以前は無限にリロードを繰り返してフリーズしていたが、同じデータでの再試行に上限を設け、潔く諦めるようになった（描画プロセス自体がクラッシュした際の自動復帰はそのまま機能する）。メニュー表が届く前にウエイターが仮の値段を伝票にボールペンで書き込み、後で安くなった本物のメニューが届いても修正してくれなかったのが、今は鉛筆で書き、新しいメニューが来たらすぐ直してくれるようなものである。さらに、開かない引き出しを日が暮れるまでムキになって引っ張り続けることもなくなった。

ko

가격이 업데이트되면 비용이 알아서 다시 계산되며, 웹 패널이 무한 새로 고침에 빠지지 않는다

이제 비용 계산이 최신 가격을 자동으로 추적하며, 웹 패널이 무한 새로고침 굴레에 빠져 멈추는 일도 없다. 예전엔 시스템이 최신 가격표를 가져오기 전에 로컬에 있는 기본 가격으로 비용을 먼저 짐작해 놓고는, 그 임시 숫자를 기록에 아예 '못 박아' 버렸다. 그 바람에 나중에 진짜 가격표를 가져와도 다시 계산하지 않아 비용이 계속 어긋나는 문제가 있었다(특히 Codex처럼 기록 자체에 가격 정보가 없는 경우 심했다). 이제는 숫자를 박아두지 않고 새 가격표가 오면 그 즉시 바로잡는다. 또한 웹 패널이 데이터 업데이트(state injection, 화면에 최신 데이터를 밀어 넣는 과정)에 계속 실패할 때 예전엔 '실패, 새로고침, 또 실패'의 무한 굴레에 빠져 멈춰버렸는데, 이제는 같은 데이터로 몇 번 시도해보고 안 되면 깔끔하게 포기할 줄 안다(물론 화면을 그리는 프로세스 자체가 죽었을 때는 예전처럼 알아서 살려낸다). 새 메뉴판이 인쇄되기도 전에 종업원이 대충 부른 값을 영수증에 볼펜으로 적어놓고는, 나중에 진짜 가격이 더 싸게 나왔는데도 안 고쳐주던 것에서, 이제는 연필로 적어두었다가 새 메뉴판이 오면 바로 다시 계산해 주는 것과 같다. 게다가 안 열리는 서랍을 해 질 때까지 미련하게 쥐고 흔드는 일도 없어졌다.
