zh-TW

離席時，讓下一個 5 小時額度窗口先起跑

要離開電腦吃飯或出門，卻不想回來才開始等下一輪 Claude 額度嗎？usage 新增可自行開啟的「Auto-start 5-hour Session」選項。它發現 5 小時 quota window（Claude 額度每 5 小時重新計算的一段時間）剛重設、而且目前沒有執行中的 session（一次持續運作的工作階段）時，會在背景送出一次 `claude -p ok --model haiku`，立刻替下一個窗口起跑。像替洗衣機設好延後啟動：你人不在，倒數仍先開始；每個窗口最多只送一次，不會反覆叫醒它。這項功能預設關閉，不會碰 Anthropic 的 quota API（查詢額度的程式介面）；找不到 `claude` CLI（從終端機下指令操作 Claude 的工具）也會安靜略過。開啟時會先說明 usage 和 Mac 必須保持醒著，但闔上筆電仍照常讓 Mac 睡眠。

Original changelog
**New opt-in menu toggle: Auto-start 5-hour Session.** When enabled, usage detects that your Claude 5-hour quota window has just reset and no session is currently running, then fires a single `claude -p ok --model haiku` in the background to immediately start the next window — useful if you're stepping away and want the next 5 hours counting down before you're back. Off by default; sends at most once per 5-hour window (self-throttled), never touches the Anthropic quota API, and silently no-ops if the `claude` CLI can't be found. Enabling it shows a one-time dialog explaining that it needs usage (and the Mac) to stay awake to fire — closing the lid always sleeps the Mac regardless of this setting.

en

Start your next 5-hour window while you're away

Heading out for lunch or errands and don't want your next Claude allowance window to start only when you return? usage now has an opt-in Auto-start 5-hour Session toggle. When it sees that a 5-hour quota window has reset and no session is running, it sends one background `claude -p ok --model haiku` command to start the next window immediately. Think of it like scheduling a washing machine: the countdown can begin while you're away, but it runs at most once per window rather than repeatedly nudging the machine. It is off by default, never calls Anthropic's quota API, and quietly does nothing if the `claude` CLI is unavailable. On setup, it explains that usage and your Mac need to stay awake; closing the lid still puts your Mac to sleep as usual.

ko

자리를 비운 사이 다음 5시간 창을 먼저 시작한다

밥을 먹으러 가거나 외출할 때, 돌아온 뒤에야 다음 Claude 사용량 창이 시작되는 것은 아쉽다. usage에 직접 켤 수 있는 ‘Auto-start 5-hour Session’ 토글이 생겼다. 5시간 quota window(Claude 사용량을 5시간마다 새로 계산하는 구간)가 막 초기화됐고 실행 중인 session(계속 돌아가는 작업 단위)이 없음을 감지하면, 백그라운드에서 `claude -p ok --model haiku`를 한 번 실행해 다음 창을 곧바로 시작한다. 세탁기 예약 시작처럼 자리를 비운 동안에도 카운트다운은 먼저 시작되지만, 같은 창에서 반복해 깨우지는 않는다. 기본값은 꺼짐이며 Anthropic의 quota API를 건드리지 않고, `claude` CLI를 찾지 못하면 조용히 아무 일도 하지 않는다. 켤 때 usage와 Mac이 깨어 있어야 함을 알려 주지만, 노트북 덮개를 닫으면 Mac은 평소처럼 잠든다.
