遠端開發更安全順暢，跨平台環境原汁原味

透過遠端連線跑程式碼變得更強大且安全了。現在遠端執行器（executor，幫你跑程式的背後引擎）預設採用端到端加密（Noise relay），而且就算你掛網不動，連線也不會隨便斷開。如果你用 Mac 連到 Windows 伺服器，它能完美保留對方的原生資料夾路徑、Shell 環境與檔案權限，不會再因為作業系統不同而水土不服。此外，系統現在也支援企業級代理伺服器常用的 P-521 憑證簽章，並修復了資料庫（SQLite）的毀損問題。就像你從台灣遠端遙控國外的無塵室，現在不僅通訊全加密、連線不中斷，連你對機械手臂下的指令都會自動翻譯成當地規格，不怕摔壞東西。

Original changelog
Remote executors now use authenticated, end-to-end encrypted Noise relay channels. (#26242, #26245)
Cross-platform remote execution now preserves executor-native working directories and shells, including filesystem permission paths across app-server and exec-server boundaries. (#27819, #27995, #28032, #28122, #28165, #28367)
TLS connections now support P-521 certificate signatures commonly used by enterprise proxies. (#27706)
Idle exec-server relays remain connected, and steered user input immediately interrupts `wait_agent`. (#28286, #28341)
Bundled SQLite is pinned to a version containing the WAL-reset corruption fix. (#27992)

外掛市集大擴編，專屬清單與自創分類上線

找外掛（plugin，擴充系統功能的附加元件）的體驗變得更像逛現代化商城了。現在市集裡多了一個「我建立的（created-by-me）」專區，讓你快速找到自己寫的工具；同時系統會根據你的登入身分，推薦專屬的精選外掛目錄。如果遇到不同外掛撞名（宣告衝突），系統現在懂得自動去重複並保持市集排序不亂掉。此外，特定的執行器外掛現在可以在每個執行緒（thread，獨立的工作流程）裡單獨啟動對應的 MCP（讓 AI 控制外部工具的協定）伺服器。就像百貨公司重新裝潢，不僅有了你的專屬品牌櫃位和量身訂做的推薦專刊，連不同專櫃的同名商品都不會再標錯價錢了。

如果你的專案塞滿了各種工具，現在跑起來會順暢非常多。系統改善了底層邏輯，把工具搜尋的結果快取起來，並避免重複拷貝歷史對話，大幅降低了延遲與記憶體消耗。同時，它也為圖片快取設定了 64 MiB 的上限，並限制回饋上傳最多只牽涉八個相關執行緒（thread，獨立的工作流程），避免系統不知節制把記憶體吃光。另外，終端機重新調整視窗大小時的文字重排功能現在強制開啟，不會再有過時設定卡住畫面了。就像一間總是把資料庫印出來歸檔的舊公司，現在學會了用電腦索引，不僅辦公桌（記憶體）不再被公文山壓垮，找資料的速度也變快了。

Smoother performance and reduced memory usage in heavy sessions

Heavy projects loaded with tools will now run significantly smoother. It also put a hard 64 MiB cap on the prompt-image cache and restricted feedback uploads to a maximum of eight related threads (independent workflows), preventing the system from ravenously eating up all your RAM. Additionally, terminal text reflow upon resizing is now always enabled, overriding any obsolete settings that might freeze the view. Like an old office that used to print out every database record for archiving, but finally learned to use digital indexes — not only is the desk (RAM) no longer crushed by paper mountains, but finding information is much faster.
