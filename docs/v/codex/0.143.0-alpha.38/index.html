修正日誌寫入頻率過高的問題，拯救你的固態硬碟壽命

為了避免 AI 運作時在背景瘋狂寫入日誌縮短你的硬碟壽命，系統移除了過於頻繁的資料庫記錄。以前系統在處理資料庫日誌時（TRACE，最詳細、連每一秒傳遞了什麼細微資料都記錄下來的偵錯等級），會把巨量的繁雜日誌事件反覆寫入本機 `~/.codex/logs_2.sqlite` 資料庫，產生嚴重的寫入放大（write amplification，寫入硬碟的資料量遠大於實際檔案大小的現象），一年下來可能寫入高達 640 TB 的巨量資料，嚴重威脅 SSD（固態硬碟，電腦儲存資料的硬碟）的壽命；現在把這些囉唆的通訊過程從資料庫裡拿掉，停止了這類冗長事件的橋接，以降低寫入放大與 SSD 耗損。這就像以前助理把跟你講的每一字、呼吸的每一口氣都拿筆刻在木板上，一小時就堆滿一整個倉庫；現在改成只記重要結論，省下了大片森林。

en

Fixed high-frequency logging that generated massive disk writes, saving your SSD

System stopped saving noisy connection events. Previously, the database logging (TRACE, the most verbose debugging level that records every tiny bit of activity second-by-second), continuously saving verbose log events to `~/.codex/logs_2.sqlite`. This created massive write amplification (write amplification, a phenomenon where the actual volume of data written to physical storage is far larger than the logical file size), racking up to 640 TB of disk writes a year and risking premature solid-state drive (SSD, the high-speed storage drive in your computer) failure. These verbose bridge events are now ignored, stopping the bridging of these verbose log events to reduce write amplification and SSD wear. It's like an assistant writing down every single word you speak and breath you take by carving them into wooden blocks, filling a whole warehouse in an hour; now he only logs key decisions, saving a forest.

Original changelog
Fixed critical SQLite TRACE-level database logging causing severe write amplification and high SSD wear (up to 640TB/year) by stopping bridging verbose log events.

Debug logs no longer save raw connection payloads, protecting your code privacy

Debug logs no longer save raw connection messages. Previously, the app recorded full WebSocket payloads (WebSocket payloads, the raw data packets transmitted over the network containing your code and messages) directly inside local plain-text trace logs (trace logs, detailed text files that record the application's step-by-step runtime operations) during debug sessions; now these are automatically stripped. It's like a courier company's package tracking system that used to print the exact contents of your box on the shipping label; now it only shows a tracking number to keep your items private.

ログの超高頻度な書き込みバグを修正、あなたの SSD の寿命を守る

不要なデータ保存処理が廃止された。以前はデータベースの記録が TRACE レベル（TRACE、1 秒ごとの極小のデータ通信や動作まで全て詳細に記録する、最もおしゃべりなデバッグ設定）になっていたため、膨大な冗長なログイベントが `~/.codex/logs_2.sqlite` データベースへ絶えず書き込まれていた。これにより深刻な書き込み増幅（write amplification，実際に書き込まれた物理データ量が元のファイルサイズより肥大化する現象）が発生し、1 年間で最大 640 TB もの書き込みが行われ、ソリッドステートドライブ（SSD、データを保存するパソコンの高速ドライブ）の寿命を縮めていた。今は不要な通信記録を保存しないように修正され、この冗長なイベントのブリッジを停止し、書き込み増幅と SSD の消耗を抑えた。これは、アシスタントがあなたの発言だけでなく呼吸の一つ一つまで木板に彫り刻んで記録し、1 時間で倉庫をいっぱいにしていたのを、重要な決定事項だけをメモ帳に書くように改めて、森林資源を節約したようなものである。

로그의 초고빈도 쓰기 버그를 수정하여 소중한 SSD 의 수명을 보호함

너무 빈번하게 이루어지던 데이터베이스 쓰기 동작을 제거했다. 예전에는 데이터베이스 로그가 TRACE 레벨（TRACE, 매초 발생하는 미세한 연결 상태나 파일 변경까지 전부 낱낱이 기록하는 가장 세밀한 디버깅 등급）로 켜진 탓에, 막대한 장황한 로그 이벤트가 로컬 `~/.codex/logs_2.sqlite` 데이터베이스에 쉴 새 없이 쓰였다. 이로 인해 디스크 쓰기 증폭（write amplification, 실제 파일 크기보다 물리적 디스크에 쓰이는 데이터 용량이 훨씬 불어나는 현상）이 일어났고, 연간 최대 640 TB 에 달하는 용량을 쓰고 지우기를 반복해 솔리드 스테이트 드라이브（SSD, 컴퓨터 내에서 파일 저장을 담당하는 고속 하드디스크） 수명을 단축하고 있었다. 이제 상세 이벤트 영구 저장이 해제되어 이러한 장황한 이벤트의 브리지를 중단하여 쓰기 증폭과 SSD 마모를 줄였다. 마치 조수가 당신이 하는 모든 말과 숨소리까지 컴퓨터 내부의 데이터베이스에 일일이 깎아 새기는 바람에 한 시간 만에 창고를 가득 채우던 것을, 이제 핵심 메모만 노트에 적도록 바꾸어 자원을 아끼게 된 것과 같다.
