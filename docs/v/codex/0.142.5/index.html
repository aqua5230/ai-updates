日誌不再記錄完整對話，保護隱私不佔空間

你的對話與程式碼內容不會再被完整印在系統底層的追蹤日誌（trace logs，開發者用來除錯的深層紀錄）裡了。以前當系統透過 WebSocket（讓客戶端與伺服器保持長連線的通訊協定）發送請求時，會不小心把整坨請求內容原封不動寫進日誌中，這不僅佔空間，還可能有隱私風險；現在系統已經把這部分的紀錄擋下了。就像你去銀行辦事，原本櫃檯會在公開的作業紀錄本上抄下你的完整提款明細，現在他們懂得只記個流水號，把你的隱私保護好。

en

Trace logs no longer record full response payloads

Your conversations and code will no longer be printed entirely in the trace logs (deep diagnostic records used by developers for debugging). Previously, when the system sent requests via WebSocket (a communication protocol that keeps a persistent connection between client and server), it would accidentally write the full request payloads into the logs, which wasted space and posed privacy risks; the system now prevents this. Like visiting a bank where the teller used to copy your entire withdrawal slip into a public ledger, but now they only jot down a reference number, keeping your private business safe.

日志不再记录完整对话，保护隐私不占空间

你的对话与代码内容不会再被完整印在系统底层的追踪日志（trace logs，开发者用来调试的深层记录）里了。以前当系统透过 WebSocket（让客户端与服务器保持长连线的通讯协定）发送请求时，会不小心把整坨请求内容原封不动写进日志中，这不仅占空间，还可能有隐私风险；现在系统已经把这部分的记录挡下了。就像你去银行办事，原本柜台会在公开的作业记录本上抄下你的完整提款明细，现在他们懂得只记个流水号，把你的隐私保护好。

ja

トレースログに完全な応答内容が記録されなくなり、プライバシーを保護

会話やコードの内容が、システム底層のトレースログ（trace logs、開発者がデバッグに使う深層記録）に丸ごと印字されることはなくなった。以前はシステムが WebSocket（クライアントとサーバー間で持続的な接続を保つ通信プロトコル）を通じてリクエストを送信する際、誤ってリクエスト内容をそっくりそのままログに書き込んでおり、容量を食ううえにプライバシーのリスクもあったが、今はその記録をブロックするようになった。銀行の窓口で、以前は公開の作業記録簿に引き出し明細がすべて書き写されていたのが、今は整理番号だけをメモしてプライバシーを守ってくれるようになったようなものである。

ko

추적 로그에 전체 응답 내용이 기록되지 않아 프라이버시를 보호

대화와 코드 내용이 시스템 밑바닥의 추적 로그(trace logs, 개발자가 디버깅할 때 쓰는 깊은 기록)에 통째로 찍히는 일이 없어졌다. 예전엔 시스템이 WebSocket(클라이언트와 서버가 계속 연결을 유지하는 통신 규약)을 통해 요청을 보낼 때 실수로 요청 내용 전체를 로그에 그대로 적어버려 용량도 차지하고 프라이버시 위험도 있었지만, 이제는 이 기록을 막아버렸다. 은행 창구에 갔을 때 예전엔 공개된 작업 장부에 출금 내역 전체를 베껴 적던 직원이 이제는 접수 번호만 적어서 개인 정보를 지켜주는 것과 같다.
