en

GPT-5.6 model guidance refreshed, with context corrected to 272,000 tokens

When you are about to hand Codex a long plan, meeting notes, and code together, the key question is how much it can keep in view. This release refreshes bundled guidance for GPT-5.6 Sol, Terra, and Luna, and corrects each model's context window—the amount of material one conversation can hold—to 272,000 tokens. A token is a unit AI uses to process text. Think of it as the number of sheets that fit on one worktable: label the table correctly, and you can plan your pile of material properly. The two follow-up changes also backport that model-metadata and prompt correction to 0.144, limited to GPT-5.6; they do not announce new abilities for these models.

ja

GPT-5.6の3モデル向け説明を更新、文脈容量を27.2万トークンに訂正

長い企画書、会議メモ、コードをまとめて Codex に渡す前に気になるのは、どれだけの内容を覚えておけるかである。今回、GPT-5.6 Sol、Terra、Luna に同梱する操作説明を更新し、3モデルの context window（1回の会話で保持できる内容の容量）を272,000 tokenに訂正した。token はAIが文章を処理する数え方の単位である。同じ作業机に置ける紙の枚数だと考えればよい。机の大きさが正しく示されれば、載せる資料も取り違えずに済む。続く2件の変更は、このモデル情報とプロンプトの訂正を0.144へ戻したもので、対象はGPT-5.6だけであり、3モデルの新機能を告知したものではない。
