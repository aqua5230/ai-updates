이제 macOS 사용자도 네트워크 통제가 엄격한 곳에서 순조롭게 로그인 인증을 할 수 있다. respect_system_proxy 설정을 켜면 시스템이 macOS에 있는 PAC, WPAD(프록시를 자동으로 설정하는 스크립트와 규약) 등 시스템 프록시(system proxy, 네트워크 요청을 대신 전달해 주는 중간 기착지) 설정을 알아듣는다. 경비가 삼엄한 건물에서 전용 통행증을 얻은 것처럼, 이제 애플 컴퓨터를 쓸 때도 출입 통제 구역을 자동으로 통과해 인터넷에 연결할 수 있는 것과 같다.

이미지 및 자격 증명 오류가 명확해져, 실패 시 해결 방법을 바로 알려줌

오류가 났을 때 시스템이 더 이상 알아볼 수 없는 코드만 뱉어내지 않는다. Amazon Bedrock을 쓸 때 자격 증명(credentials, 합법적인 사용자임을 증명하는 열쇠)이 만료되면, 뭉뚱그려 '권한 없음'이라고 뜨는 대신 이제는 어떻게 복구해야 하는지 구체적인 단계를 바로 알려준다. 원격 HTTP 이미지를 불러오다 실패할 때도 AI 모델이 읽고 무엇이 문제인지 알려줄 수 있는 명확한 검증 오류를 반환한다(로컬 이미지와 인라인 데이터는 영향을 받지 않음). 은행에서 서류가 반려될 때 예전엔 그저 차갑게 '불합격' 도장만 찍어줬다면, 이제는 '신분증이 만료되었으니 구청에 가서 새로 발급받으세요'라고 적힌 안내장을 건네주는 것과 같다.

PowerShell의 복잡한 명령어 실행을 안전하게 막고, 서식 정리가 성공하면 조용히 넘어감

PowerShell(Windows의 강력한 스크립트 환경) 명령어를 실행하기가 더 안전해졌다. 보안 검사기가 읽어낼 수 없는 복잡한 실행 블록(AST regions)이 명령어에 숨어 있으면, 악성 프로그램이 몰래 들어오는 것을 막기 위해 시스템이 무조건 가로막고 당신이 직접 승인하도록 요구한다. 또한, 코드 서식 정리 도구(formatter)가 성공적으로 끝나면 조용히 넘어가고 실패했을 때만 진단 메시지를 찍어주며, 코드 모드(Code Mode)에서 선택한 모델의 규격이 맞지 않을 때도 미리 경고해 준다. 공항 보안 검색대에서 엑스레이가 꿰뚫어 보지 못하는 수상한 금속 상자는 반드시 직접 열어 내용물을 확인하라고 요구하는 반면, 무사히 통과한 승객은 조용히 지나가고 진짜 금지품을 잡았을 때만 경보를 울리는 것과 같다.

원격 경로 자동 변환 및 더 풍성해진 안전 버퍼링 인터페이스

클라이언트와 서버의 운영체제가 다를 때(예: Mac에서 Windows로 연결) 경로 문제로 골머리를 앓지 않아도 된다. 원격 연결된 MCP 서버는 이제 원격 플랫폼 고유의 방식대로 쓴 절대 경로를 알아듣고 그대로 받아들여, 직접 슬래시 방향을 고칠 필요가 없다. 이와 함께 앱 화면도 서버에서 가시성 및 모델 관련 데이터를 받아와 훨씬 더 풍성한 안전 버퍼링(safety-buffering, 위험한 작업이 바로 실행되는 것을 막는 확인 화면) 인터페이스를 보여줄 수 있게 되었다. 국제 소포를 보낼 때 예전엔 현지 주소 형식을 직접 찾아 적어야 했지만, 이제는 우체국 시스템이 알아서 번역해 주고 훨씬 알아보기 쉬운 파손 주의 스티커까지 붙여주는 것과 같다.
