改舊問題不會抹掉原對話

想把早先的提問改個字、或重試被安全機制暫緩的回合，現在不必拿整段原對話冒險。Codex 會建立帶上下文的分支，保留原本對話、附件與提及的綁定。這像在書稿影印一份後改稿：新稿能走另一條路，原稿與夾在裡面的資料仍完整留著。附件和提及的對象也會跟著原路徑保存，避免分支後還要重新交代背景。你可以比較兩種方向，而非只能覆寫前一個選擇。

Original changelog
Editing an earlier prompt or retrying a safety-buffered turn now creates a contextual branch, preserving the original conversation, attachments, and mention bindings. (#33201, #33207, #33211)

長對話少重畫，終端機更跟手

終端機聊久了，畫面一再重畫往往比模型回答更惱人。新版以增量 Markdown 渲染、減少重畫、快取（cache，把算過或查過的結果暫存起來直接用）與限制指令輸出大小，改善長對話和串流輸出的反應速度。它像收銀員不再每掃一件商品就重印整張明細，只補上新的一行；畫面少忙，輸入與閱讀就更順。舊結果能沿用時就不必重新算或查，串流中的新內容也不用讓整頁翻新。限制輸出也避免一條指令把整個視窗塞爆。

Original changelog
Improved terminal responsiveness for long conversations and streamed output through incremental Markdown rendering, fewer redraws, caching, and bounded command output. (#34045, #34049, #34216, #34223, #34359)

MCP 連線不再被登入流程卡住

外部工具很多時，MCP（Model Context Protocol，讓 AI 連接外部工具的協定）一啟動就同時登入、刷新授權，容易慢下來甚至互相衝突。新版替啟動設逾時、不讓 OAuth 探索（尋找登入授權入口的流程）阻塞整體、把刷新排隊，並安全重用工具目錄。像機場把登機口檢查分流：太久沒來的人不會堵住全隊，同一張航班看板也不用每人重新印。結果是 MCP 啟動與驗證較穩，並非新增工具功能。

Original changelog
Prevented slow or conflicting MCP startup and authentication flows by enforcing startup timeouts, avoiding blocking OAuth discovery, serializing refreshes, and reusing tool catalogs safely. (#32229, #32781, #32825, #33184, #33297)

Windows 執行與沙盒更可靠

在 Windows 上執行 Codex 時，這次集中補強執行與沙盒（sandbox，隔離程式可碰範圍的安全空間）可靠性：加入原生 exec-server 沙盒化、強制網路代理規則、隱藏輔助主控台，也正確處理 hook（掛鉤，特定事件發生時被觸發的小程式）指令的引號。像工地把圍籬、出入口與工作人員通訊一起整理好；你看不見的細節，正是程式能照預期跑、網路規則不被繞過的基礎。

Original changelog
Improved Windows execution and sandbox reliability, including native exec-server sandboxing, network-proxy enforcement, hidden helper consoles, and correctly quoted hook commands. (#32849, #32857, #33926, #34423)

安裝與版本資料少一層故障點

更新或安裝本不該因為外部宿主程式一時不可用就停住。新版修正精簡版 release metadata（發行版本的描述資料）解析與 macOS code mode 安裝；若外部 code-mode host 無法使用，會改走同一程序內的備援。像自動門的外接感應器壞了，門仍有內建按鈕可開。換句話說，原本仰賴外部元件的那一步遇到障礙，程式仍有自己完成工作的路。這是既有安裝流程的修復與備援，不代表另推出一種新安裝方式。

Original changelog
Fixed compact release-metadata parsing and macOS code-mode installation, with an in-process fallback when the external code-mode host is unavailable. (#31667, #31876, #31899)

Original changelog
