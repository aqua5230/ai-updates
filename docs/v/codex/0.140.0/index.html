查詢用量、無縫搬家，指令工具大擴充

想要管好自己的帳號與專案變得超容易。現在輸入 `/usage` 就能看你每天、每週或累積用了多少代幣（token）；想清掉不需要的紀錄時，只要用 `codex delete` 或 `/delete` 就能永久刪除對話，系統還會貼心地幫你把底下的代理人（subagent）清乾淨並加上防呆確認。如果你是從 Claude Code 跳槽過來的，現在也能透過 `/import` 輕鬆把設定、專案組態跟最近的聊天紀錄無縫搬家過來。就像你有了專屬的管家，隨時能查這個月花了多少錢、幫你把不要的雜物連同地下室一起清掉，連搬家時都能直接把舊公寓的傢俱原封不動搬到新家。

Original changelog
Added `/usage` views for daily, weekly, and cumulative account token activity. (#27925)
Added permanent session deletion through `codex delete`, `/delete`, and app-server `thread/delete`, with confirmation safeguards and subagent cleanup. (#25018, #27476)
Added `/import` for selectively importing setup, project configuration, and recent chats from Claude Code. (#27070, #27071, #27703)

聊天介面更自由，貼上大量文字與圖片不再出包

終端機裡的聊天介面變得更寬容、更方便了。使用 `/goal` 指令時，就算你貼上超長的文章、龐大的程式碼區塊或是附加圖片，它都能完好保留不會吃字。現在只要打出 `@` 符號，就會自動跳出一個整合式的選單，讓你輕鬆指定要參考哪個檔案、外掛（plugin）或技能。此外，如果你之前按過「不再提醒更新」，系統現在會乖乖記住不再煩你，且對話結束後也不會殘留奇怪的執行中指標。對了，實驗性質的 `/realtime` 語音功能已經功成身退，從介面裡移除了。就像你換了個大容量信箱，以前塞不下的大包裹跟照片現在都能輕鬆投遞；寫信時只要打個代號，通訊錄就會自動幫你填好所有聯絡人，而且它終於記住你貼在門上「謝絕推銷」的貼紙了。

Original changelog
`/goal` now preserves oversized text, large pasted blocks, and image attachments, including in remote app-server sessions. (#27508, #27509, #27510)
Typing `@` now opens the unified mentions menu for files, plugins, and skills by default. (#27499)
Persisted “Don’t remind me” update dismissals reliably and cleared stale running-hook indicators after completed turns. (#27619, #27783)
Removed the experimental `/realtime` voice controls and related audio dependencies from the TUI. (#27801)

安全儲存憑證，外掛與工具連線更穩固

你的登入身分與外掛連線現在受到更高級的保護。系統新增了 Amazon Bedrock 的 API 金鑰認證，並用加密的方式把你的憑證（credentials，證明你是合法使用者的金鑰）安全地鎖在本地端。在連線外部工具（MCP）時，如果一開始沒連上系統會聰明地重試，如果發現憑證不能用也會直接回報你「已登出」，而且不會再偷偷把你自己停用的伺服器打開。遠端外掛的解除安裝現在也能順利運作，並清楚提示哪些應用程式需要先認證。就像你把家裡鑰匙放進了指紋保險箱，保全系統遇到接觸不良會自己多試兩下，確認門禁卡消磁就會請你重新登記，而且你不想要的警報器絕對不會半夜自己響起來。

Original changelog
Added managed Amazon Bedrock API-key authentication and encrypted local storage for CLI and MCP OAuth credentials. (#27443, #27689, #27504, #27535, #27539, #27541)

防呆與防當機，壞掉的資料庫會自己修好

如果底層紀錄狀態的 SQLite 資料庫不幸毀損（甚至資料夾壞成不明檔案），系統現在會自動備份並從歷史紀錄把它原地復活。當你在自動審查（`/review`）遇到指引時按下 `Esc` 退出，畫面不會再直接當機，且你取消審查後指引還會乖乖留著。跑在背景的非互動指令，現在也能讓你用 `Ctrl-C` 隨時中斷，而且還能完好保留最後的輸出結果；面對超大專案跟落落長的對話，系統也學會了快取跟減少重複讀取，反應速度大躍進。就像你的車子裝了黑盒子跟自動修復烤漆，不管是不小心按錯按鈕還是被石頭砸到，都不會拋錨，甚至還能自己把刮痕補好、讓引擎跑得更順暢。

Original changelog
Corrupted SQLite state databases are now backed up and rebuilt automatically from rollout data, including malformed database-directory cases. (#26859, #27719)
Prevented `/review` from crashing when `Esc` is pressed with queued guidance, while preserving that guidance when the review is canceled. (#22879)
Non-TTY background commands can now be interrupted with Ctrl-C while preserving their final output and exit status. (#26734)
Improved responsiveness for large repositories and long sessions by preserving Git’s built-in filesystem monitor, avoiding duplicate history reads, accelerating archive lookup, and caching turn-diff rendering. (#26880, #27031, #27276, #27489)

Auto-repairing databases and crash-proof safety nets

For massive codebases and marathon chat sessions, it also learned to cache UI rendering and skip duplicate history reads, delivering a massive speed boost. 사용량 조회, 깔끔한 정리 및 매끄러운 환경 이전을 위한 명령어 대거 추가

계정과 프로젝트를 관리하기가 엄청나게 쉬워졌다. 이제 `/usage`를 입력하면 매일, 매주 또는 누적해서 토큰을 얼마나 썼는지 볼 수 있다. 필요 없는 기록을 지우고 싶을 땐 `codex delete`나 `/delete`로 대화를 영구 삭제할 수 있는데, 시스템이 알아서 아래에 딸린 하위 에이전트(subagent)까지 깨끗이 치워주고 혹시나 실수할까 봐 확인 창도 띄워준다. Claude Code에서 넘어온 사람이라면 이제 `/import`를 통해 설정, 프로젝트 구성, 최근 대화 기록을 손쉽게 옮겨올 수 있다. 이번 달에 돈을 얼마나 썼는지 언제든 알려주고, 필요 없는 잡동사니를 지하실까지 싹 비워주며, 이사할 때 옛날 아파트의 가구를 새집으로 고스란히 옮겨주는 전속 집사가 생긴 것과 같다.

Added `/import` for selectively importing setup, project configuration, and recent chats from Claude Code. (#27070, #27071, #27703)

채팅 화면이 더 자유로워지고, 거대한 텍스트와 이미지 붙여넣기가 매끄러워짐

터미널의 채팅 화면이 훨씬 너그럽고 쓰기 편해졌다. `/goal` 명령어를 쓸 때, 엄청나게 긴 글이나 방대한 코드 덩어리, 첨부 이미지를 통째로 붙여넣어도 글자가 잘리지 않고 고스란히 남는다. 이제 `@` 기호만 치면 통합 메뉴가 알아서 튀어나와 파일, 플러그인, 스킬을 쉽게 지정할 수 있다. 또한, 예전에 '업데이트 다시 보지 않기'를 눌렀다면 시스템이 이를 얌전히 기억해 더는 귀찮게 굴지 않으며, 대화가 끝난 뒤에도 실행 중 표시가 남던 찌꺼기도 지워진다. 덧붙여 실험적으로 제공되던 `/realtime` 음성 제어 기능은 제 역할을 다하고 인터페이스에서 사라졌다. 우편함을 대형으로 바꾼 덕에 예전엔 쑤셔 넣을 수 없었던 커다란 소포나 사진첩을 이제 거뜬히 넣을 수 있게 된 것과 같다. 편지를 쓸 때 기호 하나만 치면 주소록이 알아서 완성되고, 마침내 대문에 붙여둔 '잡상인 출입 금지' 딱지를 무시하지 않게 되었다.

자격 증명의 안전한 보관 및 플러그인과 도구 연결이 더 튼튼해짐

로그인 신분과 플러그인 연결이 한층 더 철저한 보호를 받는다. Amazon Bedrock의 API 키 인증이 추가되었고, 자격 증명(credentials, 합법적인 사용자임을 증명하는 열쇠)은 암호화되어 로컬에 안전하게 보관된다. 외부 도구(MCP)에 연결할 때, 처음에 실패하더라도 시스템이 똑똑하게 다시 시도하며, 자격 증명을 쓸 수 없으면 '로그아웃됨'으로 확실히 알려주고, 당신이 직접 꺼둔 서버를 몰래 다시 켜는 일도 없다. 원격 플러그인을 삭제하는 기능도 이제 제대로 작동하며, 설치할 때 어떤 앱이 인증을 먼저 받아야 하는지 또렷하게 띄워준다. 집 열쇠를 지문 인식 금고에 넣어둔 것과 같다. 보안 시스템이 접촉 불량일 땐 알아서 다시 시도하고, 출입 카드가 망가진 걸 확인하면 새로 발급받으라고 알려주며, 당신이 원치 않아 꺼둔 경보기는 한밤중에 절대 혼자 울리지 않는다.

Original changelog
Added managed Amazon Bedrock API-key authentication and encrypted local storage for CLI and MCP OAuth credentials. (#27443, #27689, #27504, #27535, #27539, #27541)
Improved MCP reliability by retrying transient startup failures, reporting unusable OAuth credentials as logged out, and preserving explicitly disabled servers. (#25147, #26713, #27414)
Fixed remote plugin uninstall requests and correctly surfaced apps requiring authentication during installation. (#27085, #27223)

고장 방지와 안전장치 강화, 깨진 데이터베이스가 알아서 고쳐짐

밑바탕 상태를 기록하는 SQLite 데이터베이스가 깨지더라도(심지어 폴더가 알 수 없는 파일로 망가진 경우까지) 이제 시스템이 알아서 백업하고 과거 기록에서 완벽하게 되살려낸다. 자동 리뷰(`/review`) 중 안내가 떴을 때 `Esc`를 눌러 빠져나가도 더 이상 화면이 먹통이 되지 않으며, 리뷰를 취소해도 안내 문구는 고스란히 남아 있다. 백그라운드에서 도는 비대화형 명령어 역시 언제든 `Ctrl-C`로 끊을 수 있고, 이때 마지막 출력 결과물도 온전히 남겨준다. 엄청나게 큰 프로젝트나 끝없이 이어지는 대화에서도 캐시를 활용하고 중복 읽기를 줄여 반응 속도가 크게 빨라졌다. 차에 블랙박스와 자동 복원 페인트를 칠한 것과 같다. 버튼을 잘못 누르거나 돌멩이에 맞아도 도로에 퍼지지 않고, 스스로 흠집을 메우며 엔진을 부드럽게 굴리는 셈이다.

Corrupted SQLite state databases are now backed up and rebuilt automatically from rollout data, including malformed database-directory cases. (#26859, #27719)
Prevented `/review` from crashing when `Esc` is pressed with queued guidance, while preserving that guidance when the review is canceled. (#22879)
Non-TTY background commands can now be interrupted with Ctrl-C while preserving their final output and exit status. (#26734)
