It's like having to memorize every book title on a whole library floor just to find one useful book — now you just search for what you want and get pointed straight to the shelf.

When a remote image link fails, the AI now understands why instead of failing silently

If you point the AI at a remote image link (HTTP/HTTPS image, a picture that lives on a server elsewhere and only loads via its URL), problems with the format or source used to silently stall or fail the whole flow with no clear reason. PowerShell commands with unreadable executable code now get held for your approval, not waved through

The safety classifier (safety classifier, the system that automatically judges whether a command is safe and decides whether to ask your permission first) used to sometimes wave through PowerShell (Windows's scripting language) commands containing executable code regions (AST, abstract syntax tree, a structured breakdown of code that computers can parse) it couldn't actually parse — treating 'can't read it' as 'must be safe.' Now anything the classifier can't read gets stopped for your manual approval instead, closing that 'unreadable so let it through' loophole. It's like customs that used to sometimes wave through suspicious items it couldn't identify; now anything it can't identify gets pulled aside and opened up first.

Original changelog
PowerShell commands containing executable AST regions the safety classifier cannot inspect now require approval.

Picking an incompatible model in Code Mode now gets you a warning up front

Code Mode (a mode where the AI calls tools by writing code directly instead of issuing commands one at a time) needs the model to supply specific metadata (metadata, extra information describing which features a model supports) to work properly. AI 가 플러그인 도구를 찾을 때 이제 먼저 “검색”한 뒤 쓰며, 더 정확하고 빠름

MCP（Model Context Protocol，AI 가 외부 도구와 연결되는 규약）서버를 많이 연결해두면, AI 는 예전엔 모든 도구의 설명서를 통째로 외워야만 뭐가 쓸 수 있는지 알 수 있었다. 이제는 지원되는 환경에서 AI 가 도구 검색（tool search，통째로 외우는 대신 필요할 때 질의로 필요한 도구를 찾는 방식）으로 도구를 찾아 더 정확하게 찾아내며, 구형 모델과 구버전 플러그인도 그대로 잘 작동한다. 마치 예전엔 도서관 한 층의 책 제목을 전부 외워야 쓸만한 책을 찾을 수 있었던 것을, 이제는 검색 시스템에 “이걸 찾는다”고 물으면 바로 그 칸을 가리켜주는 것과 같다.

Original changelog
MCP tools now use tool search by default when supported, improving tool discovery while preserving compatibility with older models and providers.

다크 모드에서 플러그인 아이콘이 더 이상 튀지 않으며, 전용 다크 버전을 따로 준비 가능

플러그인（plugin，Codex 에 추가 기능을 더해주는 부가 패키지）개발자는 이제 전용 다크 모드（dark mode，배경이 어둡고 야간 사용에 적합한 표시 모드）아이콘을 따로 준비할 수 있다. 플러그인 자체 매니페스트나 원격 플러그인 스토어를 통해 제공되며, 다크 화면으로 전환하면 자동으로 이 아이콘으로 바뀌어, 어두운 배경에 밝은 아이콘이 튀어 보이는 일이 없어진다.

Original changelog
Plugins can provide dedicated dark-mode logos through local manifests and remote catalogs.

Amazon Bedrock 자격 증명이 만료돼도 이제 고치는 법을 정확히 알려줌, 단순히 "권한 없음"으로 끝나지 않음

Amazon Bedrock（아마존의 클라우드 AI 플랫폼）을 통해 모델에 연결하는 경우, 예전엔 자격 증명（credentials，서비스 사용 권한이 있음을 증명하는 키）이 만료되면 그냥 막연한 “권한 없음” 오류만 떴고 무엇이 문제인지 직접 추측해야 했다. 이제는 구체적으로 어떻게 해야 하는지, 유효한 자격 증명을 어떻게 다시 받는지 정확히 알려준다. 예전엔 ATM 이 “카드에 문제가 있다”라고만 표시하던 것을, 이제는 “카드가 만료됐으니 창구에서 새 카드를 발급받으세요”라고 또박또박 알려주는 것과 같다.

Original changelog
Expired Amazon Bedrock credentials now produce actionable recovery guidance instead of a generic authorization error.

원격 이미지 링크가 실패해도 AI 가 이제 이유를 이해함, 조용히 실패하지 않음

원격 이미지 링크（HTTP/HTTPS image，다른 서버에 있어 URL 을 통해야만 보이는 이미지）를 AI 에게 줬을 때, 예전엔 형식이나 출처에 문제가 있으면 이유도 모른 채 과정 전체가 조용히 멈추거나 실패하곤 했다. 이제는 시스템이 명확한 검증 오류를 만들어내, AI 스스로도 “왜 이 이미지가 안 되는지”를 읽고 이해할 수 있다. 직접 붙여넣은 이미지 데이터（data URL）나 로컬 파일 이미지는 영향 없이 그대로 작동한다.

Original changelog
Remote HTTP(S) image inputs now return clear model-visible validation errors; inline data URLs and local images remain supported.

PowerShell 명령에 분석 못 하는 실행 가능 코드가 있으면 이제 먼저 막고 승인을 구함

안전 분류기（safety classifier，명령이 안전한지 자동으로 판단해 사전에 동의를 구할지 정하는 시스템）는 예전엔 PowerShell（Windows 의 스크립트 언어）명령 안에 자신이 분석할 수 없지만 실제로는 실행 가능한 코드 영역（AST，abstract syntax tree，코드를 컴퓨터가 이해하기 쉽게 구조화한 트리）이 있으면 안전하다고 오판해 그냥 통과시키는 경우가 있었다. 이제는 분류기가 읽지 못하고 판단할 수 없으면 무조건 먼저 막고 직접 승인을 받도록 바뀌어, “못 읽으니 통과”라는 허점이 사라졌다. 세관이 예전엔 식별 못 하는 의심 물품을 가끔 그냥 통과시키던 것을, 이제는 식별 못 하면 무조건 먼저 막고 열어서 검사하는 것으로 바꾼 것과 같다.

Original changelog
PowerShell commands containing executable AST regions the safety classifier cannot inspect now require approval.

Code Mode 에서 호환되지 않는 모델을 고르면 이제 미리 경고해줌

Code Mode（AI 가 명령을 하나씩 부르는 대신 코드를 작성하는 방식으로 도구를 호출하는 모드）가 제대로 작동하려면 모델이 특정 메타데이터（metadata，이 모델이 어떤 기능을 지원하는지 설명하는 부가 정보）를 제공해야 한다. 예전엔 지원하지 않는 모델을 고르면 영문도 모른 채 조용히 문제가 생기곤 했다. 이제는 시스템이 미리 “이 모델은 이렇게 쓰기에 적합하지 않을 수 있다”라고 경고를 띄워준다.

Original changelog
Code Mode now warns when the selected model lacks the required metadata.
