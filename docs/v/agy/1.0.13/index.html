Antigravity 1.0.13

發布日期：2026-07-13

zh-TW

指令放行機制更聰明，認得輸出轉向也更安全

現在「永遠允許執行」的指令名單變得更嚴謹也更聰明了。以前只要指令名字有部分對得上，系統可能就會放行，現在預設改成必須「完全一模一樣」才算數，除非你自己加上 regex: 前綴來指定使用正規表示式（regular expression，一種用符號表示文字特徵的寫法）。同時，它也沒那麼死板了：如果你只是把原本已經允許的指令結果輸出存進檔案（像 指令 > 檔案 這樣），系統不會因為你多加了輸出轉向的寫法就把你擋下來。就像大樓警衛，以前只要名牌上有一樣的字就放人，現在預設只認全名，但如果你只是手上多拿了個要存進倉庫的箱子，他也不會故意刁難你。

Original changelog
Improved command permission security by making "Always Approve" rule matching strict (non-regex) by default, while allowing users to explicitly opt-in to regex matching by prepending rules with `regex:`.
Improved command permission usability by relaxing redirection checks, allowing safe commands with output redirection (e.g., `tool > file`) to match without requiring strict full-command approval.

修好畫面切換死當與打字復原失效問題

切換畫面、快速復原不再卡死。現在當你在不同畫面間切換（例如打開子代理詳情或登出）時，鍵盤不會再突然完全沒反應卡死，輔助說明的面板也會正確關閉。如果你在輸入框裡打字太快並頻繁按復原（undo）或重做（redo），歷史紀錄也不會再亂掉。打上帶有斜線的特殊捷徑指令時，送出的瞬間那個斜線也不會因為畫面更新而短暫消失，你打什麼它就顯示什麼。此外，退出程式時也不會再囉嗦地印出兩行重複的提示，清爽多了；而叫代理去瀏覽網頁時，它也不會因為忘記帶上瀏覽器專用的內部提示詞而失敗。就像開車，以前切換排檔時方向盤偶爾會卡死，音響也會短暫消音，現在整台車的操控變得順暢無比。

Original changelog
Resolved bugs during UI transitions (such as opening subagent details or logging out) by introducing a unified synchronization mechanism that prevents key lockups and ensures overlay panels like the /help view are properly reset.
Fixed a bug in the CLI prompt editor where undo and redo history stacks could become desynchronized during rapid mutations by decoupling the history state into a unified, pointer-backed structure.
Fixed a bug where the CLI would temporarily render skill commands without their slash prefix during optimistic updates by deferring prefix stripping to the serialization boundary, ensuring the UI always displays exactly what the user typed.
Fixed a redundant CLI exit message by removing the "Resume in the same project" hint line, leaving only the standard resume command to simplify exit output.
Fixed a bug where browser-related prompt sections were missing from the agent's prompt registry, ensuring browser-based tasks execute reliably.

Smarter command approvals: strict matching and redirection awareness

The "Always Approve" command list is now both stricter and smarter. Previously, partial name matches might slip through, but now the default requires an exact, full-string match unless you explicitly prepend rules with regex: to use regular expressions (a pattern-matching syntax). At the same time, it's less rigid about minor additions: if you just pipe the output of an already-approved command into a file (like command > file), the system will no longer block you simply because you added redirection syntax. Like a building security guard who used to let people in if their badge had any matching words, but now strictly checks the full name — yet won't maliciously stop you just because you're holding a box destined for the warehouse.

Improved command permission usability by relaxing redirection checks, allowing safe commands with output redirection (e.g., `tool > file`) to match without requiring strict full-command approval.

Fixes UI transition lockups and editor undo history failures

A bunch of annoying hiccups that made navigating the terminal feel clunky have been fixed. Now, when transitioning between different views (like opening subagent details or logging out), the keyboard won't suddenly lock up and stop responding, and help overlay panels will close properly. If you type rapidly in the input box and frequently mash undo or redo, your edit history stack won't get hopelessly desynchronized anymore. When you type special slash commands, the slash prefix will no longer flicker and momentarily disappear from the screen right as you hit send; it exactly displays what you typed. Moreover, exiting the program no longer spams redundant reminder lines, making the output much cleaner, and asking agents to browse the web won't randomly fail because it forgot to load the browser-specific internal prompts. Like driving a car: shifting gears used to occasionally lock the steering wheel and briefly mute the radio, but now the entire driving experience handles flawlessly.

'항상 허용'하는 명령어 목록이 더 깐깐하고 똑똑해졌다. 예전엔 명령어 이름이 대충 비슷하기만 해도 시스템이 통과시켜 줬을 수 있지만, 이제는 기본적으로 '글자 하나 안 틀리고 똑같아야' 인정하며 정규 표현식(regular expression, 기호를 써서 글자 패턴을 나타내는 방식)을 쓰려면 규칙 앞에 굳이 regex:를 붙여야 한다. 그러면서도 융통성이 생겨서, 이미 허락받은 명령어의 결과를 파일에 저장하려고(예를 들어 명령어 > 파일처럼) 출력 방향을 바꾸는 기호를 덧붙였다고 해서 시스템이 무작정 막아서지 않는다. 빌딩 경비원이 예전엔 이름표에 비슷한 글자만 있어도 들여보내 주다가 이제는 무조건 전체 이름을 깐깐하게 확인하지만, 그렇다고 창고에 넣을 상자 하나를 들고 있다고 해서 일부러 딴지를 걸며 막지는 않는 것과 같다.

터미널에서 작업할 때 답답하게 만들던 자잘한 오류들을 한꺼번에 고쳤다. 이제 하위 에이전트의 자세한 정보를 보거나 로그아웃하는 등 다른 화면으로 넘어갈 때, 갑자기 키보드가 아예 먹통이 되어 멈추는 일이 없고 도움말 화면도 제대로 닫힌다. 입력 칸에서 글자를 너무 빨리 치면서 되돌리기(undo)나 다시 실행(redo)을 마구 눌러도 기록이 엉망진창으로 꼬이지 않는다. 슬래시(/)가 붙은 특수 단축 명령어를 칠 때도, 전송하는 찰나에 화면이 새로 고쳐지며 슬래시가 잠깐 사라지던 일이 없어 당신이 친 그대로 똑똑히 보여준다. 게다가 프로그램을 끄고 나갈 때 똑같은 안내 문구를 두 줄이나 구시렁대며 띄우지 않아 한결 깔끔해졌고, 에이전트에게 웹 서핑을 시켰을 때 브라우저 전용 내부 지시어를 빼먹는 바람에 실패하는 일도 없어졌다. 차를 몰 때 예전엔 기어를 바꾸면 가끔 운전대가 굳어버리거나 오디오가 잠깐 끊기곤 했는데, 이제는 차 전체가 막힘없이 부드럽게 움직이는 것과 같다.
