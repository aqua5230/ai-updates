Antigravity 1.0.6

發布日期：2026-07-13

zh-TW

打字與找指令更聰明，送出訊息零延遲

在終端機裡打字和下指令變得流暢很多了。現在送出訊息的瞬間，你打的字會直接出現在畫面上，不再需要乾等背後連線處理，感覺就像沒有延遲一樣。找快捷指令（像 /help）也變聰明了，就算你只打中間幾個字母（例如 /el），系統也能模糊搜尋並幫你找出來；在 /open 或 /add-dir 時，也會像普通終端機一樣幫你自動補齊檔案路徑。另外，以前如果你在括號後面緊接著打 @ 想標記檔案，選單會裝死出不來，或者你剛按 Esc 打斷代理講話後馬上打字，字會被系統吞掉；現在這些惱人的打字小蟲都被抓掉了。就像手機鍵盤升級，以前打字要等網路發送，稍微打錯字還找不到功能，現在不但能隨便縮寫就搜到，而且字一打完就馬上在螢幕上看到，絕不漏字。

Original changelog
Added shell-style path auto-completion for `/open` and `/add-dir`.
Added optimistic rendering for user chat prompt submissions, injecting messages immediately into the viewport to eliminate perceived input lag.
Added fuzzy and partial substring matching across slash commands. E.g. `/el` -> shows `/help` and `/model` while previous no suggested completions.
Fixed a bug when suggestion was not triggered when `@` is typed after `(`. Enabled unconditional typeahead suggestions whenever `@` is typed without preceding whitespace, streamlining mention workflows.
Fixed a bug where entering a prompt immediately after pressing `Esc` (to interrupt an active agent stream) caused the newly typed input to be swallowed or rejected.

歷史清單不再被子代理洗版，狀態列與沙盒更可靠

你的歷史對話清單現在乾淨多了。打開 /resume 時，系統會自動把子代理（背後自動跑的機器人）的對話紀錄隱藏起來，只專心留著你自己開的對話，不會再被洗版。如果你有自訂終端機底下的狀態列，現在可以透過設定把「官方預設狀態」跟你自己的「自訂狀態」上下疊在一起顯示，不用再面臨二選一的抉擇。對於進階使用者，當你使用不跳視窗的純文字模式（--print）時，加上 --sandbox 的安全隔離指令也終於能正確生效了。就像整理信箱，以前廣告信會把你的私人信件全部淹沒，現在系統幫你把機器人的信件自動過濾掉；而且你的辦公桌上現在可以同時放著公司的公用月曆和你自己的私人筆記本，兩個都能一眼看到。

Original changelog
Skipped subagent conversations from `/resume`, keeping the standalone conversation picker focused purely on direct user initiated conversations.
Added a `stack_with_default` flag to the `statusLine` configuration to render both the default Antigravity status line and custom status line output vertically stacked.
Fixed `--sandbox` flag propagation in headless print mode (`-p` / `--print`), ensuring sandbox isolation is correctly enforced during non-interactive execution.

en

Zero-latency message submission with smarter typing and path autocompletion

Typing and running commands in the terminal now feels incredibly fluid. The moment you submit a message, your text instantly appears on the screen without waiting for the backend to process it, completely eliminating perceived input lag. Finding slash commands (like /help) is also much smarter: even if you only type a few letters from the middle of the word (like /el), the system will use fuzzy matching to find what you mean. The /open and /add-dir commands now support shell-style path autocompletion just like a normal terminal. Additionally, we've squashed several annoying typing bugs: the autocomplete menu will no longer play dead if you type @ immediately after a parenthesis to mention a file, and your typed text will no longer be swallowed if you start typing right after pressing Esc to interrupt a rambling agent. Like upgrading your phone keyboard: you used to wait for the network every time you hit send and couldn't find apps if you misspelled them; now, you can find anything with a sloppy abbreviation, and your texts instantly show up on the screen the moment you tap send.

Clean history lists, stacked status lines, and reliable headless sandboxing

Your conversation history list is now much cleaner. When you open /resume, the system automatically hides all subagent (background bots) conversations, focusing purely on the chats you explicitly started so your list doesn't get relentlessly spammed. If you customized the status line at the bottom of the terminal, you can now toggle a setting to vertically stack the "default official status" with your "custom status" together, so you don't have to choose one or the other. For advanced users, when running in the non-interactive text-only mode (--print), the --sandbox isolation flag will finally enforce its security boundaries correctly. Like organizing your inbox: spam and automated emails used to drown out your personal letters, but now the system filters out the robot mail for you; additionally, you can now keep both the corporate calendar and your personal notebook on your desk at the same time, seeing both at a glance.

会話履歴の一覧がはるかに綺麗になった。/resume を開いた際、システムがサブエージェント（裏で自動で動くボット）の会話記録を自動で隠し、あなたが直接始めた会話だけを残すようになったため、履歴が埋め尽くされることはなくなった。端末下部のステータス行をカスタマイズしている場合、設定を変えれば「公式のデフォルトステータス」と「自作のステータス」を上下に重ねて同時に表示できるようになり、どちらかを選ぶ必要がなくなった。また、上級者向けに、対話画面を出さない純粋なテキストモード（--print）で実行した際も、--sandbox による安全な隔離機能が正しく働くようになった。郵便受けの整理のようなもので、以前はダイレクトメールが私信を完全に埋め尽くしていたのが今はロボットからの手紙を自動で除外してくれ、さらに会社のカレンダーと自分の手帳を両方とも机に並べて、一目で確認できるようになったわけである。

ko

타이핑과 명령어 찾기가 똑똑해지고, 메시지를 보내는 즉시 화면에 뜬다.

터미널에서 글을 치고 명령어를 내리는 느낌이 훨씬 매끄러워졌다. 이제 메시지를 보내는 순간 당신이 친 글자가 화면에 바로 나타나, 뒤에서 통신 처리가 끝날 때까지 멍하니 기다릴 필요 없이 랙(지연)이 싹 사라진 느낌을 준다. 단축 명령어(/help 등)를 찾을 때도 똑똑해져서, 단어 중간에 있는 글자 몇 개(예: /el)만 쳐도 시스템이 찰떡같이 알아듣고 찾아주며, /open이나 /add-dir을 쓸 때도 일반 터미널처럼 파일 경로를 알아서 채워준다. 게다가 예전엔 괄호 바로 뒤에 @를 쳐서 파일을 부르려 하면 메뉴가 죽은 척 안 나오거나, 에이전트의 말을 Esc로 끊자마자 글을 치면 글자가 통째로 씹히는 등 성가신 타이핑 벌레(버그)들도 모조리 잡아냈다. 스마트폰 키보드를 업그레이드한 것과 같다. 예전엔 글을 칠 때마다 네트워크를 기다려야 하고 조금만 틀려도 기능을 못 찾았지만, 이제는 대충 줄여 쳐도 다 찾아주고 전송 버튼을 누르자마자 글자가 화면에 바로 박히며 글자가 씹히는 일도 없게 됐다.

하위 에이전트가 기록을 도배하지 않고, 상태 줄과 샌드박스가 더 믿음직해졌다.

과거 대화 목록이 훨씬 깨끗해졌다. /resume를 열면 시스템이 하위 에이전트(뒤에서 알아서 도는 로봇)의 대화 기록을 자동으로 숨겨주고 당신이 직접 연 대화만 깔끔하게 남겨두어, 목록이 잡다한 대화로 꽉 차는 일이 없다. 터미널 아래쪽의 상태 줄을 직접 고쳐 쓰는 경우, 이제 설정을 통해 '공식 기본 상태'와 '내가 만든 상태'를 위아래로 쌓아 동시에 띄울 수 있어 둘 중 하나만 고르는 아쉬움이 사라졌다. 고급 사용자의 경우, 대화 창을 띄우지 않는 글자 모드(--print)로 실행할 때 --sandbox(보안 격리 명령)를 붙여도 드디어 안전하게 제구실을 한다. 우편함을 정리하는 것과 같다. 예전엔 광고 우편물이 내 개인 편지를 다 덮어버렸지만 이제는 기계가 보낸 편지를 알아서 걸러주고, 내 책상 위에 회사의 공용 달력과 내 개인 수첩을 동시에 올려놓고 한눈에 볼 수 있게 된 셈이다.
