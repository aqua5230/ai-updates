zh-TW

啟動神速不留垃圾，對話紀錄全面擁抱資料庫

程式啟動速度變快了，而且也不會再亂製造垃圾檔案。以前系統會在你每個專案資料夾裡塞一個隱藏的 .antigravitycli 檔案來記住專案，不但礙眼還拖慢搜尋速度；現在它改把這些紀錄集中收在一個快取檔裡，專案資料夾終於乾淨了。同時，像「檢查 Git 版本控制」或「載入外部工具（MCP 伺服器）」這些本來很耗時的啟動步驟，現在都改到背景平行處理了，就算某個工具連線卡住，也不會害你連打字視窗都開不了。另外，終端機的對話紀錄正式改用 SQLite 資料庫格式（.db）儲存，如果你之前有從 Antigravity 2.0 匯入對話失敗的毛病，現在也都修好了。就像家裡的玄關，以前你要把所有鞋子擺出來慢慢挑才能出門，現在都收進隱藏的鞋櫃裡，而且警衛幫你開門跟司機發動車子是同時進行的，不再需要傻傻排隊等。

Original changelog
Added SQLite (.db) conversation support and will be CLI’s conversation format. Fixed a bug when importing SQLite conversation from Antigravity 2.0 to CLI.
Decoupled project discovery from local `.antigravitycli` workspace directories. The CLI now stores workspace-to-project mappings in a centralized `~/.gemini/antigravity-cli/cache/projects.json` file, eliminating repository clutter and speeding up project discovery to a single-map lookup.
Moved VCS detection out of the synchronous CLI startup path to prevent slow initialization.
Parallelized the MCP server initialization sequence, preventing slow or hanging custom MCP servers from blocking independent, fast-starting servers (like local plugins) from loading on startup or configuration reloads.

數學公式漂漂亮亮，排版與反白更完美

終端機現在看得懂也畫得出數學公式了。系統現在支援 LaTeX 數學語法渲染，不管是微積分還是複雜的矩陣公式，都能直接在終端機裡漂亮地印出來（如果你不喜歡，也可以用環境變數關掉它）。在看歷史對話清單時，如果對話標題裡面有奇怪的換行或一堆空白，系統現在會把它們壓平成一行，不會再把選單撐破。當你在看程式碼差異（/diff）時，以前行號跟程式碼中間會有一條尷尬的未上色縫隙，現在背景標示顏色會完美地橫跨整個畫面，不再有破綻。就像以前在終端機裡看數學公式只能看一堆看不懂的符號亂碼，現在就像在看排版精美的教科書；而且看螢光筆畫重點時，筆跡也不會在中途詭異地斷掉了。

Original changelog
Added LaTeX math rendering, enabling the CLI to display beautiful mathematical formulas directly in the terminal viewport. Set `AGY_CLI_DISABLE_LATEX` environment variable to turn off LaTeX rendering globally if desired.
Collapses all newlines and consecutive whitespaces in conversation previews and titles before rendering list items, preventing visual UI layout breaks in the picker rows.
Styled the separator space between the line number column and diff content to match the text blocks, ensuring background highlights stretch seamlessly across the viewport width in tool outputs and `/diff` details.

解決卡死當機與規則不理人的問題

網路不穩時介面不再卡死。以前網路不穩或代理跑太快時，畫面有時會直接卡死再也不動；現在修好了這個回呼（callback）的問題，不會再死當。如果你自己寫了快捷指令（skill），以前在選單裡按 Tab 鍵補齊後，它竟然會直接清空輸入框裝死不執行；現在補齊後終於會乖乖送出了。另外，以前你在 rules.json 裡設定了「哪些規則要忽略、哪些要白名單」，系統啟動時卻會當作沒看到，傻傻把所有規則都載入；現在它終於學會聽話照做了。還有 /changelog 指令寫入快取時因為太急著關閉程式而失敗的毛病也修好了。就像你去點餐，以前你照著菜單點隱藏菜色，店員卻直接把點菜單撕掉當沒聽到，或是你明文規定不吃的配菜廚師還是全加進去，現在這些荒唐的服務態度都已經糾正完畢了。

Original changelog
Resolved sporadic and permanent UI hangs caused by a stateful callback streamer race condition during network drops or extremely fast agent steps.
Resolved inconsistent behavior where selecting skill-derived slash commands from autocompletion suggestions cleared the input without executing. Autocompleted skill commands are now correctly submitted to the backend.
Aligned the interactive `/changelog` and `agy changelog` cache paths to both use `antigravity-cli`, and made the caching process synchronous to resolve a race condition where immediate process exit terminated the cache write.
Resolved an issue where exclusion rules and allowlists configured in rules.json were silently ignored, causing the discovery engine to load every .md rule file unconditionally at boot.

en

Fast startup with zero repository clutter and full SQLite database support

Program startup is much faster now, and it no longer litters your projects with junk files. Previously, the system would drop a hidden .antigravitycli file in every project folder to remember it, which cluttered your repository and slowed down discovery; it now stores these mappings centrally in a single cache file, leaving your project folders perfectly clean. At the same time, time-consuming startup steps like checking Git (VCS) or loading external tools (MCP servers) have been moved to parallel background processes—so even if one custom tool hangs, it won't block you from opening the chat window to start typing. Additionally, terminal conversations now officially use the SQLite database format (.db), and the bug that previously failed to import your chats from Antigravity 2.0 has been fixed. Like your home's entryway: you used to have to leave all your shoes scattered out to find a pair before leaving, but now they're neatly organized in a hidden cabinet; plus, the guard opening the gate and the driver starting the car now happen simultaneously instead of waiting in line.

The terminal can now understand and draw math formulas. The system fully supports LaTeX math rendering, meaning everything from calculus to complex matrices will print beautifully right in your terminal viewport (you can also turn this off globally with an environment variable if you prefer). When viewing your conversation history, if a chat title contains awkward line breaks or a ton of consecutive whitespace, the system flattens it into a single line so it doesn't break the menu layout. When viewing code differences (/diff), there used to be an awkward uncolored gap between the line numbers and the code; the background highlight now stretches seamlessly across the entire width of the screen, leaving no visual flaws. Like trying to read math equations in the terminal: you used to stare at an unreadable mess of symbols, but now it looks like a beautifully typeset textbook; and when using a highlighter to mark important lines, the ink no longer randomly skips a spot in the middle.

Fixes UI hangs, autocomplete execution bugs, and ignored rules

We've eradicated several bugs that made you want to smash your keyboard. Previously, if your network dropped or an agent ran extremely fast, the UI could completely freeze due to a stateful callback race condition; this has been resolved so it won't permanently hang anymore. If you wrote a custom slash command (skill), selecting it via Tab autocomplete used to inexplicably clear your input box and do absolutely nothing; it now properly submits to the backend. Also, if you configured allowlists or exclusion rules in rules.json, the system used to blindly ignore them on startup and load every rule anyway; it has finally learned to follow your instructions. A bug where the /changelog cache failed to save because the program exited too quickly has also been fixed. Like ordering at a restaurant: you used to order a secret menu item, but the waiter would just rip up your ticket and play dumb, or the chef would toss in the exact ingredients you explicitly asked to exclude; these ridiculous service issues have finally been corrected.

터미널이 이제 수학 공식을 알아보고 그릴 줄 안다. 시스템이 LaTeX 수학 문법을 지원하게 되어 미적분이나 복잡한 행렬 공식도 터미널 안에서 아주 예쁘게 찍혀 나온다(원하지 않으면 환경 변수로 이 기능을 끌 수도 있다). 대화 기록 목록을 볼 때 대화 제목에 이상한 줄바꿈이나 빈칸이 잔뜩 들어 있어도, 시스템이 알아서 한 줄로 납작하게 눌러 메뉴 모양이 망가지는 걸 막아준다. 코드의 바뀐 부분(/diff)을 볼 때 예전엔 줄 번호와 코드 사이에 칠해지지 않은 뻘쭘한 틈이 있었지만, 이제는 바탕색이 화면 전체를 빈틈없이 덮어 한결 깔끔하다. 터미널에서 수학 공식을 볼 때 예전엔 알아먹기 힘든 기호 뭉치만 봐야 했다면, 이제는 예쁘게 편집된 교과서를 보는 것과 같으며, 형광펜으로 밑줄을 그을 때 잉크가 중간에 뚝 끊기던 문제도 사라진 셈이다.

키보드를 내리치고 싶게 만들던 짜증 나는 버그들을 몇 개 잡았다. 예전엔 인터넷이 불안정하거나 에이전트가 너무 빨리 돌아갈 때 화면이 완전히 굳어버리곤 했는데, 이제 꼬여있던 회신(callback) 문제를 풀어 더 이상 멈추지 않는다. 내가 직접 만든 단축 명령어(기술)를 메뉴에서 Tab 키로 자동 완성하면, 어이없게도 입력 칸을 싹 비우고 모른 척 넘어가 버렸지만 이제는 제대로 입력되어 실행된다. 또, 예전엔 rules.json에 '어떤 규칙은 빼고 어떤 것만 허락할지'를 뻔히 적어두어도 시스템이 켜질 때 못 본 척 모든 규칙을 다 불러왔는데, 이제야 말을 제대로 듣는다. /changelog 명령어가 너무 급하게 프로그램을 끄느라 임시 파일(캐시)을 쓰지 못하던 버그도 고쳤다. 식당에 비유하자면 예전엔 비밀 메뉴를 시켰더니 종업원이 주문표를 찢어버리고 모른 척하거나, 절대 넣지 말라고 신신당부한 재료를 요리사가 몽땅 넣어버리는 식이었다면, 이제는 이런 황당한 태도가 싹 고쳐진 셈이다.
