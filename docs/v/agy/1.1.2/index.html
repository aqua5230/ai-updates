zh-TW

終端機不再卡頓，介面顯示與預覽更完善

當你長對話執行到 5000 步（step，AI 動作次數）以上時，終端機不再卡頓或吃滿 CPU。以前在不閃爍模式（no flickering mode，防止畫面更新閃爍的機制）下，長對話會因行數計算消耗大量資源而打字卡頓；現在優化計算後已徹底解決。另外，新建檔案確認時可按 f 開啟全螢幕的差異預覽（full diff，完整的更動對照），體驗與修改現有檔案一致。最後，對話裡的工具標題不再顯示 CodeSearch(4 files found...) 等混亂內容，改為 Read(/path) 般乾淨的摘要。就像看超長小說，以前越後面越難翻且目錄凌亂，現在每一頁都很順手、目錄整齊，還能隨時拉出全螢幕對照。

Original changelog
Added an `f` (full diff) shortcut to the create-file tool review screen so new-file confirmations can open a full-screen diff view, matching the existing file-edit experience.
Improved responsiveness on large conversations (5000+ steps) in no flickering mode by switching hot-path line-count methods to pointer receivers, cutting the per-frame prefix-sum cost and eliminating sustained 99% CPU and keystroke lag.
Fixed garbled builtin tool headers such as CodeSearch(4 files found...) by mapping generic tool steps back to clean summaries like Read(/path) and CodeSearch(query).

腳本模式更可靠，模型不偷換、授權不卡死

當你用腳本自動執行任務時，系統不再默默更換模型或卡死。在列印模式（print mode，印出答案即結束的執行方式）下，若指定的模型不存在，系統會立刻報錯中斷並列出可用清單，不再默默降級（但互動模式仍維持警告並使用備援模型（fallback，指主要方案失效時的替代方案）的舊規）。此外，用管線餵指令給腳本時，若需要登入，你可以直接在控制終端機（controlling terminal，如 /dev/tty）貼上 OAuth 授權碼；若在無頭模式（headless runs，沒有終端機視窗的執行模式）下則會立刻報錯，不再傻傻等待。就像派助理買限量便當，買不到時他會立刻打電話告知並回報菜單讓你重挑，需要刷卡時也會把刷卡機遞到你面前；若你不在場，他會直接回報失敗，而不是在櫃檯乾等。

Original changelog
Added support for pasting the OAuth authorization code in print mode (-p) via the controlling terminal (/dev/tty on POSIX and CONIN$ on Windows) when stdin is consumed by a piped prompt, and made truly headless runs fail fast with an actionable message instead of blocking.
Fixed print mode silently downgrading to the default model when --model cannot be resolved by hard-failing with a non-zero exit and listing the available models, while interactive sessions keep the fallback-with-warning behavior.

捷徑自動更新不覆蓋，指令檢查不再大驚小怪

自訂快捷鍵能隨著系統升級自動更新，指令安全檢查也更聰明了。當系統加入新的預設快捷鍵時，它會自動補上新值，同時完整保留你修改過的按鍵配置（user overrides）。另外，當你輸入包著另一層指令的巢狀指令替換（nested command substitutions，如在 echo 裡包著 git 查詢）時，只要這兩個指令都在允許清單（allowlist，白名單）裡，系統就會直接放行，不再重複計算層數（double-counting）而跳出提示詢問。就像遊戲更新後會自動加上新技能按鍵，且不影響你改過的自訂鍵；也像機場安檢看到包包裡裝著已過安檢的隨身包，不會再叫你拆開檢查一次。

Original changelog
Fixed permission checks not respecting the allowlist for nested command substitutions, so a command like echo "$(dirname $(git rev-parse --show-toplevel))" now runs without prompting when echo and git are allowlisted, instead of double-counting the nested command and prompting for review.
Fixed the CLI keybindings file staying out of sync with /keybindings when new default bindings are introduced by persisting the injected defaults while preserving user overrides.

代理分身狀態更精準，MCP 工具不留殘局

當你叫出多個代理幫忙或掛載外部工具時，進度追蹤更精準，系統也更乾淨。多個代理分身（subagents，平行處理任務的 AI 助手）同時工作時，以前會因資料競爭（data race，搶改同一筆資料）和違反寫入時複製（copy-on-write violation）導致狀態錯亂；現在修改狀態前會先複製紀錄，不再互踩。此外，載入 MCP（Model Context Protocol，讓 AI 連接外部工具的協定）工具時，以前在獨立模式下會找不到路徑且殘留背景程序（subprocesses）；現在能正確定位並在關閉時清乾淨。就像請裝潢工班，以前搶簽同一張表導致塗改混亂，現在每人發一張表；離開時也一定帶走所有工具，不留電線垃圾。

Original changelog
Fixed mcp manager failing to resolve tool schema paths in standalone mode and leaking MCP server subprocesses after shutdown, which previously caused panics and cleanup failures for custom agents loading MCP tools.
Fixed a data race and copy-on-write violation when updating subagent states by cloning their stats before in-place mutation, preventing corrupted step counts and status for parallel subagents.

Subagents report accurate states, and MCP tools stop leaking memory

Task progress is fully accurate and the system cleans up properly when you run agent teams or load external tools. 스크립트로 자동화 작업을 돌릴 때 시스템이 몰래 모델을 바꾸거나 멈춰 서지 않는다. 인쇄 모드(print mode, 출력 후 즉시 종료)에서 지정 모델이 없으면 즉시 오류를 내고 사용 가능 목록을 보여준다(대화형 모드는 경고 후 대체(fallback)하는 예전 방식을 유지한다). 또한 파이프로 입력 중에 로그인이 필요해지면 제어 터미널(controlling terminal, /dev/tty 등)에 바로 OAuth 인증 코드를 붙여넣을 수 있다. 헤드리스 실행(headless runs) 상태라면 무한정 기다리지 않고 즉시 오류로 끝낸다. 심부름꾼이 품절 시 즉시 전화해 메뉴를 읊어주고, 결제가 필요할 때 단말기를 내밀며, 부재중이면 계산대에서 대기하지 않고 곧장 돌아오는 것과 같다.

Original changelog
Added support for pasting the OAuth authorization code in print mode (-p) via the controlling terminal (/dev/tty on POSIX and CONIN$ on Windows) when stdin is consumed by a piped prompt, and made truly headless runs fail fast with an actionable message instead of blocking.
Fixed print mode silently downgrading to the default model when --model cannot be resolved by hard-failing with a non-zero exit and listing the available models, while interactive sessions keep the fallback-with-warning behavior.

단축키가 덮어쓰기 없이 자동 업데이트되고, 명령어 검사도 호들갑 떨지 않는다.

사용자 지정 단축키가 자동 업데이트되고, 명령어 안전 검사도 똑똑해졌다. 시스템에 새 기본 단축키가 추가되면 사용자 변경 설정(user overrides)을 보존하면서 새 단축키를 자동으로 추가한다. 또한 다른 명령어를 감싸는 중첩 명령어 치환(nested command substitutions)을 입력할 때, 두 명령어 모두 허용 목록(allowlist)에 있다면 중첩 층수를 이중 계산(double-counting)해 확인을 요구하는 일 없이 곧바로 실행해 준다. 게임 업데이트 시 새 단축키가 추가되어도 기존 설정이 초기화되지 않고, 보안 검색대에서 이미 검사한 가방 안의 파우치를 다시 열어보라고 요구하지 않는 것과 같다.

Original changelog
Fixed permission checks not respecting the allowlist for nested command substitutions, so a command like echo "$(dirname $(git rev-parse --show-toplevel))" now runs without prompting when echo and git are allowlisted, instead of double-counting the nested command and prompting for review.
Fixed the CLI keybindings file staying out of sync with /keybindings when new default bindings are introduced by persisting the injected defaults while preserving user overrides.

에이전트 분신 상태가 정확해지고, MCP 도구도 흔적 없이 정리된다.

여러 에이전트를 가동하거나 외부 도구를 연결할 때 진행 상황 추적이 정확해지고 시스템도 깨끗이 정리된다. 여러 에이전트 분신(subagents, 병렬 작업을 수행하는 AI 조수)이 동시 작동할 때, 이전엔 데이터 충돌(data race)이나 기록 시 복사 위반(copy-on-write violation)으로 상태가 꼬였으나 이제는 업데이트 전 기록을 복제해 충돌을 피한다. 또한 MCP(Model Context Protocol, AI를 외부 도구에 연결하는 규약) 도구를 불러올 때 독립 모드에서 경로를 찾지 못해 백그라운드 프로세스(subprocesses)가 남던 오류를 해결하고 종료 시 완전히 정리한다. 공사 인부들이 출근부 한 장으로 다투지 않고 개별판을 쓰며, 퇴근할 때 공구를 모두 챙겨가는 것과 같다.

Original changelog
Fixed mcp manager failing to resolve tool schema paths in standalone mode and leaking MCP server subprocesses after shutdown, which previously caused panics and cleanup failures for custom agents loading MCP tools.
Fixed a data race and copy-on-write violation when updating subagent states by cloning their stats before in-place mutation, preventing corrupted step counts and status for parallel subagents.
