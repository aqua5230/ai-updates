解決 Windows 貼上無效與文字編輯器當機問題

用 Windows 的人，現在可以直接按 alt+v 貼上圖片了。以前因為作業系統的終端機軟體會攔截 ctrl+v，導致貼上功能時常沒反應；現在多了個備用快捷鍵解決這問題。如果你在設定裡把文字編輯器設為「自動」但又沒裝特別的編輯器，以前系統會傻傻地去呼叫不存在的 vim 然後當掉，現在它懂得乖乖去叫預設的記事本了。此外，當你在不跳視窗的純文字模式下執行指令（例如用管線 pipe 把輸出傳給別的程式），系統也不會再把印出的字默默吃掉了。就像在國外買東西，以前用信用卡如果刷不過就沒轍，現在多提供你一個手機支付的選項，而且店員也終於認得你當地的國民身分證了。

Original changelog
Added `alt+v` as an alternative paste shortcut on Windows to resolve issues where ctrl+v is intercepted by the terminal emulator, enabling reliable image pasting.
Fixed a bug on Windows where print mode and other non-TUI command outputs were silently discarded when run in non-TTY environments (such as pipes or subprocesses).
Fixed Windows editor fallback to use "edit" or "notepad" when the editor setting is "auto" and no editor is configured, instead of attempting to use "vim".

MCP 連線更寬容，設定檔能隨時重整不蓋台

連慢啟動的 MCP 伺服器時，你現在可多等 60 秒。（讓 AI 接上外部工具的協定）時，會耐心等足 60 秒才放棄，不再因為對方動作慢就連線失敗。當你在 /permissions 權限面板看設定時，系統也會聰明地從硬碟重新讀取最新的狀態，避免你一不小心把別處剛改好的設定又蓋過去了。在審核子代理的畫面裡，畫面上提示的按鍵也會確實顯示你自己改過的快捷鍵，不會再寫死成預設按鍵騙你。就像你去辦事處抽號碼牌，以前櫃檯只等三秒沒人就跳號，現在會等你一分鐘；而且辦事員的螢幕也會隨時跟總局的資料庫同步，不會因為不同步就讓你填的資料被洗掉。

Original changelog
Improved the `/permissions` panel to dynamically reload configurations from disk and prevent accidental overwrites.
Increased the MCP connection timeout to 60 seconds to improve reliability for slow-starting custom MCP servers.
Fixed the subagent approval TUI to dynamically render user-defined custom keybindings (such as alternative approval keys) instead of showing hardcoded defaults.

Fixes ignored pastes and editor crashes on Windows

Windows users can now reliably paste images by pressing alt+v. Previously, the operating system's terminal emulator would often intercept ctrl+v, causing the paste function to mysteriously do nothing; this alternative shortcut solves the problem. If you set your text editor to "auto" but hadn't installed any fancy editors, the system used to blindly call a non-existent vim and crash; it now smartly defaults to Notepad. Furthermore, when you run commands in text-only environments (like piping the output to another program), the system will no longer silently swallow the printed text. Like shopping abroad: if your credit card got declined, you used to be out of luck, but now they offer a mobile payment alternative, and the clerk finally knows how to read your local ID card.

More forgiving MCP connections and dynamic permission reloads without overwrites

When connecting to slow-starting custom MCP servers (the protocol that connects AI to external tools), the system will now patiently wait up to 60 seconds before giving up, no longer dropping the connection just because the server took a bit to warm up. While viewing the /permissions panel, the system now smartly reloads the latest state from the disk, preventing you from accidentally overwriting settings you just updated elsewhere. Also, in the subagent approval screen, the key hints now accurately reflect any custom keybindings you set, rather than lying to you with hardcoded defaults. Like drawing a number at a government office: the clerk used to call your number and instantly skip you if you didn't run to the desk in three seconds, but now they wait a whole minute; additionally, the clerk's computer perfectly syncs with the main database so your freshly submitted forms don't get accidentally overwritten.
