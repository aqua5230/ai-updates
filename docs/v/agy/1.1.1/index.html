Antigravity 1.1.1

發布日期：2026-07-08 ~ 07-11

zh-TW

預設改成「寫檔前先給你過目」：逐行看差異、想留哪段自己挑

Antigravity 動你的檔案之前，現在預設會先停下來給你過目。新的 request-review 模式（顧名思義「請求審核」——動你的檔案前先請你過目）在每次寫檔前攤開逐行差異預覽（diff preview，把改前改後逐行並排、標紅標綠的對照畫面，按 f 開啟），你可以整批接受，也可以一段留一段丟；連新建檔案都有專屬的「Create file」預覽，讓你先看內容再點頭。就像裁縫改衣服前先讓你試穿別上珠針的版本，滿意哪段留哪段，而不是改完才給你看成品。

Original changelog
Added `request-review` (default) mode as the default execution behavior: automatically pauses before file write operations to display an interactive, line-level diff preview (`f` shortcut) where users can review, accept, or reject individual code modifications before they are saved to disk. Added a dedicated "Create file" confirmation preview for new file creations (`write_to_file` without overwrite): renders new content as an addition-only diff preview. (1.1.0)

shift+tab 一鍵循環三種執行模式，/plan 取代 /planning

執行模式切換正式開放：按 shift+tab 就能在 default（每次寫檔先問）、accept-edits（放行編輯）、plan（只規劃不動手）三檔之間循環。指令也整併了——/plan 取代舊的 /planning、/fast 直接移除，切換方式收斂成一套，不再多頭馬車。就像檯燈從三顆分開的開關變成一顆旋鈕，轉一下換一檔。

Original changelog
Agent execution mode cycling is now publicly available: `default` -> `accept-edits` -> `plan`. Added `/plan` mode to replace legacy `/planning`, and removed `/fast` slash commands: consolidated and simplified execution mode switching around `shift+tab` mode cycling and the `/plan` mode prefix. (1.1.0)

預設模式進 /settings 面板，改完立刻生效

想固定用某個執行模式，現在到 /settings 面板挑 Agent Mode 就好，改完即時生效，不用再手動編輯 settings.json（設定檔）、也不用每次啟動都掛 --mode 參數。就像冷氣終於有了遙控器上的「預設模式」鈕，不必每次開機都爬去主機上按。

Original changelog
Added an `Agent Mode` option to the `/settings` panel so users can set and persist a default execution mode (`default`, `accept-edits`, `plan`) without manually editing `settings.json` or passing `--mode` on startup, with real-time synchronization so changes take effect immediately. (1.1.0)

啟動就能指定要用哪個自訂代理

啟動時就能指定要用哪個自訂代理（custom agent，你事先寫好個性與專長的分身）：掛上 --agent 旗標直接點名，agent／agents 子指令還能列出手上有哪些分身可挑。以前得先進門再換人，現在像叫車 App 直接指定司機，上車前就選好。

Original changelog
Added the `--agent` flag and `agent/agents` subcommand, allowing users to select a custom agent at launch and list available agents. (1.1.1)

成果檢視器能按 / 搜尋、n/N 跳轉了

在成果檢視器（artifact viewer，看代理產出的檔案或報告的畫面）裡按 / 就能就地搜尋關鍵字，n 跳下一個、N 跳上一個，長報告不用再用眼睛慢慢掃。就像看紙本說明書終於改成看 PDF，Ctrl+F 一按直達你要的那行。

Original changelog
Added in-file keyword search (`/`) and jump navigation (`n/N`) to the artifact detail viewer, allowing users to find and cycle through matches without disrupting terminal escape sequences or image grids. (1.1.1)

腳本派工救星：-p 失敗會老實報錯，也不再卡住不動

用腳本呼叫 agy 的人這條最有感：print 模式（--print／-p，不開介面、印完答案就走的無頭用法）以前伺服器端失敗時會裝沒事——回傳成功代碼、輸出空白，害你的腳本以為任務完成了；現在錯誤會寫進 stderr（標準錯誤輸出，專門裝錯誤訊息的通道）並回傳非零代碼。另外在 shell 腳本或子程序裡跑 agy -p 會卡住等輸入的毛病也修了。就像外送員以前送失敗會默默把空袋子掛你門口，現在會確實打電話告訴你「餐沒送到」。

Original changelog
Fixed print mode (`--print` / `-p`) silently exiting with a success code and empty output when a request failed server-side, now writing the error to stderr and returning a non-zero exit code. Fixed `agy -p` hanging when run inside a shell script or subprocess by no longer reading stdin when a prompt is provided via a flag. (1.1.1)

放行過的寫檔不再重複問

你在 settings.json 的 permission.allow 允許清單（事先放行的白名單）裡核准過的寫檔動作，預設模式現在會直接放行，不再每次都跳審查。就像機場的常客快速通關——身家早就查核過的人，不必每趟都重新排安檢。

Original changelog
Changed the default mode to respect write_file permissions allowlisted in `settings.json` under `permission.allow`, so pre-approved file writes no longer prompt for review. (1.1.1)

介面細節總打磨：差異預覽、快捷鍵提示、表格對齊都變順眼

這輪順手把一圈介面細節磨亮了：檔案編輯的差異預覽補上前後文和分段線更好讀；各面板頁尾的快捷鍵提示會尊重你自訂的鍵位、不再寫死；/resume 選單裡重新命名對話的輸入框不再把長標題吃掉；聊天輸出裡帶檔案連結的 Markdown 表格也不再歪掉。像車子進廠保養——沒換引擎，但每個按鈕都變順手了。

Original changelog
Improved file-edit diff preview rendering: computed accurate line-level diffs with context lines (3 lines) and hunk separators. Improved UI footer keybinding hints across all panels by replacing hardcoded hint strings with centralized layout helpers that dynamically respect customized keybinding configurations. Improved the multiline conversation rename view in the `/resume` picker. Fixed misaligned markdown tables containing file links in chat output. (1.1.0–1.1.1)
