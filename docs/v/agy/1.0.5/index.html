Antigravity 1.0.5

zh-TW

權限管理更統一，還能在終端機裡直接修改

管理系統權限變得更輕鬆了。現在終端機會把「專案專屬權限」、「你和其他 Antigravity 系統共用的設定」以及「CLI 自己的設定檔」這三邊的權限規則聰明地合併在一起，讓你不用再到處找設定檔。而且我們加了全新的 /permissions 指令，讓你直接在終端機裡面就能新增、修改或刪除這些權限規則，超級方便。在啟動軟體時，你現在也能直接加上 --model 參數來指定你想用的 AI 模型，甚至可以用新的 models 指令來查看有哪些模型可以選。就像大樓的保全系統，以前你要分別去三個不同的櫃檯設定門禁卡，現在全部連線同步了，而且你只要在大廳的觸控螢幕上點兩下，就能直接幫訪客開門。

Original changelog
Added `--model` to set model when launching CLI. Also a new `models` subcommand to list available models.
Added `/permissions` command which allows to add/edit/remove permissions rules for each of the three configs above directly inside the CLI.
Integrated the permissioning system with the rest of Antigravity. CLI permissions now merges project level permissions, permissions from user settings shared with Antigravity, and permissions from the CLI `settings.json`.

審閱程式碼不卡關，畫面排版與補齊更貼心

現在你的操作流程會順暢很多。以前如果系統問你要不要放行某個指令，或者正等著你回答問題時，你就沒辦法點開「產出審閱（Artifact Review）」面板去看程式碼；現在你隨時可以按 ctrl+r 切過去看，看完切回來時原本的進度都還在，不用被迫先回答盲猜。畫面底下的狀態列也重新排版了，提示訊息和產出狀態會併成一行顯示，就算你的視窗拉得很窄，文字也會優雅地用「...」省略，不會再擠成一團破圖。在使用快捷指令時，如果你打了縮寫（例如 /se），按 Tab 鍵補齊時它會乖乖補成你想要的 /settings，不會再雞婆地硬幫你換成它背後的本名 /config。就像你在填問卷填到一半時，以前不能離開座位去翻書確認答案，現在你可以隨時去查資料，回來時寫到一半的字都還在，而且服務生也不會再亂糾正你的發音了。

Original changelog
Allowed opening the Artifact Review panel (shortcut `ctrl+r`) while answering pending questions or tool permission confirmations, preserving your current progress when toggling back.
Improved statusline layout by merging active tip and artifact status on a single line and truncating with ellipsis on narrow terminals to prevent collisions.
Improved autocomplete: tab completion for slash commands now resolves to the matched alias instead of the primary command name (e.g., `/se` autocompletes to `/settings` instead of `/config`).

歷史紀錄支援資料庫，快取與外掛工具設定修復

/resume 不再顯示空對話、也能掃描 SQLite 資料庫。打開 /resume 找歷史紀錄時，現在不會再跑出一堆沒有內容的空對話來擾亂你，載入速度也變快了，甚至還學會了掃描 SQLite 資料庫檔（.db），讓你找資料更全面。如果你用純文字的無頭模式（-p）執行，系統現在會乖乖把暫存資料（快取）寫進專用的 ~/.gemini/antigravity-cli/cache 資料夾裡，不會再隨地大小便，弄髒你當下的工作目錄。在設定外部 MCP 伺服器工具時，現在可以直接在設定檔裡填寫 url 來連線，不用再繞遠路。如果你有自訂工具資料夾，現在它們也能被當成正常的工作區來對待了。就像買了更聰明的吸塵器，以前它會把空袋子也當成垃圾收集起來，還會把吸出來的灰塵隨地亂倒，現在它不但能精準過濾空袋子，還能直接接上網路管線，把灰塵準確地倒進專屬的垃圾桶裡。

Original changelog
Fixed a bug that metadata was written in the current directory as opposed to `~/.gemini/antigravity-cli/cache` when running using `-p`.
Improved `/resume` performance: optimized lazy loading of conversation details, filtered out empty conversations, and added support for scanning SQLite database files (`.db` and `.db-wal`).
Improved customization support by allowing directories in the customization manager to be passed as workspace directories, enabling correct trajectory metadata population and `/add-dir` support.
Added support for `url` in `mcp_config.json` to configure MCP servers directly via a URL.

en

Unified permissions management with in-app editing and model launch flags

Managing system permissions is now a breeze. The terminal intelligently merges your "project-specific permissions," "settings shared with the rest of Antigravity," and the "CLI's own config file" into one unified rule set, saving you from hunting down various config files. We've also added a brand-new /permissions command, allowing you to easily add, edit, or remove these rules directly inside the terminal. When launching the software, you can now instantly select your preferred AI model by adding the --model flag, or use the new models subcommand to see a list of all available options. Like a building's security system: you used to have to visit three different desks to set up an access card, but now everything is synchronized, and you can just tap the lobby's touchscreen a few times to let a guest in.

Original changelog
Added `--model` to set model when launching CLI. Also a new `models` subcommand to list available models.
Added `/permissions` command which allows to add/edit/remove permissions rules for each of the three configs above directly inside the CLI.
Integrated the permissioning system with the rest of Antigravity. CLI permissions now merges project level permissions, permissions from user settings shared with Antigravity, and permissions from the CLI `settings.json`.

Uninterrupted code review, cleaner status lines, and smarter autocomplete

Your workflow is about to become much smoother. Previously, if the system asked you to approve a command or was waiting for you to answer a question, you were blocked from opening the "Artifact Review" panel to look at the code; now, you can press ctrl+r at any time to switch over and check, and when you switch back, your progress is perfectly preserved so you never have to blindly guess. The status line at the bottom of the screen has also been redesigned: active tips and artifact statuses now merge into a single line, and if your window is narrow, the text gracefully truncates with an ellipsis (...) instead of crushing the layout into a mess. When using slash commands, if you type an abbreviation (like /se), hitting Tab will obediently autocomplete it to the alias you wanted (/settings), rather than obnoxiously replacing it with its underlying primary name (/config). Like filling out a questionnaire: you used to be forbidden from leaving your seat to double-check a book for the answer, but now you can get up to do research anytime, and when you return, your half-written answers are exactly where you left them—and the waiter will no longer rudely "correct" your pronunciation.

Original changelog
Allowed opening the Artifact Review panel (shortcut `ctrl+r`) while answering pending questions or tool permission confirmations, preserving your current progress when toggling back.
Improved statusline layout by merging active tip and artifact status on a single line and truncating with ellipsis on narrow terminals to prevent collisions.
Improved autocomplete: tab completion for slash commands now resolves to the matched alias instead of the primary command name (e.g., `/se` autocompletes to `/settings` instead of `/config`).

SQLite support for history, fixed cache locations, and direct MCP URLs

We've exorcised a few details where the devil was hiding. When opening /resume to browse your history, the list now filters out entirely empty conversations that used to clutter the view, loads details much faster, and even learned how to scan SQLite database files (.db and .db-wal) to give you a more comprehensive view. If you run the tool in text-only headless mode (-p), it now obediently writes its temporary cache data into the dedicated ~/.gemini/antigravity-cli/cache folder instead of treating your current working directory like a litter box. When configuring external MCP server tools, you can now connect directly by placing a URL in the config file, skipping unnecessary detours. Finally, if you manage custom tool directories, they can now properly be treated as functional workspace directories. Like buying a smarter vacuum cleaner: it used to collect empty wrappers while dumping the dust it sucked up all over your living room rug; now, it filters out the useless wrappers perfectly, plugs directly into your network pipe, and dumps the dirt precisely into its designated trash bin.

Original changelog
Fixed a bug that metadata was written in the current directory as opposed to `~/.gemini/antigravity-cli/cache` when running using `-p`.
Improved `/resume` performance: optimized lazy loading of conversation details, filtered out empty conversations, and added support for scanning SQLite database files (`.db` and `.db-wal`).
Improved customization support by allowing directories in the customization manager to be passed as workspace directories, enabling correct trajectory metadata population and `/add-dir` support.
Added support for `url` in `mcp_config.json` to configure MCP servers directly via a URL.
