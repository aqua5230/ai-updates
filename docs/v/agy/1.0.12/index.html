Antigravity 1.0.12

發布日期：2026-07-13

zh-TW

預覽與留言更順手，連結能點且不怕誤刪

現在在終端機裡審閱差異與留註解變得更安全也更靈活了。以前你在註解模式打字打到一半，如果不小心按到 Esc，千辛萬苦打好的字就會瞬間灰飛煙滅；現在它會貼心跳出確認對話框攔住你。看程式碼差異時，如果你不小心按下一筆跳過了想看的地方，現在可以按 shift+n 直接「倒車」回去看上一筆。此外，現在終端機裡的網址連結真的可以點擊了，按住熱鍵點一下就能直接在瀏覽器打開；清空歷史對話畫面的快捷鍵（ctrl+o）也修好重新復活，不會再失靈。就像看實體報紙變成看電子書一樣，現在不但可以在文章上點連結，不小心翻頁還能一鍵退回，而且寫在旁邊的便條紙也不會因為風吹就飛走。

Original changelog
Added a confirmation prompt when pressing `Esc` in comment mode with unsaved modifications to prevent users from accidentally discarding their work in review views.
Introduced reverse diff cycling navigation mapped to `shift+n` in unified diff review mode to allow users to easily cycle backwards through diff blocks.
Added dynamic OSC8 terminal hyperlink support to render clickable links in supporting terminals, with automatic fallback stripping for backward compatibility.
Fixed a regression where `ctrl+o` scrollback clearing failed by restoring the use of cached fields rather than shared pointer comparisons for trajectory toggle detection.
Fixed incorrect key strings by removing the unsupported backtab default binding and correcting invalid `pgdn` references to `pgdown` to align with Bubble Tea v2 canonical names.

啟動專案更直覺，設定檔優先順序與企業連線不再雷人

你可以更直接地管理並啟動不同專案了。現在只要在啟動時加上 `--project` 或 `--new-project`，就能直接指定或建立專案，不用管你現在人在哪個資料夾。設定檔的優先順序也變聰明了，專案專屬的權限設定現在會確實蓋過全域設定，不會再發生「明明專案設定了卻不理我」的窘境。如果你在公司裡使用，以前因為企業防火牆的深度封包檢測（DPI）會誤擋系統的加密連線，導致你一直斷線；現在系統找回了最佳化設定，連線能順利穿透防火牆了。另外，系統也不會再自作聰明把你寫在程式碼區塊裡的 Makefile 語法當成數學公式亂改一通了。就像去不同的分公司上班，現在你只要報上分公司名字就能直接上工，當地的員工守則會確實蓋過總公司的規定，而且公司的門禁系統也不會再把你的正常公事包誤認為危險物品而被警衛攔下。

Original changelog
Added support for `--project` and `--new-project` launch flags to allow users to explicitly set or create projects, and updated the project resolution logic to default regardless of the active workspace.
Improved permission config merging priorities by ensuring project-specific configurations (located in `~/.gemini/config/projects/`) take precedence over global settings in `~/.gemini/antigravity-cli/settings.json`.
Fixed an enterprise network connectivity issue by restoring AES-NI compile-time optimizations, which prevents Deep Packet Inspection (DPI) firewalls from incorrectly flagging and resetting TLS connections.
Fixed a rendering bug where Makefile syntax (like `$(call ...)`) inside code blocks was mistakenly parsed and mangled by LaTeX math expansion, by introducing a state machine that restricts expansion to prose segments.

en

Smoother review mode, clickable links, and safeguarded comments

Reviewing diffs and leaving comments in the terminal is now safer and more flexible. Previously, if you accidentally hit Esc while halfway through typing a comment, your hard work would instantly vanish into thin air; now, a thoughtful confirmation dialog pops up to stop you. When reviewing code changes, if you accidentally skip past a section you wanted to see, you can now press shift+n to instantly "reverse" back to the previous diff block. Furthermore, web links in the terminal are now genuinely clickable—just hold the modifier key and click to open them straight in your browser. The shortcut for clearing the conversation history screen (ctrl+o) has also been resurrected and fixed. Like upgrading from a physical newspaper to an e-book: you can now tap links on the page, instantly flip back if you turn a page by mistake, and your sticky notes won't blow away in the wind.

Intuitive project launching, smarter config priorities, and reliable enterprise connections

Managing and launching different projects is now much more straightforward. By simply adding the `--project` or `--new-project` flags at startup, you can directly specify or create a project without worrying about which folder you are currently in. Config priorities are also smarter now: project-specific permission settings will reliably override global settings, ending the frustration of "I set it for this project but it's ignoring me." If you use the tool at a corporate office, Deep Packet Inspection (DPI) firewalls used to falsely block the system's encrypted connections causing endless disconnects; optimizations have now been restored so traffic passes smoothly through the firewall. Additionally, the system will no longer foolishly misinterpret your Makefile syntax inside code blocks as math formulas and mangle them. Like reporting to work at different branch offices: you can now just state the branch name to get straight to work, the local branch rules will properly override corporate headquarters' policies, and the building's security scanner will no longer mistake your everyday briefcase for contraband and block you at the door.

미리보기와 메모 작성이 더 편해지고, 링크를 누를 수 있으며 실수로 지워지는 일도 없다.

이제 터미널에서 차이점을 살펴보고 메모를 남기는 일이 더 안전하고 자유로워졌다. 예전엔 메모 모드에서 글을 치다 실수로 Esc를 누르면 기껏 쳐둔 글이 순식간에 날아갔지만, 이제는 확인 창이 떠서 막아준다. 코드 차이를 볼 때, 실수로 '다음'을 눌러 보고 싶던 부분을 지나쳐 버렸더라도 이제 shift+n을 눌러 바로 이전 부분으로 '후진'할 수 있다. 게다가 이제 터미널 안의 인터넷 주소도 정말로 누를 수 있게 되어, 단축키를 누른 채 클릭하면 바로 브라우저가 열린다. 이전 대화 화면을 싹 지워주는 단축키(ctrl+o)도 고쳐져 다시 제대로 돌아간다. 종이 신문을 보다가 전자책으로 바꾼 것과 같다. 이제 기사에서 링크를 누를 수 있고, 실수로 페이지를 넘겨도 버튼 한 번에 되돌릴 수 있으며, 옆에 붙여둔 포스트잇이 바람에 날아갈 일도 없어진 셈이다.

프로젝트 시작이 더 직관적이고, 설정 파일 우선순위와 기업 망 연결 문제가 해결되었다.

이제 다른 프로젝트를 관리하고 시작하는 일이 훨씬 직접적이고 쉬워졌다. 프로그램을 켤 때 `--project`나 `--new-project`를 붙이기만 하면, 지금 어느 폴더에 있든 상관없이 바로 프로젝트를 지정하거나 새로 만들 수 있다. 설정 파일의 우선순위도 똑똑해져서, 이제 프로젝트 전용 권한 설정이 전체 설정보다 확실하게 앞서 적용되므로 '분명히 프로젝트 설정을 바꿨는데 왜 말을 안 듣지' 하는 답답함이 사라졌다. 회사에서 쓸 때 예전엔 기업 방화벽의 심층 패킷 분석(DPI)이 시스템의 암호화된 통신을 오해해 자꾸 연결을 끊었지만, 이제 최적화 설정을 되찾아 방화벽을 무사히 통과한다. 또한, 코드 블록 안에 적어둔 Makefile 문법을 수학 공식으로 착각해 제멋대로 망가뜨리는 일도 더는 없다. 다른 지사에 출근할 때, 이제 지사 이름만 대면 바로 일을 시작할 수 있고 그곳의 지사 규칙이 본사 규정을 확실히 덮어쓰며, 건물 검색대에서 평범한 서류 가방을 위험한 물건으로 오해해 경비원에게 막히는 일이 없어진 것과 같다.
