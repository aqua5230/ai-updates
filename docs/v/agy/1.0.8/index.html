Antigravity 1.0.8

發布日期：2026-07-13

zh-TW

「模型與額度」頁面重新設計

終端機介面變得更實用也更美觀了。現在你可以像一般打指令一樣，按方向鍵「上」來找回之前打過的快捷指令（slash commands，像 /plan 這些），不用每次都重打，而且在選單裡打字時，也不會再發生「打 /conv 結果系統硬是幫你補全成 /conv-switch 並關掉選單」這種自作聰明的毛病。「模型與額度」頁面重新設計，畫面最底下的狀態列也會隨時告訴你現在用了多少額度和處於什麼執行模式。當你在看歷史對話或任務清單時，如果你的視窗太窄，系統會自動隱藏不重要的欄位以免版面擠爆，而且時間和狀態也會對齊得清清楚楚。就像你常去的咖啡廳重新裝潢，不但點單機終於會記住你常點的隱藏菜單，而且就算用小桌子，服務生也會幫你把餐盤排得整整齊齊。

Original changelog
Added support for capturing slash command history, allowing users to use the up arrow to replay previously entered slash commands.
Redesigned the "Models & Quota" page (enabled by default, replacing the legacy usage page) to gracefully handle disabled quota buckets by displaying a dimmed "Disabled" status and omitting the progress bar.
Added display of quota usage and execution mode in the status line.
Redesigned the `/resume` conversation picker to align the workspace column and added adaptive column dropping (workspace, time, steps) to support narrow terminals.
Redesigned the `/tasks` list and detail views for better alignment and readability, placing start times on the left, right-aligning status, and capping the panel height.
Fixed an autocomplete bug where a command that is an exact prefix of another (e.g., `/conv` vs `/conv-switch`) would aggressively auto-complete and hide the suggestions menu.

解決貼上大量文字與舊電腦當機的問題

大量單行貼上不再拖慢輸入框。以前如果你一次貼上超級長（例如好幾萬字）的一行文字，輸入框會因為來不及處理而卡住很久；現在它會聰明地把它折疊成一個小圖示，不再拖垮效能。用 /btw 更省 token，並支援串流回覆，也改用一邊生成一邊印字的方式，不再斷尾。如果你用的是很舊的電腦（沒有支援硬體加速的 CPU），以前一開程式就會直接崩潰，現在也能正常打開了。另外，Windows 與部分 Linux 系統（Wayland）的剪貼簿讀取也修好了，而且系統現在會檢查剪貼簿裡的檔案大小，防止你貼上超大檔案時把記憶體整個吃光。就像換了更強的抽水馬達，以前如果一次倒太多水進去水管就會塞住甚至爆管，現在它會先幫你打包分流，連舊房子的老水管也能撐得住，而且還順便把幾個地方漏水的問題修好了。

Original changelog
Improved `/btw` to be more token efficient and support streaming responses for a smoother user experience and fixed premature truncation.
Fixed a CPU compatibility issue (SIGILL on non-AES-NI CPUs), preventing immediate crashes on startup on older CPUs (like Intel Ivy Bridge) or VM environments that lack AES-NI support.
Added a per-line guard against extremely long single-line pastes in the TUI prompt editor to prevent performance lag, replacing them with an expandable placeholder.
Fixed a TUI hang in the artifact view during long sessions by optimizing the rendering complexity of large step histories.
Fixed potential OOM risks when reading large clipboard files by verifying file size before reading.
Fixed Windows and Wayland-only Linux distributions clipboard image and file reading.

設定檔終於寫對位置，拒絕權限後不再卡死

系統裡的設定與狀態同步現在變得更精準了。以前你用 /hooks 幫系統掛上自動執行腳本時，它會傻傻地把設定檔寫錯資料夾，導致你設定半天背後的程式卻讀不到；現在終於會乖乖寫進共用的設定檔目錄裡了。當你切換專案或加入新目錄時，你自己寫的快捷指令和技能也會立刻在自動補齊選單裡現身，不用再重開程式。如果你儲存設定失敗，狀態列現在會立刻閃紅燈警告你。另外，以前如果你拒絕了代理的某個執行權限，然後馬上又傳訊息給它，它會因為還沒從被拒絕的錯愕中回過神來而當機；現在這個「反應慢半拍」的毛病也治好了。就像你去辦公所，以前你在櫃檯填的申請單會被收進錯的檔案櫃，而且就算你剛加入新部門的通訊錄，也要等隔天大家才查得到你；現在這些資料都即時連線了，而且如果你剛拒絕推銷員，他也不會再擋在門口不讓你跟下一個客人講話。

Original changelog
Fixed a bug where the `/hooks` command wrote configurations to `~/.gemini/antigravity-cli/hooks.json` instead of the shared `~/.gemini/config/hooks.json`, ensuring hooks remain synchronized between the TUI and the backend.
Fixed dynamic reloading of custom skills and system slash commands, ensuring they are instantly discovered in autocomplete upon conversation switch or `/add-dir`.
Improved configuration saving by propagating write failures as transient error flashes on the statusline.
Improved settings inheritance by ensuring the CLI inherits the `use_ai_credits` setting from global user settings on startup.
Fixed a race condition where sending a message immediately after denying a permission request would fail due to incomplete backend cleanup.

Like your favorite cafe getting a remodel: the register finally remembers your secret menu orders, and even if you sit at a tiny table, the waiter still manages to arrange your plates perfectly.

A slew of performance issues that could freeze or crash the program have been resolved. Previously, if you pasted a ridiculously long single line of text (like tens of thousands of characters), the input box would choke and freeze for ages; now, it smartly collapses it into a little placeholder icon, keeping performance smooth. When you use /btw to ask the agent for a quick favor, its replies are now faster and stream in word by word instead of getting prematurely truncated. If you're on a very old computer (with a CPU that lacks hardware acceleration), the program used to instantly crash on startup, but now it opens perfectly. In addition, clipboard reading on Windows and certain Linux setups (Wayland) has been fixed, and the system now checks the file size before reading the clipboard so pasting massive files won't devour all your RAM. Like installing a much stronger water pump: previously, dumping too much water at once would clog or burst the pipes, but now it automatically bundles and reroutes the flow so even the old plumbing in an aging house can handle it, while fixing a few leaks along the way.

When you switch projects or add a new directory, any custom slash commands or skills you wrote will instantly appear in the autocomplete menu without you needing to restart the program. If saving a configuration fails, the status bar will now immediately flash a red warning alert. Additionally, if you denied an agent's permission request and then immediately sent it a new message, the backend used to hang because it hadn't finished cleaning up after the denial; this "slow reaction" bug is now cured. Like visiting a government office: the application forms you filled out at the counter used to be filed in the wrong cabinet, and if you just joined a new department, it took a whole day before anyone could find you in the directory; now everything is instantly synced, and if you turn away a salesperson, they no longer block the doorway preventing you from talking to the next person.

엄청나게 긴 글을 붙여넣거나 오래된 컴퓨터에서 멈추는 문제, 클립보드 오류 해결

프로그램을 멈추거나 튕기게 만들던 성능 문제를 한꺼번에 해결했다. 예전엔 수만 글자에 달하는 엄청나게 긴 글을 한 줄로 통째로 붙여넣으면, 입력 칸이 처리하지 못하고 오랫동안 멈춰 있었지만 이제는 똑똑하게 작은 아이콘 하나로 접어두어 성능을 깎아먹지 않는다. /btw를 써서 에이전트에게 간단한 일을 부탁할 때, 답변 속도가 빨라졌고 도중에 잘리는 일 없이 글자가 만들어지는 대로 바로바로 찍어준다. 하드웨어 가속을 지원하지 않는 아주 오래된 컴퓨터(CPU)를 쓰는 경우, 예전엔 프로그램을 켜자마자 뻗어버렸지만 이제는 무사히 켜진다. 아울러 Windows와 일부 Linux(Wayland)에서 클립보드를 제대로 못 읽던 문제도 고쳐졌고, 이제 시스템이 클립보드에 든 파일 크기를 먼저 확인하므로 엄청나게 큰 파일을 붙여넣어도 램(메모리)을 몽땅 잡아먹는 일이 없다. 훨씬 힘 좋은 물 펌프로 바꾼 것과 같다. 예전엔 한꺼번에 물을 너무 많이 부으면 배관이 막히거나 터졌지만, 이제는 알아서 물줄기를 나눠주어 낡은 집의 오래된 배관도 버틸 수 있게 되었고, 내친김에 곳곳의 물새는 곳도 싹 고친 셈이다.

설정 파일이 제자리에 저장되고, 권한을 거절한 뒤 멈추는 현상이 사라졌다.

시스템 안의 설정과 상태를 맞추는 일이 훨씬 정확해졌다. 예전엔 /hooks를 써서 자동 실행 스크립트를 걸어두면, 시스템이 멍청하게 설정 파일을 엉뚱한 폴더에 쓰는 바람에 기껏 설정해 두고도 뒤에서 도는 프로그램이 이를 읽지 못했지만 이제는 드디어 공용 설정 폴더에 제대로 저장된다. 프로젝트를 바꾸거나 새 폴더를 더할 때도, 당신이 직접 짠 단축 명령어와 기술이 프로그램을 다시 켤 필요 없이 자동 완성 메뉴에 바로 나타난다. 설정을 저장하다 실패하면 상태 줄이 즉시 빨간불을 반짝이며 경고해 준다. 또한 예전엔 에이전트가 권한을 달라는 걸 거절한 직후에 바로 새 메시지를 보내면, 뒤에서 거절 처리를 다 끝내기도 전에 꼬여버려 멈추곤 했는데, 이제 이 '굼뜬 반응' 문제도 고쳐졌다. 관공서에 갔을 때, 예전엔 창구에서 쓴 신청서가 엉뚱한 캐비닛에 들어가 버리고 새 부서로 옮겨도 다음 날이 되어야 연락망에서 나를 찾을 수 있었지만 이제는 이 모든 게 실시간으로 바로 연결되며, 물건 파는 사람을 거절했다고 해서 그 사람이 문 앞을 틀어막고 다음 사람과 말하는 걸 방해하는 일도 없어진 것과 같다.
