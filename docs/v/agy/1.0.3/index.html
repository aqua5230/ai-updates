zh-TW

額度用完不用怕，全新 G1 點數支援上線

當你的標準額度用光時，不用再苦苦等待了。系統現在正式支援「G1 點數（G1 credits）」，並推出全新的 UseG1Credits 設定，只要開啟它，當標準額度耗盡時，系統就會自動扣點數讓你繼續使用。你可以在畫面底下的狀態列即時看到還剩多少點數，也可以用全新的 /credits 指令打開點數面板，甚至能直接在裡面點擊連結購買充值。就像你手機的網路流量用完時，以前只能被迫斷網等下個月，現在系統會自動幫你用預付卡扣款繼續飆網，而且隨時能查餘額跟一鍵儲值。

Original changelog
Added support for G1 credits in the Antigravity CLI. Users can now utilize G1 credits when their standard quota runs out. This includes a new `UseG1Credits` setting to enable automatic credit usage and a real-time display of remaining credits in the status bar.
Added a new `/credits` panel that provides an in-CLI interface with a direct link to purchase additional G1 credits.

畫面排版更精緻，多題問卷不再忘記答案

長程式碼差異會依視窗寬度換行。如果你是用蘋果的終端機（Apple Terminal），現在會看到全新設計的標誌。在挑選或設定程式的主題顏色時，預覽畫面變得更真實了，會把「警告訊息」和「思考過程」的顏色也秀給你參考，網址底線也不會再亂畫。當你在看程式碼差異（/diff）時，如果文字太長，現在它會聰明地根據你的視窗大小自動換行，不會再被卡掉一半。當系統連環問你問題時（例如設定精靈），如果你按左鍵想回到上一題修改，以前它會把你剛剛辛辛苦苦打的答案跟選項忘得一乾二淨，現在它終於學會記住你的作答進度了。就像你在填線上表單，以前按了「上一頁」填好的資料就會全部消失重來，現在系統會貼心地幫你把寫到一半的字都保留著。

Original changelog
Redesign CLI logo on Apple Terminal.
Improved color scheme preview in settings and onboarding: added warnings and thought process examples to the preview, and corrected link styling to only underline the URL.
Fixed `/diff` detail view truncation: implemented dynamic line wrapping based on terminal viewport width and added automatic tab-to-space expansion to prevent layout overflow.
Fixed `AskQuestion` state management: memorizes selected options, write-in values, and UI states when navigating back and forth (`KeyLeft`) between questions in multi-question dialogs.

解決卡死、編輯器開不了與專案找不到的怪病

輸入框開頭按左移不再無限卡死。以前在輸入框打字時，如果你在最開頭的空白處按熱鍵往左跳，程式就會陷入死胡同無限當機，現在已經修好了。在終端機介面裡想關掉某個外部 MCP 工具時，以前按了「關閉」卻因為系統寫錯設定檔位置而關不掉，現在終於能確實把工具卸載了。如果你有自訂文字編輯器（像是在 $EDITOR 裡寫 --alternate-editor=vi），以前系統會因為不懂得處理等號而導致編輯器打不開，現在它看得懂了。另外，系統在尋找你的專案資料夾時，如果遇到壞掉的捷徑（symlink），以前會直接當機放棄尋找，現在它懂得自動跳過壞掉的捷徑，繼續找出其他正常的專案。就像你去巡房，以前只要遇到一扇壞掉打不開的門，你就會直接下班回家，現在你懂得繞過那扇門去檢查其他的房間了。

Original changelog
Fixed an infinite loop in the prompt input. Navigating left (`wordLeft`) when encountering spaces at the very beginning of the input no longer causes an infinite hang.
Fixed custom MCP server disabling via the TUI. Resolved a directory path mismatch where pressing the `[Disable]` button wrote to the legacy `mcp_config.json` path instead of the migrated `config/mcp_config.json` path, ensuring custom MCP servers can now be successfully disabled and unloaded.
Fixed `$EDITOR` environment variable parsing: resolved issues where arguments containing `=` (e.g., `--alternate-editor=vi`) were incorrectly split, causing editor launch failures.
Fixed project discovery robustness: updated the CLI to skip invalid or broken symlinks in `.antigravitycli/` rather than failing immediately, allowing discovery of valid projects.

en

Never run out of quota: G1 credits support is now live

You no longer have to wait agonizingly when your standard quota runs out. The system now officially supports "G1 credits" and introduces a new UseG1Credits setting; when enabled, it seamlessly starts using your credits the moment your standard quota is depleted. You can monitor your remaining credits in real-time right on the bottom status bar, use the brand-new /credits command to open a dedicated panel, and even click a direct link within it to purchase more. Like running out of your phone's monthly data plan: you used to get cut off from the internet and forced to wait for the next month, but now the system automatically taps into a prepaid card to keep you surfing at full speed, letting you check your balance and top up with a single click.

Original changelog
Added support for G1 credits in the Antigravity CLI. Users can now utilize G1 credits when their standard quota runs out. This includes a new `UseG1Credits` setting to enable automatic credit usage and a real-time display of remaining credits in the status bar.
Added a new `/credits` panel that provides an in-CLI interface with a direct link to purchase additional G1 credits.

Refined layouts and UI dialogs that remember your answers

We've made several thoughtful tweaks to the visual layout. If you use Apple Terminal, you'll now be greeted by a newly redesigned logo. When previewing or setting the application's color theme, the preview is much more realistic, now including samples for "warnings" and "thought processes," and fixing an issue where URL underlines were drawn haphazardly. When viewing code differences (/diff), if a line is too long, it now smartly wraps text based on your window size instead of cutting it off completely. When the system asks you a sequence of questions (like in a setup wizard) and you press the left arrow to go back and change a previous answer, it used to completely suffer from amnesia and wipe out everything you had just painstakingly typed; it finally knows how to remember your progress. Like filling out an online form: hitting "back" used to delete all your entered data and force you to start over, but now the system thoughtfully preserves everything you've written so far.

Original changelog
Redesign CLI logo on Apple Terminal.
Improved color scheme preview in settings and onboarding: added warnings and thought process examples to the preview, and corrected link styling to only underline the URL.
Fixed `/diff` detail view truncation: implemented dynamic line wrapping based on terminal viewport width and added automatic tab-to-space expansion to prevent layout overflow.
Fixed `AskQuestion` state management: memorizes selected options, write-in values, and UI states when navigating back and forth (`KeyLeft`) between questions in multi-question dialogs.

Fixes input hangs, editor launch failures, and fragile project discovery

We've eradicated several highly confusing system bugs. Previously, when typing in the input box, if you hit the shortcut to jump left while at the very beginning of the blank space, the program would get trapped in an infinite loop and completely hang; this has been fixed. When trying to disable an external MCP tool from the terminal UI, clicking "Disable" used to do absolutely nothing because the system was writing to the wrong config file path; it now correctly unloads the tool. If you use a custom text editor with complex arguments (like --alternate-editor=vi in your $EDITOR variable), the system used to crash because it didn't know how to handle the equals sign; it parses it perfectly now. Additionally, when searching for your project folders, encountering a broken shortcut (symlink) used to make the system throw a tantrum and completely give up searching; it now smartly skips the broken links and continues finding your valid projects. Like a night guard doing rounds: previously, if one door was jammed, the guard would just go home and abandon the rest of the patrol, but now they know how to walk past the broken door and check the remaining rooms.

Original changelog
Fixed an infinite loop in the prompt input. Navigating left (`wordLeft`) when encountering spaces at the very beginning of the input no longer causes an infinite hang.
Fixed custom MCP server disabling via the TUI. Resolved a directory path mismatch where pressing the `[Disable]` button wrote to the legacy `mcp_config.json` path instead of the migrated `config/mcp_config.json` path, ensuring custom MCP servers can now be successfully disabled and unloaded.
Fixed `$EDITOR` environment variable parsing: resolved issues where arguments containing `=` (e.g., `--alternate-editor=vi`) were incorrectly split, causing editor launch failures.
Fixed project discovery robustness: updated the CLI to skip invalid or broken symlinks in `.antigravitycli/` rather than failing immediately, allowing discovery of valid projects.

입력 창 멈춤, 편집기 실행 실패, 프로젝트를 찾다 포기하는 버그 해결

사람 머리에 물음표가 둥둥 뜨게 만들던 이상한 버그들을 잡았다. 예전엔 입력 칸에 글자를 칠 때 가장 맨 앞 빈칸에서 왼쪽으로 건너뛰는 단축키를 누르면, 프로그램이 뫼비우스의 띠에 갇힌 듯 무한히 멈춰버렸지만 이제는 말끔히 고쳐졌다. 터미널 창에서 외부 MCP 도구를 끄려고 '끄기(Disable)' 버튼을 눌러도 시스템이 엉뚱한 설정 파일에 글을 쓰는 바람에 꺼지지 않던 문제도 고쳐져, 이제는 도구를 확실히 치워버릴 수 있다. 만약 복잡한 설정이 달린 나만의 글 편집기를 쓴다면($EDITOR 변수에 --alternate-editor=vi처럼 적어둔 경우), 예전엔 시스템이 등호(=)를 처리할 줄 몰라 편집기를 열지 못했지만 이제는 제대로 알아보고 척척 열어준다. 또한, 시스템이 당신의 프로젝트 폴더를 찾아다닐 때 깨진 바로 가기(심볼릭 링크)를 만나면 아예 찾는 걸 포기하고 뻗어버렸지만, 이제는 고장 난 바로 가기를 알아서 건너뛰고 나머지 멀쩡한 프로젝트를 계속 찾아낸다. 건물 순찰을 도는 경비원과 같다. 예전엔 고장 나서 안 열리는 문을 하나만 만나도 바로 퇴근해버렸지만, 이제는 그 문을 지나쳐 다른 방들을 계속 확인하고 돌아다닐 줄 알게 된 셈이다.
