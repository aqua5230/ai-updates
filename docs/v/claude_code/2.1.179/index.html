Dropped connections stop spitting raw errors, the spinner stops sticking, and remote background tasks stop looking frozen

Connection hiccups no longer startle you or pretend to still be running. If the connection drops while Claude is mid-answer, you used to get a raw error (an unvarnished message meant for engineers) dumped in your face, and the spinner would stick at "running tool"; now the partial answer you'd already received is preserved instead of vanishing. In remote sessions (conversations running against another machine), background tasks used to keep reading "still running" between turns as if frozen; their status is fresh and accurate now. Remote-session plugin (add-ons that extend features) loading is faster too. Like watching a live TV broadcast that used to throw a garbled error screen the instant the signal cut, freeze its spinner, and leave the caption machine reading "now playing" after the show ended — now a cut keeps the last frame, the status stays honest, and channel changes are quicker.

Linux and WSL2 stop seizing up: scrolling works again, and huge folders no longer bloat the tool description to death

Two issues on Linux and WSL2 (the subsystem that runs Linux inside Windows) that could make an entire session unusable are fixed. Mouse-wheel scrolling in WSL2 under Windows Terminal and VS Code used to break (a regression in 2.1.172, regression meaning a change that accidentally broke something else); it works again. And on Linux, if you set a sandbox (the mechanism that traps a program inside a restricted zone) `denyRead`/`allowRead` rule as a glob (a pattern using wildcards to match many paths at once) over a huge directory tree, it used to bloat the Bash tool's description text to an enormous size and freeze the whole session; that's fixed. Like thumbing through a heavy yearbook whose index was scrambled — you couldn't turn the page, or one chapter's index suddenly ran to thousands of pages and burst the binding; now pages turn fine and the index is trim.

Ctrl+O shows a subagent's transcript again, and clicking the input box reclaims focus

Two panel operations for inspecting a subagent (an agent Claude splits off to handle small chores) are fixed. When you pressed Ctrl+O to view a subagent's transcript (the verbatim full conversation history), it used to show nothing; it displays properly now. And when focus (the target your keyboard input is currently aimed at) was sitting on the subagent or footer panel, clicking the input box used to fail to jump focus back — so your typing would go unresponsive mid-sentence; it returns to the input box properly now. Like opening a colleague's work logbook that used to come up blank, and calling them back to their desk only to find them still glued to the break room — now the log shows and they come back the moment you call.

Ctrl+O로 서브에이전트의 대화 기록(transcript, 한 글자도 빠짐없이 남긴 전체 대화 내역)을 보려 해도 예전엔 아무것도 뜨지 않았지만 이제는 정상적으로 보인다. 또 포커스(focus, 키보드 입력이 현재 향하는 대상)가 서브에이전트 패널이나 하단 정보줄에 머문 상태에서 입력창을 클릭하면, 예전엔 포커스가 입력창으로 돌아가지 않아 타이핑하다 말고 반응이 끊겼지만 이제는 입력창으로 제대로 돌아온다. 동료의 업무 수첩을 펼치면 예전엔 하얗게 비어 있고, 자리로 돌아오라 불러도 차를 마시는 곳에 들러붙어 있었지만 이제는 기록도 보이고 부르면 곧장 자리로 돌아오는 것과 같다.

설문이 한 글자 답을 평점으로 오해하지 않고, 환영 화면에 광고가 겹치지 않는다

당신의 입력을 오해하거나 너무 많은 것을 들이미는 두 가지 작은 성가심이 고쳐졌다. 턴이 끝난 직후 숫자 하나만 입력했을 때 세션 평점(rating, 1~5점식 만족도)으로 간주되는 일이 더 이상 발생하지 않는다. 환영 화면은 여러 홍보 배너(promotional banners, 새 기능을 파는 광고 띠)를 겹쳐 띄웠지만 이제는 세션당 최대 하나만 보여준다. 식사를 마친 뒤 웨이터가 무심코 뱉은 숫자 하나를 만족도 점수로 받아 적고, 출입구에는 전단 세 장이 겹쳐 시야를 가렸다면, 이제는 설문이 섣불리 추측하지 않고 출입구에도 포스터 한 장만 남는 것과 같다.
