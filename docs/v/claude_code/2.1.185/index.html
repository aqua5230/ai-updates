zh-TW

等待回覆時，提示更準也更不急

等 API（程式彼此交換資料的窗口）回覆時，畫面現在會先告訴你「正在等 API 回應，將在……後重試」，不再直接說沒有回應；而且要連續安靜 20 秒才跳出，原本 10 秒就會提醒。也就是說，短暫沒有回應時，不會那麼快跳出提示；真的久等時，仍會清楚告訴你下一步要重試。像客服暫時沒說話時，先告訴你正在查詢，而不是立刻說電話掉了。

en

Waiting for a reply now gets a calmer, clearer hint

When you are waiting for an API response, the hint now says "Waiting for API response · will retry in …" instead of declaring that there was no response. It also waits for 20 seconds of silence before appearing, rather than 10. So a brief silence does not produce the hint as quickly, while a genuinely long wait still tells you that a retry is coming. It is like a support agent saying they are checking something instead of immediately claiming the call was lost.

ja

応答待ちの案内が、より正確で慌てない表示になった

API（プログラム同士がデータをやり取りする窓口）の応答を待つとき、表示は「API からの応答を待機中・……後に再試行」と案内するようになり、いきなり応答なしとは言わなくなった。表示が出るのも無音が20秒続いてからで、従来の10秒より待つ。短い無応答では警告がすぐ出にくくなり、本当に長く待つ場合は再試行することが分かる。電話口の担当者が、切れたと言う前に確認中だと伝えるようなものだ。
