zh-TW

權限守門更嚴，危險指令不再鑽漏洞

你叫 Claude 改檔、跑終端機時，最怕它把看似無害的指令當成安全通行證。這次把權限檢查補成像大樓門禁：dir/** 不會再誤認整棟樓任何同名資料夾都能寫入；PowerShell、Bash 的重新導向、超長指令、zsh 比較式；help、man 的危險用法，以及使用 -m／--magic-file 或 -f／--files-from 的 file 指令，都會在有風險時停下來問你。Docker 或 Podman 若改連別台 daemon（背景服務程式）也得先確認。遠端工作階段更不會在你本機按同意前搶跑。原本該只限目前資料夾的 hook 規則，也不再意外套到更深處。

Original changelog
Fixed single-segment `dir/**` allow rules like `Edit(src/**)` auto-approving writes to nested `dir/` directories anywhere in the tree instead of only `<cwd>/dir`
Fixed a permission-check bypass affecting commands run in Windows PowerShell 5.1 sessions
Fixed Bash permission checks to fail closed on file-descriptor redirect forms that bash parses differently than the permission analyzer
Fixed Bash permission checks misjudging very long commands — commands over 10,000 characters now always prompt instead of running automatically
Fixed Bash permission checks treating zsh variable subscripts and modifiers in `[[ ]]` comparisons as inert text — these commands now prompt for approval
Fixed Bash permission checks to no longer auto-approve certain `help` and `man` commands that could run unsafe options, command substitutions, or backslash paths
Fixed permission prompts on remote sessions that could proceed before the local confirmation dialog
Added permission prompts for `docker` commands (including the Podman `docker` shim) carrying daemon-redirect flags (`--url`, `--connection`, `--identity`, and Podman's remote mode) that previously ran without one
Changed single-segment `dir/**` hook `if:` conditions to match only `<cwd>/dir`; write `**/dir/**` for any-depth matching. `deny`/`ask` permission rules keep their any-depth match.
Changed `file` commands using `-m`/`--magic-file` or `-f`/`--files-from` to require permission instead of being auto-allowed as read-only

對話能止損，長工也不再悄悄失聯

遇到持續辱罵或越獄（jailbreak，誘導 AI 逃避安全規則）的對話，Claude 現在可以主動結束工作階段，不必被困在拉扯裡。另一頭，長時間工具呼叫會定期送出進度心跳，讓你知道它還活著。幾個容易讓工作半途斷線的問題也補好了：feature flag（功能開關）的空值或壞資料不再讓程式崩潰或清空快取；設定檔指到裝置檔可能造成記憶體持續增加；超過 2 MiB 的設定檔則會在啟動時清楚拒絕；公司 proxy（代理伺服器）、慢速 SDK 管線、排程任務、Linux 的 pkill -f，以及 shell 設定路徑誤填成資料夾，都不再把工作悄悄弄壞。顧問思考時也不會誤跳網路警告，從家目錄執行 claude rc 則會說清楚信任設定不會被保存。像客服櫃台能請搗亂者離開，後台也會定時亮燈報平安。

追查一段工作，記錄終於接得起來

當你想查「這則訊息到底叫了哪個工具、花了多少資源」，紀錄不該像散落的收據。memory 檔案（讓 Claude 留下跨工作階段筆記的檔案）現在有 ISO modified 時間戳；OpenTelemetry（把程式執行紀錄送進監控工具的標準）則替每則訊息加上 UUID、請求編號和工具來源，也可設定內容欄位原本 60 KB 的截斷上限。自訂 agent 列表能顯示模型與 reasoning effort（推理投入程度）。串流重複送出的累積訊息不會再把成本和 token（模型處理文字的計量單位）重複算，非同步紀錄也會帶上正確追蹤脈絡。最後，memory 裡行內 # 不會再害後面內容被截掉。像把每張收據編號、蓋章、釘回同一份帳本。

背景工作不再佔著位子又刪不掉

把工作放到 background 工作階段（背景執行，不佔著你眼前終端機）後，最煩的是它明明停了卻還佔著資源，或想清掉卻找不到。這版修了整條收尾流程：舊 daemon（背景常駐程式）退場時不會刪掉接班者的 control socket（用來溝通的本機通道）；用 ← 或 /background 暫停的工作不會無限養著 worker；服務閒置、從非 Git 資料夾送出、或重新開啟時遇到不可讀目錄，都不會讓刪除或還原對話失敗。像夜班交接不再把新值班員的鑰匙丟掉，完成的工作階段也能確實清掉。

當你更新 MCP（讓 Claude 連接外部工具的協定）的提示與資源時，偶發錯誤不再把伺服器現有的斜線指令和資源清空，像工具箱短暫斷電，裡面的工具仍留在原位。hooks（特定事件發生時自動執行的指令）傳回 exit code 2 而 stdout JSON 未通過 schema validation（檢查資料格式是否合規）時，現在仍會如文件所說阻擋。/ultrareview 在沒有 merge base（兩條 Git 分支共同起點）的 Git repository 裡，不再拒絕執行，會提供審查所有 tracked files（Git 正在追蹤的檔案）。若 keep-alive connection pooling（重複使用連線的方式）遇到 stale-connection error，之後會停用它，重試改開新 socket（網路連線通道）。

MCP 서비스와 연결은 일시적 오류로 한꺼번에 사라지지 않는다

MCP(Claude를 외부 도구에 연결하는 규약)의 프롬프트와 리소스를 새로 고치다가 일시 오류가 나도 서버의 기존 슬래시 명령과 리소스는 지우지 않는다. 공구함이 잠깐 정전돼도 안의 도구는 남는 것과 같다. hooks(특정 이벤트에 자동 실행하는 명령)가 exit code 2를 내고 stdout JSON이 schema validation(데이터 형식이 맞는지 확인하는 검사)을 통과하지 못해도 문서대로 막는다. merge base(Git 분기의 공통 시작점)가 없는 Git repository에서도 /ultrareview는 거부하지 않고 추적 중인 모든 파일을 검토하는 선택지를 낸다. stale-connection error 뒤에는 keep-alive connection pooling을 끄므로 재시도는 새 socket(통신 통로)을 연다.
