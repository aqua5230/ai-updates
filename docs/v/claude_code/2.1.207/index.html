終端機不再卡頓，長回覆畫面不亂跳

現在你看 Claude 吐出落落長的表格或程式碼時，打字和捲動都會滑順無比。以前只要遇到超長的清單或段落串流（streaming，答案一邊生成一邊逐字顯示），終端機畫面就會卡住，而且等它講完還會莫名其妙往上跳回開頭；現在這些顯示問題都修好了。就像看新聞台底下的跑馬燈，以前字太多會卡住甚至整條重播，現在不管塞多少字都順順跑過。

自動模式全面開放，設定不再跟著專案跑

想放手讓系統自己做事，自動模式（auto mode，讓 Claude 不逐一問你就自己執行的模式）本來就有，但現在於 Bedrock、Vertex AI 與 Foundry（企業透過 AWS 或 Google Cloud 等雲端服務租用 Claude 的通道）上預設就能用了。以前你要先手動設定環境變數（開程式之前預先塞給它的隱藏開關，一般使用者看不到）才會開放。同時，這項開關現在只讀跟著你這個人、每個專案都適用的全域設定檔，不再去讀跟著資料夾走、別人給你的專案裡可能藏著一份的專案內設定，避免你換個專案就不小心被打開。就像巡邏機器人本來就會巡邏，只是現在出廠預設就打開這功能，而且它只聽你手上的總遙控器，不會因為走到不同倉庫就自己切換模式。

AWS 平台連線變快、升級模型，Windows 不再死當

用 Bedrock、Vertex 或 AWS 上 Claude 平台的人，系統現在預設會幫你掛上最新的 Opus 4.8 模型，而且也不會像以前那樣，每講一句話就跑去向 IAM Identity Center 要一次新憑證（credentials，證明你是合法使用者的金鑰），速度快多了。對 Windows 使用者來說，如果 AWS 憑證卡住（例如背後處理程序當掉），系統現在最多等你 60 秒就會自己中斷，不會整個程式死當在那裡。就像進遊樂園玩，以前每玩一項設施都要重新查驗一次身分證，而且驗票機當機你只能站著枯等；現在買一次票就能玩到底，驗票機卡住一分鐘就會有人來放行。

任務列表與深研功能更好讀，不卡畫面也不忘記名字

查看代理的任務清單時，介面變得聰明很多。現在你重複貼上同一段文字，它只會把原本摺疊的佔位符展開，不會刷兩筆紀錄佔空間；卡住等你的任務，會直接秀出卡在哪個問題跟「等了 3 分鐘」，不會只印兩次死板的時間戳。深研（Deep research，讓 Claude 大量爬網找資料的功能）也會明確印出它正在查哪個網站的網域，不再一律顯示「未知」。還有，背景任務不會再忘記自己因為計畫而取的名字，重開進過工作樹（worktree，讓同一個專案多開幾份平行工作副本的 Git 功能）的任務也不會一片空白了。就像醫院的掛號看板，不再只顯示冰冷的「未知病人」與時間代碼，而是會告訴你「王大明，在 3 號房，已等候 5 分鐘」，清楚明瞭。

讓多隻 Claude 互相合作的代理團隊現在更強韌了。以前只要團隊信箱（mailbox，代理們互相傳話的檔案）裡混進一則格式錯誤的壞訊息，整個團隊就會陷入無限崩潰的迴圈，每秒報錯直到你手動刪檔；現在它們懂得出錯時直接忽略並繼續工作。而針對遠端遙控，網路斷線或是憑證（證明你是合法使用者的金鑰）更新重連後不再漏接任務進度，你在手機跟網頁版上也能正常看見桌機版 App 跑的背景任務與工作流進度了。就像一間辦公室，以前收到一封亂碼傳真全公司就停擺，現在會把它扔進垃圾桶繼續辦公；出差在外的主管，也能順暢看到辦公室裡的進度白板。

補上外掛設定的破口，防範惡意指令偷塞

這次收緊了外掛（plugin，擴充 Claude 功能的附加元件）的安全鎖。以前外掛設定如果放在專案資料夾裡，心懷不軌的專案就能利用環境變數的寫法，偷偷把惡意指令混進你的終端機裡（這叫 shell injection 攻擊）；現在直接禁掉這種寫法，並且不再讀取專案層級的外掛設定，只讀取個人設定、--settings 參數以及 managed settings。此外，在非互動模式（non-interactive run，像 SDK 那樣不跳提示直接跑的模式）下，也不會再跳過安全授權對話框，把你偷偷標記成「已同意」。就像公司的門禁系統，以前會不小心讀到訪客自己印的假識別證，或者讓走後門的人免簽名就混進來；現在只認總公司發的卡，而且所有人一律得過警衛這關。

Plugin settings are battened down to stop sneaky injections and auto-consents

We've battened down the security hatches for plugins (add-ons that extend Claude's capabilities). Previously, if plugin configs lived in a project folder, a malicious repository could use variable syntax to sneak nasty commands straight into your terminal (a shell injection attack); this syntax is now completely rejected, and plugin settings are no longer read from project-level files at all, only user, --settings, and managed settings are honored. Furthermore, non-interactive runs (like SDKs that run without prompting the user) will no longer silently record you as "consented" to security policies without actually showing you the dialog. Like a corporate security gate that used to occasionally accept a visitor's home-printed badge or let back-door VIPs bypass the sign-in sheet; now it strictly honors only HQ-issued cards, and everyone signs the book.
