zh-TW

遠端任務不再因為「看起來太閒」而被誤殺

當你在無頭模式（headless session，沒有畫面的純背景執行模式）跑 Claude 時，如果剛啟動就在跑鉤子（hook，讓你在特定時間點自動觸發的腳本），以前它可能因為沒把進度回報出來，讓遠端系統以為它閒置太久（idle-reaped）而直接把它砍掉。現在進度會正常串流回報，確保系統知道它還在努力工作。就像一個在倉庫理貨的員工，以前因為沒出聲被主管當成在偷懶而直接開除，現在他會邊做事邊回報，就不會再被誤殺了。

en

Remote tasks no longer get killed for "looking idle" during startup

When running Claude in a headless session (a purely background mode with no visible interface), if it was busy running startup hooks (scripts that automatically trigger at specific times) right after launching, the system used to sometimes fail to report its progress. This silence caused remote servers to assume it was idling (idle-reaped) and ruthlessly terminate it. Progress now streams correctly, proving it is still hard at work. It's like a warehouse worker who used to get fired for "slacking" because they were working quietly in the back; now they call out their progress as they go, avoiding wrongful termination.

zh-CN

远端任务不再因为「看起来太闲」而被误杀

当你在无头模式（headless session，没有画面的纯背景执行模式）跑 Claude 时，如果刚启动就在跑钩子（hook，让你在特定时间点自动触发的脚本），以前它可能因为没把进度回报出来，让远端系统以为它闲置太久（idle-reaped）而直接把它砍掉。现在进度会正常串流回报，确保系统知道它还在努力工作。就像一个在仓库理货的员工，以前因为没出声被主管当成在偷懒而直接开除，现在他会边做事边回报，就不会再被误杀了。

ja

リモートタスクが「暇そうに見える」という理由で誤って強制終了されなくなった

ヘッドレスセッション（headless session、画面のない完全なバックグラウンド実行モード）で Claude を動かす際、起動直後にフック（hook、特定のタイミングで自動実行されるスクリプト）を処理していると、以前はその進捗が報告されず、リモートシステムに「暇している（idle-reaped）」と誤認されて強制終了されることがあった。今は進捗が正しくストリーミング報告されるようになり、システムが作業中であることを確実に把握できる。倉庫で荷解きをしている従業員が、以前は無言で働いていたために上司からサボりだと勘違いされてクビになっていたのが、今は作業しながら声を出すようになり、誤って解雇されなくなったようなものである。

ko

원격 작업이 '한가해 보인다'는 이유로 억울하게 강제 종료되지 않는다.

헤드리스 세션(headless session, 화면 없이 배경에서만 도는 모드)에서 Claude를 돌릴 때, 켜자마자 훅(hook, 특정 시점에 자동으로 실행되는 스크립트)을 실행 중이면 예전엔 진행 상황을 밖으로 알리지 않아 원격 서버가 '너무 오래 쉬고 있다(idle-reaped)'고 착각해 그대로 죽여버리곤 했다. 이제는 진행 상황을 제때 흘려보내, 시스템이 아직 열심히 일하고 있음을 알 수 있게 했다. 창고에서 짐을 정리하는 직원이, 예전엔 아무 소리 없이 일하다가 농땡이 피우는 줄 안 상사에게 해고당했다면, 이제는 일하면서 계속 진행 상황을 보고해 억울하게 잘리지 않게 된 것과 같다.
