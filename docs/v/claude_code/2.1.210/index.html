畫面不卡死、任務不迷路，介面操作更順手

現在你看著終端機（terminal，輸入指令的黑畫面）等系統跑長時間任務時，不再只能盯著靜止畫面發呆，而且在各個任務間切換也更穩定了。以前當它在背後執行很耗時的工具指令時，進度條看起來就像當機一樣不動；現在工具摘要欄位會有一秒一秒跳動的計時器，讓你知道它還在努力。同時，代理（agent，幫你自動做事的 AI 分身）清單底部的提示現在會明確顯示有幾個背景對話（background sessions，在主畫面背後默默執行的工作）正在等你回覆，數字改變時還會用顏色閃爍提醒；而且按左方向鍵（←）打開 agents 總覽畫面後，如果你切回原本的對話，先前的任務追蹤列表也不會再消失或讓畫面殘留疊影了。另外，重新接上背景對話的指令（claude attach）現在會耐心等系統準備好再連線，不會因為太心急而跳出「找不到工作」的錯誤。對使用螢幕報讀器的使用者來說，按 Shift+Tab 切換權限模式時，系統現在也會大聲把新模式唸出來了。就像你去銀行辦事，以前抽完號碼牌只能看著停滯的看板乾等，還以為機器壞了；現在看板會清楚跳秒數、輪到你時會閃燈提醒，而且就算你去別的櫃檯轉一圈回來，你的進度也妥妥地留在看板上。

關閉任務不留垃圾，編輯與貼上更乾淨乾脆

當你刪除任務或貼上文字時，系統不會再悄悄留下讓你困擾的「殘留物」了。以前，如果你強制殺掉一個背景任務，它可能會在你的版本控制系統裡留下一個永久的工作樹鎖定（git worktree lock，防止別的 process 同時搶用同一個 worktree 的管理鎖），害你之後怎麼都無法操作 Git；現在系統懂得定期巡邏，把那些主人已經死掉的鎖清乾淨。另外，當你把 Claude 裡的文字貼到外部編輯器時，也不會再莫名其妙黏著 È 或 É 這種奇怪的亂碼。如果外掛程式寫入快取（cache，暫存資料的地方）失敗，現在也不會留下一堆無用的暫存檔，並且修好了在 Windows 系統上容易撞到的檔案重新命名錯誤。在介面上，被你刪掉的對話，它原本夾帶的圖片不會再像陰魂不散一樣卡在儀表板上；而且當你直接同意一個計畫而沒有做任何修改時，系統也不會再錯誤標記「使用者已編輯」，更不會拿舊的版本把你的計畫檔覆蓋掉。就像你去飯店退房，以前房客離開後可能還把保險箱鎖死，或者在牆上留下撕不掉的膠帶殘膠；現在房務員會把保險箱解鎖、殘膠清乾淨，保證下一個客人住進來時一塵不染。

AI 不再對指令產生幻覺，讀檔與紀錄更精準

Claude 判斷執行狀態的腦袋變得更精明，不會再誤判指令的結果或偷偷略過資料了。以前如果它下了一個切換資料夾（cd）的指令但因為太耗時被移到背景執行，它會產生幻覺以為自己已經在新資料夾裡了；現在系統會明確告訴它「工作目錄並沒有改變」。同樣地，當終端機指令超時被自動放到背景時，系統給的訊息也變得更清楚，讓 AI 能分辨「這是當機卡住了」還是「只是被放到背景」。此外，如果你的自訂技能（skills）或指令裡有用來佔位的變數（像是 $1 或 $2），以前會被系統當成廢話默默吃掉，現在則會一字不漏地保留。而當它寫入記憶檔（MEMORY.md，Claude 用來記住重要事項的筆記本）的內容超過讀取上限時，以前它會悄悄把超出的部分截斷，讓你以為全都記下了；現在它會直接大聲報錯，逼著整理。甚至連搜尋檔案內容（Grep）翻頁翻到底時，也不會再傻傻地回報「找不到相符結果」了。另外，鉤子（hook，在特定事件發生時自動觸發的程式）超時的時候不會再錯誤地回報為「使用者拒絕」，害它中途停下來傻等；內建的資料視覺化技能，它的圖表顏色也換成了對色盲更友善的科學色碼標準。就像你請助理去圖書館找資料，以前他找時間太久被你叫回來，還會幻想自己已經把書借出來了，或者筆記本寫滿了就偷偷把後面的內容撕掉；現在他會老實報告還沒借到書，而且筆記本一滿就會立刻舉手跟你換一本新的。

分身代理不越界，安全防護與連線更牢固

你現在叫出的分身代理更守規矩了，不用擔心它們亂跑，而且抵禦外在惡意內容的防護網也變厚了。以前，當你設定代理在獨立的工作樹（worktree，像是專案的平行時空副本）裡隔離作業時，它竟然還能跑去修改你最主線的程式碼；現在它被嚴格限制，只能在自己的沙盒裡玩沙，絕不能動到主程式。我們也強化了代理的抵抗力，防止它在讀取網頁或檔案時，被裡面暗藏的惡意提示詞（prompt injection，駭客把偷渡指令寫在文章裡騙 AI 執行的手法）牽著鼻子走。另外，用來加開更高強度處理效果的 ultracode 關鍵字，以前只要網頁鉤子（webhook）或 GitHub 留言裡剛好提到這個詞就會被誤觸，現在只有人類親手輸入才會生效；同時也修好了在終端機指令裡指定 --effort ultracode 卻被偷偷忽略的問題。針對開發者，外掛提供的 MCP 伺服器（讓 AI 能接上外部工具的橋樑）不會再因為中途重新同步而被無辜拆除了；透過初始化請求註冊的 SDK MCP 伺服器，也不會再傻傻等到下一輪才開始連線。此外，Fable 顧問模型因為伺服器問題暫時從選擇器裡移除了，等修好就會回來。就像公司請了臨時工來幫忙，以前他們會不小心跑到總經理辦公室亂改文件，還會聽信路邊撿到的紙條亂做事；現在他們都被配發了專屬的封閉工作間，而且受過反詐騙訓練，只聽從你親自下達的指令。

啟動防呆更完善，當機風險與設定衝突大幅降低

Closing tasks leaves no trash, and copy-pasting is much cleaner

When you delete tasks or paste text, the system will no longer quietly leave behind annoying residues. Previously, if you force-killed a background session, it might leave a permanent git worktree lock (an administrative lock preventing other processes from concurrently using the same worktree) in your version control system, completely blocking your Git commands forever; now, the system regularly sweeps the area to release locks whose owning processes have died. Also, when you paste text from Claude into an external editor, it won't inexplicably come attached with weird garbage characters like È or É anymore. If a plugin fails to write to its cache (a place for temporary data), it will no longer leave behind useless temporary files, and a tricky locked-file rename error that plagued Windows users has been fixed. Visually, images pasted into abandoned reply drafts won't stubbornly haunt your agents dashboard after the session is deleted; and when you approve a plan without making any changes, the system won't falsely brand it "(edited by user)" or overwrite your plan file with a stale snapshot. It's like checking out of a hotel: a previous guest might have left the safe locked or left stubborn tape residue on the walls, but now the housekeeping staff unlocks the safe and scrubs the glue away, ensuring the room is completely spotless for the next guest.

AI no longer hallucinates command results, improving file reading and logging

Claude's brain for tracking execution states has gotten sharper, so it won't misjudge command results or quietly ignore data anymore. Previously, if it issued a change directory (cd) command that took too long and got shunted to the background, it would hallucinate that it was already inside the new folder; the system now explicitly tells it that the working directory is unchanged. Similarly, when a terminal command times out and is auto-backgrounded, the message given to the AI is much clearer, helping it distinguish whether the command is hopelessly hung or merely running in the background. Moreover, if your custom skills or commands use positional placeholders (like $1 or $2), the system used to treat them as garbage and silently devour them; they are now preserved exactly as written. When writing to the memory file (MEMORY.md, Claude's notebook for remembering important things) pushes the index past its read limit, it used to quietly truncate the overflowing parts leaving you thinking everything was saved; now it throws a loud, explicit error to force a cleanup. Even Grep (the tool for searching inside files) will no longer foolishly claim "No matches found" when you simply paginate past the end of the results. Also, when a hook (a script triggered by specific events) times out, it won't be falsely reported as a user rejection that freezes unattended sessions, and the bundled data visualization skill now uses scientifically calibrated color codes that are much friendlier to colorblind users. It's like sending an assistant to the library: previously, if you called him back because he took too long, he'd hallucinate that he had already checked the books out, or if his notebook was full he'd just rip up the remaining pages; now, he honestly reports he hasn't gotten the books yet, and the moment his notebook fills up, he immediately raises his hand to ask for a new one.

Subagents stay in bounds with hardened security and robust connections

The subagents you spawn are now much better behaved, saving you from worrying about them going rogue, and the defensive net against external malicious content is thicker. Previously, when you configured an agent to work isolated inside its own worktree (a feature acting like a parallel universe copy of your project), it could somehow still reach out and run git-mutating commands on your main repository checkout; it's now strictly confined to playing in its own sandbox and cannot touch the main codebase. We've also hardened the agent's immune system to prevent it from being manipulated by indirect prompt injection (a trick where hackers hide sneaky commands inside articles to trick the AI) when it reads web pages or files. Additionally, the ultracode keyword used to opt into higher-effort processing used to accidentally trigger if a webhook payload or a GitHub comment merely mentioned the word; it now only activates when explicitly typed by a human. The bug where specifying --effort ultracode in terminal commands was silently ignored has also been fixed. For developers, plugin-provided MCP servers (bridges that let AI connect to external tools) will no longer be unfairly torn down if they resync mid-session; and SDK MCP servers registered via an initialize request won't foolishly wait until the next turn to start connecting. Also, the Fable advisor model has been temporarily removed from the picker while a server-side issue is being patched. It's like hiring temporary workers to help out: previously, they might accidentally wander into the CEO's office to alter documents or blindly follow instructions found on a scrap of paper in the street; now, they are assigned dedicated, locked workspaces and have been given anti-fraud training so they only listen to your direct orders.
