保留網路管制，關掉檔案隔離

有些公司工具必須直接碰到電腦上的檔案，卻仍想限制它往外連網。現在可用 `sandbox.filesystem.disabled` 設定，跳過檔案系統隔離（把程式能看的檔案範圍圈起來的防護），同時保留網路出口管制。它像辦公室把檔案室的門打開給指定同事，卻仍由警衛管住所有外出通道；不是把整套沙盒（sandbox，限制程式活動範圍的保護環境）拆掉。需要讓 Claude Code 存取本機檔案、又不願放開網路限制時，這項開關提供了中間選擇。

Keep network controls while disabling file isolation

Some company tools need direct access to files on the computer while their internet access must remain restricted. The new `sandbox.filesystem.disabled` setting skips filesystem isolation (the guardrail that limits which files a program can see) but keeps network egress control. Think of opening the records room to approved staff while guards still control every exit; it does not tear down the whole sandbox (a protected boundary around what a program can do). This gives you a middle option when Claude Code needs local files but the network restrictions must stay in place.

Long chats run faster and tell you when context is full

A long-running chat used to get slower the longer it went: each new turn made message cleanup grow disproportionately, producing multi-second stalls and slow resumes. That quadratic buildup is fixed. `/context` now explicitly warns when the conversation exceeds its context window (how much of a conversation the AI can hold at once), and a failed `/compact` clearly appears as an error. It is like a desk where every added sheet once meant rereading a larger pile from the start; now sorting stays smoother, and an overfilled drawer shows a warning instead of pretending nothing is wrong.

Claude가 백그라운드 작업을 하거나 worktree(Git에서 한 프로젝트의 병렬 사본을 만드는 기능)로 나눌 때 격리가 제대로 지켜진다. 재개된 백그라운드 에이전트는 원래 프롬프트와 도구 제한을 되찾고, 시작 직후 높은 우선순위 메시지가 와도 취소되지 않는다. 격리된 하위 에이전트는 `git -C`, `--git-dir`, 환경 변수로 Git을 공유 체크아웃으로 돌릴 수 없으며 다른 프로젝트의 남은 worktree에도 들어가지 않는다. Git 저장소 없는 배경 worktree도 삭제할 수 있고 삭제한 세션이 되살아나지 않는다.
