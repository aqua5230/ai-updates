自動模式加裝防護網，攔截破壞性刪除指令

Claude 現在不會在你未明講放棄工作內容時執行破壞性的 Git 指令。如果你沒有明確叫它放棄目前的工作進度，系統會主動攔截那些危險的 Git 指令（像是 `git reset --hard`、`git clean -fd` 等會刪掉檔案的操作）；對於 `git commit --amend`，如果不是它自己這次對話做的提交，它也不准修改。如果你用的是 Terraform 或 Pulumi 等基礎設施工具，除非你點名要刪除特定環境，否則系統會嚴格禁止執行 `destroy` 把伺服器砍掉。此外，以前排程任務或 Webhook 收到的訊號會被系統誤當成你的鍵盤輸入，甚至不小心幫 AI 按下「同意」按鈕，現在這個安全漏洞也被補上了。這就像給除草機加裝了安全鎖，除非你明確解鎖並指著那片花圃，否則它絕對不會把珍貴的玫瑰花當成雜草一起絞掉；而且外頭飛進來的鳥也不會再誤觸開關啟動機器了。

Original changelog
Improved auto mode safety: destructive git commands (`git reset --hard`, `git checkout -- .`, `git clean -fd`, `git stash drop`) are now blocked when you didn't ask to discard local work, `git commit --amend` is blocked when the commit wasn't made by the agent this session, and `terraform destroy`/`pulumi destroy`/`cdk destroy` are blocked unless you asked for the specific stack
Fixed scheduled task and webhook trigger deliveries being treated as keyboard input; they now classify as task notifications and can no longer approve a pending action or set the session title in auto mode

Auto mode gets safety nets against destructive commands

Claude will no longer execute destructive Git commands unless you explicitly ask it to discard your local work. The system will now actively block destructive Git commands (like `git reset --hard` and `git clean -fd` that irreversibly delete files). It also refuses to run `git commit --amend` unless the agent itself made that specific commit during the current session. For infrastructure-as-code tools like Terraform or Pulumi, running a `destroy` command is now strictly forbidden unless you specifically name the target stack. Additionally, scheduled tasks and Webhook triggers used to be incorrectly interpreted as your keyboard inputs, sometimes accidentally 'approving' an AI action on your behalf; this dangerous loophole has been firmly closed. It's like putting a safety guard on a lawnmower: unless you explicitly flip the switch and point at a specific patch of dirt, it absolutely refuses to shred your prized rose bushes, and a stray bird landing on the handle can no longer accidentally start the engine.

Agent teams are tougher, and search/output glitches are gone

Team-based AI collaboration just got a lot more dependable. Previously, spawning a teammate in a tmux pane (a terminal multiplexer) could fail if your shell initialization was slow, and keystrokes you typed on the main screen would bizarrely leak into the new teammate's window; these chaotic overlaps are now fixed. Also, if a teammate dispatched a background task to a subagent, that task used to be ruthlessly killed the moment the teammate finished its turn—now it survives to completion. On the cognitive side, subagents using WebSearch no longer return totally blank results, and if the AI spins its wheels generating a "thinking block" but fails to produce actual visible output, it won't just silently give up anymore—it will re-prompt the model once to produce visible output. Finally, we squashed the annoying `400 Extra inputs are not permitted` errors that plagued subagent spawns on certain configurations. It's like a busy factory floor: previously, a slow-booting new robot would jam the assembly line, and any chore handed to an intern was immediately thrown in the trash the moment their manager clocked out for the day. Now, handoffs are flawless, and the interns faithfully finish their tasks even after their boss goes home.

Settings and UI cleaned up, squash screen-corrupting bugs

The interface is now noticeably cleaner and more intuitive. The settings menu (`/config`) makes sense now: you can hit Enter or Space to toggle switches, and pressing Esc actually saves and closes instead of reverting all your changes. You can also run `/config --help` to easily look up shorthand keys. We've scrubbed the constant 'setup issues' nag from the startup logo; if you want a system health check, just run `/doctor`. You'll now get clear warnings if you request a deprecated model or one that automatically updates to a newer version. We also squashed a ton of screen-corrupting bugs: the total UI meltdown (merged text and duplicated spinners) in Windows Terminal under heavy subagent loads is fixed, your cursor won't get stranded above the prompt in vim mode anymore, the noisy 'PostToolUse' timing lines cluttering focus mode are gone, and user-level skills won't infinitely duplicate themselves in your autocomplete list. Finally, headless environments (running without a screen) will no longer crash by exposing fake auth-stub tools to the AI when an MCP server demands authentication. It's like thoroughly decluttering your workstation: broken buttons now work predictably, phantom alarms have been silenced, and the flickering monitor that scrambled your spreadsheets has finally been replaced, leaving you with a distraction-free space.

자동 모드에 안전망 추가, 파괴적인 삭제 명령어 차단

Claude는 당신이 작업 내용을 버리겠다고 분명히 말하지 않는 한 파괴적인 Git 명령어를 실행하지 않는다. 당신이 지금 하던 작업을 다 버려도 좋다고 확실히 말하지 않으면, 시스템은 위험한 Git 명령어(파일을 싹 날려버리는 `git reset --hard`나 `git clean -fd` 등)를 알아서 막아낸다. 또한 이번 대화 중에 자기가 직접 한 커밋이 아니면 `git commit --amend`로 내용을 고치는 것도 금지된다. Terraform이나 Pulumi 같은 인프라 관리 도구를 쓸 때도, 당신이 특정 환경을 콕 집어 지우라고 하지 않으면 서버를 날려버리는 `destroy` 명령어를 엄격하게 차단한다. 그뿐만 아니라 예전엔 예약된 작업이나 Webhook 신호를 당신이 키보드를 누른 것으로 착각해 AI의 행동을 멋대로 '승인'해버리던 무서운 구멍도 꽉 막았다. 잔디깎이에 안전장치를 단 것과 같다. 당신이 확실히 잠금을 풀고 특정 잡초밭을 가리키지 않는 한 소중한 장미꽃을 갈아버리는 일은 절대 없으며, 날아가던 새가 스위치를 밟는다고 기계가 멋대로 켜지지도 않는다.

Original changelog
Improved auto mode safety: destructive git commands (`git reset --hard`, `git checkout -- .`, `git clean -fd`, `git stash drop`) are now blocked when you didn't ask to discard local work, `git commit --amend` is blocked when the commit wasn't made by the agent this session, and `terraform destroy`/`pulumi destroy`/`cdk destroy` are blocked unless you asked for the specific stack
Fixed scheduled task and webhook trigger deliveries being treated as keyboard input; they now classify as task notifications and can no longer approve a pending action or set the session title in auto mode

에이전트 팀이 더 튼튼해지고, 검색 및 출력 멈춤 현상 사라짐

AI들이 역할을 나눠 일하는 에이전트 팀이 훨씬 믿음직스러워졌다. 예전엔 tmux(터미널 화면 분할 도구)로 새 팀원을 부를 때 컴퓨터 로딩이 조금만 늦어도 실행에 실패하고, 심지어 메인 화면에서 당신이 친 타자가 엉뚱하게 새 팀원의 창으로 새어 나가던 간섭 문제들을 싹 해결했다. 또한 팀원이 하위 에이전트에게 맡긴 백그라운드 작업이, 지시를 내린 팀원이 자기 턴을 마치자마자 무자비하게 끊기던 버그도 고쳐져 이제는 끝까지 무사히 마친다. AI의 검색과 생각 과정도 나아졌다. 하위 에이전트가 웹 검색(WebSearch)을 하면 텅 빈 결과만 가져오거나, AI가 머리를 굴리는 시간(thinking block)만 잔뜩 쓰고 정작 내놓는 대답 없이 턴을 끝내버리던 어이없는 상황이 사라졌고, 이제는 다시 한 번 모델에게 눈에 보이는 출력을 내라고 알려준다. 마지막으로, 설정이 꼬여 하위 에이전트를 부를 때마다 `400 Extra inputs are not permitted` 오류를 뿜어대던 문제도 고쳤다. 마치 공장 생산 라인에서 새로 온 로봇이 부팅이 늦다고 라인 전체가 멈추거나, 과장님이 퇴근했다고 인턴에게 맡긴 일까지 쓰레기통에 처박히던 것이, 이제는 인수인계가 매끄러워져 과장님이 퇴근해도 인턴이 맡은 바를 끝까지 책임지고 해내는 것과 같다.

설정 및 인터페이스 정리, 화면이 깨지는 버그 싹쓸이

인터페이스가 한결 깔끔하고 손에 익게 바뀌었다. 설정 패널(`/config`) 조작이 직관적으로 변해 Enter나 스페이스바를 눌러 스위치를 바로 켜고 끌 수 있으며, Esc를 누르면 예전처럼 튕겨 나가며 취소되는 대신 제대로 저장하고 닫힌다. 단축 명령어가 궁금하면 `/config --help`로 바로 찾아볼 수도 있다. 프로그램이 켜질 때마다 로고 밑에 신경 쓰이게 떠 있던 '설정 문제' 경고도 치워버렸다. 점검이 필요하면 알아서 `/doctor`를 치면 된다. 폐기 예정이거나 자동 업데이트되는 모델을 선택하면 명확한 경고를 띄워준다. 화면을 엉망으로 만들던 버그들도 대청소했다. Windows 터미널에서 하위 에이전트를 잔뜩 띄웠을 때 글씨가 겹치고 화면이 무너지던 문제, vim 모드에서 커서가 입력창 위 허공에 갇혀 버리던 귀신 같은 현상을 모두 잡았다. 집중 모드에서 대답이 끝날 때마다 밑에 뜨던 시끄러운 후크(Hooks) 실행 시간 기록도 지웠고, 당신이 직접 만든 스킬이 자동 완성 목록에 분신술처럼 여러 번 뜨는 일도 없앴다. 마지막으로, 화면이 없는(headless) 환경에서 MCP 서버 인증이 필요할 때 시스템이 가짜 도구를 AI에게 던져주어 오류가 나던 문제도 해결했다. 어지러운 책상을 싹 정리한 것과 같다. 쓸데없이 울리던 알람을 끄고 고장 난 버튼을 고쳤으며, 화면을 지지직거리게 만들던 낡은 형광등까지 갈아 끼워 오롯이 일에만 집중할 수 있게 되었다.
