背景任務大升級：程式關閉或更新，任務依然活著

背景任務現在有了「不死之身」。以前如果你把 Claude 關掉、重新啟動，或是程式剛好在背景自動更新，你那些跑到一半的長時間指令和任務就會全部慘死；現在它們能在這些劇烈變動中存活下來，甚至在 Windows 上也不例外。如果不小心發生系統重啟導致背景任務死掉，下次你打開清單時，它們會自動從上次斷掉的地方無縫接軌繼續做。這也順便解決了以前你喚醒一個卡住的任務時，系統可能會誤刪掉它整份對話紀錄並強迫重頭開始的毛病。這就像你在煮湯時家裡突然停電，以前整鍋湯會壞掉只能倒掉重煮，現在復電後瓦斯爐會自己點火，繼續把你那鍋湯熬完。

效能與防呆優化：不再誤報錯誤、節省 25% 資源

터미널 화면의 작동이 훨씬 가볍고 깔끔해졌다. 긴 답변을 쏟아낼(streaming) 때 시스템이 알아서 내용이 바뀌지 않은 부분은 놔두고 그리지 않아 자원 낭비를 줄인다. 백그라운드 작업이 구조화된 출력에 실패했을 때도, 실패한 줄과 다시 시도하는 줄을 중복해서 찍어 눈을 어지럽히는 일이 없어졌다. 오해를 부르던 상태 표시도 여러 개 고쳤다. Bedrock 플랫폼에서 /context가 토큰을 전부 0개로 표시하던 버그를 잡았고, 심층 연구(/deep-research) 기능이 검증에 실패했을 때 '모든 주장이 반박됨'이라고 오버하던 대신 '검증되지 않음'이라고 정직하게 알려준다. 또한 외부 플러그인(plugin)을 내 컴퓨터에 두거나 Git으로 관리할 때 검사를 건너뛰거나 버전을 무시하던 문제, MCP 인증 시 권한(scopes)을 너무 많이 요구해 GitLab 등에서 접속을 거부당하던 문제 등 검증 절차의 구멍들을 모두 메웠다. 마지막으로, Anthropic 외의 다른 서버를 쓰도록 설정하면 쓸데없는 원격 제어 기능은 알아서 꺼준다. 자동차의 고장 난 기름 게이지를 고치고 에어컨 효율을 높인 것과 같으며, 출입증을 발급받을 때 서류 순서가 조금 다르다고 쫓아내던 경비원이 이제는 아무 탈 없이 통과시켜 주는 것과 같다.
