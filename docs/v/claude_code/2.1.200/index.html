預設改為手動模式，不會再趁你不注意偷偷繼續

現在 Claude 做事前預設會先問你，放著不回覆也不會自己繼續。不管是指令行、VS Code 還是 JetBrains，權限模式都改成了「手動（Manual）」。當它彈出對話框問你問題（AskUserQuestion）時，以前如果放著不管，它可能會自己當作你同意而繼續跑；現在預設會一直等你回答，除非你自己去設定檔裡開啟「閒置超時自動繼續」的功能。就像教練本來會自己決定要不要換球員，現在預設改成換人前必須請示總教練，而且沒拿到簽名絕對不會自己放行。

Original changelog
Changed `AskUserQuestion` dialogs to no longer auto-continue by default; opt into an idle timeout via `/config`
Changed the "default" permission mode to "Manual" across the CLI, `--help`, VS Code, and JetBrains; `--permission-mode manual` and `"defaultMode": "manual"` are accepted alongside `default`

背景任務更清醒：電腦休眠醒來或當機後不再裝死

這次把背景代理（agent）從休眠中喚醒的毛病治好了。以前你的電腦如果睡著又醒來，正在跑的任務可能會默默停住不幹活；如果它卡住被系統重新啟動，還會把之前你按 Esc 取消掉的步驟又拿出來跑一次，現在這些亂象都沒了。另外，如果程式崩潰留下了一個寫著進程代號的鎖定檔（daemon.lock），而作業系統剛好又把這個代號分給別的程式，以前代理就會因為誤認而永遠不肯啟動，現在修好這個檢查邏輯了；被限流（rate limit）打斷的子任務現在也會明確報錯，不會假裝自己跑完卻丟出空結果。這就像警衛打瞌睡醒來，以前會忘記自己巡邏到哪，或是看到地上掉著上一班的臂章就以為別人還在上班而拒絕接班，現在他醒來後會確實確認身分並正確接手工作。

Background tasks wake up properly after sleep and survive crashes

We've cured background agents of their bad waking habits. Previously, if your computer went to sleep and woke back up, a running task might silently stall and refuse to work; and if a stalled task respawned, it might foolishly re-run a step you had explicitly cancelled with Esc. These confusing behaviors are gone. Additionally, if the program crashed and left behind a stale lockfile (daemon.lock) whose process ID got recycled by the OS, the agent used to get confused and refuse to ever start again; this detection logic is now fixed. Subagents cut off by a rate limit will also now fail properly instead of pretending they finished successfully and returning an empty void. It's like a security guard waking up from a nap: they used to forget their patrol route or see an old name badge on the floor and refuse to start their shift; now they properly verify their status and resume duty flawlessly.

Accessibility upgrades and a flicker-free terminal experience

The experience for users relying on screen readers and magnifiers is now vastly improved. The system automatically hides purely decorative symbols, reads complex transcript symbols as short, descriptive labels, and parses nested tables line-by-line as Header: value, making it much easier to digest. The MCP (a protocol letting AI use external tools) server list also now correctly tracks focus for accessibility tools. For those using tmux (a terminal multiplexer), we enabled synchronized output so the screen no longer flickers like a strobe light; and raw control codes will no longer bleed into the agent view. It's like an audio tour guide who used to confusingly describe every decorative carving on the wall; now they gracefully summarize the important exhibits, and the museum's flickering lights have finally been replaced.

Foolproofing: Silent audio doesn't error, and stalled installs explain why

You will now see accurate error reasons when voice dictation captures silence or an installation fails due to out-of-memory. If your recording ends up capturing pure silence, the system used to misleadingly throw a "Voice connection failed" error; now it understands you just didn't record any sound, and the installer script will explicitly explain what happened if the OS kills it for memory instead of failing mysteriously. For plugins, the --plugin-dir flag now properly loads agents even if placed after the agents command, and project-scoped plugins won't get lost just because you are using a Git worktree (parallel working copies). Additionally, misformatting the MCP server list in your settings file will no longer crash the entire program at startup. It's like going to the store: the cashier used to just say "connection failed" when your card was empty, but now they directly tell you "insufficient funds" so you aren't left guessing.

백그라운드 작업이 잠에서 더 잘 깨고, 컴퓨터 절전이나 충돌 후에도 멈추지 않는다.

백그라운드 에이전트(agent)가 절전 모드에서 깨어날 때 겪던 고질병을 고쳤다. 예전엔 컴퓨터가 잠들었다 깨어나면 돌고 있던 작업이 소리 없이 멈춰서 일하지 않곤 했고, 멈춰서 다시 켜질 때 예전에 Esc를 눌러 취소했던 단계까지 다시 실행해 버리는 황당한 일도 있었지만 이제는 모두 해결되었다. 또한 프로그램이 충돌하며 프로세스 번호가 적힌 잠금 파일(daemon.lock)을 남겼을 때, 운영체제가 우연히 그 번호를 다른 프로그램에 줘버리면 에이전트가 오해하고 영영 켜지지 않던 문제도 고쳤다. 횟수 제한(rate limit)에 걸려 끊긴 하위 작업도 이제는 끝난 척 빈 결과를 던지는 대신 명확하게 오류를 뿜어낸다. 꾸벅꾸벅 졸다 깬 경비원이 예전엔 어디까지 순찰했는지 까먹거나, 바닥에 떨어진 이전 근무자의 명찰을 보고 아직 누군가 일하고 있는 줄 알고 교대하기를 거부했다면, 이제는 깨어나서 정확히 상황을 파악하고 무사히 업무를 이어받는 것과 같다.

화면 읽기 프로그램(screen-reader)과 돋보기에 의지하는 사용자들을 위해 훨씬 친절해졌다. 이제 화면을 꾸미기 위한 특수 기호는 알아서 숨겨주고, 대화 기록 속의 기호들은 짧은 꼬리표로 바꿔 읽어주며, 표 역시 '제목: 내용' 형식으로 한 줄 한 줄 정성껏 읽어준다.
