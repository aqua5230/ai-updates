終端機輸入直接對答，外部工具驗證不卡關

使用終端機指令的體驗大升級了！現在你在終端機使用 bash 模式（打 ! 開頭加上指令）時，Claude 不會只默默把結果記下來，而是會主動對執行結果做出回應（你如果不喜歡，也可以去設定關掉）。對於需要登入的外部工具（MCP），你現在可以直接在指令列輸入 claude mcp login 來登入或登出，不需要再切換到互動選單；如果你是在別台電腦透過 SSH 連線過來，還能用無瀏覽器模式完成認證。萬一你打錯了工具伺服器的名字，系統現在會聰明地猜測你是不是拼錯了，並把太長的伺服器清單縮短顯示。這就像你的助理原本只會幫你抄寫跑腿回來的發票，現在他跑完腿會順便跟你報告結果；而且現在你要進金庫，不用再跑回大廳填表，在門口直接刷卡就能進去，就連你名字寫錯一點點，門禁系統也會認得你。

Original changelog
`!` bash commands now trigger Claude to respond to the output automatically; set `"respondToBashCommands": false` in settings.json to keep the previous context-only behavior
Added `claude mcp login <name>` and `claude mcp logout <name>` to authenticate MCP servers from the CLI without opening the interactive `/mcp` menu, with `--no-browser` stdin redirect support for completing over SSH
Improved `claude mcp get` and `claude mcp remove` to suggest the closest configured server name on a typo and truncate long server lists

子代理人與背景任務現在變得更好管教，畫面也不再亂閃了。以前在背景跑的子代理人遇到需要權限時，要嘛自己亂拒絕，要嘛卡在那邊；現在它會把權限請求丟回你的主對話框裡，並清楚標示「是哪個小助手在要權限」，你按 Esc 就能單獨拒絕那個工具。如果你派出去的小助手因為要求特定格式（JSON schema）連續五次失敗，系統現在會自動中止它，不會讓它陷入無限重試的死迴圈。我們也修好了一大堆畫面干擾：像是從代理人介面退出來時畫面不會再黏著舊紀錄、背景任務的預覽框不會再把原始的工具名稱閃爍出來、切換深淺色終端機時不再被突如其來的白光或黑畫面閃瞎，連同在主視窗按 Esc 或 Ctrl+C 時，系統現在也能正確煞車了。這就像是你派出去實習的菜鳥，以前遇到問題只會自己瞎忙或裝死，現在懂得打電話回來請示；而且他們傳回來的監視器畫面終於修好了，不會再看到殘影或莫名其妙的雪花雜訊。

網路與系統休眠穩健化，企業與平台連線升級

面對電腦休眠或網路環境變化，系統的防護力更強了。以前如果你在 AI 正在產生文字（streaming）時把電腦蓋上休眠，醒來後它會跳出「找不到內容」或格式解析錯誤；現在它能完美熬過這段時間，醒來繼續運作。對於企業用戶，系統現在會正確顯示按照用量計費的對話花費；如果是透過 AWS 登入的企業版，/login 介面也新增了「重新整理憑證」的選項，不用再等到憑證徹底過期才被踢下線。如果你使用 Chrome 擴充功能並且關掉了權限警告，多開終端機時不會再發生分頁群組無法隔離的問題。而對於帶領團隊的主代理人，現在如果開出新隊友（透過 tmux 等背景視窗），新隊友也會乖乖繼承隊長所設定的努力度（--effort）。此外，系統的無限重試限制最高設為 15 次，避免失控浪費。這就像你把辦公室電腦關掉睡個午覺，以前醒來資料會全毀，現在它會自己保存在雲端；而且大老闆交代專案的認真程度，現在也會準確傳達給每一個下屬，不會有人偷懶。

Direct responses to bash commands and seamless MCP authentication

The terminal command experience just got a massive upgrade! When you use bash mode (starting with !), Claude will no longer just silently absorb the terminal output as context; it will now proactively respond and analyze the result right away (if you prefer the old silent behavior, you can toggle it off in your settings). For external MCP tools that require authentication, you can now seamlessly log in or out directly from your shell using claude mcp login without being forced into an interactive menu. If you're working over an SSH connection, there's even a headless, no-browser fallback. And if you happen to typo a tool server's name, the CLI is now smart enough to suggest the closest match and cleanly truncate unwieldy lists. It's like having an assistant who used to just silently file away the receipts after running an errand, but now actively hands you a summary report; plus, you no longer have to walk all the way back to the lobby to fill out a security form to access the vault—you just swipe your card at the door, and the guard even recognizes you if you slightly misspell your own name.

Background tasks and subagents behave predictably without visual glitches

Subagents and background tasks are now far more obedient and visually stable. Previously, a background subagent hitting a permissions barrier would either quietly auto-deny itself or get stuck. Now, it neatly surfaces the permission prompt back to your main session, clearly identifying which assistant is asking, and pressing Esc will cleanly deny just that specific tool. We've also added a failsafe: if an agent fails a structured JSON schema validation 5 times in a row, it now automatically aborts instead of spiraling into an infinite loop. On the visual side, we've squashed a swarm of UI glitches: closing the agents view no longer leaves ghost text painted on your screen, background task previews won't flash ugly raw tool names before loading, switching into the agents view on light terminals won't blind you with a sudden dark-theme flash, and pressing Esc or Ctrl+C in the main view while agents run in the background is now perfectly responsive. It's like managing a team of interns: they used to panic silently or spin in circles when they hit a locked door, but now they text you for permission; and the broken security monitors tracking them have finally been fixed so you no longer see weird static flashes or lingering burn-in images.

Stronger network and sleep recovery, with enterprise platform updates

The system is much more resilient when your computer goes to sleep or network conditions shift. Previously, if you closed your laptop while the AI was streaming an answer, you'd wake it up to an ugly 'Content block not found' or a JSON parsing error; the system can now flawlessly survive the hibernation cycle. For Enterprise and Team users, usage-based session costs are now correctly displayed again. If you use the Claude Platform on AWS, the /login menu now provides a 'refresh credentials' option so you can proactively renew your token before getting kicked out. We also fixed a bug with the Chrome extension where disabling the permissions gate broke tab-group isolation when running multiple terminal sessions. In agent teams, new teammates spawned into background window panes (like tmux) will now obediently inherit the leader's designated effort level (--effort). Finally, we've hard-capped API retries at 15 to prevent run-away loops from burning your wallet. It's like taking a nap while a massive document is printing: you used to wake up to a jammed machine and shredded paper, but now it pauses perfectly and finishes the job when you wake up. Plus, the boss's strict instructions on how thoroughly to do the job now successfully trickle all the way down to the newest interns without anyone slacking off.
