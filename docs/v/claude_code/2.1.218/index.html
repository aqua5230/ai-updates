程式碼審查留在背景，結果更貼近你的分支

你請 Claude 檢查一串改動，還想繼續問別的事時，/code-review 現在會交給背景 subagent（在旁邊獨立做事的小幫手）處理，不再把整個對話塞滿；連著輸入的 slash commands 也仍會以這次審查為目標。/ultrareview 可接受「檢查我的登入改動」這種說法：它會審查目前分支，並把這句當成結果備註。非互動執行 /code-review ultra 也會確實啟動雲端審查，而非悄悄改做本機審查；參數不對時，Claude 會拿到可修正的錯誤訊息。像把文件送去後台校稿，你仍能在前台寫下一封信，校稿員也知道該看哪一份。

Original changelog

螢幕閱讀器能跟上刪字與游標

使用 --ax-screen-reader 模式改句子時，按 Option+Delete、Ctrl+W 或 Ctrl+U 刪掉一個詞或一整行，螢幕閱讀器現在會唸出剛刪掉的文字，不必猜自己刪了什麼。輸入列尾端打一個空白，VoiceOver 也會回報空白，而不是誤唸「換行」。在 plugin 與 settings 面板用方向鍵移動時，終端機游標會跟著停在焦點列，讓螢幕閱讀器和放大鏡都找得到目前位置。這像有人一邊替你讀稿，一邊用手指指出你剛擦掉的字與正在看的那一行。

Original changelog
Added screen-reader announcements of deleted text for word and line deletions (`Option+Delete`, `Ctrl+W`, `Cmd+Backspace`, `Ctrl+U`, `Ctrl+K`) in `--ax-screen-reader` mode
Fixed VoiceOver reading "new line" instead of echoing the typed space at the end of the input in `--ax-screen-reader` mode

貼上多行不變形，誤按左方向鍵可回頭

把好幾行指令貼進某些終端機時，換行曾被編成 Ctrl+J，Claude 卻把它當成字母 j，整段內容擠成一行；現在會保留原本的換行。剛編輯完就按左方向鍵，也不再無預警丟掉整段對話：它會先請你確認。若你進入 agent view（把工作放到背景看的畫面），按 Esc 則能回到剛背景化的對話。像寫信時貼上的段落不再被壓成一行，手肘碰到返回鍵前也有人先問你要不要真的把信丟掉。

Original changelog
Fixed the left arrow key discarding the conversation with no undo: presses right after editing now ask to confirm, and Esc in the agent view returns to the conversation it backgrounded
Fixed multi-line paste collapsing into one line with `j` in place of newlines in terminals that encode pasted newlines as Ctrl+J

MCP 連不上時，終於看得到原因

外部工具突然不能用時，claude mcp list 或 /mcp 現在會顯示 HTTP 狀態與錯誤文字，不再只留一句「連不上」。MCP（Model Context Protocol，讓 AI 連接外部工具的協定）設定值前後若藏了看不見的空白，也會先警告你；這種空白常讓看似相同的網址或金鑰失效。啟動時「N 個 MCP servers 需要驗證」的通知，則不會再把尚未在 claude.ai 連接的 connector 多算進去。好比客服不只說電話不通，還告訴你是號碼錯、對方忙線，並把名單上的幽靈聯絡人劃掉。

Original changelog
Added HTTP status and error text to `claude mcp list` and `/mcp` when a server fails to connect, and a warning for MCP config values with hidden leading or trailing whitespace
Fixed the "N MCP servers need authentication" startup notice over-counting claude.ai connectors that aren't connected in claude.ai

Windows 路徑與長選取文字不再亂碼

Windows 使用者若要讓工具讀取 C:\Users\unicorn 這類路徑，\u 開頭的片段以前可能被誤當成 Unicode 編碼，變成中日韓字，於是檔案像被改了地址般找不到；現在會保留原路徑。IDE 裡選了一大段文字，若內容剛好在 emoji 中間被截斷，也不再產生亂碼。另有工具執行器出錯卻被悄悄吞掉的情形也已修正。這像郵差終於照信封地址送件，不會把 \u 當成暗號改寫，信封破到一半時也會說清楚，而不是裝作沒收到。

Original changelog
Fixed Windows paths with `\u`-prefixed segments (like `C:\Users\unicorn`) being corrupted into CJK characters in tool inputs, which made those files inaccessible
Fixed mojibake when a long IDE selection was truncated mid-emoji, and a case where a tool executor error could be silently dropped

對話紀錄與背景工作少出現幽靈狀態

整理過長對話後查看 /context，token（AI 處理與計費的文字單位）用量不再顯示整理前的舊數字。工具中斷後，也不會亂留「使用者中斷請求」或缺了另一半的 tool_use 紀錄；prompt history 寫入碰撞或失敗時，歷史紀錄不再容易漏掉或重複。遇到 context overflow（對話塞超過可容納的量）又有很大的 thinking budget 時，重試不會一直重送同一個註定失敗的請求。背景化與 headless／SDK 的 fork session，整理後也保得住來源關係；壞掉的 delta attachment 不再讓續接對話每回合失敗。像一本帳本不再重複記帳、掉頁，換了書封後仍看得出每張收據從哪裡來。

Fixed an engine teardown race that could start and abandon a phantom turn, and made input pushed after close consistently rejected
Fixed spurious "[Request interrupted by user]" messages after interrupted tool calls, and an unpaired `tool_use` block left in the transcript when a tool aborted mid-response
Fixed prompt history entries being dropped or duplicated when history writes raced or failed
Fixed a retry loop that re-sent identical doomed requests after a context-overflow error with a large thinking budget; `Ctrl+B` backgrounding now applies the same background-shell caps as other paths

雲端模型計費、PR 與 AWS 設定更可靠

透過 Bedrock 使用模型時，gateway 花費計量現在會依你設定的模型費率計算 application-inference-profile ARN 與其他對應到上游模型的 ID，不會因識別碼不同而算錯價。剛建立或連結 pull request（PR，請求把一組程式改動併入專案的單據）就結束 session，也較不會遺失 PR 事件。Bedrock 設定精靈對分區 AWS 區域的 assume-role profile（暫時借用另一個 AWS 身分的設定），或只能走 proxy 的網路，能正確驗證。系統時鐘被調整後，回合耗時也不會罕見地變負數或失真。就像計程車改按可靠的里程表計價，收據、身分證明與行車時間都不會因儀表跳動而弄錯。

Original changelog
Fixed gateway spend metering to price Bedrock application-inference-profile ARNs and other config-mapped upstream model IDs at the configured model's rates
Fixed pull request events occasionally being lost when a session exited immediately after creating or linking a PR
Fixed the Bedrock setup wizard failing profile verification for assume-role profiles in partitioned AWS regions and on proxy-only networks
Fixed rare negative or incorrect turn duration measurements after a system clock adjustment by timing turns with a monotonic clock

自動模式少打擾，信任範圍更清楚

自動模式遇到危險的 rm、背景 & 或可疑 Windows 路徑時，不會一律跳出權限視窗；會交由 auto-mode classifier（判斷是否安全的自動分類器）決定。plan mode 裡，靜態分析器無法證明是唯讀的 Bash 指令，也同樣交給它判斷，少掉不必要的打斷。IDE 互動的 sandbox（把程式限制在安全範圍的隔間）指令限制也加強了。信任對話框會明說授權涵蓋哪個 repository root（專案最外層資料夾），agent frontmatter 的 hook（特定事件觸發的小程式）則必須放在已接受工作區信任的資料夾。伺服器管理設定中的無害功能與成本開關，也不再每次都要求批准。這像門禁把一般訪客、危險包裹和哪棟樓的通行證分清楚。

Agent 與 skill 設定更寬容，也避開名稱撞車

Original changelog

In non-interactive runs, /code-review ultra now truly starts a cloud review instead of silently falling back to a local one, and bad arguments return useful feedback. It is like sending a document to an editor while you keep writing at your desk; the editor still knows which document to read.
