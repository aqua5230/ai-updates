自動模式與背景任務大清倉，不再產生幽靈任務

這次讓自動模式（auto mode）跟背景任務變得更聰明。現在自動模式拒絕你執行的動作時，會把「為什麼拒絕」的理由清楚寫在紀錄與權限面板裡；你也可以開啟新設定，讓所有指令都經過安全分類，而不只是針對特定程式碼。對於背景代理人，它現在學會了平行處理，不會再笨笨地為了啟動代理人而要求主對話中斷。系統也新增了自動記憶體清理機制（memory-pressure reaping），會把閒置的背景指令清掉以節省資源。此外，以前你切換到背景時，常會被假警報恐嚇說「任務會被放棄」，或是莫名其妙生出一個重複執行對話的幽靈任務；在更新過後，這些假警報、幽靈任務，以及代理人面板會把其他兄弟代理人藏起來的毛病，全都修好了。就像請了一個精明的管家，以前他拒絕你做事只會搖頭，而且每次分派任務就要你閉嘴等他；現在他會清楚告訴你原因，並且能一邊聽你說話一邊指揮傭人，還會自動把沒在用的房間燈關掉省電。

跟外部系統打交道變得更順暢了。現在你在終端機使用 bash 模式（打 `!` 開頭）時，系統會即時幫你自動補齊檔案路徑，不用再辛苦地一個字一個字打。如果你的 MCP（外部工具伺服器）需要登入認證，系統一啟動就會溫馨提示你去 `/mcp` 處理；而且當工具呼叫因為權限不足（401/403 錯誤）被擋下時，MCP 輔助程式現在會自動重新連線並重試，不會直接死給你看。此外，當外部套件（plugin）在市集上改名時，你的設定檔也會自動跟著更新名字。登入後查看模型選單時，也不會再看到過期或空白的假畫面了。就像你的專屬助理，現在打字會自動幫你接下去，去別的部門調資料遇到門禁會自己想辦法重新刷卡，別人改名了也懂得幫你更新通訊錄，而且名片本打開永遠是最新狀態。

企業觀測日誌升級，可直接紀錄 AI 的回答內容

企業管理員現在有了更透明的監控工具。系統在 OpenTelemetry（企業用來收集日誌的標準框架）中新增了專門記錄 Claude 回答內容的事件。只要你原本就有設定記錄使用者的提示詞，系統升級後預設就會連同 AI 的回答一起記錄下來，讓你一眼看穿 AI 到底產出了什麼。當然，如果你有資安考量，也可以手動設定變數（`OTEL_LOG_ASSISTANT_RESPONSES=0`）來關閉這個功能，讓日誌只保留使用者的指令。就像公司的監視器原本只錄得到員工對對講機講的話，現在連對講機另一頭的回應也能一併錄音存證，當然，你想把喇叭靜音也完全沒問題。

Original changelog

Redacted unless `OTEL_LOG_ASSISTANT_RESPONSES=1`; when that var is unset it follows `OTEL_LOG_USER_PROMPTS`, so deployments that already log prompt content will start receiving response content on upgrade — set `OTEL_LOG_ASSISTANT_RESPONSES=0` to keep prompts-only.

Auto mode and background agents cleaned up without ghost tasks

Auto mode and background tasks just got a lot smarter. When auto mode denies an action, it now explicitly lists the reason in the transcript and permissions panel; you can also opt to route all terminal commands through the safety classifier instead of just specific code-execution patterns. For background agents, launching one no longer instructs the main AI to "end your response"—it can keep working on other things while the agent runs in parallel. We also added an automatic memory-pressure reaping system that sweeps up idle background shells to save resources. Moreover, several annoying bugs are squashed: sending tasks to the background no longer spooks you with false "tasks would be abandoned" warnings, nor does it spawn a phantom subagent that re-runs your entire conversation, and the agent panel won't erroneously hide sibling agents anymore. It's like hiring a savvy butler: previously, he just shook his head without explaining why you couldn't do something, and told you to stop talking while he dispatched chores; now he explains his logic, listens to you while delegating tasks, and remembers to turn off the lights in empty rooms to save power.

Enterprise logs upgraded to record AI response content

Enterprise admins now have a more transparent monitoring tool. The system added a new OpenTelemetry (a standard logging framework) event that specifically captures Claude's response text. If your deployment was already configured to log user prompts, the upgrade will seamlessly start logging the AI's answers as well, giving you full visibility into what the AI actually produced. Naturally, if you have privacy or security concerns, you can explicitly set a variable (`OTEL_LOG_ASSISTANT_RESPONSES=0`) to disable this and keep the logs strictly prompts-only. It's like a company security camera that previously only recorded employees speaking into an intercom; now it records the voice coming back from the other side too, though you always have the switch to mute the speaker if needed.

MCP와 플러그인 설정이 더 똑똑해지고, Bash 모드에서 경로 자동 완성 지원

외부 시스템과 손발을 맞추는 일이 훨씬 매끄러워졌다. 터미널에서 bash 모드(`!`로 시작)를 쓸 때, 이제 파일 경로를 실시간으로 자동 완성해 주어 일일이 타자를 치는 수고를 덜어준다. MCP(외부 도구 서버)가 로그인을 요구하면 프로그램이 켜질 때 `/mcp`를 확인하라고 친절하게 알려주며, 권한이 없어(401/403 오류) 도구 사용이 막히더라도 보조 프로그램이 알아서 다시 연결해 재시도하고 쉽게 포기하지 않는다. 마켓플레이스에서 외부 플러그인(plugin)의 이름이 바뀌면 당신의 설정 파일도 알아서 새 이름으로 바뀐다. 로그인을 막 마친 뒤 모델 메뉴를 열었을 때 텅 빈 화면이나 옛날 정보가 뜨던 문제도 고쳐졌다. 똑똑한 비서가 글씨를 마저 써 주고, 다른 부서에 자료를 가지러 갔다가 출입증이 막히면 알아서 다시 찍어보며, 거래처 사람이 개명하면 주소록도 알아서 고쳐 명함첩을 항상 최신으로 유지하는 것과 같다.

기업용 로그 업그레이드, AI의 답변 내용 직접 기록 가능

기업 관리자에게 더 투명한 감시 도구가 생겼다. 시스템에 OpenTelemetry(기업이 로그를 수집하는 표준 프레임워크)를 위한 이벤트가 추가되어 Claude의 답변 내용까지 따로 기록할 수 있게 되었다. 원래 사용자의 프롬프트를 기록하도록 설정해 두었다면, 업데이트 후에는 알아서 AI의 답변까지 함께 기록해 AI가 어떤 결과물을 내놓았는지 훤히 들여다볼 수 있다. 물론 보안이 걱정된다면 변수(`OTEL_LOG_ASSISTANT_RESPONSES=0`)를 설정해 이 기능을 끄고 프롬프트만 남기도록 할 수도 있다. 회사의 CCTV가 예전엔 무전기에 대고 말하는 직원의 목소리만 녹음했다면, 이제는 무전기 너머에서 돌아오는 대답까지 통째로 녹음해 증거로 남기는 것과 같다. 원한다면 언제든 스피커 녹음만 꺼둘 수도 있다.
