zh-TW

權限管控更嚴格，沙盒防護升級

這次針對安全與管控做了重要升級。現在你可以透過設定 sandbox.credentials，防止在沙盒（sandbox，隔離的安全執行環境）裡跑的指令偷看你的密碼檔或環境變數機密。如果你在企業環境裡用，現在只要選到公司禁用的模型，介面上會明確跳出「受組織設定限制」的提示，不會讓你白忙一場。此外，對於外部工具（MCP）連線，以前如果對方毫無反應，系統會傻傻地無限期等下去；現在只要卡住 5 分鐘就會自動中斷並報錯（你也可以自己改設定），不讓整個流程被拖垮。這就像幫公司的實驗室加裝了防盜門，不但會阻擋來路不明的軟體偷拿機密資料，而且如果連線到外部廠商一直沒人接，五分鐘後電話就會自動掛斷，不會佔線一整天。

Original changelog
Added `sandbox.credentials` setting to block sandboxed commands from reading credential files and secret environment variables
Added org-configured model restrictions to the model picker, `--model`, `/model`, and `ANTHROPIC_MODEL`, with a "restricted by your organization's settings" message when a restricted model is selected
Fixed remote MCP tool calls that hang with no response for 5 minutes — they now abort with an error instead of blocking indefinitely (override with `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT`)

代理人管理更精準，不再無意義空轉

讓多個 AI 代理人（agent）一起工作時，現在更穩定了。當你要求代理人以特定格式（JSON schema）輸出結果時，它現在給出正確答案後就不會再陷入無限重覆呼叫工具的死迴圈。如果在背景工作的代理人做完事卻沒吐出結果，也不會再一直卡在「執行中」騙你。對於分支出來的子代理人，系統現在會正確計算它們的層級深度，不會讓代理人無限制地「子生孫、孫生子」下去。而且代理人被停掉時，通知現在會正確寫出是誰停的，並把用語改成更白話的「已結束」或「已停止」。如果你強制砍掉代理人，它遺留在 Git 工作樹（worktree）裡的垃圾檔案，現在系統也會自動幫你清乾淨了。這就像工廠裡的機器人，以前交差後還會原地一直轉圈圈不肯停，甚至做了一堆複製人塞滿走道，現在它們只要完成任務就會乖乖回報「停工」，而且離職時還會順手把自己的工作桌收乾淨。

Original changelog
Fixed `--json-schema` and workflow `agent({schema})` structured output: the model can no longer re-call `StructuredOutput` indefinitely after a successful call, and follow-up turns now reliably return structured output
Fixed background jobs in the agents view getting stuck in "working" indefinitely when the agent ended a turn without producing structured output
Fixed agent stop notifications not correctly attributing who stopped the agent, and improved wording ("finished"/"stopped" instead of "came to rest")
Fixed subagent depth tracking: resumed subagents now restore their original spawn depth, and forked subagents now count toward the depth cap
Fixed leaked agent worktree registrations: locked `.git/worktrees/` entries from killed agents are now cleaned up automatically

en

Tighter permissions and upgraded sandbox security

We've rolled out major upgrades for security and administrative control. You can now use the new sandbox.credentials setting to explicitly block commands running inside the sandbox (an isolated secure environment) from peeking at your credential files or secret environment variables. For enterprise users, the model picker now respects organization-configured restrictions and will clearly tell you if a model is "restricted by your organization's settings" rather than letting you guess why it failed. Additionally, if an external MCP tool server goes silent and hangs, the system will no longer wait around forever; it will automatically abort the call and throw an error after 5 minutes (a timeout you can adjust). It's like installing a vault door on the company lab to prevent rogue experiments from stealing passwords, and implementing a strict five-minute rule on hold times so an unresponsive vendor doesn't tie up the phone line all day.

Fixed remote MCP tool calls that hang with no response for 5 minutes — they now abort with an error instead of blocking indefinitely (override with `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT`)

Agent management is more precise, avoiding endless spinning

Coordinating multiple AI agents is now much more stable. When you require an agent to produce structured output (like a JSON schema), it will no longer get stuck in an endless loop re-calling the output tool after already succeeding. If a background agent ends its turn without producing the requested structure, it won't deceptively stay stuck displaying a 'working' status forever. For spawned subagents, the system now accurately tracks how deep the chain goes, ensuring agents don't infinitely spawn subagents and violate the depth cap. When an agent gets halted, the notification now correctly attributes who stopped it and uses clearer wording like 'finished' or 'stopped' instead of cryptic phrasing. Also, if you kill an agent, any locked entries it left behind in your Git worktree are now automatically cleaned up. It's like factory robots that used to mindlessly spin in circles after finishing a job, or endlessly build clones of themselves until the floor ran out of space; now they properly clock out, report exactly who shut them off, and perfectly tidy up their workstations before they leave.

For users of CJK (Chinese/Japanese/Korean) languages, pasting multi-byte characters in certain terminals used to result in a garbled mess of mojibake; this is fixed and your text will paste perfectly. It's like an old vending machine that only took stiff button presses upgrading to a modern touchscreen; it no longer jams when you insert foreign coins, and the cancel button actually works if you change your mind mid-purchase.
